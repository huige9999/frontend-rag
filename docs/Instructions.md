# frontend-rag 内容组织规则

## 1. 当前阶段目标

当前阶段不整理完整课件内容，也不做全局前端知识地图。

只做一件事：

每学到一门课，就把这门课自己的「内部结构」和「学习主线」整理出来。

目标是以后复习、讲课、查漏补缺时，能快速知道：

- 这门课在讲什么
- 它按什么逻辑展开
- 每一章在整体里是什么位置
- 遇到问题该回到哪一章

---

## 2. 文件粒度

一个 md 对应一门课程。

例如：

- Node.md
- Vue3.md
- Vue2.md
- 微信小程序.md
- React.md
- 工程化.md

暂时不把每个小节拆成独立 md。

---

## 3. 每门课的 md 模板

# 课程名

## 1. 目录结构整理

用树形结构整理这门课的章节。

## 2. 章节逻辑说明

说明这门课是按照什么顺序讲的。

重点回答：

- 先讲什么？
- 后讲什么？
- 为什么这么安排？
- 这门课的主线是什么？

## 3. 使用场景等元信息沉淀

可选。

可以记录：

- 这门课学完能用在哪？
- 和实际项目有什么关系？
- 和其他课程有什么关系？
- 面试或教学中有没有高频场景？

## 4. 临时补充

记录临时想到的问题、疑惑、关联知识，不要求结构完美。

---

## 4. 整理规则

1. 只整理当前课程内部体系。
2. 不做跨课程知识地图。
3. 不整理完整课件内容。
4. 不追求一次性完美。
5. 优先保留原课程目录顺序。
6. 如果原课程目录太散，可以按语义合并成大模块。
7. 每个大模块最多补充一句说明。
8. 使用场景是可选项，不强制补全。

---

## 5. 后续目标

当课程 md 积累足够多、查询不方便时，再单独建立知识索引文件。

知识索引可以按照前端知识领域重新组织，例如：

- JavaScript
- TypeScript
- HTML / CSS / Web API
- Vue
- React
- Node
- 工程化
- 小程序
- 性能优化
- 架构设计
- AI + 前端

但这属于第二阶段，当前阶段不展开。
