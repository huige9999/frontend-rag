# vue-i18n@next 快速上手

## 课程概述

本节课是前端国际化（i18n）系列课程的第五节，主要讲解如何在 Vue 3 项目中快速集成和使用 `vue-i18n@next`（即 vue-i18n v9），包括安装配置、i18n 实例创建、插件注册、语言包定义以及在模板和组合式 API 中的基本使用方法。

---

## 一、项目初始化与依赖安装

### 1.1 创建 Vue 3 + TypeScript 项目

使用 Vite 创建 Vue 3 项目，项目结构如下：

```
intl-demo/
├── index.html
├── package.json
├── vite.config.ts
├── src/
│   ├── main.ts
│   ├── App.vue
│   ├── style.css
│   ├── vite-env.d.ts
│   ├── i18n/
│   │   ├── index.ts
│   │   └── locales/

│   │       └── zh-CN.json
│   ├── routes/

│   ├── components/
│   │   └── Nav.vue
│   └── views/
│       ├── HomeView.vue
│       └── AboutView.vue
```

**说明：**

### 1.3 Vite 配置

`vite.config.ts`：

```ts
import { fileURLToPath, URL } from 'node:url'
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [
    vue(),
  ],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})
```

**说明：**

- 使用 `@vitejs/plugin-vue` 插件支持 `.vue` 单文件组件
- 配置路径别名 `@` 指向 `src` 目录，方便模块导入

## 二、语言包定义

### 2.1 英文语言包

`src/i18n/locales/en-US.json`：

```json
{
  "nav": {
    "home": "Home",
    "about": "About"
  },
  "home": {
    "header": "Welcome to Vue3 I18n Tutorial",
    "content": "This tutorial teaches you how to use vue-I18n in Vue3---{company}"
  }
}
```

### 2.2 中文语言包

`src/i18n/locales/zh-CN.json`：

```json
{
  "nav": {
    "home": "首页",
    "about": "关于"
  },
  "home": {
    "header": "欢迎使用Vue3 I18n 教学",
    "content": "这个教学教导大家在Vue3中使用vue-I18n---{company}"
  }
}
```

**说明：**

- 语言包采用 JSON 格式，使用**嵌套对象**组织翻译键值
- 键名采用 `模块.字段` 的层级结构，如 `nav.home`、`home.header`
- `{company}` 是**插值占位符**，用于在翻译文本中动态插入变量值
- 语言文件按 `语言代码-国家代码` 命名（如 `zh-CN`、`en-US`），符合 BCP 47 标准

## 三、创建 i18n 实例

`src/i18n/index.ts`：

```ts
import { createI18n } from 'vue-i18n'
import enUS from "./locales/en-US.json"
import zhCN from "./locales/zh-CN.json"

export default createI18n({
  locale: import.meta.env.VITE_DEFAULT_LOCALE,
  fallbackLocale: import.meta.env.VITE_FALLBACK_LOCALE,
  legacy: false,
  messages: {
    'en-US': enUS,
    'zh-CN': zhCN,
  },
  globalInjection: true,
})
```

**关键配置项说明：**

| 配置项 | 说明 |
|--------|------|
| `locale` | 当前使用的语言，通过环境变量 `VITE_DEFAULT_LOCALE` 设置默认值 |
| `fallbackLocale` | 回退语言，当当前语言的翻译缺失时，使用该语言的翻译作为备选 |
| `legacy` | 设为 `false` 表示使用**组合式 API 模式**（Vue 3 推荐），设为 `true` 为选项式 API 模式 |
| `messages` | 语言包对象，键为 locale 标识，值为对应的翻译数据 |
| `globalInjection` | 设为 `true` 后，可以在所有组件模板中直接使用 `$t()` 方法，无需逐个组件引入 |

### 3.1 内联定义语言包（备选方式）

## 四、注册 i18n 插件

`src/main.ts`：

```ts
import { createApp } from 'vue'
import App from './App.vue'
import router from './routes'
import i18n from './i18n'

const app = createApp(App)
app.use(i18n)
app.use(router)
app.mount('#app')
```

**说明：**

- 通过 `import i18n from './i18n'` 导入上一步创建的 i18n 实例
- 使用 `app.use(i18n)` 将 i18n 插件注册到 Vue 应用中
- 注册后，所有组件都可以使用 `$t()` 方法和 `useI18n()` 组合式函数
- 入口 HTML 文件 `index.html` 通过 `<div id="app"></div>` 提供挂载点：

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Vite + Vue + TS</title>
  </head>
  <body>
    <div id="app"></div>
    <script type="module" src="/src/main.ts"></script>
  </body>
</html>
```

## 五、在模板中使用翻译

### 5.1 基本翻译 —— `$t()` 方法

`src/components/Nav.vue`：

```vue
<template>
  <div>
    <nav>
      <ul>
        <li>
          <router-link to="/">{{ $t("nav.home") }}</router-link>
        </li>
        <li>
          <router-link to="/about">{{ $t("nav.about") }}</router-link>
        </li>
      </ul>
    </nav>
  </div>
</template>
```

**说明：**

- `$t("nav.home")` 会根据当前 locale 自动返回对应的翻译文本
- 当 locale 为 `zh-CN` 时，`$t("nav.home")` 返回 `"首页"`
- 当 locale 为 `en-US` 时，`$t("nav.home")` 返回 `"Home"`
- 之所以能直接在模板中使用 `$t()`，是因为在 `createI18n` 中设置了 `globalInjection: true`

### 5.2 带插值的翻译

`src/views/HomeView.vue`：

```vue
<template>
  <main>
    <h2>{{ $t("home.header") }}</h2>
    <p>{{ $t("home.content", { company: "DuYi-Edu" }) }}</p>
    <p>该网站已经被浏览 30 次</p>
    <p>2023年07月29日</p>
  </main>
</template>

<script setup lang="ts">
// import { useI18n } from 'vue-i18n';
// const { t } = useI18n();
</script>
```

**说明：**

- `$t("home.content", { company: "DuYi-Edu" })`：第二个参数传入对象，用于替换翻译文本中的 `{company}` 占位符
- 渲染结果（中文）：`"这个教学教导大家在Vue3中使用vue-I18n---DuYi-Edu"`
- 渲染结果（英文）：`"This tutorial teaches you how to use vue-I18n in Vue3---DuYi-Edu"`
- `<script setup>` 中注释的代码展示了**组合式 API 用法**：通过 `useI18n()` 解构出 `t` 函数，在 JS/TS 逻辑中调用 `t("key")` 获取翻译文本

### 5.3 About 页面（待国际化示例）

`src/views/AboutView.vue`：

```vue
<template>
  <div>
    <h1>关于我们</h1>
    <p>收到贡献:￥1234.56</p>
  </div>
</template>
```

**说明：** 该页面中 `关于我们`、`收到贡献` 以及金额 `￥1234.56` 均为硬编码文本，尚未进行国际化处理。这可以作为后续练习——将这些硬编码文本替换为 `$t()` 调用，并使用 `Intl.NumberFormat` 处理货币格式。

## 六、路由配置

`src/routes/index.ts`：

```ts
import { createRouter, createWebHistory } from 'vue-router'
import HomeView from '../views/HomeView.vue'

const router = createRouter({
  history: createWebHistory(import.meta.env.VITE_BASE_URL),
  routes: [
    {
      path: '/',
      name: 'Home',
      component: HomeView
    },
    {
      path: '/about',
      name: 'About',
      component: () => import('../views/AboutView.vue')
    },
  ]
})
export default router
```

**说明：**

- 使用 Vue Router 4 的 `createRouter` 和 `createWebHistory` 创建路由实例
- `import.meta.env.VITE_BASE_URL` 通过 Vite 环境变量设置路由基础路径
- About 页面使用**懒加载**（`() => import(...)`），按需加载组件

## 八、核心知识点总结

### 8.1 vue-i18n@next 集成步骤

1. **安装依赖**：`npm install vue-i18n@next`
2. **创建语言包**：在 `src/i18n/locales/` 目录下编写 JSON 翻译文件
3. **创建 i18n 实例**：使用 `createI18n()` 并配置 `locale`、`messages`、`legacy`、`globalInjection` 等选项
4. **注册插件**：在 `main.ts` 中通过 `app.use(i18n)` 注册
5. **使用翻译**：在模板中使用 `$t("key")` 或在 `<script setup>` 中使用 `useI18n()` 的 `t` 函数

### 8.2 关键 API 速查

| API | 用途 | 使用场景 |
|-----|------|---------|
| `createI18n(options)` | 创建 i18n 实例 | `src/i18n/index.ts` 中调用 |
| `app.use(i18n)` | 注册 i18n 插件 | `main.ts` 中调用 |
| `$t(key, params)` | 模板中获取翻译文本 | 组件 `<template>` 中使用 |
| `useI18n()` | 获取 i18n 组合式函数 | `<script setup>` 中解构出 `t` |
| `globalInjection: true` | 全局注入 `$t` 方法 | `createI18n` 配置项 |
| `legacy: false` | 启用组合式 API 模式 | `createI18n` 配置项（Vue 3 推荐） |

### 8.3 翻译键的命名规范

- 使用**点号分隔**的层级结构：`模块.字段`（如 `nav.home`、`home.header`）
- 键名应具有**语义化**，避免使用无意义的编号
- 相同模块的键归类在同一个对象下，便于管理和维护
