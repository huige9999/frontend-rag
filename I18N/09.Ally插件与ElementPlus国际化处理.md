本节课在前几节课的基础上，重点讲解两方面的内容：

1. **Vue I18n Ally 插件**：使用 `@intlify/unplugin-vue-i18n` 插件实现翻译文件的自动加载，告别手动 import。
2. **ElementPlus 国际化**：将 ElementPlus 组件库与 vue-i18n 集成，实现 UI 组件的多语言切换。

## 二、@intlify/unplugin-vue-i18n 插件（Ally 插件）

### 2.1 为什么需要 Ally 插件？

在之前的课程中，我们需要手动导入每个翻译文件：

```ts
// 传统方式：手动导入每个语言文件
import enUS from "./locales/en-US.json"
import zhCN from "./locales/zh-CN.json"

const messages = {
  'en-US': enUS,
  'zh-CN': zhCN,
}
```

这种方式存在明显问题：
- 每新增一种语言，都需要手动 import 并注册
- 翻译文件越来越多时，管理起来非常繁琐

`@intlify/unplugin-vue-i18n` 插件可以**自动扫描指定目录下的翻译文件并加载**，无需手动导入。

### 2.2 安装依赖

```json
// package.json（devDependencies 中）
{
  "devDependencies": {
    "@intlify/unplugin-vue-i18n": "^0.12.2"
  }
}
```

> 安装命令：`npm install @intlify/unplugin-vue-i18n -D`

### 2.3 在 vite.config.ts 中配置插件

```ts
// vite.config.ts
import { fileURLToPath, URL } from 'node:url'
import { resolve } from 'node:path'
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import VueI18nPlugin from '@intlify/unplugin-vue-i18n/vite'

export default defineConfig({
  plugins: [
    vue(),
    // 配置 Vue I18n Ally 插件
    VueI18nPlugin({
      // 指定翻译文件的扫描路径
      include: resolve(__dirname, './src/i18n/locales/**'),
      // runtimeOnly: false 表示包含完整的 vue-i18n 运行时（含编译器）
      runtimeOnly: false,
      // strictMessage: false 允许翻译消息中不使用 @.xxx 的键名格式
      strictMessage: false
    }),
  ],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})
```

**配置项说明：**

| 配置项 | 说明 |
|--------|------|
| `include` | 使用 glob 模式指定翻译文件路径，`**` 表示匹配所有子目录下的文件 |
| `runtimeOnly` | 设为 `false` 包含完整运行时（支持动态编译翻译消息）；设为 `true` 则只包含运行时（体积更小，但需预编译） |
| `strictMessage` | 设为 `false` 允许更灵活的消息格式，不强制 ICU 消息格式规范 |

### 2.4 在 i18n/index.ts 中使用自动加载的翻译消息

```ts
// src/i18n/index.ts
import { createI18n } from 'vue-i18n'
// 关键：通过虚拟模块自动导入所有翻译文件
import messages from "@intlify/unplugin-vue-i18n/messages";
import numberFormats from './rules/number'
import datetimeFormats from './rules/datetime'

export default createI18n({
  locale: import.meta.env.VITE_DEFAULT_LOCALE,
  fallbackFormat: import.meta.env.VITE_FALLBACK_LOCALE,
  legacy: false,           // 使用组合式 API 模式
  messages,                // 自动加载的翻译消息，无需手动 import 每个文件
  globalInjection: true,   // 全局注入 $t 等方法
  numberFormats,           // 数字格式化规则
  datetimeFormats          // 日期时间格式化规则
})
```

**关键改动：**

```ts
// 之前（手动导入）:
import enUS from "./locales/en-US.json"
import zhCN from "./locales/zh-CN.json"
const messages = { 'en-US': enUS, 'zh-CN': zhCN }

// 现在（Ally 插件自动加载）:
import messages from "@intlify/unplugin-vue-i18n/messages";
```

`@intlify/unplugin-vue-i18n/messages` 是一个**虚拟模块**，插件会在编译时自动将 `locales` 目录下的所有 JSON 文件合并为一个 messages 对象。之后新增语言文件时，只需要在 `locales` 目录中新增一个 JSON 文件即可，**无需修改任何代码**。

## 三、ElementPlus 国际化配置

### 3.1 安装 ElementPlus

```json
// package.json（dependencies 中）
{
  "dependencies": {
    "element-plus": "^2.3.8"
  }
}
```

### 3.2 在 main.ts 中全局注册 ElementPlus

```ts
// src/main.ts
import { createApp } from 'vue'
import App from './App.vue'
import router from './routes'
import i18n from './i18n'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'

const app = createApp(App);
app.use(i18n);
app.use(ElementPlus);      // 全局注册 ElementPlus
app.use(router);
app.mount('#app')
```

### 3.3 在 App.vue 中使用 ElConfigProvider 配置语言

ElementPlus 组件（如日期选择器、分页器等）内部有自带的多语言文本（如"今天"、"上一月"等）。要让这些文本跟随 vue-i18n 的语言切换，需要使用 `ElConfigProvider` 组件。

```vue
<!-- src/App.vue -->
<script setup lang="ts">
import Nav from './components/Nav.vue'
import LanguageSwitcher from './components/LanguageSwitcher.vue'
import { ElConfigProvider } from 'element-plus'

// 导入 ElementPlus 内置的语言包
import zhCn from 'element-plus/dist/locale/zh-cn.mjs'
import en from 'element-plus/dist/locale/en.mjs'

import Trans from "@/i18n/translation"
import { computed } from 'vue'

// 根据当前 vue-i18n 的语言设置，动态计算 ElementPlus 应使用的语言包
const locale = computed(() => {
  const lang = Trans.currentLocale;
  return lang === 'zh-CN' ? zhCn : en
})
</script>

<template>
  <div>
    <Nav />
    <LanguageSwitcher />
    <div>
      <!-- 使用 ElConfigProvider 包裹 router-view，让所有 ElementPlus 组件自动使用对应语言 -->
      <el-config-provider :locale="locale">
        <router-view />
      </el-config-provider>
    </div>
  </div>
</template>
```

**核心原理：**

1. ElementPlus 为每种语言提供了独立的语言包（`element-plus/dist/locale/` 目录下）
2. `ElConfigProvider` 组件通过 provide/inject 机制，将语言配置向下传递给所有子组件
3. 使用 `computed` 计算属性，当 vue-i18n 的语言切换时，ElementPlus 的语言包也自动跟随切换

**ElementPlus 语言包映射：**

| vue-i18n locale | ElementPlus 语言包 | 说明 |
|-----------------|-------------------|------|
| `zh-CN` | `element-plus/dist/locale/zh-cn.mjs` | 简体中文 |
| `en-US` | `element-plus/dist/locale/en.mjs` | 英文 |

### 3.4 ElConfigProvider 的作用范围

`el-config-provider` 采用 Vue 的 provide/inject 机制，因此它会影响其**所有后代组件**中的 ElementPlus 组件。将 `<router-view />` 放在 `<el-config-provider>` 内部，就能让所有页面中的 ElementPlus 组件都实现国际化。

## 四、ElementPlus 组件国际化实战 —— ElementView

### 4.1 ElementView.vue 完整代码

```vue
<!-- src/views/ElementView.vue -->
<template>
  <!-- 尺寸选择按钮组：文本使用 i18n 翻译 -->
  <el-radio-group v-model="size" label="size control" size="small">
    <el-radio-button label="large">{{ $t('element.large') }}</el-radio-button>
    <el-radio-button label="default">{{ $t('element.default') }}</el-radio-button>
    <el-radio-button label="small">{{ $t('element.small') }}</el-radio-button>
  </el-radio-group>

  <div class="demo-date-picker">
    <!-- 默认日期选择器 -->
    <div class="block">
      <span class="demonstration">{{ $t('element.default') }}</span>
      <el-date-picker
        v-model="value1"
        type="date"
        :placeholder="$t('element.pick_a_day')"
        :size="size"
      />
    </div>

    <!-- 带快捷选项的日期选择器 -->
    <div class="block">
      <span class="demonstration">{{ $t("element.quick_select") }}</span>
      <el-date-picker
        v-model="value2"
        type="date"
        :placeholder="$t('element.pick_a_day')"
        :disabled-date="disabledDate"
        :shortcuts="shortcuts"
        :size="size"
      />
    </div>
  </div>
</template>

<script lang="ts" setup>
import { ref } from 'vue'
import { useI18n } from 'vue-i18n'

const { t } = useI18n()

const size = ref<'default' | 'large' | 'small'>('default')
const value1 = ref('')
const value2 = ref('')

// 快捷选项：使用 t() 函数进行翻译
const shortcuts = [
  {
    text: t('element.today'),
    value: new Date(),
  },
  {
    text: t('element.yesterday'),
    value: () => {
      const date = new Date()
      date.setTime(date.getTime() - 3600 * 1000 * 24)
      return date
    },
  },
  {
    text: t('element.last_week'),
    value: () => {
      const date = new Date()
      date.setTime(date.getTime() - 3600 * 1000 * 24 * 7)
      return date
    },
  },
]

// 禁用今天之后的日期
const disabledDate = (time: Date) => {
  return time.getTime() > Date.now()
}
</script>
```

### 4.2 关键知识点解析

**1. 模板中使用 `$t()` 翻译 ElementPlus 组件的文本属性**

```vue
<!-- placeholder 属性国际化 -->
<el-date-picker :placeholder="$t('element.pick_a_day')" />

<!-- 按钮文本国际化 -->
<el-radio-button label="large">{{ $t('element.large') }}</el-radio-button>
```

**2. 在 JS/TS 中使用 `t()` 函数翻译**

`shortcuts` 数组中的 `text` 字段需要在 `<script setup>` 中使用 `useI18n()` 提供的 `t()` 函数进行翻译：

```ts
const { t } = useI18n()

const shortcuts = [
  { text: t('element.today'),    value: new Date() },
  { text: t('element.yesterday'), value: () => { /* ... */ } },
  { text: t('element.last_week'), value: () => { /* ... */ } },
]
```

> 注意：`shortcuts` 中使用的是 `t()` 函数而非 `$t()`，因为在 `<script setup>` 中没有 `$t` 的直接访问方式，需要通过 `useI18n()` 解构获取。

**3. ElementPlus 组件内置文本的自动国际化**

日期选择器中像"年"、"月"、"日"、"确定"、"清空"等内置文本，由 `ElConfigProvider` 提供的语言包自动翻译，**无需手动处理**。这部分就是第三节中 `ElConfigProvider` 配置的作用。

**两种国际化文本的区分：**

| 类型 | 处理方式 | 示例 |
|------|---------|------|
| 开发者自定义的文本（placeholder、label 等） | 使用 `$t()` 或 `t()` 函数 | `:placeholder="$t('element.pick_a_day')"` |
| ElementPlus 组件内置文本 | `ElConfigProvider` + 语言包自动处理 | 日期选择器的"年/月"、分页器的"条/页"等 |

## 五、翻译工具类（translation.ts）

翻译工具类是整个国际化方案的核心，封装了语言检测、切换、路由守卫等功能。

```ts
// src/i18n/translation.ts
import { NavigationGuardNext, RouteLocationNormalized } from "vue-router"
import { LANGUAGES } from "./constants"
import i18n from "@/i18n"
import axios from "axios"

const Translation = {
  // 获取支持的语言列表
  get supportedLocales() {
    return LANGUAGES
  },

  // 获取 / 设置当前语言
  get currentLocale() {
    return i18n.global.locale.value
  },
  set currentLocale(newLocale: string) {
    i18n.global.locale.value = newLocale
  },

  // 获取 .env 中的默认语言
  get defaultLocale() {
    return import.meta.env.VITE_DEFAULT_LOCALE
  },

  // 判断语言是否被支持
  isLocaleSupported(locale: string) {
    return locale in Translation.supportedLocales
  },

  // 获取用户浏览器语言
  getUserLocale() {
    let locale = navigator.language ? navigator.language : Translation.defaultLocale
    return {
      locale,                          // zh-CN, en-US
      localeNoRegion: locale.split('-')[0]  // zh, en
    }
  },

  // 获取用户本地持久化的语言
  getPersistedLocale() {
    let persistedLocale = localStorage.getItem("user-locale") || ""
    if (Translation.isLocaleSupported(persistedLocale)) {
      return persistedLocale
    } else {
      return null
    }
  },

  // 猜测默认语言（优先级：本地持久化 > 浏览器语言 > 环境变量默认值）
  guessDefaultLocale() {
    const userPersistedLocale = Translation.getPersistedLocale()
    if (userPersistedLocale) return userPersistedLocale

    const userPreferredLocale = Translation.getUserLocale()
    if (Translation.isLocaleSupported(userPreferredLocale.locale)) {
      return userPreferredLocale.locale
    }
    if (Translation.isLocaleSupported(userPreferredLocale.localeNoRegion)) {
      return userPreferredLocale.localeNoRegion
    }
    return Translation.defaultLocale
  },

  // 路由守卫：确保 URL 中始终包含语言参数
  routerMiddleware(to: RouteLocationNormalized, _from: RouteLocationNormalized, next: NavigationGuardNext) {
    const paramLocale = to.params.locale as string
    if (!Translation.isLocaleSupported(paramLocale)) {
      const defaultLocale = Translation.guessDefaultLocale()
      return next(defaultLocale)
    }
    Translation.switchLanguage(paramLocale)
    return next()
  },

  // 生成带语言参数的路由对象
  i18nRoute(to: any) {
    return {
      ...to,
      params: { locale: Translation.currentLocale },
      ...to.params
    }
  },

  // 切换语言（核心方法）
  switchLanguage(newLocale: string) {
    Translation.currentLocale = newLocale
    document.documentElement.lang = newLocale                // 更新 HTML lang 属性
    localStorage.setItem("user-locale", newLocale)           // 持久化到 localStorage
    axios.defaults.headers.common["Accept-Language"] = newLocale  // 设置请求头
  }
}

export default Translation
```

**`switchLanguage` 方法做了四件事：**

1. 更新 vue-i18n 的当前 locale
2. 设置 `<html>` 标签的 `lang` 属性（有利于 SEO 和浏览器识别）
3. 将语言偏好持久化到 `localStorage`（下次访问时恢复）
4. 设置 axios 的 `Accept-Language` 请求头（后端根据此头返回对应语言的数据）

## 八、路由配置

```ts
// src/routes/index.ts
import { createRouter, createWebHistory, RouterView } from 'vue-router'
import HomeView from '../views/HomeView.vue'
import Trans from "@/i18n/translation"

const router = createRouter({
  history: createWebHistory(import.meta.env.VITE_BASE_URL),
  routes: [
    {
      // :locale? 为可选的语言参数，如 /zh-CN/about 或 /en-US
      path: "/:locale?",
      component: RouterView,
      beforeEnter: Trans.routerMiddleware,  // 路由守卫：自动处理语言参数
      children: [
        { path: '', name: 'Home', component: HomeView },
        { path: 'about', name: 'About', component: () => import('../views/AboutView.vue') },
        { path: 'post', name: 'Post', component: () => import('../views/PostView.vue') },
        { path: 'element', name: 'Element', component: () => import('../views/ElementView.vue') }
      ]
    }
  ]
})

export default router
```

**关键点：**

- `/:locale?` 使用可选参数，URL 形如 `/zh-CN/about` 或 `/en-US`
- `beforeEnter: Trans.routerMiddleware` 在进入路由前检查语言参数的合法性并切换语言
- 子路由继承了父路由的 `locale` 参数，无需在每个子路由中重复定义

### 9.2 语言切换组件（LanguageSwitcher.vue）

```vue
<!-- src/components/LanguageSwitcher.vue -->
<template>
  <div>
    <select v-model="locale" @change="handleChange">
      <option v-for="(value, key) in Trans.supportedLocales" :key="key" :value="key">
        {{ value }}
      </option>
    </select>
  </div>
</template>

<script setup lang="ts">
import { useI18n } from "vue-i18n"
import { useRouter } from "vue-router"
import Trans from "@/i18n/translation"

const router = useRouter()
const { locale } = useI18n()

const handleChange = async (e: Event) => {
  const value = (e.target as HTMLSelectElement).value
  Trans.switchLanguage(value)

  try {
    // 更新 URL 中的语言参数
    await router.replace({ params: { locale: value } })
    // 刷新页面以确保所有组件重新渲染
    router.go(0)
  } catch (e) {
    console.log(e)
    router.push("/")
  }
}
</script>
```

**切换流程：**

1. 用户选择新语言
2. `Trans.switchLanguage()` 更新 vue-i18n locale、localStorage、HTML lang 属性、axios 请求头
3. `router.replace()` 更新 URL 中的语言参数
4. `router.go(0)` 刷新页面，确保所有组件（包括 ElementPlus）都使用新语言重新渲染

### 11.3 路由层 —— 根据 Accept-Language 返回对应语言数据

```js
// intl-server/routes/index.js
var express = require('express')
var router = express.Router()
var postDao = require('../dao/postDao')

router.get('/list', async function (req, res, next) {
  let { page, limit } = req.query
  // 从请求头中获取前端设置的语言
  let locale = req.headers['accept-language']

  console.log(locale)

  // 统一语言标识
  if (locale.includes('zh-CN')) {
    locale = 'zh-CN'
  } else if (locale.includes('en-US')) {
    locale = 'en-US'
  }

  // 根据语言过滤数据
  const filter = { 'locale': locale }
  let result = await postDao.list(filter, page, limit)

  res.json(result)
})

module.exports = router
```

**后端国际化策略：** 前端在 `switchLanguage` 中设置了 `axios.defaults.headers.common["Accept-Language"]`，后端读取该请求头，按照语言标识从数据库中过滤出对应语言的文章数据返回。

## 十三、本节课知识点总结

### Ally 插件（@intlify/unplugin-vue-i18n）

| 步骤 | 说明 |
|------|------|
| 1. 安装 | `npm install @intlify/unplugin-vue-i18n -D` |
| 2. 配置 vite.config.ts | 引入 `VueI18nPlugin`，设置 `include` 指定翻译文件目录 |
| 3. 使用虚拟模块 | `import messages from "@intlify/unplugin-vue-i18n/messages"` 自动加载所有翻译文件 |
| 4. 新增语言 | 只需在 locales 目录下新增 JSON 文件，无需修改代码 |

### ElementPlus 国际化

| 步骤 | 说明 |
|------|------|
| 1. 安装 | `npm install element-plus` |
| 2. 全局注册 | `app.use(ElementPlus)` |
| 3. 导入语言包 | `import zhCn from 'element-plus/dist/locale/zh-cn.mjs'` |
| 4. ElConfigProvider | 用 `<el-config-provider :locale="locale">` 包裹 `<router-view />` |
| 5. 动态切换 | 使用 `computed` 根据 vue-i18n locale 返回对应的 ElementPlus 语言包 |
| 6. 组件文本国际化 | 开发者自定义文本用 `$t()` / `t()`，组件内置文本由语言包自动处理 |

### 两种国际化方案的协作

```
用户切换语言
    |
    v
Trans.switchLanguage(newLocale)
    |
    +---> vue-i18n locale 更新 ----> $t() 翻译文本更新
    |
    +---> computed(locale) 更新 ---> ElConfigProvider 语言包更新 ---> ElementPlus 组件内置文本更新
    |
    +---> localStorage 持久化 ----> 下次访问自动恢复语言
    |
    +---> Accept-Language 请求头 --> 后端返回对应语言的数据
```
