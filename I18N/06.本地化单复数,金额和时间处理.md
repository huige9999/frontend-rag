# 本地化单复数、金额和时间处理

本节课通过 `intl-demo` 项目，演示如何在 Vue 3 + vue-i18n 中利用 **Intl API** 处理本地化中的三大核心场景：**单复数**、**金额格式化**和**时间格式化**。

## 一、环境变量与默认语言

文件：`.env`

```env
VITE_BASE_URL=/
VITE_DEFAULT_LOCALE=zh-CN
VITE_FALLBACK_LOCALE=en-US
```

**说明：**

- `VITE_DEFAULT_LOCALE`：设置默认语言为 `zh-CN`（简体中文），项目启动时 i18n 会使用该语言。
- `VITE_FALLBACK_LOCALE`：设置回退语言为 `en-US`，当某个翻译 key 在当前语言中找不到时，会自动使用英文翻译。
- 这些值通过 `import.meta.env` 在运行时读取。

## 二、Vite 配置集成 vue-i18n 插件

文件：`vite.config.ts`

```ts
import { fileURLToPath, URL } from 'node:url'
import { resolve } from 'node:path'
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import VueI18nPlugin from '@intlify/unplugin-vue-i18n/vite'

export default defineConfig({
  plugins: [
    vue(),
    VueI18nPlugin({
      include: resolve(__dirname, './src/i18n/locales/**'),
      runtimeOnly: false,
      strictMessage: false
    }),
  ],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})
```

**说明：**

- `VueI18nPlugin` 是 `@intlify/unplugin-vue-i18n` 提供的 Vite 插件，用于在编译时自动加载 JSON 语言包文件。
- `include`：指定语言包的 glob 路径，插件会自动扫描 `src/i18n/locales/` 下的所有 JSON 文件并合并为 messages 对象。
- `runtimeOnly: false`：包含完整的 vue-i18n 运行时，支持编译时和运行时的消息处理。
- `strictMessage: false`：关闭严格消息检查，允许使用不包含命名参数的插值表达式。
- 在代码中通过 `import messages from "@intlify/unplugin-vue-i18n/messages"` 即可获取所有语言包的聚合对象，无需手动逐个 import。

## 三、i18n 实例配置

文件：`src/i18n/index.ts`

```ts
import { createI18n } from 'vue-i18n'
import messages from "@intlify/unplugin-vue-i18n/messages";
import numberFormats from './rules/number'
import datetimeFormats from './rules/datetime'

export default createI18n({
  locale: import.meta.env.VITE_DEFAULT_LOCALE,
  fallbackFormat: import.meta.env.VITE_FALLBACK_LOCALE,
  legacy: false,
  messages,
  globalInjection: true,
  numberFormats,
  datetimeFormats
})
```

**说明：**

- `locale`：从环境变量读取默认语言（`zh-CN`）。
- `fallbackFormat`：从环境变量读取回退语言（`en-US`）。
- `legacy: false`：使用 Composition API 模式，支持 `useI18n()` 组合式函数。
- `messages`：由 `@intlify/unplugin-vue-i18n` 插件自动聚合的语言包，包含 `en-US` 和 `zh-CN` 的所有翻译。
- `globalInjection: true`：全局注入 `$t`、`$n`、`$d` 等方法到 Vue 组件中，可在模板中直接使用。
- **`numberFormats`**：自定义的数字格式化规则，用于 `$n()` 方法进行金额等数字的本地化格式化。
- **`datetimeFormats`**：自定义的日期时间格式化规则，用于 `$d()` 方法进行日期时间的本地化格式化。

## 四、单复数处理

### 4.1 语言包中的复数定义

单复数是国际化中一个非常容易被忽视的问题。不同语言有不同的复数规则：英语有单数和复数两种形式，阿拉伯语有六种，而中文基本不区分单复数。

vue-i18n 使用管道符 `|` 分隔不同的复数形式，通过传入一个数字来自动选择匹配的文本。

**英文语言包** (`src/i18n/locales/en-US.json`)：

```json
{
  "home": {
    "num_visits": "This page hasn't been visited :( | This page has been visited {n} time | This page has been visited {n} times"
  }
}
```

**中文语言包** (`src/i18n/locales/zh-CN.json`)：

```json
{
  "home": {
    "num_visits": "该网站还没有人浏览:( | 该网站首次被浏览了 | 该网站已经被浏览了{n}次"
  }
}
```

**说明：**

- vue-i18n 使用管道符 `|` 来分隔不同复数形式的文本。
- 对于英语，有三种形式：
  - 第1段（n=0）：`This page hasn't been visited :(`
  - 第2段（n=1）：`This page has been visited {n} time`（单数用 `time`）
  - 第3段（n>1）：`This page has been visited {n} times`（复数用 `times`）
- 对于中文，同样定义了三种形式，但表述方式不同：
  - 第1段（n=0）：`该网站还没有人浏览:(`
  - 第2段（n=1）：`该网站首次被浏览了`
  - 第3段（n>1）：`该网站已经被浏览了{n}次`
- `{n}` 是占位符，会被替换为实际传入的数字。
- vue-i18n 底层利用 `Intl.PluralRules` API 来判断当前语言的复数规则，自动选择正确的文本段。

### 4.2 在模板中使用复数

文件：`src/views/HomeView.vue`

```vue
<template>
  <main>
    <p>{{ $t("home.num_visits", 30) }}</p>
  </main>
</template>
```

**说明：**

- `$t("home.num_visits", 30)` 的第二个参数 `30` 是一个数字。
- vue-i18n 根据这个数字和当前 locale 的复数规则，自动选择对应的文本段。
- 当前 locale 为 `en-US` 时，30 对应复数（other），输出：`This page has been visited 30 times`。
- 当前 locale 为 `zh-CN` 时，30 对应第三段，输出：`该网站已经被浏览了30次`。
- 如果传入 `0`（英文）：`This page hasn't been visited :(`。
- 如果传入 `1`（英文）：`This page has been visited 1 time`。

### 4.3 Intl.PluralRules 原理

vue-i18n 的复数处理底层依赖 JavaScript 内置的 `Intl.PluralRules` API。它根据不同的 locale 和数值返回对应的复数类别：

```javascript
const rule = new Intl.PluralRules('en-US');
rule.select(0);   // "other" → 选择第1段
rule.select(1);   // "one"   → 选择第2段
rule.select(2);   // "other" → 选择第3段
rule.select(30);  // "other" → 选择第3段
```

不同语言的复数规则差异很大：

| 语言 | 0 | 1 | 2 | 3-10 | 11-99 | 100+ |
|------|---|---|---|------|-------|------|
| 英语 | other | one | other | other | other | other |
| 中文 | other | one | other | other | other | other |
| 阿拉伯语 | zero | one | two | few | many | other |
| 俄语 | many | one | few | few | many | many |

## 五、金额（数字）格式化

### 5.1 数字格式化规则定义

文件：`src/i18n/rules/number.ts`

```ts
export default {
  "en-US": {
    currencyFormat: {
      style: "currency",
      currency: "USD"
    }
  },
  "zh-CN": {
    currencyFormat: {
      style: "currency",
      currency: "CNY"
    }
  }
}
```

**说明：**

- 该文件导出了一个对象，键名为 locale，值为该 locale 下的数字格式化规则集合。
- `currencyFormat` 是自定义的格式名称，可以在模板中通过 `$n(number, "currencyFormat")` 引用。
- `style: "currency"`：指定格式化类型为货币格式。
- `currency: "USD"`（en-US）：使用美元符号 `$`。
- `currency: "CNY"`（zh-CN）：使用人民币符号 `¥`。
- 这些选项与 `Intl.NumberFormat` 的 `options` 参数完全对应，vue-i18n 底层就是调用 `Intl.NumberFormat` 来实现格式化。

### 5.2 在模板中使用金额格式化

文件：`src/views/AboutView.vue`

```vue
<template>
  <div>
    <h1>{{ $t("about.header") }}</h1>
    <p>{{ $t("about.donations", {donations: $n(1234.56, "currencyFormat")}) }}</p>
  </div>
</template>
```

**说明：**

- `$n(1234.56, "currencyFormat")`：调用 vue-i18n 的数字格式化方法，第一个参数是要格式化的数字，第二个参数是在 `numberFormats` 中定义的格式名称。
- 当 locale 为 `en-US` 时，`$n(1234.56, "currencyFormat")` 输出 `$1,234.56`。
- 当 locale 为 `zh-CN` 时，`$n(1234.56, "currencyFormat")` 输出 `¥1,234.56`。
- 然后将格式化后的字符串作为 `donations` 参数传入 `$t()` 进行翻译插值：
  - 英文：`Donations raised:$1,234.56`
  - 中文：`收到贡献:¥1,234.56`

### 5.3 Intl.NumberFormat 原理

vue-i18n 的 `$n()` 方法底层调用 `Intl.NumberFormat`：

```javascript
// 等价于 $n(1234.56, "currencyFormat") 在 en-US 环境下
new Intl.NumberFormat('en-US', {
  style: 'currency',
  currency: 'USD'
}).format(1234.56);
// 输出: "$1,234.56"

// 等价于 $n(1234.56, "currencyFormat") 在 zh-CN 环境下
new Intl.NumberFormat('zh-CN', {
  style: 'currency',
  currency: 'CNY'
}).format(1234.56);
// 输出: "¥1,234.56"
```

不同地区的数字表示差异包括：货币符号、千位分隔符、小数点符号等。

## 六、时间（日期）格式化

### 6.1 日期格式化规则定义

文件：`src/i18n/rules/datetime.ts`

```ts
interface DatetimeFormats {
  [locale: string]: {
    [formatName: string]: Intl.DateTimeFormatOptions
  }
}

const datetimeFormats: DatetimeFormats = {};

['en-US', 'zh-CN', 'ja-JP'].forEach(locale => {
  datetimeFormats[locale] = {
    shortFormat: {
      year: 'numeric', month: 'short', day: 'numeric'
    },
    longFormat: {
      year: 'numeric', month: 'short', day: 'numeric',
      weekday: 'short', hour: 'numeric', minute: 'numeric', second: 'numeric'
    }
  }
})

export default datetimeFormats;
```

**说明：**

- 定义了 `DatetimeFormats` 接口，使用索引签名确保类型安全，值的类型为 `Intl.DateTimeFormatOptions`（TypeScript 内置类型）。
- 使用 `forEach` 循环为 `en-US`、`zh-CN`、`ja-JP` 三个 locale 统一生成格式化规则，避免重复代码。
- `shortFormat`：短格式，只包含年月日。例如：
  - `en-US`：`Jan 1, 2024`
  - `zh-CN`：`2024年1月`
  - `ja-JP`：`2024年1月`
- `longFormat`：长格式，包含年月日 + 星期 + 时分秒。例如：
  - `en-US`：`Mon, Jan 1, 2024, 12:30:00 PM`
  - `zh-CN`：`2024年1月周一 下午12:30:00`
  - `ja-JP`：`2024年1月月曜日 12:30:00`
- 格式化选项说明：
  - `year: 'numeric'`：年份以数字形式显示（如 2024）
  - `month: 'short'`：月份以缩写形式显示（如 Jan、1月）
  - `day: 'numeric'`：日期以数字形式显示（如 1）
  - `weekday: 'short'`：星期以缩写形式显示（如 Mon、周一）
  - `hour/minute/second: 'numeric'`：时/分/秒以数字形式显示

### 6.2 在模板中使用日期格式化

文件：`src/views/HomeView.vue`

```vue
<template>
  <main>
    <p>{{ $d(new Date(), "longFormat") }}</p>
  </main>
</template>
```

**说明：**

- `$d(new Date(), "longFormat")`：调用 vue-i18n 的日期格式化方法，第一个参数是 `Date` 对象，第二个参数是在 `datetimeFormats` 中定义的格式名称。
- 当 locale 为 `zh-CN` 时，输出类似：`2024年6月周五 下午1:30:00`。
- 当 locale 为 `en-US` 时，输出类似：`Fri, Jun 7, 2024, 1:30:00 PM`。
- 当 locale 为 `ja-JP` 时，输出类似：`2024年6月金曜日 13:30:00`。
- 同一个日期对象，在不同 locale 下自动呈现该地区习惯的日期格式。

### 6.3 Intl.DateTimeFormat 原理

vue-i18n 的 `$d()` 方法底层调用 `Intl.DateTimeFormat`：

```javascript
// 等价于 $d(new Date(), "longFormat") 在 zh-CN 环境下
new Intl.DateTimeFormat('zh-CN', {
  year: 'numeric',
  month: 'short',
  day: 'numeric',
  weekday: 'short',
  hour: 'numeric',
  minute: 'numeric',
  second: 'numeric'
}).format(new Date());
```

## 八、应用入口与依赖安装

文件：`src/main.ts`

```ts
import { createApp } from 'vue'
import App from './App.vue'
import router from './routes'
import i18n from './i18n'

const app = createApp(App);
app.use(i18n);
app.use(router);
app.mount('#app')
```

**说明：**

- 通过 `app.use(i18n)` 将 i18n 实例注册到 Vue 应用中，使所有组件都能使用 `$t`、`$n`、`$d` 等方法。
- 注册顺序为先 i18n 后 router。

文件：`package.json` 关键依赖：

```json
{
  "dependencies": {
    "vue": "^3.3.4",
    "vue-i18n": "^9.3.0-beta.24",
    "vue-router": "^4.2.4"
  },
  "devDependencies": {
    "@intlify/unplugin-vue-i18n": "^0.12.2"
  }
}
```

**说明：**

- `vue-i18n` v9 是专为 Vue 3 设计的国际化插件，支持 Composition API。
- `@intlify/unplugin-vue-i18n` 是编译时插件，用于自动加载和优化语言包。

## 九、总结

本节课涉及的核心 API 对照表：

| vue-i18n 方法 | 底层 Intl API | 用途 | 在项目中的使用 |
|---------------|--------------|------|---------------|
| `$t(key, number)` | `Intl.PluralRules` | 单复数处理 | `HomeView.vue` 中页面访问次数 |
| `$n(number, format)` | `Intl.NumberFormat` | 数字/金额格式化 | `AboutView.vue` 中捐赠金额 |
| `$d(date, format)` | `Intl.DateTimeFormat` | 日期时间格式化 | `HomeView.vue` 中当前日期 |

**关键配置流程回顾：**

1. 在 `rules/number.ts` 和 `rules/datetime.ts` 中定义各 locale 的格式化规则。
2. 在 `i18n/index.ts` 中将 `numberFormats` 和 `datetimeFormats` 传入 `createI18n`。
3. 在组件模板中使用 `$n()` 和 `$d()` 调用对应的格式化规则。
4. 在语言包 JSON 中用管道符 `|` 定义复数形式，在模板中通过 `$t(key, number)` 触发复数选择。
