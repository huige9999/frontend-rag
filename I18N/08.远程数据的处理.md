# 08. 远程数据的处理

本节课讲解如何在国际化项目中处理远程数据。核心思路是：**前端通过请求头 `Accept-Language` 将当前语言传递给后端，后端根据语言标识从数据库中查询对应语言的数据并返回**，从而实现远程数据的国际化。

---

## 一、整体架构概览

```
┌─────────────────────┐       GET /post/list        ┌─────────────────────┐
│     intl-demo       │ ──────────────────────────► │     intl-server     │
│   (Vue3 前端)       │  Header: Accept-Language     │   (Express 后端)    │
│                     │                              │                     │
│  axios 发送请求      │ ◄────────────────────────── │  根据 locale 查询    │
│  展示多语言文章      │   返回对应语言的文章数据       │  MongoDB 数据库      │
└─────────────────────┘                              └─────────────────────┘
```

- **前端**：Vue3 + vue-i18n + axios，在切换语言时将语言标识写入 axios 默认请求头
- **后端**：Express + Mongoose，读取请求头中的语言标识，按语言过滤数据库数据
- **数据库**：MongoDB，每条数据都有一个 `locale` 字段标识所属语言

#### 3.2.3 翻译管理模块 (i18n/translation.ts)

这是整个国际化项目的核心模块，负责语言切换、路由集成和**远程请求的语言传递**：

```typescript
import { NavigationGuardNext, RouteLocationNormalized } from "vue-router";
import { LANGUAGES } from "./constants";
import i18n from "@/i18n";
import axios from "axios";

const Translation = {
  // 获取支持的语言列表
  get supportedLocales() {
    return LANGUAGES;
  },

  // 获取 / 设置当前语言
  get currentLocale() {
    return i18n.global.locale.value
  },
  set currentLocale(newLocale: string) {
    i18n.global.locale.value = newLocale;
  },

  // 获取环境变量中的默认语言
  get defaultLocale() {
    return import.meta.env.VITE_DEFAULT_LOCALE;
  },

  // 判断语言是否被支持
  isLocaleSupported(locale: string) {
    return locale in Translation.supportedLocales;
  },

  // 获取浏览器默认语言
  getUserLocale() {
    let locale = navigator.language ? navigator.language : Translation.defaultLocale;
    return {
      locale,              // zh-CN, en-US
      localeNoRegion: locale.split('-')[0]  // zh, en
    }
  },

  // 获取本地持久化的语言
  getPersistedLocale() {
    let persistedLocale = localStorage.getItem("user-locale") || "";
    if (Translation.isLocaleSupported(persistedLocale)) {
      return persistedLocale;
    } else {
      return null;
    }
  },

  // 猜测默认语言（优先级：本地存储 > 浏览器语言 > 环境变量默认值）
  guessDefaultLocale() {
    const userPersistedLocale = Translation.getPersistedLocale();
    if (userPersistedLocale) {
      return userPersistedLocale;
    }
    const userPreferredLocale = Translation.getUserLocale();
    if (Translation.isLocaleSupported(userPreferredLocale.locale)) {
      return userPreferredLocale.locale;
    }
    if (Translation.isLocaleSupported(userPreferredLocale.localeNoRegion)) {
      return userPreferredLocale.localeNoRegion;
    }
    return Translation.defaultLocale;
  },

  // 路由守卫：确保路由参数包含语言标识
  routerMiddleware(to: RouteLocationNormalized, _from: RouteLocationNormalized, next: NavigationGuardNext) {
    const paramLocale = to.params.locale as string;
    if (!Translation.isLocaleSupported(paramLocale)) {
      const defaultLocale = Translation.guessDefaultLocale();
      return next(defaultLocale);
    }
    Translation.switchLanguage(paramLocale);
    return next();
  },

  // 生成带语言参数的路由对象
  i18nRoute(to: any) {
    return {
      ...to,
      params: { locale: Translation.currentLocale },
      ...to.params
    }
  },

  // ★ 核心：切换语言（包括设置远程请求头）
  switchLanguage(newLocale: string) {
    Translation.currentLocale = newLocale;
    // 将 html 标签的 lang 属性设置为新的语言
    document.documentElement.lang = newLocale;
    // 将当前语言保存到 localStorage 中
    localStorage.setItem("user-locale", newLocale);
    // ★ 关键：在 axios 默认请求头中加上 Accept-Language
    axios.defaults.headers.common["Accept-Language"] = newLocale;
  }
}

export default Translation
```

**重点说明 —— `switchLanguage` 方法中的远程数据国际化关键：**

```typescript
axios.defaults.headers.common["Accept-Language"] = newLocale;
```

这一行代码的作用是将当前语言标识设置到 axios 的**全局默认请求头**中。之后前端所有通过 axios 发出的请求，都会自动携带 `Accept-Language` 请求头，后端即可据此判断应返回哪种语言的数据。

这是远程数据国际化的**核心桥梁**——前端告诉后端"我需要什么语言的数据"。

#### 3.6.2 语言切换组件 (components/LanguageSwitcher.vue)

```vue
<template>
  <div>
    <select v-model="locale" @change="handleChange">
      <option v-for="(value,key) in Trans.supportedLocales" :key="key" :value="key">
        {{ value }}
      </option>
    </select>
  </div>
</template>

<script setup lang="ts">
import { useI18n } from "vue-i18n";
import { useRouter } from "vue-router";
import Trans from "@/i18n/translation";

const router = useRouter();
const { locale } = useI18n();

const handleChange = async (e: Event) => {
  const value = (e.target as HTMLSelectElement).value;
  Trans.switchLanguage(value);

  try {
    await router.replace({ params: { locale: value } });
    router.go(0);
  } catch (e) {
    console.log(e);
    router.push("/")
  }
}
</script>
```

**说明：**
- 切换语言时调用 `Trans.switchLanguage(value)`，该方法会：
  1. 更新 vue-i18n 的当前语言
  2. 更新 `<html>` 标签的 `lang` 属性
  3. 将语言持久化到 `localStorage`
  4. **设置 axios 全局请求头 `Accept-Language`**（关键步骤）
- 切换后刷新页面 `router.go(0)`，确保远程数据重新请求并以新语言展示

## 五、后端核心代码

### 5.1 路由层 —— 读取语言标识 (routes/index.js)

```javascript
var express = require('express');
var router = express.Router();
var postDao = require('../dao/postDao')

router.get('/list', async function (req, res, next) {
  let { page, limit } = req.query;
  let locale = req.headers['accept-language'];

  console.log(locale)

  // 规范化语言标识
  if (locale.includes('zh-CN')) {
    locale = 'zh-CN';
  }
  else if (locale.includes('en-US')) {
    locale = 'en-US'
  }

  // ★ 关键：根据语言标识构建过滤条件
  const filter = { 'locale': locale };

  let result = await postDao.list(filter, page, limit);

  res.json(result)
});

module.exports = router;
```

**重点说明 —— 远程数据国际化的后端处理：**

1. **读取请求头**：`req.headers['accept-language']` 获取前端设置的语言标识
2. **规范化语言标识**：将语言标识统一为 `zh-CN` 或 `en-US`
3. **构建查询过滤条件**：`const filter = { 'locale': locale }`，按语言字段过滤数据
4. **返回对应语言的数据**：调用 DAO 层查询数据库，只返回匹配当前语言的文章

这就是后端处理远程数据国际化的**完整流程**——从请求头中读取语言，据此过滤数据库查询。

### 5.2 数据模型 —— 多语言数据存储 (models/PostModel.js)

```javascript
const mongoose = require("mongoose");
const Schema = mongoose.Schema;

const postSchema = new Schema({
  title: {
    type: String,
    require: true
  },
  content: {
    type: String,
    require: true,
  },
  author: {
    type: String,
    required: false,
  },
  locale: {
    type: String,
    required: false,
  }
}, { versionKey: false });

module.exports = mongoose.model("PostModel", postSchema);
```

**重点说明 —— 多语言数据存储方案：**

每条文章记录都包含一个 `locale` 字段，标识该条数据所属的语言。数据库中的数据结构大致如下：

```
// 中文文章
{ title: "Vue3入门教程", content: "本文介绍...", author: "张三", locale: "zh-CN" }

// 英文文章
{ title: "Vue3 Getting Started", content: "This article introduces...", author: "John", locale: "en-US" }
```

这种方案的特点是：
- **同一集合中存储所有语言的数据**，通过 `locale` 字段区分
- 查询时通过 `{ locale: "zh-CN" }` 过滤，只返回对应语言的数据
- 优点：实现简单，扩展新语言只需添加数据即可
- 适用场景：内容型数据（文章、商品描述等）的国际化

## 六、远程数据国际化的完整数据流

```
1. 用户在前端切换语言为 "zh-CN"
       │
       ▼
2. LanguageSwitcher 调用 Trans.switchLanguage("zh-CN")
       │
       ├── 更新 vue-i18n 的 locale
       ├── 更新 <html lang="zh-CN">
       ├── 存入 localStorage
       └── ★ 设置 axios.defaults.headers.common["Accept-Language"] = "zh-CN"
       │
       ▼
3. 用户访问文章页面，PostView 调用 getPostList()
       │
       ▼
4. axios 发送 GET 请求，自动携带 Header:
   Accept-Language: zh-CN
       │
       ▼
5. 后端 routes/index.js 接收请求
   从 req.headers['accept-language'] 读取 "zh-CN"
       │
       ▼
6. 构建过滤条件: { locale: "zh-CN" }
   调用 postDao.list({ locale: "zh-CN" }, 1, 10)
       │
       ▼
7. MongoDB 查询: PostModel.find({ locale: "zh-CN" })
   只返回中文文章数据
       │
       ▼
8. 后端将中文文章数据 JSON 返回给前端
       │
       ▼
9. PostView 渲染中文标题、内容、作者
```

## 七、总结

### 远程数据国际化的关键点

| 环节 | 关键操作 | 涉及文件 |
|------|---------|---------|
| 前端设置语言 | `axios.defaults.headers.common["Accept-Language"]` | `i18n/translation.ts` |
| 前端发送请求 | axios 自动携带 `Accept-Language` 请求头 | `apis/postApis.ts` |
| 后端读取语言 | `req.headers['accept-language']` | `routes/index.js` |
| 数据库存储 | 每条记录包含 `locale` 字段 | `models/PostModel.js` |
| 后端过滤查询 | `{ locale: "zh-CN" }` | `dao/postDao.js` |
| 模拟数据生成 | 根据语言变量生成对应语言的数据 | `mock/mockPost.js` |

### 静态文本 vs 远程数据的国际化分工

| 类型 | 国际化方式 | 示例 |
|------|-----------|------|
| 静态文本（导航栏、按钮、提示） | 前端翻译文件 (JSON) | `$t("nav.home")` |
| 动态远程数据（文章、商品） | 后端按语言返回不同数据 | `Accept-Language` + `locale` 过滤 |

两种方式配合使用，才能实现完整的国际化体验。
