# 🌐 国际化（i18n）思维导图知识梳理

> 本文档整理自课程配套的 `国际化.xmind` 思维导图，涵盖国际化的完整知识体系。

---

## 一、术语

| 术语 | 英文 | 说明 |
|------|------|------|
| **国际化** | Internationalization (i18n) | 可以快速地本地化；借助本地化的数据，相同的程序可以运行在世界各地 |
| **本地化** | Localization (l10n) | 货币、日期等信息的表现形式，适应于用户所在的地区和语言；目的是让不同地区的用户都能感受到这个网站像本地网站一样亲切、易用 |
| **全球化** | Globalization (g11n) | 国际化 + 本地化的统称 |

> 💡 **核心本质**：国际化需求的本质是在保证本地化质量的前提下，尽可能的提升效率。

## 二、特征 — 需要本地化的内容

### 2.1 语言相关
- **语言** — 文案的翻译与切换
- **文字** — 不同书写系统（汉字、拉丁、阿拉伯等）
- **音频** — 语音、配音的本地化
- **视频** — 字幕、配音、画面中的文案

### 2.2 视觉与标识
- **图片和颜色** — 不同文化对颜色的理解不同
- **名字和称谓** — 姓名、敬语的地区差异

### 2.3 编码与格式
- **相关编码** — 身份证、护照等地区性编号规则
- **电话号码** — 国际区号、号码格式差异
- **地址和邮政区号** — 各国地址格式不同
- **货币** — 符号、位置、精度
- **度量单位** — 公制 vs 英制
- **相关标准** — 纸张大小（A4 vs Letter）等

## 三、文化因素

| 类别 | 说明 |
|------|------|
| **时间日期格式** | `YYYY/MM/DD`、`MM/DD/YYYY`、`DD.MM.YYYY` 等 |
| **时区** | UTC、GMT、各地区标准时间 |
| **数字格式** | 千分位分隔符（`,` vs `.`）、小数点表示 |
| **文字方向** | LTR（左到右）、RTL（右到左） |
| **习惯** | 用户使用习惯、交互偏好 |
| **内容** | 内容的本地化策略 |
| **运营方式** | 不同地区的运营策略差异 |
| **法律法规** | 数据隐私、Cookie 政策等法律合规 |
| **范围** | 本地化的覆盖范围决策 |

## 五、前置知识

- **字符编码** — ASCII → Unicode / UTF-8 的演进
- **区域设置（Locale）** — 语言编码（ISO 639-1）+ 地区编码（ISO 3166），如 `zh-CN`、`en-US`
- **新的 JS 国际化 API** — `Intl.PluralRules`、`Intl.DateTimeFormat`、`Intl.NumberFormat` 等

## 六、技术方案

### 6.1 前端方案

#### 语言来源判断
- **IP 地址** — 根据用户 IP 推断所在地区
- **程序默认配置** — 应用预设的默认语言
- **缓存** — 之前访问产生的 Cookie 或 LocalStorage 等

#### 核心能力
- 针对来源判断用户区域
- 用户切换地区 / 语言 / 货币 / 数字表示的功能

> ⚠️ **原则**：本地化优先于技术方案

#### i18n 工具库
- **vue-i18n** — Vue.js 生态
- **react-intl** — React 生态
- **i18next** — 框架无关
- 其他 i18n 相关工具库

### 6.2 后端方案
- **后端模板** — 服务端渲染时注入多语言内容
- **映射表** — 不同区域读取不同的内容

### 6.3 翻译方式
- **人工翻译** — 专业翻译团队
- **工具辅助** — CAT 工具、翻译记忆库
- **翻译平台** — Crowdin、Phrase、Transifex 等

## 七、数据处理 — 固定内容 vs 动态数据

### 7.1 固定内容
- 固定内容可以**响应式切换**
- 通常通过前端 i18n 库管理语言包（JSON 文件）

### 7.2 动态数据
- **同样结构不同语言内容的表** — 数据库中同一表包含多语言字段
- **接口添加标准地区语言配置** — 区分获取不同数据
- **后端数据库批量翻译** — 存入数据库
- **不同区域是不同的数据库** — 物理隔离方案
- **切换语言环境后，刷新页面重新请求**
- **后端直接返回所有语言环境数据，由前端控制渲染**

## 九、国际化处理

### 9.1 文案
- **文案内容改变** — 翻译后的文案与原文长度、语义差异
- **内容长度** — 不同语言的文本长度差异（德语通常比英语长 30%）

### 9.2 样式和外观
- **字体大小、间隔、对齐方式** — 不同文字系统的排版需求
- **方向** — LTR / RTL 布局适配
- **图片、颜色等** — 文化敏感的视觉元素

### 9.3 图片内容改变
- **重要位置和活动图片** — 首页 Banner、活动页面
- **常规图片与 icon**
  - 常规图片和 icon 的切换
  - 常规图片中显示的文案、活动内容、价格等

### 9.4 本地化格式
- 货币格式化
- 数字格式化
- 日期格式化

## 十、CSS 中的国际化

### 10.1 文案内容导致的 CSS 变化

| 解决方案 | 说明 |
|---------|------|
| 使用字体族 | 设置 `font-family` 适配多语言字体 |
| 宽度或高度由内容决定 | 使用 `fit-content` 或不固定尺寸 |
| 设置最小或最大宽度 | `min-width` / `max-width` 防止布局溢出 |
| 使用 `text-overflow` 属性 | `ellipsis` 截断过长文本 |
| 调整行高 | 不同语言的 `line-height` 需求不同 |
| CSS Grid | 弹性网格布局自适应内容 |

### 10.2 语言选择器
- `[lang=locale]` — 属性选择器
- `:lang(locale)` — 伪类选择器

### 10.3 文字方向 direction
设置文本、表格列和水平溢出的方向：
- `ltr` — 从左到右（Left-To-Right）
- `rtl` — 从右到左（Right-To-Left）

### 10.4 书写模式 writing-mode
块和行级内容的流动方向：
- `horizontal-tb` — 水平，从上到下
- `vertical-rl` — 垂直，从右到左
- `vertical-lr` — 垂直，从左到右
- `sideways-rl` — 侧向，从右到左
- `sideways-lr` — 侧向，从左到右

### 10.5 文本朝向 text-orientation
行中字符的方向（仅影响纵向模式）：
- `mixed` — 混合方向（默认）
- `upright` — 字符直立
- `sideways` — 字符侧放

### 10.6 竖行横书 text-combine-upright
- 竖排文字中的横向数字/字母组合

### 10.7 逻辑属性
- 用 `inline-size` / `block-size` 替代 `width` / `height`
- 用 `margin-inline-start` / `margin-block-end` 等替代物理方向属性
- 适应不同书写方向的布局

### 10.8 OpenType 字体特性指南
- `font-variant-east-asian` — 东亚字体变体（简繁切换等）
