# 07. 路由与默认语言的处理

本节课讲解如何在 Vue3 + vue-i18n 项目中实现**路由级别的国际化**，包括：通过 URL 路径参数标识当前语言、默认语言自动检测（浏览器语言 / 本地持久化）、路由守卫中间件、以及语言切换与路由参数的联动。

## 三、Vite 构建配置

### vite.config.ts

```typescript
import { fileURLToPath, URL } from 'node:url'
import { resolve } from 'node:path'
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import VueI18nPlugin from '@intlify/unplugin-vue-i18n/vite'

export default defineConfig({
  plugins: [
    vue(),
    VueI18nPlugin({
      include: resolve(__dirname, './src/i18n/locales/**'),
      runtimeOnly: false,
      strictMessage: false
    }),
  ],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})
```

**说明：**

- `VueI18nPlugin` 的 `include` 选项指定翻译 JSON 文件的路径，插件会自动读取并合并所有语言包。
- `runtimeOnly: false`：保留完整的 i18n 运行时，支持动态功能。
- `strictMessage: false`：关闭严格消息检查，允许更灵活的翻译格式（如多复数形式的 `|` 分隔）。
- `@` 路径别名方便在代码中使用 `@/i18n/...` 代替相对路径。

## 八、路由配置（核心）

### src/routes/index.ts

```typescript
import { createRouter, createWebHistory, RouterView } from 'vue-router'
import HomeView from '../views/HomeView.vue'
import Trans from "@/i18n/translation"

const router = createRouter({
  history: createWebHistory(import.meta.env.VITE_BASE_URL),
  routes: [
    {
      path: "/:locale?",
      component: RouterView,
      beforeEnter: Trans.routerMiddleware,
      children: [
        {
          path: '',
          name: 'Home',
          component: HomeView
        },
        {
          path: 'about',
          name: 'About',
          component: () => import('../views/AboutView.vue')
        }
      ]
    }
  ]
})
export default router;
```

**说明：**

- **`path: "/:locale?"`**：这是路由国际化的关键设计。使用动态路径参数 `:locale` 来标识当前语言，`?` 表示该参数可选。这意味着：
  - 访问 `/` 时，`locale` 参数为空，路由守卫会自动补上默认语言并跳转到 `/en-US`。
  - 访问 `/zh-CN/about` 时，`locale` 为 `zh-CN`，显示中文的关于页面。
  - 访问 `/en-US` 时，`locale` 为 `en-US`，显示英文首页。

- **`beforeEnter: Trans.routerMiddleware`**：路由独享守卫，每次进入该路由时都会执行 `Trans.routerMiddleware`，负责：
  1. 检查 URL 中的 `locale` 参数是否是支持的语言。
  2. 如果不支持，自动推算默认语言并重定向。
  3. 如果支持，调用 `switchLanguage` 切换 i18n 的当前语言。

- **`component: RouterView`**：父路由使用 `RouterView` 作为占位组件，子路由的内容通过它渲染。

- **子路由**：`path: ''` 表示根路径（如 `/en-US`），`path: 'about'` 表示 `/en-US/about`。

## 九、Translation 模块（核心逻辑）

### src/i18n/translation.ts

这是本节课最核心的模块，封装了所有与语言检测、路由守卫、语言切换相关的逻辑。

```typescript
import { NavigationGuardNext, RouteLocationNormalized } from "vue-router";
import { LANGUAGES } from "./constants";
import i18n from "@/i18n";

const Translation = {
  get supportedLocales() {
    return LANGUAGES;
  },

  get currentLocale() {
    return i18n.global.locale.value
  },

  set currentLocale(newLocale: string) {
    i18n.global.locale.value = newLocale;
  },

  get defaultLocale() {
    return import.meta.env.VITE_DEFAULT_LOCALE;
  },

  isLocaleSupported(locale: string) {
    return locale in Translation.supportedLocales;
  },

  getUserLocale() {
    let locale = navigator.language ? navigator.language : Translation.defaultLocale;
    return {
      locale,
      localeNoRegion: locale.split('-')[0]
    }
  },

  getPersistedLocale() {
    let persistedLocale = localStorage.getItem("user-locale") || "";

    if (Translation.isLocaleSupported(persistedLocale)) {
      return persistedLocale;
    }
    else {
      return null;
    }
  },

  guessDefaultLocale() {
    const userPersistedLocale = Translation.getPersistedLocale();

    if (userPersistedLocale) {
      return userPersistedLocale;
    }

    const userPreferredLocale = Translation.getUserLocale();

    if (Translation.isLocaleSupported(userPreferredLocale.locale)) {
      return userPreferredLocale.locale;
    }

    if (Translation.isLocaleSupported(userPreferredLocale.localeNoRegion)) {
      return userPreferredLocale.localeNoRegion;
    }

    return Translation.defaultLocale;
  },

  routerMiddleware(to: RouteLocationNormalized, from: RouteLocationNormalized, next: NavigationGuardNext) {
    const paramLocale = to.params.locale as string;

    if (!Translation.isLocaleSupported(paramLocale)) {
      const defaultLocale = Translation.guessDefaultLocale();
      return next(defaultLocale);
    }

    Translation.switchLanguage(paramLocale);

    return next();
  },

  i18nRoute(to: any) {
    return {
      ...to,
      params: { locale: Translation.currentLocale },
      ...to.params
    }
  },

  switchLanguage(newLocale: string) {
    Translation.currentLocale = newLocale;
    document.documentElement.lang = newLocale;
    localStorage.setItem("user-locale", newLocale);
  }
}

export default Translation
```

### 逐一详解

#### 1. `supportedLocales` — 支持的语言列表

```typescript
get supportedLocales() {
  return LANGUAGES;
}
```

通过 getter 代理 `LANGUAGES` 常量，提供统一的访问入口。

#### 2. `currentLocale` — 当前语言的读写

```typescript
get currentLocale() {
  return i18n.global.locale.value
}

set currentLocale(newLocale: string) {
  i18n.global.locale.value = newLocale;
}
```

使用 getter/setter 封装对 vue-i18n 全局 `locale` 的读写操作，其他方法统一通过此属性操作语言状态。

#### 3. `defaultLocale` — 环境变量中的默认语言

```typescript
get defaultLocale() {
  return import.meta.env.VITE_DEFAULT_LOCALE;
}
```

从 `.env` 文件中读取 `VITE_DEFAULT_LOCALE` 的值（`en-US`）。

#### 4. `isLocaleSupported()` — 判断语言是否被支持

```typescript
isLocaleSupported(locale: string) {
  return locale in Translation.supportedLocales;
}
```

使用 `in` 操作符检查给定的语言标签是否存在于 `LANGUAGES` 对象的 key 中。例如 `isLocaleSupported("zh-CN")` 返回 `true`，`isLocaleSupported("fr-FR")` 返回 `false`。

#### 5. `getUserLocale()` — 获取浏览器语言偏好

```typescript
getUserLocale() {
  let locale = navigator.language ? navigator.language : Translation.defaultLocale;
  return {
    locale,                    // 完整的语言标签，如 "zh-CN"、"en-US"
    localeNoRegion: locale.split('-')[0]  // 不带地区的语言代码，如 "zh"、"en"
  }
}
```

- `navigator.language` 返回浏览器的首选语言标签（如 `"zh-CN"`）。
- `localeNoRegion` 去掉地区后缀，用于模糊匹配。例如用户浏览器语言是 `"zh-TW"`，虽然不被直接支持，但去掉地区后 `"zh"` 可以用于进一步匹配。

#### 6. `getPersistedLocale()` — 获取本地持久化的语言

```typescript
getPersistedLocale() {
  let persistedLocale = localStorage.getItem("user-locale") || "";

  if (Translation.isLocaleSupported(persistedLocale)) {
    return persistedLocale;
  }
  else {
    return null;
  }
}
```

从 `localStorage` 读取用户之前手动选择过的语言。如果该语言仍在支持列表中则返回，否则返回 `null`。这样做是为了防止之前支持的语言后来被移除导致出错。

#### 7. `guessDefaultLocale()` — 智能推测默认语言（核心）

```typescript
guessDefaultLocale() {
  // 优先级1：本地持久化的语言
  const userPersistedLocale = Translation.getPersistedLocale();
  if (userPersistedLocale) {
    return userPersistedLocale;
  }

  // 优先级2：浏览器语言的完整标签
  const userPreferredLocale = Translation.getUserLocale();
  if (Translation.isLocaleSupported(userPreferredLocale.locale)) {
    return userPreferredLocale.locale;
  }

  // 优先级3：浏览器语言的不带地区标签
  if (Translation.isLocaleSupported(userPreferredLocale.localeNoRegion)) {
    return userPreferredLocale.localeNoRegion;
  }

  // 优先级4：环境变量中的默认语言
  return Translation.defaultLocale;
}
```

**优先级从高到低：**

1. **localStorage 持久化** — 用户之前手动切换过的语言，记忆下来。
2. **浏览器语言完整匹配** — 如浏览器是 `zh-CN`，项目也支持 `zh-CN`，直接使用。
3. **浏览器语言模糊匹配** — 如浏览器是 `zh-TW`，去掉地区后匹配到 `zh`（前提是 `zh` 也在支持列表中）。
4. **环境变量兜底** — 以上都不匹配，使用 `.env` 中配置的 `en-US`。

#### 8. `routerMiddleware()` — 路由守卫中间件（核心）

```typescript
routerMiddleware(to: RouteLocationNormalized, from: RouteLocationNormalized, next: NavigationGuardNext) {
  const paramLocale = to.params.locale as string;

  if (!Translation.isLocaleSupported(paramLocale)) {
    const defaultLocale = Translation.guessDefaultLocale();
    return next(defaultLocale);
  }

  Translation.switchLanguage(paramLocale);

  return next();
}
```

**执行流程：**

1. 从目标路由 `to.params.locale` 中取出语言参数。
2. 如果语言参数为空或不被支持（如用户直接访问 `/` 或输入了错误的语言 `/fr-FR/about`），则调用 `guessDefaultLocale()` 推测合适的语言，并通过 `next(defaultLocale)` 重定向（例如跳转到 `/en-US`）。
3. 如果语言参数合法，调用 `switchLanguage()` 切换当前语言，然后 `next()` 放行。

#### 9. `i18nRoute()` — 为路由链接注入语言参数

```typescript
i18nRoute(to: any) {
  return {
    ...to,
    params: { locale: Translation.currentLocale },
    ...to.params
  }
}
```

**说明：**

- 此方法用于 `<router-link>` 的 `:to` 属性，自动在路由参数中注入当前的 `locale`。
- 先展开 `params: { locale: Translation.currentLocale }`，再用 `...to.params` 覆盖。这样如果 `to` 中没有指定 `locale`，就使用当前语言；如果指定了，则优先使用指定的值。
- 例如当前语言是 `zh-CN`，`Trans.i18nRoute({name:'About'})` 会返回 `{name:'About', params: {locale: 'zh-CN'}}`，最终渲染为 `/zh-CN/about`。

#### 10. `switchLanguage()` — 切换语言

```typescript
switchLanguage(newLocale: string) {
  Translation.currentLocale = newLocale;           // 更新 vue-i18n 的 locale
  document.documentElement.lang = newLocale;        // 更新 <html> 标签的 lang 属性
  localStorage.setItem("user-locale", newLocale);   // 持久化到 localStorage
}
```

同时做三件事：
1. 切换 vue-i18n 的当前语言，所有 `$t()` 翻译立即更新。
2. 设置 `<html lang="...">`，对 SEO 和浏览器辅助功能（如屏幕阅读器）非常重要。
3. 将语言偏好保存到 `localStorage`，下次访问时自动恢复。

## 十三、语言切换组件

### src/components/LanguageSwitcher.vue

```vue
<template>
  <div>
    <select v-model="locale" @change="handleChange">
      <option v-for="(value, key) in Trans.supportedLocales" :key="key" :value="key">
        {{ value }}
      </option>
    </select>
  </div>
</template>

<script setup lang="ts">
import { useI18n } from "vue-i18n";
import { useRouter } from "vue-router";
import Trans from "@/i18n/translation";

const router = useRouter();

const { locale } = useI18n();

const handleChange = async (e: Event) => {
  const value = (e.target as HTMLSelectElement).value;
  Trans.switchLanguage(value);

  try {
    await router.replace({ params: { locale: value } });
  } catch (e) {
    console.log(e);
    router.push("/")
  }
}
</script>
```

**说明：**

- `<select>` 使用 `v-model="locale"` 绑定 vue-i18n 的当前语言，`v-for` 遍历 `Trans.supportedLocales` 生成选项。
- `handleChange` 是切换语言的核心逻辑：
  1. 从 `<select>` 获取用户选择的新语言值。
  2. 调用 `Trans.switchLanguage(value)` 切换 i18n 语言、更新 HTML lang 属性、持久化到 localStorage。
  3. 使用 `router.replace({ params: { locale: value } })` 更新 URL 中的语言参数（不产生新的历史记录）。
  4. 如果路由替换失败（如导航到同一路由会抛出异常），则回退到首页。

```vue
    <p>{{ $t("home.num_visits", 30) }}</p>
    <p>{{ $d(new Date(), "longFormat") }}</p>
  </main>
</template>
```

**说明：**

- `$t("home.content", {company: "DuYi-Edu"})`：使用插值变量 `{company}`，将公司名称动态插入翻译文本。
- `$t("home.num_visits", 30)`：根据数字 30 自动选择复数形式。英文中 30 对应第三条 "This page has been visited 30 times"；中文中 30 对应第三条 "该网站已经被浏览了30次"。
- `$d(new Date(), "longFormat")`：按照 `datetime.ts` 中定义的 `longFormat` 格式化当前日期。

## 十五、整体数据流总结

```
用户访问 URL
    |
    v
路由守卫 routerMiddleware
    |
    +-- URL 中有合法的 locale？--> switchLanguage(locale) --> 放行
    |
    +-- URL 中无 locale 或不合法？--> guessDefaultLocale()
                                          |
                                          +-- 优先 localStorage
                                          +-- 其次 navigator.language
                                          +-- 兜底 .env 配置
                                          |
                                          v
                                     重定向到 /{defaultLocale}
```

**语言切换流程：**

```
用户在下拉框选择新语言
    |
    v
LanguageSwitcher.handleChange()
    |
    +-- Trans.switchLanguage(newLocale)  --> 更新 i18n / html lang / localStorage
    |
    +-- router.replace({ params: { locale: newLocale } })  --> 更新 URL
```

**导航链接生成流程：**

```
<router-link :to="Trans.i18nRoute({name:'About'})">
    |
    v
i18nRoute() 注入当前 locale
    |
    v
生成 href="/zh-CN/about" 或 "/en-US/about"
```

## 十六、本节要点回顾

1. **路由参数化语言**：使用 `/:locale?` 动态路径参数在 URL 中标识语言，使每个 URL 都明确对应一种语言，有利于 SEO 和链接分享。

2. **路由守卫中间件**：通过 `beforeEnter` 守卫拦截每次路由进入，自动检测和修正语言参数，确保用户始终在正确的语言路径下。

3. **默认语言智能推测**：`guessDefaultLocale()` 按照"本地持久化 > 浏览器语言完整匹配 > 浏览器语言模糊匹配 > 环境变量兜底"的优先级链，为用户提供最合适的语言体验。

4. **语言切换与路由联动**：切换语言时同步更新 i18n locale、HTML lang 属性、localStorage 和 URL 路径，保持全局一致性。

5. **导航链接自动注入语言**：`i18nRoute()` 工具方法确保所有 `<router-link>` 生成的链接始终携带当前语言参数。
