# CSS 中的国际化

## 课程概述

本节课是前端国际化（i18n）系列课程的第三节，主要探讨 CSS 在国际化场景中的关键能力，包括文字方向控制、语言样式选择、开放字体特性（OpenType）、书写模式、语言转换引发的 CSS 问题，以及逻辑属性与逻辑值的使用。这些 CSS 特性是构建真正国际化前端页面的重要基础。

---

## 一、文字方向（direction）

### 1.1 主题说明

CSS 的 `direction` 属性用于控制文本的书写方向。世界上有些语言（如阿拉伯语、希伯来语）是从右到左（RTL）书写的，而大多数语言是从左到右（LTR）书写的。`direction` 属性与 `text-align` 配合使用时，需要注意它们的优先级关系。

同时，HTML 元素的 `dir` 属性可以在标签级别设置文字方向，`dir="auto"` 可以让浏览器根据内容自动判断方向。

> 对应代码文件：`代码/direction.html`

### 1.2 关键代码

```html
<html lang="en" dir="auto">
```

```css
/* 默认方向（LTR） */
div {
  border: 2px solid #ccc;
  margin: 15px;
  padding: 15px;
}

/* 设置从右到左方向 */
div {
  direction: rtl;
}

/* direction 与 text-align 的组合 */
div {
  text-align: left;
  direction: rtl;
}

div {
  text-align: right;
}
```

### 1.3 要点说明

- `direction: rtl` 会使文本从右侧开始排列，适用于阿拉伯语、希伯来语等 RTL 语言
- `text-align` 的优先级高于 `direction`：当同时设置 `text-align: left` 和 `direction: rtl` 时，文本会左对齐，但双向文本的排列顺序仍受 `direction` 影响
- `dir="auto"` 放在 `<html>` 标签上，可以让浏览器根据页面内容自动判断文字方向

## 二、语言样式（:lang 伪类）

### 2.1 主题说明

CSS 的 `:lang()` 伪类选择器可以根据元素的语言属性（`lang`）来应用不同的样式。这在多语言页面中非常有用，可以为不同语言的内容设置不同的字体、颜色、排版等样式。

> 对应代码文件：`代码/lang.html`

### 2.2 关键代码

```css
/* 英文内容的样式 */
p:lang(en) {
  color: blue;
  background: white;
}

/* 中文内容的样式 */
p:lang(zh) {
  color: white;
  background: red;
}
```

```html
<p lang="en">Some English text.</p>
<p lang="zh">这是中文字.</p>
```

### 2.3 要点说明

- `:lang()` 选择器匹配基于 `lang` 属性的值，无需在元素上添加 class
- 可以根据语言差异设置不同的字体族、字号、行高、颜色等，以适应各语言的排版需求
- 结合 HTML 的 `lang` 属性使用，语义清晰，便于维护

## 三、开放字体特性（OpenType / font-variant-east-asian）

### 3.1 主题说明

CSS 的 `font-variant-east-asian` 属性用于控制东亚文字（中文、日文、韩文）的字体变体，可以在简体中文和繁体中文之间切换显示。这利用了 OpenType 字体的特性，无需更换字体文件即可实现简繁转换的视觉效果。

> 对应代码文件：`代码/opentype.html`

### 3.2 关键代码

```css
/* 简体中文变体 */
.zhs {
  font-variant-east-asian: simplified;
}

/* 繁体中文变体 */
.zht {
  font-variant-east-asian: traditional;
}
```

```html
<p class="zh-content">
  一个简体字可能对应多个繁体字，如简体字「发」，其相应的繁体字可能为「發」或者「髮」。
  一个繁体汉字对应多个简体汉字的情况与前者相比数量较少，但是仍需注意。
  如繁体字「乾」可能对应简体字「干」或者「乾」
</p>

<button class="zhsBtn">简体</button>
<button class="zhtBtn">繁体</button>
```

```javascript
const zhsBtn = document.querySelector(".zhsBtn");
const zhtBtn = document.querySelector(".zhtBtn");
const zhContent = document.querySelector(".zh-content");

zhsBtn.addEventListener('click', () => {
  zhContent.classList.remove("zht");
  zhContent.classList.add("zhs");
});

zhtBtn.addEventListener('click', () => {
  zhContent.classList.remove("zhs");
  zhContent.classList.add("zht");
});
```

### 3.3 要点说明

- `font-variant-east-asian: simplified` 强制使用简体字形显示
- `font-variant-east-asian: traditional` 强制使用繁体字形显示
- 此功能依赖字体本身包含简繁两种字形（如思源宋体等 OpenType 字体），并非所有字体都支持
- 简繁转换的视觉效果由浏览器和字体共同决定，复杂的简繁对应关系（如"发"对应"發"和"髮"）需要上下文判断，`font-variant-east-asian` 仅做字形切换

### 3.4 简繁汉字的地区差异示例

代码中还包含了一张中国大陆与中国台湾省在计算机术语上的用词差异对照表：

| 中国大陆 | 中國臺灣省 |
|---------|-----------|
| 程序     | 程式       |
| 图标     | 圖示       |
| 变量     | 變數       |
| U盘      | 隨身碟     |
| 硬盘     | 硬碟       |
| 内存     | 記憶體     |
| 信息     | 資訊       |
| 设置     | 設定       |
| 比特     | 位元       |
| 屏幕     | 熒幕       |

这说明国际化不仅是文字翻译，还包括同一语言在不同地区的用词差异。

## 四、书写模式（writing-mode）

### 4.1 主题说明

CSS 的 `writing-mode` 属性控制文本的书写方向和排列方式。在中文、日文、韩文等语言中，传统的书写模式是从上到下、从右到左的竖排排版。`writing-mode` 使得网页能够支持这种排版方式。

与之配合的还有 `text-orientation`（控制字符方向）和 `text-combine-upright`（控制直立文本中的横向组合）。

> 对应代码文件：`代码/write-model.html`

### 4.2 关键代码

```css
:root {
  /* 设置竖排书写模式：从上到下，从右到左 */
  writing-mode: vertical-rl;

  /* 控制字符方向为正立（仅当 writing-mode 非 horizontal-tb 时有效） */
  text-orientation: upright;
}

.content {
  margin: 10px;
  padding: 8px;
  font-size: 20px;
  line-height: 2;
}

/* 将多个字符直立排列在一个字符宽度内（如英文缩写、数字） */
.abbr {
  text-combine-upright: all;
}
```

```html
<div class="content">
  <p>英雄联盟职业联赛（英语：League of Legends Pro League，简称：<span class="abbr">LPL</span>），
    是中国最高级别的英雄联盟职业比赛，
    同时也是中国赛区通往每年英雄联盟季中冠军赛和英雄联盟全球总决赛的唯一渠道。
    <span class="abbr">2013</span>年<span class="abbr">1</span>月<span class="abbr">29</span>日，
    第一届英雄联盟职业联赛（<span class="abbr">LPL</span>）开始季前赛...
  </p>
</div>
```

### 4.3 要点说明

- `writing-mode: vertical-rl` 设置竖排从右到左书写，`vertical-lr` 设置竖排从左到右书写，`horizontal-tb` 为默认水平从上到下
- `text-orientation: upright` 使所有字符正立显示（默认情况下拉丁字母会旋转 90 度）
- `text-combine-upright: all` 将连续的拉丁字符或数字压缩到一个中文字的宽度内，常用于竖排排版中的英文缩写和日期数字
- 竖排书写模式是中文、日文等 CJK 语言的传统排版方式，在国际化场景中需要考虑支持

## 五、语言转换出现的 CSS 问题

### 5.1 主题说明

在国际化实践中，语言切换会带来一系列 CSS 布局问题。由于不同语言的文本长度差异很大（如英文文本通常比中文长 1.5-2 倍），切换语言后可能出现文本溢出、布局错位、间距不合理等问题。

> 对应代码文件：`代码/语言转换出现的CSS问题.html`

### 5.2 关键代码

**文本溢出与省略号截断：**

```css
.long-text {
  width: 300px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
  margin: 20px;
}
```

```html
<div class="long-text">
  This is a long text that will be truncated with an ellipsis when it overflows the container.
</div>
<div class="long-text">
  这是一个长文本，溢出时将用省略号截断
</div>
```

**Flex 布局中的文本溢出：**

```css
.container1 {
  display: flex;
  justify-content: space-between;
  align-items: center;
  height: 200px;
  border: 1px solid #000;
}

.container1 div {
  max-width: 300px;
  border: 1px solid #000;
  padding: 30px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

**Grid 布局中的多语言内容：**

```css
.container2 {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  grid-auto-rows: minmax(50px, auto);
}

.container2 h1 {
  grid-column: 1 / span 12;
  font-size: 36px;
}

.container2 p {
  grid-column: 1 / span 12;
  font-size: 18px;
  line-height: 1.5;
}
```

**导航栏简繁切换导致布局变化：**

```css
.header {
  position: relative;
  font-size: 17px;
  margin: 30px;
}

.header span {
  position: relative;
  margin: 30px;
  border: 1px solid red;
}

/* 繁体模式下的不同间距 */
.header_tw {
  position: relative;
  font-size: 14px;
  margin: 30px;
}

.header_tw span {
  position: relative;
  margin: 15px;
  border: 1px solid blue;
}
```

```javascript
const btn = document.getElementById('btn');
const nav = document.getElementById('nav');

btn.addEventListener('click', (e) => {
  const val = e.target.innerText;
  if (val === '繁體') {
    e.target.innerText = '简体';
    nav.style.fontVariantEastAsian = 'traditional';
    nav.className = 'header_tw';
  } else {
    e.target.innerText = '繁體';
    nav.style.fontVariantEastAsian = 'simplified';
    nav.className = 'header';
  }
});
```

### 5.3 要点说明

- **文本长度差异**：同一含义的文本在不同语言中长度差异显著，需要使用弹性布局来适应
- **文本溢出处理**：使用 `text-overflow: ellipsis` + `overflow: hidden` + `white-space: nowrap` 组合来处理溢出文本
- **Flex/Grid 布局**：更适合国际化场景，因为它们天然具有弹性，能更好地适应不同文本长度
- **导航栏间距**：切换语言后，导航项的宽度和间距可能需要调整，建议使用相对单位或自适应方案
- **字体大小调整**：不同语言可能需要不同的基础字号，如繁体中文可能需要比简体中文稍小的字号来保持视觉平衡

## 六、逻辑属性（Logical Properties）

### 6.1 主题说明

CSS 逻辑属性与逻辑值模块引入了一套基于逻辑方向（而非物理方向）的属性体系。传统的 CSS 使用 `top`、`right`、`bottom`、`left` 等物理方向来描述盒子模型的各个边，但在不同的书写模式下，这些物理方向的含义会变得混乱。

逻辑属性使用 **块向（block）** 和 **行向（inline）** 两个抽象方向来描述，它们的具体物理含义取决于当前的书写模式：

- **块向（Block）**：与行内文本流向垂直的方向。横排书写模式中指上下方向，竖排书写模式中指左右方向
- **行向（Inline）**：与行内文本流向平行的方向。横排书写模式中指左右方向，竖排书写模式中指上下方向

> 对应代码文件：`代码/逻辑属性/logic-properties.html`

### 6.2 关键代码

**块向与行向边框示例：**

```html
<!-- border-block：块向（横排时为上下，竖排时为左右） -->
<div style="writing-mode: horizontal-tb; border-block: 5px solid #e54d42; width: 500px; padding: 20px; margin: 20px;">
  文字水平方向，border-block 是上下方向
</div>
<div style="writing-mode: vertical-lr; border-block: 5px solid #e54d42; width: 500px; padding: 20px; margin: 20px;">
  文字垂直方向，border-block 是左右方向
</div>

<!-- border-inline：行向（横排时为左右，竖排时为上下） -->
<div style="writing-mode: horizontal-tb; border-inline: 5px solid #e54d42; width: 500px; padding: 20px; margin: 20px;">
  文字水平方向，border-inline 是左右方向
</div>
<div style="writing-mode: vertical-rl; border-inline: 5px solid #e54d42; width: 500px; padding: 20px; margin: 20px;">
  文字垂直方向，border-inline 是上下方向
</div>
```

**物理方向 vs 逻辑方向对比：**

在代码中，针对同一种四色边框效果，分别使用物理方向和逻辑方向实现：

物理方向（需要根据每种书写模式单独调整）：

```css
/* 水平 LTR（默认） */
.phy-box1 {
  border-top-color: #FF6347;    /* 提香红 */
  border-right-color: #2e8b57;  /* 马尔斯绿 */
  border-bottom-color: #00a0b0; /* 蒂芙尼蓝 */
  border-left-color: #F9DC24;   /* 申布伦黄 */
}

/* RTL 方向 - 需要交换左右颜色 */
.phy-box2 {
  border-top-color: #FF6347;
  border-left-color: #2e8b57;   /* 原来的右边颜色 */
  border-bottom-color: #00a0b0;
  border-right-color: #F9DC24;  /* 原来的左边颜色 */
}

/* 竖排从左到右 - 需要完全重排颜色 */
.phy-box3 {
  border-left-color: #FF6347;   /* 原来的上 */
  border-bottom-color: #2e8b57; /* 原来的右 */
  border-right-color: #00a0b0;  /* 原来的下 */
  border-top-color: #F9DC24;    /* 原来的左 */
}
```

逻辑方向（一套代码适用所有书写模式）：

```css
/* 逻辑方向 - 同一套规则适用于所有书写模式和文字方向 */
.log-box {
  border-block-start-color: #FF6347;   /* 提香红 */
  border-inline-end-color: #2e8b57;    /* 马尔斯绿 */
  border-block-end-color: #00a0b0;     /* 蒂芙尼蓝 */
  border-inline-start-color: #F9DC24;  /* 申布伦黄 */
}
```

### 6.3 要点说明

- 使用物理方向时，每当书写模式改变（如从 LTR 切换到 RTL，或从横排切换到竖排），都需要重新编写边框颜色规则
- 使用逻辑方向时，**只需编写一套 CSS 规则**，即可自动适应所有书写模式和文字方向
- 逻辑属性的核心优势：**一次编写，自动适配所有书写模式**，大幅降低国际化样式维护成本

## 七、逻辑属性进行布局

### 7.1 主题说明

在实际项目布局中，逻辑属性不仅能用于边框，还能用于控制 margin、padding、float、尺寸等布局属性。本示例展示了一个完整的页面布局（页头 + 主内容区 + 侧边栏），通过逻辑属性实现横排与竖排书写模式的自动适配。

> 对应代码文件：`代码/逻辑属性进行布局/index.html` + `代码/逻辑属性进行布局/style.css`

### 7.2 关键代码

**使用逻辑属性的外层容器：**

```css
.common-stracture {
  max-inline-size: 1100px;          /* 逻辑属性，替代 max-width */
  margin-inline-start: auto;        /* 逻辑属性，替代 margin-left */
  margin-inline-end: auto;          /* 逻辑属性，替代 margin-right */
}
```

**页头区域：**

```css
.main-header {
  padding: var(--common-padding);
  font-size: 40px;
  background: #000;
  color: #fff;
}

/* float 的逻辑属性写法 */
.push-end {
  float: inline-end;   /* 逻辑属性，替代 float: right */
}
```

**Grid 主内容区：**

```css
.main-content {
  display: grid;
  grid-template-columns: auto 300px;
  padding-block-start: var(--common-padding);   /* 逻辑属性，替代 padding-top */
  padding-block-end: var(--common-padding);     /* 逻辑属性，替代 padding-bottom */
  background: #eee;
}

.main-content .main {
  padding-inline-start: var(--common-padding);  /* 逻辑属性，替代 padding-left */
  padding-inline-end: var(--common-padding);    /* 逻辑属性，替代 padding-right */
}
```

**侧边栏图文列表：**

```css
.articles-template-item .image {
  display: block;
  float: inline-start;            /* 逻辑属性，替代 float: left */
  inline-size: 40px;              /* 逻辑属性，替代 width */
  block-size: 40px;               /* 逻辑属性，替代 height */
  margin-inline-end: var(--common-padding-small);   /* 逻辑属性 */
  margin-block-end: var(--common-padding-small);    /* 逻辑属性 */
}
```

**标题的块向间距：**

```css
.main-title {
  font-size: 30rem;
  margin-block-end: var(--common-padding);   /* 逻辑属性，替代 margin-bottom */
}
```

**基于逻辑尺寸的媒体查询：**

```css
/* 基于行向尺寸（而非物理宽度）的媒体查询 */
@media (max-inline-size: 1000px) {
  .main-content {
    background: red;
    grid-template-columns: auto;
  }
}
```

**书写模式切换脚本：**

```html
<select class="lang push-end">
  <option value="horizontal-tb">English</option>
  <option value="vertical-rl">Chinese</option>
</select>
```

```javascript
$(document).ready(function() {
  $(".lang").on("change", function() {
    $(".lang option:selected").each(function() {
      $("html").css("writing-mode", $(this).prop("value"));
    });
  });
});
```

### 7.3 要点说明

- **逻辑尺寸属性**：`inline-size` 替代 `width`，`block-size` 替代 `height`；在竖排模式下它们会自动交换
- **逻辑外边距/内边距**：`margin-inline-start/end`、`padding-block-start/end` 等替代物理方向的 margin/padding
- **逻辑浮动**：`float: inline-start` 和 `float: inline-end` 替代 `float: left` 和 `float: right`
- **逻辑媒体查询**：`@media (max-inline-size: ...)` 替代 `@media (max-width: ...)`，能正确响应竖排模式下的视口变化
- 通过下拉框切换 `writing-mode`，页面的所有逻辑属性会自动适配新的书写方向，无需修改任何 CSS 规则
- 这展示了逻辑属性在实际布局中的完整应用：**一次编写布局代码，横排竖排自动适配**

## 总结

CSS 中的国际化特性主要围绕以下几个层面：

| 层面 | 核心属性/特性 | 用途 |
|------|-------------|------|
| 文字方向 | `direction`、`dir` HTML 属性 | 支持 RTL/LTR 语言 |
| 语言样式 | `:lang()` 伪类 | 按语言差异化样式 |
| 字体特性 | `font-variant-east-asian` | 东亚文字简繁变体切换 |
| 书写模式 | `writing-mode`、`text-orientation`、`text-combine-upright` | 支持竖排等非水平书写模式 |
| 语言切换问题 | 弹性布局、文本溢出处理 | 应对文本长度差异导致的布局问题 |
| 逻辑属性 | `border-block-*`、`border-inline-*`、`inline-size`、`block-size`、`margin-inline-*` 等 | 一次编写，自动适配所有书写模式 |

**核心原则**：在国际化项目中，应优先使用逻辑属性替代物理方向属性，使用弹性布局应对文本长度差异，并为不同书写模式做好适配准备。
