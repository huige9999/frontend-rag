## 2. 章节逻辑说明

正如课件开头所说：「对前端而言，网络部分代码不多，以概念为主」。这门课主线是**「概念打底（02）→ 工具（07）→ 原理（08）→ 请求能力（09）→ 综合项目（10~12）」**，最终落在一个「聊天机器人」全流程小项目上。

### 先讲什么：网络基本概念（02）

- **02 基本概念-客户端和服务器**：客户端/服务器、请求/响应、C/S 与 B/S、url、http 报文等。这一章是整门课的「概念地基」，决定后面能否看懂接口文档。

**为什么先讲概念**：网络课「概念重于代码」，先把通信的最基本词汇和 url/http 报文讲透，后面的 AJAX、接口才有抓手。

### 再讲什么：工具与浏览器原理（07~08）

- **07 关于 Apifox 的使用**：接口测试工具，对应实际开发里「怎么调接口、怎么看返回」。
- **08 浏览器页面处理流程**：从输入 url 到页面渲染出来的完整流程（DNS、请求、解析、渲染）。

**为什么这么安排**：概念之后，先给「调接口的工具」（07），再讲「请求在浏览器里到底经历了什么」（08），一外一里把请求的生命周期补全。

### 然后讲什么：AJAX 请求能力（09）

- **09 AJAX**：原生 XHR 的请求写法（demo1/2/3）。这是前端「主动发起网络请求」的起点。

### 最后讲什么：聊天机器人综合项目（10~12）

用 3 章把前面所有概念 + 请求能力串成一个真实小项目：

- **10 聊天机器人-接口封装**：把接口请求统一封装。
- **11 聊天机器人-登录和注册**：登录/注册流程。
- **12 聊天机器人-首页**：聊天首页交互。

### 主线一句话总结

**网络概念地基（02）→ 接口工具 Apifox（07）→ 浏览器请求全流程（08）→ AJAX 请求能力（09）→ 聊天机器人综合项目（10~12）**，目标是从「不懂网络」到「看得懂接口文档、会发起请求、能做一个带登录和接口交互的小项目」。

## 3. 使用场景等元信息沉淀

- **学完能用在哪**：看懂任何接口文档、用 Apifox 调接口、用原生 AJAX/后续 axios 发起请求、做带登录注册的接口驱动型小项目。
- **和实际项目的关系**：所有前后端分离项目的「请求层」基础；浏览器页面处理流程是排查「首屏慢、白屏」类问题的理论依据。
- **和其他课程的关系**：上承《程序思维训练》，下接《第三方库》（其中的 Axios 就是这套请求能力的工程化封装）；聊天机器人项目是后续 Vue 项目里「获取远程数据、登录鉴权」的雏形。
- **面试/教学高频场景**：从输入 url 到页面渲染流程（08）、http 报文结构、C/S 与 B/S（02）。
