## 2. 章节逻辑说明

这门课是一条**「用 Egg 框架搭建一个规范的后端服务」主线**，从「认识 Egg、写出 Hello World」开始，沿 MVC 链路（路由 → 控制器 → 模板 → 中间件 → service → 数据库）把核心开发能力补齐，再用「环境/部署/定时/日志/会话/异常/安全」等工程化能力把生产可用性补全，最后进入「扩展、插件、生命周期、鉴权」的框架定制层。

### 先讲什么：认识 Egg 与 MVC 主干（1-1 ~ 1-8）

- **1-1 起步**：讲清 Egg 是什么（一站式整合 + 规范 + 插件扩展 + 约定优于配置 + 多进程 + MVC）、目录约定（`app/controller`、`app/router.js`、`config`）、运行流程（生成配置 → 启动 → 匹配路由 → Controller Action 处理）。这是整门课的地基。
- **1-2 路由和控制器**：路由是请求与处理函数的桥梁。讲普通路由、命名路由、重定向、路由拆分、子目录控制器、RESTful `resources`；以及 Controller/Action 规范、`ctx` 获取、Controller 处于洋葱最里层（不给 `next`）。
- **1-3 静态资源插件**：`app/public` 约定、`egg-static`、引出**插件**概念（命名 `egg-*`、`plugin.js` 启用、`config.default.js` 配置、集中配置集中管理）。
- **1-4 模板引擎**：服务端渲染需求、`egg-view` 统一管理引擎、安装/启用/统一配置/渲染（`render/renderView/renderString`）。至此形成完整 MVC。
- **1-6 中间件**：高阶函数写法、`app/middleware` 约定、全局中间件（`config` 配置 + `match/ignore`）、路由中间件、内置中间件、AOP 横切关注点。
- **1-7 通用模型的提取**：`app.locals`（全局）/`ctx.locals`（上下文）/局部 model 的分层。
- **1-8 抽离业务逻辑**：三层架构 + MVC（视图层 → 业务逻辑层 → 数据访问层），引入 `app/service`、`Service` 基类、`ctx.service` 访问、别名 `this.service`。

**为什么这么安排**：严格顺着「一个请求进来怎么被处理」的 MVC 数据流走——路由匹配 → Controller → 模板渲染（V）→ 中间件拦截 → 数据模型 → Service 业务层。先把主干走通，后面才谈数据、工程化与定制。

### 再讲什么：接数据 + 上线相关（1-9 ~ 1-13）

- **1-9 使用数据库**：mysql / sequelize / mongoose / redis 多种方式，对接数据访问层。
- **1-10 环境配置**：`config/env` 与 `EGG_SERVER_ENV`、`app.config.env`、按环境加载 `config.{env}.js` 覆盖默认配置、自定义环境。
- **1-11 部署**：`egg-cluster` 多进程模型（Master / worker / Agent，无需 pm2）、`egg-scripts` 启停命令与参数。
- **1-12 定时任务**：`app/schedule` 约定、两种写法、`schedule` 配置（interval/cron/type/immediate/env/disable）。
- **1-13 日志记录**：`egg-logger`、日志路径、四类日志（app/core/error/agent）、日志级别、配置、自定义日志。

**为什么这么安排**：MVC 骨架立起来后，先解决「数据从哪来（1-9）」，再解决「如何区分环境、如何上线、如何在后台跑任务、如何记日志（1-10~1-13）」——这是把一个能跑的 demo 变成可维护、可运维的真实服务的关键一步。

### 然后讲什么：会话、异常、安全等运行时能力（1-14 ~ 1-19）

- **1-14 egg-session**：`egg-session` + `koa-session`，默认 cookie 序列化加密存储（区别于传统服务端存储）、配置、可选自定义 `sessionStore` / `egg-session-redis`。
- **1-15 异常处理**：koa 异常模式、`egg-onerror` 插件按格式/环境分流处理、404 处理与 `notfound` 配置。
- **1-16 egg的其他功能**：文件上传、国际化 I18n、helper 对象、参数校验（导览性质）。
- **1-17 安全**：`egg-security`，XSS 防护（`escape`/`sjs`/`shtml` + 白名单）、CSRF 防护（csrf token 原理与 `ctx.csrf`）、`match/ignore/enable`。
- **1-18 整合单页应用**：开发阶段双端口并行、部署阶段把 vue 打包结果作为静态资源托管、视图目录 + 通配路由兜底到单页。
- **1-19 使用脚手架**：`create-egg` 导览。

**为什么这么安排**：会话/异常/安全是任何上线服务都绕不开的运行时关注点；「整合单页应用」和「脚手架」是从「纯后端」走向「前后端协同」的收尾，承上启下到第2章的定制能力。

### 最后讲什么：框架定制与扩展（2-1 ~ 2-4）

- **2-1 框架扩展**：`app/extend` 对 application/context/request/response/helper 五类对象的扩展，并给出**扩展决策树**（是否在请求处理中发生 → 决定扩展目标）。
- **2-2 自定义插件**：插件文件夹结构、与普通应用目录的异同、插件无 router/controller、无 `plugin.js`、`package.json` 的 `eggPlugin` 节点（name/dependencies/optionalDependencies/env）。
- **2-3 生命周期函数**：`app.js` 中的启动钩子（configWillLoad → didLoad → willReady → didReady → serverDidReady → beforeClose）及各阶段适用场景。
- **2-4 passport 鉴权**：`egg-passport` 统一第三方登录方案、`app.passport.mount`、创建 GitHub 应用、统一 API（`ctx.user` / `ctx.isAuthenticated()` / `ctx.logout()`）。

**为什么放最后**：这些是「会用了之后才知道框架边界在哪、怎么改它」的内容。扩展、插件、生命周期是理解 Egg「约定 + 插件」架构的关键，鉴权则是综合运用扩展与插件的实战案例。

### 主线一句话总结

**认识 Egg（是什么/约定/MVC 流程）→ 搭通 MVC 主干（路由→控制器→模板→中间件→service→数据库）→ 补齐工程化（环境/部署/定时/日志）→ 补齐运行时（会话/异常/安全/单页整合）→ 定制框架（扩展/插件/生命周期/鉴权）**，最终目标是能用 Egg 按规范交付一个生产可用的后端服务。

## 3. 使用场景等元信息沉淀

- **学完能用在哪**：独立搭建符合 Egg 约定的后端服务（RESTful API + 数据库 + 模板渲染 + 单页托管）；为中后台/后台管理类项目提供 BFF 层；做代理、Mock、中间服务器。
- **和实际项目的关系**：MVC + 三层架构（路由 → service → 数据访问）是真实项目可直接复用的分层范式；环境配置、部署、日志、异常、安全是上线必备的工程化能力。
- **和其他课程的关系**：是《Node》课程（Express 那条线）的「规范化框架」延伸——Express 讲机制，Egg 讲约定与插件化；session、CSRF/XSS、跨域等概念在两门课中互通，可与《Node》对照复习。
- **面试/教学高频场景**：Egg 的约定与目录结构（1-1）、多进程模型 Master/worker/Agent（1-11）、中间件与洋葱模型（1-6）、session 默认 cookie 存储原理（1-14）、CSRF token 防护原理（1-17）、生命周期钩子顺序（2-3）、扩展决策树（2-1）。
