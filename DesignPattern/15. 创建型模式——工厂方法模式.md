# 工厂方法模式

前面所介绍的简单工厂模式实际上并不是 GoF 所提出的 23 种设计模式里面的一种，更像是一种编程习惯。

但是这种简单工厂模式的编写习惯存在一定的问题。



## 简单工厂存在的问题

假设现在我们有这样的一个需求：

>假设我们有一个日志记录系统，需要根据不同的环境（如文件日志、数据库日志）来记录日志。

那么根据我们上节课所介绍的简单工厂模式，写出来的代码如下：

```ts
// 产品对应的统一接口
interface Logger {
  log(message: string): void;
}

// 文件的形式记录日志
// 实现了 Logger 接口
class FileLogger implements Logger {
  log(message: string) {
    console.log(`File log: ${message}`);
  }
}

// 数据库的形式记录日志
// 实现了 Logger 接口
class DatabaseLogger implements Logger {
  log(message: string) {
    console.log(`Database log: ${message}`);
  }
}

// 一个工厂类
// 提供了 createLogger 方法，根据不同参数来创建不同的产品
class LoggerFactory {
  static createLogger(type: string): Logger {
    switch (type) {
      case "file":
        return new FileLogger();
      case "database":
        return new DatabaseLogger();
      default:
        throw new Error("Unknown logger type");
    }
  }
}

const logger = LoggerFactory.createLogger("file");
logger.log("This is a message.");
```

简单工厂模式所存在的问题：

工厂类的职责过重，每一次需要新增一个类型的产品的时候，都需要修改工厂类内部的逻辑，不符合设计原则里面的开放-封闭原则。



## 工厂方法类图结构

标准的工厂方法模式，其核心就是一个产品对应一个工厂，而不是将创建不同类型的产品的逻辑放入到一个方法里面。

假设有产品 A，B，C，那么在标准的工厂方法模式里面，就有对应的工厂 A，B，C，假设现在需要新增一个产品 D，我们只需要新增一个对应的工厂 D 即可，原来的工厂是不受任何影响的。

下面是标准工厂模式的类图结构：

![image-20240322104208915](https://xiejie-typora.oss-cn-chengdu.aliyuncs.com/2024-03-22-024209.png)

1. 产品接口：对所有的产品进行统一的接口限制
2. 具体产品：每一种产品都是实现了产品接口的
3. 创建者接口：用于约束多个工厂的接口
4. 具体的创建者：对应的就是多个工厂，每个工厂都实现了工厂接口的

## 课堂代码

### 产品接口 — ILogger

日志记录器的统一接口，所有具体的日志产品都必须实现该接口。

```typescript
// 产品的统一接口

export interface ILogger {
  log(message: string): void;
}
```

### 具体产品 — FileLogger

以文件形式记录日志的具体产品，实现了 ILogger 接口。

```typescript
import { ILogger } from "./ILogger";

export class FileLogger implements ILogger {
  log(message: string): void {
    console.log(`Log to file: ${message}`);
  }
}
```

### 具体产品 — DatabaseLogger

以数据库形式记录日志的具体产品，实现了 ILogger 接口。

```typescript
import { ILogger } from "./ILogger";

export class DatabaseLogger implements ILogger {
  log(message: string): void {
    console.log(`Log to database: ${message}`);
  }
}
```

### 具体产品 — RemoteLogger

以远程服务形式记录日志的具体产品，实现了 ILogger 接口。

```typescript
import { ILogger } from "./ILogger";

export class RemoteLogger implements ILogger {
  log(message: string): void {
    console.log(`Remote log to file: ${message}`);
  }
}
```

### 工厂接口 — IFactory

所有工厂类的统一接口，定义了创建日志产品的方法。

```typescript
// 工厂接口
// 所有工厂类都要实现这个接口

import { ILogger } from "../Logger/ILogger";

export interface IFactory {
  createLogger(): ILogger;
}
```

### 具体工厂 — FileLoggerFactory

文件日志工厂，专门负责创建 FileLogger 产品。

```typescript
import { IFactory } from "./IFactory";
import { ILogger } from "../Logger/ILogger";
import { FileLogger } from "../Logger/FileLogger";

export class FileLoggerFactory implements IFactory {
  createLogger(): ILogger {
    return new FileLogger();
  }
}
```

### 具体工厂 — DatabaseLoggerFactory

数据库日志工厂，专门负责创建 DatabaseLogger 产品。

```typescript
import { IFactory } from "./IFactory";
import { ILogger } from "../Logger/ILogger";
import { DatabaseLogger } from "../Logger/DatabaseLogger";

export class DatabaseLoggerFactory implements IFactory {
  createLogger(): ILogger {
    return new DatabaseLogger();
  }
}
```

### 具体工厂 — RemoteLoggerFactory

远程日志工厂，专门负责创建 RemoteLogger 产品。

```typescript
import { IFactory } from "./IFactory";
import { ILogger } from "../Logger/ILogger";
import { RemoteLogger } from "../Logger/RemoteLogger";

export class RemoteLoggerFactory implements IFactory {
  createLogger(): ILogger {
    return new RemoteLogger();
  }
}
```

### 客户端调用 — Client

客户端通过不同的工厂来获取对应的产品实例，无需关心具体的产品创建细节。每种产品对应一个专属工厂，新增产品时只需新增工厂类，无需修改已有代码。

```typescript
import { FileLoggerFactory } from "./Factory/FileLoggerFactory";
import { RemoteLoggerFactory } from "./Factory/RemoteLoggerFactory";

// 通过不同产品所对应的工厂来得到产品

const fileFactory = new FileLoggerFactory();
const logger = fileFactory.createLogger();
logger.log("Hello World!");

const remoteFactory = new RemoteLoggerFactory();
const remoteLogger = remoteFactory.createLogger();
remoteLogger.log("Hello World!");
```
