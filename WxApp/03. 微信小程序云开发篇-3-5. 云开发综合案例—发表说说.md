# 云开发综合案例—发表说说

## 案例概述

本案例综合运用微信小程序云开发的三大核心能力（云数据库、云存储、云函数），实现一个"发表说说"功能。用户可以在小程序中输入文字内容、选择图片，点击发布后将图片上传到云存储、将说说数据存储到云数据库，并在首页展示所有说说列表。

**项目结构：**

```
cloudprogram/
├── app.js                          # 小程序入口，初始化云服务
├── app.json                        # 全局配置，TabBar、Vant 组件注册
├── app.wxss                        # 全局样式
├── project.config.json             # 项目配置，云函数目录
├── utils/
│   └── util.js                     # 工具函数（日期格式化）
├── components/
│   └── message/                    # 说说列表项组件
├── pages/
│   ├── index/                      # 首页（说说列表）
│   └── publish/                    # 发表说说页
├── functions/
│   └── calc/                       # 云函数示例（计算器）
└── imgs/                           # TabBar 图标
```

**使用到的云开发能力：**

- **云数据库**：存储和读取说说数据（messages 集合）
- **云存储**：上传和下载说说图片
- **云函数**：云端计算示例

## 二、首页 — 说说列表展示

### index.js

```js
const db = wx.cloud.database();
const messages = db.collection('messages');

Page({
  data: {
    list: [], // 存储说说列表
  },
  // 显示首页的时候，从云端数据库拉取说说
  async onShow() {
    const {
      data
    } = await messages.get({
      limit: 5
    });
    this.setData({
      list : data
    })
  }
})
```

**要点说明：**

- 通过 `wx.cloud.database()` 获取云数据库实例，再通过 `db.collection('messages')` 获取 messages 集合
- 在 `onShow` 生命周期中从云数据库拉取说说数据，每次最多拉取 5 条
- 拉取到的数据通过 `setData` 更新到页面的 `list` 中

- 每条说说通过自定义组件 `<message>` 进行展示，通过 `item` 属性将数据传递给组件

## 三、说说列表项组件 — message

### message.js

```js
// 引入工具函数
const {
  formateDate
} = require("../../utils/util.js");

Component({
  properties: {
    item: Object
  },
  data: {
    dateInfo : null, // 存储处理后的时间
    imgArr : [], // 用来存储下载后的图片
  },
  methods: {
    // 预览图片
    previewHandle(ev){
      const src = ev.target.dataset.src;
      wx.previewImage({
        urls: this.data.imgArr,
        current : src
      })
    }
  },
  lifetimes : {
    async attached() {
      const item = this.properties.item;
      // 重新设置时间，将时间戳转为可显示格式
      this.setData({
        dateInfo : formateDate(item.publishDate)
      });
      // 接下来要处理的是图片了
      // 图片我们现在拿到的是云端的路径
      // 我们需要通过云端路径下载图片
      const arr = [...this.data.imgArr];
      for(let i=0;i<item.picPath.length;i++){
        const res = await wx.cloud.downloadFile({
          fileID : item.picPath[i]
        });
        arr.push(res.tempFilePath);
        // 等下载到最后一张的时候，再统一更新 imgArr
        if(i === item.picPath.length - 1){
          this.setData({
            imgArr: arr
          })
        }
      }
    }
  }
})
```

**要点说明：**

- 通过 `properties` 接收父组件传入的 `item` 数据（单条说说对象）
- 在 `lifetimes.attached` 生命周期中处理数据：
  1. 调用 `formateDate` 工具函数将时间戳转为可显示的日期格式
  2. 通过 `wx.cloud.downloadFile()` 将云端图片下载到本地临时路径，用于页面展示
- `previewHandle` 方法调用 `wx.previewImage` 实现图片预览功能
- 下载图片采用逐张下载的方式，等最后一张下载完毕后统一更新 `imgArr`

## 四、发表说说页

### publish.js

```js
const db = wx.cloud.database();
const messages = db.collection('messages');

Page({
  data: {
    content : "" , // 用户输入的新的说说内容
    fileList : [], // 说说对应的图片
  },
  // 读取用户选择的图片
  // 回填到 fileList 里面
  // 从而让用户选择的图片能够显示出来
  afterRead(event){
    const newFileList = [...this.data.fileList];
    const {file} = event.detail;
    for(let i=0;i<file.length;i++){
      newFileList.push({
        url : file[i].url
      })
    }
    this.setData({
      fileList : newFileList
    })
  },
  // 删除当前的图片
  deleteHandle(event){
    const index = event.detail.index;
    const newFileList = [...this.data.fileList];
    newFileList.splice(index, 1);
    this.setData({
      fileList : newFileList
    })
  },
  // 发表说说
  async publishHandle(){
    const picArr = []; // 存储云端返回的云端路径
    // 1. 既然已经点击了【发布按钮】
    // 说明要上传什么图片已经确定了
    // 先把图片上传到云端服务器
    wx.showLoading({
      title: '发表中',
    });
    for(let i=0;i<this.data.fileList.length;i++){
      // 拿到 url
      const url = this.data.fileList[i].url;
      const name = Math.random() * 10000;
      // 存储到云端的文件名
      const cloudPath = name + url.match(/\.[^.]+?$/)[0];
      // 开始上传到云端
      const uploadResult = await wx.cloud.uploadFile({
        cloudPath,
        filePath : url
      });
      picArr.push(uploadResult.fileID);
      if(i === this.data.fileList.length - 1){
        // 说明已经上传完最后一张图片了
        // 组装这一条说说数据，准备存储到云数据库
        const data = {
          content : this.data.content, // 这条说说的内容
          picPath : picArr, // 这条说说所有图片的云端路径
          publishDate : new Date().getTime().toString()
        }
        // 存储到云数据库
        await messages.add({data});
        // 4. 清空文本
        this.setData({
          content : "",
          fileList : []
        });
        wx.hideLoading();
        wx.showToast({
          title: '发表成功',
          icon : "success"
        })
      }
    }
  }
})
```

**要点说明：**

这是本案例最核心的页面，完整展示了云开发的三大能力：

1. **`afterRead`** — 用户选择图片后的回调，将图片信息回填到 `fileList` 中展示
2. **`deleteHandle`** — 删除已选择的图片，从 `fileList` 中移除指定索引的图片
3. **`publishHandle`** — 发表说说的核心逻辑：
   - 依次将每张图片通过 `wx.cloud.uploadFile()` 上传到云存储，生成随机的 `cloudPath` 避免文件名冲突
   - 上传完成后获得 `fileID`（云端文件唯一标识），收集到 `picArr` 数组中
   - 所有图片上传完毕后，组装说说数据（文字内容、图片云端路径、发布时间戳），通过 `messages.add()` 存入云数据库
   - 清空输入框和图片列表，提示发表成功

## 五、工具函数

### utils/util.js — 日期格式化

```js
const formateDate = timestamp => {
  if (!timestamp) {
    return;
  }
  let date = new Date(parseInt(timestamp));

  let year = date.getFullYear(); // 年
  let month = date.getMonth() + 1; // 月
  let day = date.getDate(); // 日

  let hour = date.getHours(); // 时
  let minutes = date.getMinutes(); // 分
  let seconds = date.getSeconds(); // 秒

  let weekArr = [
    "星期日",
    "星期一",
    "星期二",
    "星期三",
    "星期四",
    "星期五",
    "星期六",
  ];
  let week = weekArr[date.getDay()];

  if (day >= 0 && day <= 9) {
    day = "0" + day;
  }
  if (hour >= 0 && hour <= 9) {
    hour = "0" + hour;
  }
  if (minutes >= 0 && minutes <= 9) {
    minutes = "0" + minutes;
  }
  if (seconds >= 0 && seconds <= 9) {
    seconds = "0" + seconds;
  }

  return {
    year,
    month,
    day,
    hour,
    minutes,
    seconds,
    week
  };
}

module.exports = {
  formateDate
}
```

该函数将时间戳转换为包含年、月、日、时、分、秒、星期的对象，用于在说说列表中展示友好的日期格式。

## 六、云函数示例

### functions/calc/index.js — 计器云函数

```js
// 在该文件中来封装云函数逻辑

const cloud = require('wx-server-sdk');

exports.main = async(event) => {
  let result = 0;
  // 根据传递过来的参数做不同的计算
  // num1、num2、tag（+ - * /）
  switch(event.tag){
    case "+": {
      result = ~~event.num1 + ~~event.num2;
      break;
    }
    case "-":{
      result = ~~event.num1 - ~~event.num2;
      break;
    }
    case "*":{
      result = ~~event.num1 * ~~event.num2;
      break;
    }
    case "/":{
      result = ~~event.num1 / ~~event.num2;
      break;
    }
  }
  return {
    calcResult : result
  }
}
```

**要点说明：**

- 云函数通过 `exports.main` 导出，接收 `event` 参数（小程序端传入的数据）
- 根据 `event.tag` 判断运算类型（加减乘除），使用 `~~` 双取反运算符将字符串转为整数
- 返回计算结果对象 `{ calcResult: result }`

## 总结

本案例完整演示了微信小程序云开发的实际应用流程：

| 步骤 | 涉及的云能力 | 说明 |
|------|-------------|------|
| 初始化 | `wx.cloud.init()` | 在 app.js 中初始化云环境 |

| 展示列表 | **云数据库** | 从云数据库的 messages 集合中读取数据 |
| 展示图片 | **云存储** | 通过 fileID 从云存储下载图片到本地展示 |
| 云端计算 | **云函数** | 云函数处理计算逻辑，返回结果给小程序端 |
