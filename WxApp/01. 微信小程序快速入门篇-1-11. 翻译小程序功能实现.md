# 翻译小程序功能实现

本节课在上一节课搭建的结构基础上，实现翻译小程序的核心功能，包括：调用百度翻译 API 进行翻译、语言选择交互、翻译历史记录的存储与展示。

---

## 一、翻译工具函数 — utils/util.js

翻译功能封装在 `utils/util.js` 中，通过 Promise 封装 `wx.request` 调用百度翻译 API。

```javascript
import md5 from "./md5.min.js";

// 百度翻译的 APPID 以及 key
// 请填写你自己的 appid 和 key
const appid = "";
const key = "";

/**
 *
 * @param {*} q 要翻译的文本
 * @param {*} from 原来的语言
 * @param {*} to 目标语言
 */
function transFunc(q, from = "auto", to) {
  return new Promise((resolve, reject) => {
    // 生成随机数
    const salt = Date.now(); // 随机数
    // 制作签名 appid+q+salt+密钥的MD5值
    const sign = md5(`${appid}${q}${salt}${key}`);
    // 发送请求进行翻译
    wx.request({
      url: 'https://fanyi-api.baidu.com/api/trans/vip/translate',
      data: {
        q, // 要翻译的文本
        from, // 原来的语言
        to, // 目标语言
        appid,
        salt, // 随机数
        sign // 签名
      },
      success(res) {
        console.log(res)
        if (res.data && res.data.trans_result) {
          // 说明翻译成功
          // 返回翻译的文本
          resolve(res.data.trans_result[0].dst)
        }
      },
      fail() {
        wx.showToast({
          title: '网络异常',
          icon: "error",
          duration: 2000
        })
        reject({
          status: "error",
          msg: "翻译失败"
        })
      }
    })
  })
}

module.exports = transFunc;
```

**要点说明：**
- 使用 `md5` 对 `appid + q + salt + key` 进行加密生成签名（sign），这是百度翻译 API 的安全验证方式
- `from` 参数默认为 `"auto"`，表示自动检测源语言
- 通过 `Promise` 封装，使调用方可以使用 `.then()` 的方式获取翻译结果
- 翻译成功时，从返回数据的 `trans_result[0].dst` 中提取译文并通过 `resolve` 返回
- 翻译失败时，通过 `wx.showToast` 显示错误提示，并通过 `reject` 返回错误信息

## 二、全局入口 — app.js

相比上一节课，`app.js` 新增了 `onLaunch` 生命周期中加载本地缓存历史记录的逻辑，以及 `globalData` 中的 `history` 数组。

```javascript
// app.js
App({
  onLaunch() {
    // 一开始启动应用的时候，拿出本地缓存进行加载
    wx.getStorage({
      key : "history",
      success:(res)=>{
        // 我们需要将其赋值给 globalData 里面的 history
        this.globalData.history = res.data;
      }
    })
  },
  // 全局的数据，所有页面都可以使用这个数据
  globalData: {
    // 所有语言的语言列表
    lanList : [
      {'chs': '英文','lang': 'en',"index": 0},
      {'chs': '中文','lang': 'zh',"index": 1},
      {'chs': '日语','lang': 'jp',"index": 2},
      {'chs': '韩语','lang': 'kor',"index": 3},
      {'chs': '法语','lang': 'fra',"index": 4},
      {'chs': '德语','lang': 'de',"index": 5},
      {'chs': '俄语','lang': 'ru',"index": 6},
      {'chs': '泰语','lang': 'th',"index": 7},
      {'chs': '西班牙语','lang': 'spa',"index": 8},
      {'chs': '阿拉伯语','lang': 'ara',"index": 9},
      {'chs': '意大利语','lang': 'it',"index": 10},
      {'chs': '葡萄牙语','lang': 'pt',"index": 11},
      {'chs': '荷兰语','lang': 'nl', 'index':12}
    ],
    // 当前默认是什么语言，默认设置为英语
    curLan : {'chs': '英文','lang': 'en',"index": 0},
    // 历史记录
    history : []
  }
})
```

**要点说明：**
- `onLaunch` 中使用 `wx.getStorage` 异步读取本地缓存中的 `history` 数据
- 读取成功后，将缓存数据赋值给 `globalData.history`，使历史记录在应用启动时即可用
- `globalData` 新增 `history` 数组，用于存储翻译历史记录

## 三、首页 — index（翻译页）

### 1. index.wxml — 页面结构（更新）

```xml
<!-- 最外层容器 -->
<view class="container">
  <!-- 目标语言的选择 -->
  <view class="lanSelectContainer">
    <text bindtap="goToChange">翻译成{{curLanTxt}}</text>
    <text class="iconfont icon-down"></text>
  </view>
  <!-- 文本输入框 -->
  <view class="tranlateTxt">
    <textarea
      placeholder="请输入您要翻译的文本"
      bindblur="tanslateHandler"
      model:value="{{query}}"
      bindinput="inputHandle"
    ></textarea>
  </view>
  <!-- 翻译结果 -->
  <view class="resultTxt">
    <view>译文</view>
    <view class="content">{{content}}</view>
  </view>
</view>
```

**相比上一节课的变化：**
- 语言选择文字从硬编码的"翻译成英语"改为动态的 `翻译成{{curLanTxt}}`
- `textarea` 新增三个属性：
  - `bindblur="tanslateHandler"`：输入框失焦时触发翻译
  - `model:value="{{query}}"`：双向绑定用户输入的文本
  - `bindinput="inputHandle"`：输入事件处理
- 翻译结果从硬编码的"Hello"改为动态的 `{{content}}`

### 2. index.js — 页面逻辑（更新）

```javascript
import transFunc from "../../utils/util.js"
const app = getApp()

Page({
  data: {
    query : "", // 用户输入的要翻译的文本
    content : "", // 翻译的文本
    curLanTxt : app.globalData.curLan.chs
  },
  // 跳转到选择语言
  goToChange(){
    wx.navigateTo({
      url: '/pages/change/change',
    })
  },
  // 翻译事件
  tanslateHandler(){
    // 1. 拿到用户输入的值  2. 调用接口进行翻译
    transFunc(this.data.query,"auto", app.globalData.curLan.lang)
      .then(txt=>{
        // 1）修改 content 的值，以便翻译结果能够显示出来
        this.setData({
          content: txt
        });
        // 2）将此次翻译结果存储到全局的 history 里面
        app.globalData.history.unshift({
          sourceTxt : this.data.query, // 翻译的原文
          resultTxt : txt
        });
        // 3）将 history 存储到本地，方便下一次进入小程序的时候，能够加载之前的历史记录
        wx.setStorage({
          key : 'history',
          data : app.globalData.history
        })
      })
  },
  onShow(){
    this.setData({
      curLanTxt : app.globalData.curLan.chs
    })
  },
  inputHandle(){}
})
```

**要点说明：**
- 引入 `transFunc` 翻译工具函数
- `data` 中新增 `query`（用户输入文本）、`content`（翻译结果）、`curLanTxt`（当前语言中文名）
- `tanslateHandler` 翻译处理函数，在输入框失焦时触发：
  1. 调用 `transFunc` 发送翻译请求，源语言为 `auto`（自动检测），目标语言为全局的 `curLan.lang`
  2. 翻译成功后，通过 `this.setData` 更新翻译结果到页面
  3. 将翻译记录（原文 + 译文）通过 `unshift` 插入到全局 `history` 数组的头部（最新的在最前面）
  4. 使用 `wx.setStorage` 将历史记录持久化到本地存储
- `onShow` 生命周期钩子：每次页面显示时，更新语言选择栏的文字，确保从语言选择页返回后文字是最新的

## 四、语言选择页 — change

### 1. change.wxml — 页面结构（更新）

```xml
<!-- 一项一项的语言 -->
<view
  class="container"
  wx:for="{{lanList}}"
  wx:key="this"
  data-id="{{index}}"
  bindtap="selectHandle"
>
  <view
    wx:if="{{curLanIndex === index}}"
  >
    {{item.chs}}
    <text class="select iconfont icon-duihao"></text>
  </view>
  <view wx:else>
    {{item.chs}}
  </view>
</view>
```

**相比上一节课的变化：**
- 外层 `view` 新增 `bindtap="selectHandle"`，点击任意语言项时触发选择事件

### 2. change.js — 页面逻辑（更新）

```javascript
const app = getApp(); // 获取实例
Page({
  data: {
    lanList : app.globalData.lanList, // 从全局获取所有的语言
    curLanIndex : app.globalData.curLan.index // 当前语言的索引
  },
  selectHandle(options){
    const index = options.currentTarget.dataset.id;
    // 接下来需要修改全局的当前语言
    for(let i=0;i<app.globalData.lanList.length;i++){
      if(app.globalData.lanList[i].index === index){
        app.globalData.curLan = app.globalData.lanList[i];
        // 再重新设置一下 curLanIndex
        this.setData({
          curLanIndex : app.globalData.curLan.index // 当前语言的索引
        })
      }
    }
  }
})
```

**要点说明：**
- 新增 `selectHandle` 方法处理语言选择：
  1. 通过 `options.currentTarget.dataset.id` 获取点击项的索引（对应 `wxml` 中的 `data-id`）
  2. 遍历语言列表找到匹配项，更新全局 `curLan` 为选中的语言
  3. 通过 `this.setData` 更新页面的 `curLanIndex`，使勾选图标跟随更新

## 五、历史记录页 — history

### 1. history.wxml — 页面结构（更新）

```xml
<scroll-view class="container">
  <!-- 头部 -->
  <view class="header">
    <text>翻译历史</text>
    <text class="iconfont icon-close" bindtap="clearHistory">清除历史记录</text>
  </view>
  <!-- 历史记录 -->
  <view class="list">
    <view class="item" wx:for="{{historyList}}" wx:key="this">
      <view>{{item.sourceTxt}}</view>
      <view>{{item.resultTxt}}</view>
    </view>
  </view>
</scroll-view>
```

**相比上一节课的变化：**
- "清除历史记录"文字新增 `bindtap="clearHistory"` 点击事件
- 历史记录列表从静态数据改为 `wx:for="{{historyList}}"` 动态渲染
- 每条记录显示 `item.sourceTxt`（原文）和 `item.resultTxt`（译文）

### 2. history.js — 页面逻辑（更新）

```javascript
const app = getApp();
Page({
  data: {
    historyList : app.globalData.history
  },
  /**
   * 生命周期函数--监听页面显示
   */
  onShow() {
    this.setData({
      historyList : app.globalData.history
    })
  },
  // 清除历史记录
  clearHistory(){
    // 1. 清除全局 history 的值，清除 historyList
    this.setData({
      historyList : ""
    });
    app.globalData.history = [];
    // 2. 清除本地缓存，下次进来的时候，历史记录也是空的
    wx.removeStorage({
      key: 'histroy',
      success(){
        console.log("本地缓存已清除")
      }
    })
  }
})
```

**要点说明：**
- `data` 中 `historyList` 初始化为全局的 `history` 数据
- `onShow` 生命周期钩子：每次页面显示时重新从全局数据加载历史记录，确保切换 tab 回来时数据是最新的
- `clearHistory` 清除历史记录方法：
  1. 通过 `this.setData` 清空页面数据
  2. 清空全局 `globalData.history` 数组
  3. 使用 `wx.removeStorage` 清除本地缓存

## 六、本节课小结

本节课完成了翻译小程序的全部核心功能：

1. **翻译功能**：通过 `utils/util.js` 封装百度翻译 API 调用，使用 Promise 处理异步结果
2. **语言选择**：在 `change` 页面点击语言项后，更新全局 `curLan`，返回首页后自动更新显示
3. **历史记录**：
   - 每次翻译成功后，将原文和译文存入全局 `history` 数组
   - 通过 `wx.setStorage` 将历史记录持久化到本地存储
   - 应用启动时通过 `onLaunch` 中的 `wx.getStorage` 恢复历史记录
   - 支持通过 `wx.removeStorage` 清除历史记录

涉及的核心知识点回顾：
- `wx.request`：发送网络请求调用翻译接口
- `wx.setStorage` / `wx.getStorage` / `wx.removeStorage`：本地缓存的读写与删除
- `this.setData`：更新页面数据驱动视图刷新
- `getApp().globalData`：通过全局数据实现跨页面数据共享
- `onShow` 生命周期钩子：确保页面每次显示时数据都是最新的
