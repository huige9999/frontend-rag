# 自定义组件案例—待办事项

本节课通过一个完整的待办事项（TodoList）案例，综合运用了自定义组件、页面路由、TabBar、全局数据管理等知识，实现了一个包含任务清单、未完成任务、已完成任务三个页面的完整应用。

---

## 项目结构

```
todolist/
  app.js          -- 全局数据（存储所有待办事项）
  app.json        -- 全局配置（页面路由、TabBar）
  app.wxss        -- 全局样式
  components/
    item/          -- 自定义组件：任务条目
      item.js
      item.json
      item.wxml
      item.wxss
  pages/
    index/         -- 任务清单页面（添加任务 + 显示所有任务）
    unfinish/      -- 未完成任务页面
    finished/      -- 已完成任务页面
```

---

## 全局数据（app.js）

```javascript
// app.js
App({
  globalData: {
    // 存储所有的待办事项
    list : [{
      content : '看电影',
      isComplete : true
    },{
      content : '学习小程序',
      isComplete : false
    }]
  }
})
```

### 要点说明

- 所有待办事项数据存储在 `globalData.list` 中，方便多个页面共享数据
- 每条任务包含 `content`（任务内容）和 `isComplete`（是否完成）两个字段

## 自定义组件：item（任务条目）

### 组件模板（components/item/item.wxml）

```xml
<view class="item">
  <view class="content">
    <checkbox checked="{{item.isComplete}}" class="checkbox" bindtap="checkHandle"></checkbox>
    <text class="{{item.isComplete ?  'finished' : ''}}">{{item.content}}</text>
  </view>
  <image class="delete" src="/imgs/delete.png" bindtap="deleteHandle"></image>
</view>
```

### 组件逻辑（components/item/item.js）

```javascript
const app = getApp();
Component({
  properties: {
    item: Object
  },
  methods: {
    checkHandle() {
      // 1. 拿到用户点击的这一项
      const item = this.properties.item;
      // 2. 从全局数据中找到这一项
      const listItem = app.globalData.list.filter(i => i.content === item.content)[0];
      // 3. 修改状态，修改完毕后给出提示
      listItem.isComplete = !listItem.isComplete;
      if (listItem.isComplete) {
        wx.showToast({
          title: '标记【完成】',
          icon: 'success'
        })
      } else {
        wx.showToast({
          title: '标记【未完成】',
          icon: 'success'
        })
      }
      // 4. 修改当前这一项的完成状态
      this.setData({
        item: {
          isComplete: !item.isComplete
        }
      })
      // 5. 需要触发父组件的 fresh 事件，从而更新页面的 list
      this.triggerEvent('fresh');
    },
    deleteHandle() {
      wx.showModal({
        title: '确认删除',
        content: "是否要删除此项任务？",
        success: res => {
          if (res.confirm) {
            // 1. 拿到用户点击的这一项
            const item = this.properties.item;
            // 2. 从全局数据中找到这一项的下标
            const index = app.globalData.list.findIndex(i => i.content === item.content);
            // 3. 从全局中删除这一项
            app.globalData.list.splice(index, 1);

            wx.showToast({
              title: '删除成功',
            });

            this.triggerEvent("fresh");
          }
        }
      })
    }
  }
})
```

### 要点说明

- 使用 `Component` 构造器创建自定义组件
- `properties` 接收父组件传入的 `item` 对象（包含任务内容和完成状态）
- **checkHandle**：点击复选框切换任务完成状态，同时更新全局数据和当前组件数据，并通过 `this.triggerEvent('fresh')` 通知父组件刷新列表
- **deleteHandle**：点击删除图标，弹出确认对话框，确认后从全局数据中删除该任务，并通知父组件刷新
- 完成的任务文字会添加删除线样式（`.finished`）

## 任务清单页面（pages/index）

### 页面模板（pages/index/index.wxml）

```xml
<view class="inputContainer">
  <input
    type="text"
    class="input"
    placeholder="请输入新的待办事项..."
    model:value="{{newContent}}"
    bindinput="inputHandle"
  />
</view>
<button
  class="btn"
  style="width: 70%;"
  bindtap="add"
>创建一个任务</button>
<view class="title">共 {{list.length}} 项任务</view>
<!-- 下面会根据是否有任务有一个判断 -->
<view wx:if="{{list.length === 0}}" class="tip">- 请添加任务 -</view>
<view wx:else class="listContainer">
  <!-- 根据任务的长度来显示对应的任务条目 -->
  <item wx:for="{{list}}" wx:key="this" item="{{item}}" bindfresh="fresh"></item>
</view>
```

### 页面逻辑（pages/index/index.js）

```javascript
// index.js
// 获取应用实例
const app = getApp()

Page({
  data: {
    newContent : "", // 用户输入新的待办事项
    list: app.globalData.list // 从全局 list 中获取所有任务
  },
  inputHandle(){},
  add(){
    // 添加新的待办事项
    if(this.data.newContent){
      // 添加到全局数据
      app.globalData.list.unshift({
        content : this.data.newContent,
        isComplete: false
      });
      this.setData({
        newContent: '',
        list: app.globalData.list
      });
      wx.showToast({
        title: '新增任务成功',
        icon:'success'
      })
    } else {
      wx.showToast({
        title: '请输入内容',
        icon:'error'
      })
    }
  },
  fresh(){
    // 更新当前页面的 list
    // 因为在不同页面，针对 item 不同的状态会有不同的处理
    // 所以每次点击了 item 之后必须要更新 list
    this.setData({
      list : app.globalData.list
    })
  },
  onShow(){
    this.fresh();
  },
})
```

### 要点说明

- 通过 `model:value` 实现输入框与 `newContent` 的双向绑定
- `add` 方法：将新任务添加到全局数据的头部（`unshift`），清空输入框并刷新列表
- `fresh` 方法：从全局数据刷新当前页面的列表，由子组件的 `triggerEvent('fresh')` 触发
- `onShow` 生命周期：每次页面显示时刷新列表，确保从其他 Tab 切换回来时数据是最新的
- 使用 `wx:if / wx:else` 条件渲染：无任务时显示提示文字，有任务时显示任务列表
