# 微信API—扫码案例演示

本节课程演示微信小程序的扫码功能，使用 `wx.scanCode` API 调起扫码界面，识别二维码或条形码中的内容。

## 核心知识点

- `wx.scanCode`：调用客户端扫码界面进行扫码
- `onlyFromCamera` 参数：是否只允许相机扫码，不允许从相册选择
- 扫码成功后通过 `result` 获取扫码结果

## 页面逻辑（index.js）

```javascript
Page({
  data: {
    img : "",  // 存储扫码结果（这里扫码结果是一个图片地址）
  },
  // 从相机和相册扫码
  scanCode1(){
    wx.scanCode({
      // 扫码成功后，就能够得到该条码上面的信息
      success :({result})=>{
        this.setData({
          img : result
        })
      }
    })
  },
  // 只允许从相机扫码
  scanCode2(){
    wx.scanCode({
      onlyFromCamera : true,
      success :({result})=>{
        this.setData({
          img : result
        })
      }
    })
  }
})
```

## 页面结构（index.wxml）

```xml
<view class='container'>
  <view class='section'>
    <button bindtap='scanCode1'>从相机和相册扫码</button>
    <button bindtap='scanCode2'>只允许从相机扫码</button>
    <image 
      src="{{img}}"
      mode="aspectFill"
    ></image>
  </view>
</view>
```

## 小结

本节介绍了 `wx.scanCode` 的基本用法。通过设置 `onlyFromCamera` 参数可以控制是否只允许通过相机扫码。扫码成功后，`result` 字段包含了二维码或条形码中存储的内容，开发者可以根据内容进行后续处理。
