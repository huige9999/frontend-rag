# movable组件案例

本节课通过一个仿微信消息列表滑动删除的案例，演示了 `movable-area` 和 `movable-view` 组件的使用，实现了消息条目左右滑动显示"删除"按钮的交互效果。

---

## 页面结构（index.wxml）

```xml
<view class="title">基础示例</view>
<!-- 可移动区域 -->
<movable-area class="area">
  <movable-view
    class="view"
    direction="all"
    inertia
    out-of-bounds
  ></movable-view>
</movable-area>

<view class="title">案例演示</view>
<movable-area class="container" wx:for="{{messages}}" wx:key="this">
  <!-- 消息 -->
  <movable-view
    class="movable-view"
    direction="horizontal"
    out-of-bounds
    friction="150"
    x="{{item.x}}"
    bindtouchstart="touchstartHandle"
    bindtouchend="touchendHandle"
    data-index="{{index}}"
  >
    <!-- 头像 -->
    <image src="{{item.avatar}}" mode=""/>
    <!-- 消息 -->
    <view class="content">
      <view class="top">
        {{item.nickname}}
      </view>
      <view class="bottom">
        {{item.msg}}
      </view>
    </view>
    <!-- 时间 -->
    <view class="timeBox">
      <view class="top">
        {{item.time}}
      </view>
      <view class="bottom">
        ...
      </view>
    </view>
  </movable-view>
  <view class="delete">
    删除
  </view>
</movable-area>
```

### 要点说明

- **movable-area**：可移动区域，必须作为 `movable-view` 的父容器
- **movable-view**：可移动视图，主要属性：
  - `direction`：移动方向，`horizontal`（水平）、`vertical`（垂直）、`all`（全方向）
  - `inertia`：是否带有惯性
  - `out-of-bounds`：是否可以超出可移动区域
  - `friction`：摩擦系数，值越大滑动越不灵敏
  - `x`：设置 x 坐标偏移量，用于控制滑动位置
- 每条消息都由一个 `movable-area` 包裹，内部有一个 `movable-view`（消息内容）和一个"删除"按钮
- 通过 `bindtouchstart` 和 `bindtouchend` 监听触摸事件来判断滑动方向和距离
- 通过 `data-index` 传递当前项的索引，用于在事件处理中定位具体消息

## 页面逻辑（index.js）

```javascript
Page({
  data: {
    messages: [{
        avatar: "/imgs/1.jpeg",
        nickname: "大杨",
        msg: "几点到学校集合？",
        time: "12:07",
        x: 0
      },
      {
        avatar: "/imgs/2.jpeg",
        nickname: "进哥",
        msg: "那道题我解决了，我一会儿给你讲",
        time: "11:05",
        x: 0
      },
      {
        avatar: "/imgs/3.jpeg",
        nickname: "小恐龙",
        msg: "哥，明天有空没有，一起去世贸看电影呗",
        time: "10:37",
        x: 0
      },
      {
        avatar: "/imgs/4.jpeg",
        nickname: "雅静",
        msg: "你最好就在那儿等着我，我到时候过去找你",
        time: "05:20",
        x: 0
      },
      {
        avatar: "/imgs/5.jpeg",
        nickname: "小黄",
        msg: "真的哭死啦 orz",
        time: "03:47",
        x: 0
      },
      {
        avatar: "/imgs/6.jpeg",
        nickname: "婉静",
        msg: "你明天几点的课？",
        time: "01:02",
        x: 0
      }
    ],
    start_x : 0, // 记录开始拖动时的 x 值
  },
  touchstartHandle(event){
    this.setData({
      start_x : event.touches[0].clientX
    })
  },
  touchendHandle(event){
    // 拖动结束时的 clientx
    const current_x = event.changedTouches[0].clientX;
    // 根据起始和结束的 clientx
    // 我们就可以做一个拖动方向以及拖动距离的判断
    const direction = current_x - this.data.start_x;

    const index = ~~event.currentTarget.dataset.index;
    const arr = [...this.data.messages];

    if(direction < 0){
      if(direction < -30){
        // 我们认为用户已经拖动了一段距离
        // 哪怕他没有拖动满，我们应该将删除显示出来
        arr[index].x = -120;
      } else {
        arr[index].x = 0;
      }
      this.setData({
        messages : arr
      })
    } else {
      arr[index].x = 0;
      this.setData({
        messages : arr
      })
    }
  }
})
```

### 要点说明

- 每条消息数据包含 `avatar`（头像）、`nickname`（昵称）、`msg`（消息内容）、`time`（时间）、`x`（水平偏移量）
- `touchstartHandle`：记录触摸开始时的 x 坐标
- `touchendHandle`：触摸结束时，根据滑动方向和距离决定消息条目的位置：
  - 向左滑动超过 30px：将 `x` 设为 -120，露出删除按钮
  - 向左滑动未超过 30px 或向右滑动：将 `x` 设为 0，恢复原位
- `~~` 是取整的简写方式，等同于 `Math.floor` 对正数取整

- 容器 `container` 通过 `left: -120rpx` 和 `width: calc(100% + 120rpx)` 隐藏右侧的删除按钮区域

- 向左滑动时，`movable-view` 移开，露出底部的红色"删除"按钮
- 删除按钮宽度 120rpx，与 `x = -120` 的偏移量相对应
