# 16. 发布组件库

## 概述

代码的**开源**和**发布**是两个不同的概念：

- **开源**：将源码托管到公开平台（GitHub 或 Gitee），方便其他开发者查看和学习源代码
- **发布**：将打包后的代码发布到 npm（或其他包管理服务器），方便其他项目通过 `npm install` 安装使用。发布时一般只发布打包后的代码（dist 目录）

发布到 npm 的两个步骤：

1. 打包代码
2. 进行发布

## 黑白名单的配置

### 黑名单模式

在项目根目录下创建 `.npmignore` 文件，类似于 `.gitignore`，记录不需要发布的文件或目录：

```bash
# .npmignore
src
tests
```

**弊端**：源码发生更改（增加文件或目录）时，`.npmignore` 也需要及时更新，比较繁琐。

### 白名单模式（推荐）

在 `package.json` 中增加 `files` 字段，只有在白名单中出现的文件或目录才会被上传到 npm：

```json
"files": [
  "dist"
]
```

这种方式更加简洁，只需要维护需要发布的内容，不用担心遗漏新增的文件。

## 代码发布

### 1. 检查镜像源

发布到 npm 需要确保当前镜像源为 npm 官方镜像：

```bash
# 查看当前镜像
npm config get registry

# 设置为淘宝镜像
npm config set registry=https://registry.npm.taobao.org

# 设置为 npm 官方镜像
npm config set registry=https://registry.npmjs.org
```

### 2. 登录 npm 账号

```bash
npm login
```

按照提示输入用户名、密码和邮箱。

### 3. 检查包名是否已存在

在发布之前，需要检查要发布的库在 npm 上是否已存在同名包。包名取决于 `package.json` 中的 `name` 字段。如果已存在同名包，则无法发布。

### 4. 发布

```bash
npm publish
```

发布成功后，其他开发者就可以通过 `npm install duyi-plus2` 或 `pnpm add duyi-plus2` 来安装使用你的组件库了。

```json
{
  "name": "duyi-plus2",
  "version": "1.0.0",
  "description": "this is a Vue3 components library",
  "type": "module",
  "license": "MIT",
  "keywords": [
    "vue3",
    "components",
    "library"
  ],
  "files": [
    "dist"
  ],
  "module": "dist/es/duyi-plus.js",
  "main": "dist/umd/duyi-plus.umd.cjs",
  "exports": {
    ".": {
      "import": "./dist/es/duyi-plus.js",
      "require": "./dist/umd/duyi-plus.umd.cjs"
    },
    "./style.css": {
      "import": "./dist/style.css",
      "require": "./dist/umd/style.css"
    }
  },
  "peerDependencies": {
    "vue": "^3.3.4"
  }
}
```
