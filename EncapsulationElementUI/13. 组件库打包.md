# 13. 组件库打包

## 概述

在组件库开发完成后，需要将其打包为可供其他项目使用的格式。常见的模块格式有两种：

- **ES Module (ESM)**：现代打包工具（Vite、Webpack 等）优先使用的格式，支持 Tree Shaking
- **UMD (Universal Module Definition)**：兼容性更好的格式，可以直接在浏览器中通过 `<script>` 标签引入

## 书写入口文件

入口文件 `src/bundle.js` 是打包的起点，负责引入所有组件并提供统一的注册方法。

```js
// 该文件是用于打包的入口文件

/* import the fontawesome core */
import { library } from '@fortawesome/fontawesome-svg-core'
/* import font awesome icon component */
import { FontAwesomeIcon } from '@fortawesome/vue-fontawesome'
/* import all icons */
import { fas } from '@fortawesome/free-solid-svg-icons'

/* add icons to the library */
library.add(fas)

// 引入样式
import './styles/index.scss'

// 引入自定义组件
import Icon from './components/Icon/Icon.vue'
import Button from './components/Button'
import Card from './components/Card/Card.vue'
import Dialog from './components/Dialog/Dialog.vue'
import Pager from './components/Pager/Pager.vue'
import Collapse from './components/Collapse/Collapse.vue'
import CollapseItem from './components/Collapse/CollapseItem.vue'
import Tooltip from './components/Tooltip/Tooltip.vue'
import Dropdown from './components/Dropdown/Dropdown.vue'

// 接下来就和 main.js 不一样了，接下来的重点是提供一个 install 方法

const componentsArr = [Icon, Button, Card, Dialog, Pager, Collapse, CollapseItem, Tooltip, Dropdown]

// 这个就是我们要对外提供的 install 方法
const install = (app) => {
  // 这里要做的事情很简单，就是遍历注册所有的组件
  componentsArr.forEach((component) => {
    app.component(component.name, component)
  })
  // 有一个组件要单独注册
  app.component('font-awesome-icon', FontAwesomeIcon)
}

// 最后一步，就是导出
// 注意这里的导出分为两种：全部导出和按需导出

// 全部导出
export default {
  install
}

// 按需导出
export { install, Icon, Button, Card, Dialog, Pager, Collapse, CollapseItem, Tooltip, Dropdown }
```

### 组件单独注册入口

每个组件也可以提供单独的 `install` 方法，以支持按需引入。以 Button 组件为例：

```js
// src/components/Button/index.js
import Button from './Button.vue'

// 为组件单独提供 install 安装方法，供按需引入
Button.install = function (app) {
  app.component(Button.name, Button)
}

export default Button
```

## 书写 ES Module 打包配置

创建 `vite.es.config.js`，专门用于打包 ES Module 格式：

```js
// 打包 ES 模块所用的配置文件

import { fileURLToPath, URL } from 'node:url'
import { resolve, dirname } from 'node:path'
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import vueJsx from '@vitejs/plugin-vue-jsx'

const __dirname = dirname(fileURLToPath(import.meta.url))

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [vue(), vueJsx()],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  },
  publicDir: 'empty',
  build: {
    outDir: 'dist/es',
    lib: {
      // 打包的入口文件
      entry: resolve(__dirname, 'src/bundle.js'),
      name: 'DuyiPlus',
      filename: 'duyi-plus',
      formats: ['es']
    },
    // vite 内部使用的是 rollup 进行打包
    // rollupOptions 可以书写 rollup 的配置
    rollupOptions: {
      // 下面的这些依赖都是不需要打包到我们的组件库里面的
      external: [
        'vue',
        '@fortawesome/fontawesome-svg-core',
        '@fortawesome/free-solid-svg-icons',
        '@fortawesome/vue-fontawesome',
        '@popperjs/core',
        'lodash-es'
      ]
    }
  }
})
```

**关键配置说明：**

- `publicDir: 'empty'`：不复制 public 目录下的静态资源
- `lib.formats: ['es']`：只输出 ES Module 格式
- `rollupOptions.external`：将 vue、fontawesome、popper、lodash 等依赖标记为外部依赖，不打包进组件库

## 书写 UMD 打包配置

创建 `vite.umd.config.js`，专门用于打包 UMD 格式：

```js
// 打包 UMD 模块所有的 vite 配置

import { fileURLToPath, URL } from 'node:url'
import { resolve, dirname } from 'node:path'
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import vueJsx from '@vitejs/plugin-vue-jsx'

const __dirname = dirname(fileURLToPath(import.meta.url))

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [vue(), vueJsx()],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  },
  publicDir: 'empty',
  build: {
    outDir: 'dist/umd',
    lib: {
      // 打包的入口文件
      entry: resolve(__dirname, 'src/bundle.js'),
      name: 'DuyiPlus',
      filename: 'duyi-plus',
      formats: ['umd']
    },
    // vite 内部使用的是 rollup 进行打包
    // rollupOptions 可以书写 rollup 的配置
    rollupOptions: {
      // 下面的这些依赖都是不需要打包到我们的组件库里面的
      external: ['vue'],
      output: {
        exports: 'named',
        // globals 对应的是上面 external 中的依赖
        // 指令了通过什么名字去全局引入这个依赖
        globals: {
          vue: 'Vue'
        }
      }
    }
  }
})
```

**与 ES 配置的区别：**

- `formats: ['umd']`：输出 UMD 格式
- `external` 只排除 `vue`，其他依赖（如 fontawesome）会打包进 UMD 产物，因为 UMD 主要用于浏览器直接引入，无法像 npm 那样自动安装依赖
- `output.globals`：指定外部依赖 `vue` 在浏览器中对应的全局变量名 `Vue`

## 修改 package.json

```json
{
  "name": "duyi-plus2",
  "version": "1.0.0",
  "description": "this is a Vue3 components library",
  "type": "module",
  "license": "MIT",
  "keywords": [
    "vue3",
    "components",
    "library"
  ],
  "files": [
    "dist"
  ],
  "module": "dist/es/duyi-plus.js",
  "main": "dist/umd/duyi-plus.umd.cjs",
  "exports": {
    ".": {
      "import": "./dist/es/duyi-plus.js",
      "require": "./dist/umd/duyi-plus.umd.cjs"
    },
    "./style.css": {
      "import": "./dist/style.css",
      "require": "./dist/umd/style.css"
    }
  }
}
```

**关键字段说明：**

| 字段 | 说明 |
|------|------|
| `files` | 白名单模式，只发布 `dist` 目录到 npm |
| `module` | 指向 ES Module 入口，供现代打包工具使用 |
| `main` | 指向 UMD 入口，供旧版工具或 CommonJS 方式使用 |
| `exports` | 定义子路径导出，`./style.css` 允许用户单独引入样式文件 |
| `peerDependencies` | 对等依赖，声明本库需要宿主项目安装 vue@^3.3.4 |

### peerDependencies 说明

有一些依赖需要修改为 `peerDependencies`（对等依赖），表示组件库需要宿主项目安装这些依赖才能正常工作，但安装组件库时不会自动安装它们：

```json
"peerDependencies": {
  "vue": "^3.3.4"
}
```

## 配置打包命令

在 `package.json` 的 `scripts` 中添加打包相关命令：

```json
"scripts": {
  "dev": "vite",
  "build": "pnpm build-es && pnpm build-umd && pnpm move-style",
  "build-es": "vite build --config vite.es.config.js",
  "build-umd": "vite build --config vite.umd.config.js",
  "move-style": "move-file dist/es/style.css dist/style.css"
}
```

**打包流程：**

1. `pnpm build-es`：使用 ES 配置打包，输出到 `dist/es/`
2. `pnpm build-umd`：使用 UMD 配置打包，输出到 `dist/umd/`
3. `pnpm move-style`：将 `dist/es/style.css` 移动到 `dist/style.css`，方便用户统一引用

执行 `pnpm build` 即可完成全部打包。
