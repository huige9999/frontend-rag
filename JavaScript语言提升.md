## 2. 章节逻辑说明

这门课是「JS 语言基础」之后的 **ES6+ 语言提升**，主线是**「语法糖（01~04）→ 异步原理（05）→ 异步方案 Promise（06~08）→ 现代异步语法（09）→ 面试冲刺（10）」**，层层递进、收口在面试题。

### 先讲什么：ES6+ 语法糖四件套（01~04）

先用 4 章把日常写代码最高频的语法变化过一遍：

- **01 零碎的改动**：`let/const`、箭头函数、解构、模板字符串、展开运算符等零散但常用的 ES6+ 语法。
- **02 数组 / 03 对象 / 04 函数**：分别围绕数组、对象、函数三大数据载体讲 ES6+ 新增的能力与写法。

**为什么先讲语法糖**：这些是「立刻能用、改写老代码」的部分，先建立对新语法的肌肉记忆，后面看异步代码才不会被语法绊住。

### 再讲什么：事件循环（05）

- **05 事件循环**：调用栈、宿主环境、消息队列、微任务队列——讲清 JS 单线程下「代码到底按什么顺序执行」。

**为什么单独把事件循环插在这里**：它是后面理解 Promise、async/await 执行时序的**共同地基**。不先讲事件循环，Promise 的「为什么先同步后异步、微任务先于宏任务」就讲不透。

### 然后讲什么：异步主线 Promise（06~09）

这是这门课最重的一段，按「基础 → 链式 → 静态方法 → 语法糖」严格递进：

- **06 Promise基础**：什么是 Promise、三种状态、then 的基本用法。
- **07 Promise的链式调用**：链式 then、返回值穿透、错误冒泡。
- **08 Promise的静态方法**：`all` / `race` / `allSettled` 等并发控制方法。
- **09 async和await**：Promise 的同步式语法糖，try/catch 捕获。

**为什么这么安排**：Promise 是 JS 异步的核心方案，必须先吃透原生 Promise（06~08）才能理解 async/await（09）这个语法糖背后在做什么。

### 最后讲什么：面试题冲刺（10）

- **10 Promise相关面试题**：把前面异步知识压成面试题集中训练。

### 主线一句话总结

**ES6+ 语法糖（01~04）→ 事件循环原理（05）→ Promise 异步方案（06~08）→ async/await 语法糖（09）→ 面试题冲刺（10）**，目标是从「会写老 JS」进阶到「熟练用 ES6+ 语法 + 真正理解异步、能答异步面试题」。

## 3. 使用场景等元信息沉淀

- **学完能用在哪**：日常业务代码全面使用 ES6+ 写法；独立处理任意异步流程（请求并发、串行、错误兜底）；应对 JS 异步面试。
- **和实际项目的关系**：Promise / async-await 是所有接口请求、定时任务、动画序列的基础；事件循环是排查「为什么顺序不对」的根本依据。
- **和其他课程的关系**：上承《JavaScript 语言基础》，下接《程序思维训练》《网络》；与《ES6详细版》《EcmaScript核心原理》在深度上互补——本课偏实战，那两门偏原理（含手写 Promise 等）。
- **面试/教学高频场景**：事件循环输出题（05）、Promise 执行顺序（06~08）、async/await（09），是本课全部的价值落点。
