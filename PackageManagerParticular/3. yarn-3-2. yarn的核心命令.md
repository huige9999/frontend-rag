# yarn 的核心命令

## 1. 初始化

```shell
yarn init [--yes/-y]
```

## 2. 安装

添加指定包：

```shell
yarn [global] add package-name [--dev/-D] [--exact/-E]
```

安装 package.json 中的所有依赖：

```shell
yarn install [--production/--prod]
```

## 3. 脚本和本地CLI

运行脚本：

```shell
yarn run 脚本名
```

> start、stop、test 可以省略 run

运行本地安装的 CLI：

```shell
yarn run CLI名
```

### 示例：package.json 中的脚本配置

```json
{
  "name": "chapter3",
  "version": "1.0.0",
  "main": "index.js",
  "license": "MIT",
  "scripts": {
    "abc": "dir",
    "start": "node index.js"
  },
  "dependencies": {}
}
```

上述配置中，定义了两个脚本：

- `abc`：执行 `dir` 命令（列出当前目录文件）
- `start`：执行 `node index.js`（启动入口文件）

由于 `start` 属于特殊脚本名，可以省略 `run`，直接使用：

```shell
yarn start
# 等同于 yarn run start
```

而 `abc` 不是特殊脚本名，必须使用完整写法：

```shell
yarn run abc
```

### 入口文件示例

```js
// index.js
console.log("Hello")
```

## 4. 查询

查看 bin 目录：

```shell
yarn [global] bin
```

查询包信息：

```shell
yarn info 包名 [子字段]
```

列举已安装的依赖：

```shell
yarn [global] list [--depth=依赖深度]
```

> yarn 的 list 命令和 npm 的 list 不同，yarn 输出的信息更加丰富，包括顶级目录结构、每个包的依赖版本号

## 5. 更新

列举需要更新的包：

```shell
yarn outdated
```

更新包：

```shell
yarn [global] upgrade [包名]
```

## 6. 卸载

卸载包：

```shell
yarn remove 包名
```
