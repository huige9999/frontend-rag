# 其他 npm 命令

## 安装

1. **精确安装最新版本**（不使用语义版本，直接锁定确切版本号）

```shell
npm install --save-exact 包名
npm install -E 包名
```

2. **安装指定版本**

```shell
npm install 包名@版本号
```

例如：`npm install jquery@3.4.1` 会安装 jquery 的 3.4.1 版本，并在 package.json 中记录为确切版本号 `"jquery": "3.4.1"`（不带 `^` 或 `~` 前缀）。

### package.json 版本对比示例

```json
{
  "dependencies": {
    "jquery": "3.4.1",       // 使用 -E 精确安装，没有前缀
    "lodash": "^2.4.2"       // 普通安装，带有 ^ 前缀
  }
}
```

## 查询

1. **查询包安装路径**

```shell
npm root [-g]
```

不加 `-g` 查看本地安装路径，加 `-g` 查看全局安装路径。

2. **查看包信息**

```shell
npm view 包名 [子信息]
# view aliases：v info show
```

例如：
- `npm view lodash` - 查看 lodash 的完整信息
- `npm view lodash versions` - 查看 lodash 的所有历史版本
- `npm view lodash repository` - 查看 lodash 的仓库地址

3. **查询已安装的包**

```shell
npm list [-g] [--depth=依赖深度]
# list aliases: ls la ll
```

例如：
- `npm list` - 查看当前项目安装的包
- `npm list --depth=0` - 只显示顶层依赖
- `npm list -g --depth=0` - 查看全局安装的顶层包

## 更新

1. **检查有哪些包需要更新**

```shell
npm outdated
```

2. **更新包**

```shell
npm update [-g] [包名]
# update aliases: up、upgrade
```

例如：
- `npm update` - 更新所有包
- `npm update lodash` - 只更新 lodash
- `npm update -g` - 更新全局包

## 卸载包

```shell
npm uninstall [-g] 包名
# uninstall aliases: remove, rm, r, un, unlink
```

例如：
- `npm uninstall lodash` - 卸载本地 lodash
- `npm uninstall -g nodemon` - 卸载全局 nodemon

> 卸载包的同时，npm 会自动从 package.json 中移除对应的依赖记录。

## npm 配置

npm 的配置会对其他命令产生或多或少的影响

安装好 npm 之后，最终会产生两个配置文件，一个是用户配置，一个是系统配置。当两个文件的配置项有冲突的时候，用户配置会覆盖系统配置

通常，我们不关心具体的配置文件，而只关心最终生效的配置

通过下面的命令可以查询目前生效的各种配置：

```shell
npm config ls [-l] [--json]
```

另外，可以通过下面的命令操作配置：

1. **获取某个配置项**

```shell
npm config get 配置项
```

2. **设置某个配置项**

```shell
npm config set 配置项=值
```

3. **移除某个配置项**

```shell
npm config delete 配置项
```

> **关键要点**：npm 提供了丰富的命令来管理包的生命周期。常用的有 `npm view` 查看包信息、`npm list` 查看已安装包、`npm outdated` 检查更新、`npm update` 更新包、`npm uninstall` 卸载包。配置管理方面，`npm config set/get` 可以方便地操作 npm 的各种配置项（如 registry 镜像源地址）。
