# 发布包

## 准备工作

1. **移除淘宝镜像源**（发布包需要连接 npm 官方 registry）
2. 到 npm 官网注册一个账号，并完成邮箱认证
3. 本地使用 npm cli 进行登录
   1. 使用命令 `npm login` 登录
   2. 使用命令 `npm whoami` 查看当前登录的账号
   3. 使用命令 `npm logout` 注销
4. 创建工程根目录
5. 使用 `npm init` 进行初始化

## 发布

1. 开发
2. 确定版本
3. 使用命令 `npm publish` 完成发布

## 发布包示例

### 包的代码 (test12345/index.js)

以下是一个简单的发布包示例，该包封装了 lodash 的 `compact` 方法（用于去除数组中的假值）：

```js
var _ = require("lodash");

module.exports = function (arr) {
    return _.compact(arr);
};
```

### 包的配置 (test12345/package.json)

```json
{
  "name": "testabcd1234",
  "version": "1.0.1",
  "description": "this is test package",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [
    "test"
  ],
  "author": "bangbangji <2782170452@qq.com>",
  "license": "MIT",
  "dependencies": {
    "lodash": "^4.17.15"
  }
}
```

需要注意的配置项：
- **name**：包名必须唯一，不能与 npm 上已有的包名重复
- **version**：每次发布新版本时需要更新版本号
- **main**：指定包的入口文件
- **author**：格式为 `账户名 <邮箱>`，必须是有效的 npm 账户
- **license**：开源协议，通常使用 MIT

### 包的说明文档 (test12345/README.md)

发布包时建议提供 README 文件：

```markdown
# 说明文档

啥也没有
```

### 使用发布的包 (another/index.js)

发布成功后，其他人可以通过 `npm install` 安装并使用你的包：

```js
var func = require("testabcd1234");
var newArr = func([45, 7, 0, false, "", 21]);
console.log(newArr); // 输出: [45, 7, 21]
```

使用该包的项目配置 (another/package.json)：

```json
{
  "name": "another",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC",
  "dependencies": {
    "testabcd1234": "^1.0.1"
  }
}
```

## 开源协议

发布包时需要选择一个合适的开源协议，常见的开源协议包括：

- **MIT**：最宽松的协议之一，允许任何人自由使用、修改、分发，只需保留版权声明
- **Apache License 2.0**：类似 MIT，但包含专利授权条款
- **GPL**：要求衍生作品也必须开源

可以通过网站 http://choosealicense.online/appendix/ 选择协议，并复制协议内容
