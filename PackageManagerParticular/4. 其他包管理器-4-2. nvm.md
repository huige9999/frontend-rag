# nvm

nvm并非包管理器，它是用于管理多个node版本的工具

在实际的开发中，可能会出现多个项目分别使用的是不同的node版本，在这种场景下，管理不同的node版本就显得尤为重要

nvm就是用于切换版本的一个工具

## 下载和安装

最新版下载地址：https://github.com/coreybutler/nvm-windows/releases

下载nvm-setup.zip后，直接安装

## 使用nvm

nvm提供了CLI工具，用于管理node版本

在终端中输入nvm，以查看各种可用命令

> 为了加快下载速度，建议设置淘宝镜像
> node淘宝镜像：https://npm.taobao.org/mirrors/node/
> npm淘宝镜像：https://npm.taobao.org/mirrors/npm/

## 常用命令

```shell
# 查看可用的node版本
nvm list available

# 安装指定版本的node
nvm install <version>

# 切换到指定版本的node
nvm use <version>

# 查看已安装的node版本
nvm list

# 卸载指定版本的node
nvm uninstall <version>
```

## 设置镜像加速

```shell
# 设置node镜像
nvm node_mirror https://npm.taobao.org/mirrors/node/

# 设置npm镜像
nvm npm_mirror https://npm.taobao.org/mirrors/npm/
```

## 总结

nvm 的核心价值在于：

1. **多版本管理**：在同一台机器上安装和切换多个 Node.js 版本
2. **项目隔离**：不同项目可以使用不同的 Node.js 版本，互不影响
3. **便捷切换**：通过简单的命令即可在版本之间切换
