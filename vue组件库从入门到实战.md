## 2. 章节逻辑说明

这门课是《vue 从入门到实战》的进阶实战，主线是**「基于 vue-element-admin 后台脚手做一个完整的 CMS 后台管理系统，并部署上线」**：**项目准备（03~05）→ 登录鉴权（06~08）→ 内容模块开发（09~17）→ 上线部署（18）**。它教的核心不是 Vue 语法（那由前一门课解决），而是「如何用成熟的组件库 + 后台脚手，按模块把一个中后台系统搭起来」。

### 第一段：项目准备（03~05）

- **03 介绍 vue-element-admin**：认识这套业内最常用的 Vue 后台脚手及其目录结构、约定。
- **04 搭建项目架子**：基于 element-admin 把 `background-system2` 后台架子搭起来。
- **05 服务器和接口**：配独立后端 `mysite-server` + MongoDB，搞清接口地址（配 Mongodb 安装文档）。

**为什么先讲准备**：本课前置假设你已会 Vue，所以直接从「认识脚手 + 搭架子 + 接通后端接口」开始，为后续按模块开发铺好地基。

### 第二段：登录与鉴权（06~08）

- **06 登录界面制作 / 07 登录功能制作 / 08 鉴权功能制作**：登录界面 → 登录逻辑 → token 鉴权流程（element-admin 的鉴权流程是重点，配鉴权.md 课件）。

**为什么登录鉴权排第二**：中后台系统「进来要先登录」，登录鉴权是所有后续模块的前置门槛，必须先打通。

### 第三段：内容模块开发（09~17）

打通登录后，按「站点内容」逐模块开发后台 CRUD：

- **09~10 首页标语 part1/2**：首页标语渲染 + 编辑 + 图片上传。
- **11~14 文章模块 part1~4**：列表渲染/分页/删除 → 文章分类 → 添加编辑文章（配 Markdown 编辑器笔记）→ 优化添加编辑。
- **15 项目管理模块 / 16 评论管理和留言板模块 / 17 关于我、设置和个人信息模块**：把后台其余业务模块补全。

**为什么这么安排**：每个 part 都对应一个真实后台 CRUD 场景，从「列表 + 分页 + 删除」到「分类」到「新增编辑」再到「优化」，是中后台最典型的开发节奏。

### 第四段：上线部署（18）

- **18 项目部署到阿里云**：把后台 + 前台 + 后端部署到阿里云（配项目部署.md + 补充内容.md）。

### 主线一句话总结

**认识 vue-element-admin + 搭架子 + 接接口（03~05）→ 登录鉴权（06~08）→ 内容模块 CRUD（09~17）→ 部署上线（18）**，目标是学会「用成熟后台脚手 + 组件库，按模块从零搭出一个可上线的中后台管理系统」。

## 3. 使用场景等元信息沉淀

- **学完能用在哪**：基于 element-admin 独立开发并部署一个带登录鉴权的后台管理系统；理解后台 CMS 的通用结构与鉴权套路。
- **和实际项目的关系**：mysite2（后台 + 前台 + 后端）是一套可直接做作品集的全栈级项目；element-admin 的鉴权/动态路由模式是业内中后台的事实范式。
- **和其他课程的关系**：前置是《vue 从入门到实战》（要先会用 Vue）；后端 `mysite-server` 与《Node》《MongoDB》相关；部署（18）涉及服务器运维。它和《从零封装类 ElementUI 组件库》互补——本课「用」组件库做后台，那门课「造」组件库。
- **面试/教学高频场景**：vue-element-admin 鉴权流程（08）、登录 token 流程（07）、中后台 CRUD + 分页 + 图片上传（09~14）、部署（18）。
