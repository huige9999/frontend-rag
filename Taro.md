## 2. 章节逻辑说明

这门课是一条**「环境与框架认知 → React 式组件与 JSX → 基础能力 → 内置组件库大全 → 高阶组件模式 → 设备/媒体 API → 综合实战项目」**的递进主线。先建立「Taro 是什么、怎么起项目」，再用 React 的组件思维写小程序，接着把小程序原生组件库全过一遍，然后补齐函数式组件/RenderProps/Refs 等组件进阶能力，之后系统讲解设备 API 与媒体 API，最后用大量真实案例（聊天机器人、涂鸦板、音乐播放器、云存储、社交圈）把所有能力落地。

### 第一篇 入门与环境配置（1~8）：先知道 Taro 是什么、怎么跑起来

- **1 认识Taro**：Taro 是一套「React 语法写一次、编译到多端（微信/支付宝/H5/RN 等）小程序」的框架，无源码。
- **2 下载与安装 / 3 框架简介**：脚手架初始化项目，认识 Taro 的工程结构与编译思路。
- **4 全局生命周期 / 5~6 页面配置与页面间处理函数**：App 与页面的生命周期、页面配置（`window`/`tabBar` 等）、页面间数据传递。
- **7 Taro规范 / 8 项目配置**：Taro 的代码规范与 `project.config.json`/`config` 等项目配置。

**为什么先讲**：不先把「Taro 是什么、项目怎么起、生命周期在哪」搞清楚，后面写组件会无从下手。这是把 React 开发者带进小程序世界的前置认知。

### 第二篇 组件与 JSX（9~14）：用 React 思维写小程序

- **9 组件开发 / 10 组件的生命周期**：class 组件写法、组件挂载/更新/卸载生命周期。
- **11 JSX在Taro中的使用**：Taro 里 JSX 的用法与限制。
- **12 组件化开发与props**：组件拆分、父子通过 props 通信。
- **13 计时器案例开发**：用计时器把组件 + 状态串起来。
- **14 事件处理**：事件绑定、`e.stopPropagation`、传参。

**为什么这么安排**：Taro 的核心卖点就是「用 React 写小程序」，所以入门之后立刻把「组件 + JSX + props + 事件」这套 React 基本盘讲透，是后面所有章节的基础。

### 第三篇 基础能力（15~20）：补齐写业务必用的能力

- **15 路由功能 / 16 静态资源引用 / 17 外部样式与全局样式**：跳转传参、图片/字体引用、样式作用域。
- **18 条件渲染 / 19 列表渲染**：JSX 的 `&&`/三元、`map` 渲染列表。
- **20 案例-图片新闻**：把路由 + 列表 + 图片组合成一个图文列表小案例。

**为什么这么安排**：路由、资源、样式、条件/列表渲染是写任何小程序页面都绕不开的能力，补齐后才有资格做后面的组件库和实战。

### 第四篇 组件库大全（21~31）：系统过一遍小程序原生组件

- **21 视图容器**（view/scroll-view/swiper 等）/ **22 基础容器**（text/icon/progress 等）。
- **23~24 表单容器**（input/textarea/picker/switch/checkbox/radio 等）+ **25 案例-表单综合操作**。
- **26 导航组件**（navigator）/ **27~28 媒体组件**（image/audio/video/camera 等）/ **29 地图**（map）/ **30 画布**（canvas）/ **31 开放能力**（open-data 等）。

**为什么这么安排**：Taro 本质是对小程序原生组件的封装，这一篇相当于「小程序组件字典」。先视图/基础/表单打地基，再用一个表单综合案例巩固，再按媒体/地图/画布/开放能力分门别类扫尾。

### 第五篇 高阶组件能力（32~35）：React 式的组件复用模式

- **32 函数式组件**：无状态函数组件写法。
- **33 Children与组合**：用 `props.children` 做组合。
- **34 RenderProps**：用 render prop 复用逻辑（如 `MouseTracker`）。
- **35 Refs**：通过 ref 拿到组件/元素实例。

**为什么这么安排**：Taro 既然是 React 思维，React 的复用三板斧（组合 / RenderProps / Refs）自然要用得上。这一篇是「能写组件」到「会设计可复用组件」的进阶，从源码目录看，34/35 章的 `pages` 已经累积了大量前面各篇的组件，说明这些模式是在已有组件体系上演进的。

### 第六篇 设备 API（36~45）：调用手机硬件与系统能力

- **36 API使用说明**：Taro API 的统一调用规范（`Taro.xxx`，返回 Promise）。
- **37~45 设备 API**：加速度计、设备方向、剪切板、屏幕亮度、罗盘、联系人与网络、扫码、电话/振动/截屏、系统信息与 Wifi。

**为什么这么安排**：小程序区别于普通 H5 的关键就是能调硬件。先讲 API 调用规范，再按传感器/系统能力逐类过，是「能力扫盲」式的安排。

### 第七篇 其他基础 API（46~52）：文件/动画/导航/刷新/键盘

- **46 文件API / 47 动画API**：文件读写与 `Taro.createAnimation`。
- **48~49 导航与交互**：页面导航栈、交互反馈（toast/modal/loading 等）。
- **50 案例-滚动API和回到最上 / 51 各种刷新操作 / 52 键盘菜单和按钮监控**：滚动、下拉刷新/上拉加载、键盘与按键。

**为什么这么安排**：把不属于「设备/媒体」但又常用的小程序能力补齐，并穿插滚动、刷新这类高频交互案例。

### 第八篇 媒体 API（53~60）：录音、音频、相机

- **53 录音功能 + 54~56 案例-录音机（1~3）**：录音 API 与完整录音机案例。
- **57 背景音频控制 + 58 案例-酷狗音乐播放器**：背景音频 API 与仿酷狗播放器。
- **59 加载新字体 / 60 相机拍照**：自定义字体与相机。

**为什么这么安排**：媒体是小程序里最复杂的能力之一，每讲一个 API 立刻配一个大案例（录音机、音乐播放器）巩固，是「API + 实战」交替的典型节奏。

### 第九篇 综合实战项目（61~79）：把所有能力做成真实项目

- **61 在线聊天机器人 / 62~63 涂鸦板**：聊天与画板（综合网络 + canvas）。
- **64~67 云数据/电视云操作**：基于云开发的增删改查与视频云。
- **68 物流查询 / 69~70 个人云存储图集**：网络请求 + 云存储图片。
- **71~79 我的社交圈（part1~9）**：压轴大项目，9 节做完一个类朋友圈/社交应用。

**为什么放最后**：这是全课的「毕业项目」段，把前面所有零散能力（网络、组件、API、云开发、媒体、动画）综合到一个个完整项目里，尤其是「我的社交圈」9 连讲，是课程的总练兵场。

### 主线一句话总结

**环境与认知（1~8）→ React 组件与 JSX（9~14）→ 基础能力（15~20）→ 组件库大全（21~31）→ 高阶组件模式（32~35）→ 设备/媒体 API（36~60）→ 综合实战项目（61~79）**，目标是掌握「用 React 语法写小程序」的完整能力栈，并能独立做出聊天机器人、音乐播放器、社交圈等真实项目。

## 3. 使用场景等元信息沉淀

- **学完能用在哪**：用 Taro（React 语法）独立开发并发布一个微信小程序，并能一套代码编译到多端；掌握小程序原生组件库、设备/媒体 API、云开发的常见用法。
- **和实际项目的关系**：聊天机器人、音乐播放器、云存储图集、社交圈等都是可直接迁移到真实业务的项目雏形；云开发相关章节（64~67）对小程序「免服务器后端」场景很实用。
- **和其他课程的关系**：上承《React》（JSX/class 组件/RenderProps/Refs 思维直接来自 React）与《微信小程序》（组件库与 API 与原生小程序一一对应）；React 基础弱的话建议先补 React 再学本课。
- **面试/教学高频场景**：Taro 多端编译原理（1/3）、组件生命周期（10）、路由传参（15）、列表/条件渲染（18/19）、表单组件（23~25）、函数式组件 vs class（32）、RenderProps（34）、refs（35）、设备 API 调用（37~45）、云开发（64~67）。

## 4. 临时补充

- **注意版本**：本课是 **Taro 1.x**，写法是 `import { Component } from '@tarojs/taro'` 的 class 组件（底层 Nerv），不是 Taro 3.x 的 hooks + 多框架支持。复习或迁移到新项目时，要留意新版 Taro（3.x+）在写法、配置（`config/index.js`）、编译上的差异，不能照抄。
- **pages 累积式目录**：从第 32 章往后，每章 `src/pages` 会保留前面所有章的页面（如 35 章能看到 `MouseTracker/MyRefs/RefFunction` 等），这其实是「同一个工程不断加页面」的教学方式，复习某个 API 时去对应章节看新增的那个 page 即可，不必被一长串目录吓到。
- **第 1 章无源码**，只有 readme，属正常（理论介绍课）。
- 第九篇的「我的社交圈」是 9 节连讲的大项目，复习时建议当成一个整体看，单看某一 part 容易断片。
- 本课与独立的《微信小程序》课内容高度重叠（组件库、API 基本一致），区别只在「用 Taro/React 语法还是用原生 wxml/js 语法」——如果两门都学，可对照复习「同一能力在两种写法下怎么实现」。
