# 42. 补充-404页面

## 知识点说明

在单页面应用（SPA）中，404页面是一个重要的用户体验优化点。当用户访问不存在的路由路径时，应该展示一个友好的提示页面，而不是空白页面或错误信息。

## 什么是404页面

404页面是当用户访问的URL路径在路由配置中找不到对应匹配时显示的页面。在Vue Router中，通过配置通配符路由`*`来实现404页面的处理。

## 配置404路由

在路由配置数组的最末尾添加一个通配符路由：

```javascript
// routes.js
import NotFound from "@/views/NotFound.vue";

export default [
  // ... 其他路由配置
  
  {
    name: "NotFound",
    path: "*",  // 通配符，匹配所有未定义的路由
    component: NotFound,
  },
];
```

**重要提示**：
- 通配符路由`*`必须放在路由配置数组的最后
- Vue Router按照配置顺序进行匹配，通配符会匹配所有路径
- 如果放在前面，会导致所有路由都匹配到404页面

## 创建404页面组件

创建一个简单但友好的404提示页面：

```vue
<!-- NotFound.vue -->
<template>
  <div class="not-found-container">
    <img src="@/assets/404.jpg" alt="404页面提示图片" />
  </div>
</template>

<script>
export default {
  // 简单的404页面组件，不需要复杂逻辑
  name: "NotFound"
};
</script>

<style scoped>
.not-found-container {
  background: #90cbc0;
  width: 100%;
  height: 100%;
  display: flex;
  justify-content: center;
  align-items: center;
}
</style>
```

## 设计要点

### 1. 视觉友好
- 使用柔和的背景颜色
- 展示友好的提示图片或文字
- 保持页面简洁美观

### 2. 用户引导
- 提示用户当前页面不存在
- 提供返回首页的链接
- 或者自动跳转到首页

### 3. 样式处理
```css
.not-found-container {
  background: #90cbc0;        /* 柔和的背景色 */
  width: 100%;                  /* 占满整个容器宽度 */
  height: 100%;                 /* 占满整个容器高度 */
  display: flex;                /* 使用flex布局 */
  justify-content: center;      /* 水平居中 */
  align-items: center;          /* 垂直居中 */
}
```

## 完整路由配置示例

```javascript
// routes.js
import NotFound from "@/views/NotFound.vue";

// ... 其他import

export default [
  {
    name: "Home",
    path: "/",
    component: getPageComponent(() =>
      import(/* webpackChunkName: "home" */ "@/views/Home")
    ),
    meta: {
      title: "首页",
    },
  },
  {
    name: "About",
    path: "/about",
    component: getPageComponent(() =>
      import(/* webpackChunkName: "about" */ "@/views/About")
    ),
    meta: {
      title: "关于我",
    },
  },
  // ... 其他正常路由配置
  
  // 通配符路由必须放在最后
  {
    name: "NotFound",
    path: "*",
    component: NotFound,
  },
];
```

## 测试404页面

配置完成后，可以通过以下方式测试：

1. **直接访问不存在的路径**：
   - 访问 `/abc` 
   - 访问 `/test/123`
   - 访问任何未定义的路径

2. **预期结果**：
   - 显示404页面组件
   - 不会出现空白页面
   - 控制台不会报路由匹配错误

## 注意事项

1. **通配符路由的位置**：必须放在路由配置的最后，否则会覆盖所有其他路由

2. **模式兼容性**：
   - History模式：通配符`*`正常工作
   - Hash模式：通配符`*`同样适用

3. **SEO考虑**：对于需要SEO的项目，服务端也需要配置404页面

## 扩展功能

### 添加返回首页链接

```vue
<template>
  <div class="not-found-container">
    <img src="@/assets/404.jpg" alt="404页面提示图片" />
    <router-link to="/" class="back-home">返回首页</router-link>
  </div>
</template>

<style scoped>
.back-home {
  margin-top: 20px;
  color: #fff;
  text-decoration: none;
  padding: 10px 20px;
  background: rgba(0,0,0,0.2);
  border-radius: 4px;
}
</style>
```

### 自动跳转到首页

```vue
<script>
export default {
  name: "NotFound",
  mounted() {
    // 3秒后自动跳转到首页
    setTimeout(() => {
      this.$router.push("/");
    }, 3000);
  }
};
</script>
```

## 总结

- 404页面是单页面应用中重要的用户体验优化
- 通过通配符路由`*`捕获所有未定义的路径
- 通配符路由必须放在路由配置的最后
- 404页面应该友好地提示用户并提供返回选项
- 可以根据需要添加自动跳转等增强功能
