# 第10章 插槽

## 10.1 什么是插槽？

在某些组件的模板中，有一部分区域需要父组件来指定。这就好比我们制作了一个相框，但相框里的照片需要使用者自己提供。

### 典型应用场景

```html
<!-- message组件：一个弹窗消息 -->
<div class="message-container">
  <div class="content">
    <!-- 这里是消息内容，可以是一个文本，也可能是一段html，具体是什么不知道，需要父组件指定 -->
  </div>
  <button>确定</button>
  <button>关闭</button>
</div>
```

在上面的例子中，消息弹窗的结构和按钮是固定的，但消息的具体内容需要由父组件来决定。这时就需要使用插槽。

## 10.2 插槽的基本用法

### 定义插槽

在子组件中使用 `<slot>` 内置组件来定义插槽位置：

```html
<!-- Message.vue - 消息弹窗组件 -->
<template>
  <div class="message-container">
    <div class="content">
      <!-- slot是vue的内置组件，作为占位符使用 -->
      <slot></slot>
    </div>
    <button>确定</button>
    <button>关闭</button>
  </div>
</template>

<script>
export default {
  props: ["msg"],
};
</script>

<style scoped>
.message-container {
  position: fixed;
  left: 50%;
  top: 50%;
  transform: translate(-50%, -50%);
  border: 2px solid;
  padding: 20px;
}
</style>
```

### 使用插槽

在父组件中向插槽传递内容：

```html
<!-- MessageTest.vue - 测试组件 -->
<template>
  <Message>
    <div>
      <a href="">查看详情</a>
      <p>这是一条重要的系统消息</p>
    </div>
  </Message>
</template>

<script>
import Message from "./Message.vue";
export default {
  components: {
    Message,
  },
};
</script>
```

### 最终渲染结果

```html
<div class="message-container">
  <div class="content">
    <div>
      <a href="">查看详情</a>
      <p>这是一条重要的系统消息</p>
    </div>
  </div>
  <button>确定</button>
  <button>关闭</button>
</div>
```

## 10.3 具名插槽

如果某个组件中需要父元素传递多个区域的内容，就需要提供多个插槽。为了避免冲突，就需要给不同的插槽赋予不同的名字。

### 定义具名插槽

使用 `name` 属性为插槽命名：

```html
<!-- Layout.vue - 布局组件 -->
<template>
  <div class="layout-container">
    <header style="background: #f40">
      <!-- 提供页头插槽，名为header -->
      <slot name="header"></slot>
    </header>
    <main style="background: lightblue">
      <!-- 提供主要内容插槽，默认名称为default -->
      <slot></slot>
    </main>
    <footer style="background: yellow">
      <!-- 提供页脚插槽，名为footer -->
      <slot name="footer"></slot>
    </footer>
  </div>
</template>

<script>
export default {};
</script>
```

### 使用具名插槽

使用 `v-slot:` 指令或其简写 `#` 来指定内容对应的插槽：

```html
<!-- LayoutTest.vue -->
<template>
  <Layout>
    <!-- 使用 # 简写形式 -->
    <template #header>
      <h1>网站标题</h1>
    </template>

    <!-- 默认插槽不需要指定名称 -->
    <p>A paragraph for the main content.</p>
    <p>And another one.</p>

    <!-- 使用完整的 v-slot: 形式 -->
    <template v-slot:footer>
      <p>Here's some contact info</p>
    </template>
  </Layout>
</template>

<script>
import Layout from "./Layout.vue";
export default {
  components: {
    Layout,
  },
};
</script>
```

### 实际项目中的布局组件

```html
<!-- my-site/src/components/Layout/index.vue -->
<template>
  <div class="layout-container">
    <!-- 左侧区域插槽 -->
    <div class="left">
      <slot name="left"></slot>
    </div>
    <!-- 中间主区域插槽 -->
    <div class="main">
      <slot></slot>
    </div>
    <!-- 右侧区域插槽 -->
    <div class="right">
      <slot name="right"></slot>
    </div>
  </div>
</template>

<script>
export default {};
</script>

<style scoped lang="less">
.layout-container {
  width: 100%;
  height: 100%;
  display: flex;
  .left,
  .right {
    flex: 0 0 auto;     /* 不放大不缩小，按内容自适应 */
    overflow: hidden;
  }
  .main {
    flex: 1 1 auto;     /* 可放大可缩小，占据剩余空间 */
    overflow: hidden;
  }
}
</style>
```

## 10.4 插槽语法总结

### 插槽定义

| 语法 | 说明 |
|------|------|
| `<slot></slot>` | 定义默认插槽（匿名插槽） |
| `<slot name="名称"></slot>` | 定义具名插槽 |

### 插槽使用

| 语法 | 说明 |
|------|------|
| `<组件>内容</组件>` | 向默认插槽传递内容 |
| `<template v-slot:插槽名>内容</template>` | 向具名插槽传递内容 |
| `<template #插槽名>内容</template>` | 向具名插槽传递内容（简写形式） |

## 10.5 插槽的作用域

**注意**：插槽的内容是在父组件中编译的，因此可以访问父组件的数据，而不能访问子组件的数据。

## 10.6 最佳实践

1. **合理使用默认插槽**：当组件只有一个可变区域时，使用默认插槽即可
2. **命名要语义化**：具名插槽的名称应该清晰表达其用途（如 header、footer、left、right）
3. **提供后备内容**：可以在 `<slot>` 标签内部添加后备内容，当父组件不提供内容时显示
4. **保持组件独立性**：插槽的内容应该与组件自身的逻辑分离

## 10.7 常见应用场景

- **布局组件**：提供灵活的页面布局框架
- **容器组件**：定义外观和结构，内容由使用者决定
- **弹窗组件**：弹窗的样式和行为固定，内容由使用者传入
- **列表组件**：定义列表容器和分页，列表项内容由使用者提供

通过插槽机制，Vue组件可以实现高度的可复用性和灵活性，是组件化开发的重要工具。
