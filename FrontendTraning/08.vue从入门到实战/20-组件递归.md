## 什么是组件递归

组件递归是指组件在模板中调用自身的情况。这种技术通常用于处理具有树形结构的数据，比如：

- 文件目录树
- 组织架构图
- 分类导航菜单
- 评论回复系统

### 基本原理

```javascript
// 递归组件的关键要素：
// 1. 组件在模板中使用自身
// 2. 必须有终止条件（通常通过 props 控制）
// 3. 需要处理递归数据结构
```

## 递归组件的实现

### 基础示例

```vue
<template>
  <div class="tree-node">
    <div class="node-content" @click="toggle">
      {{ item.title }}
    </div>
    <div v-if="isOpen && item.children" class="node-children">
      <!-- 递归调用自身 -->
      <tree-node
        v-for="child in item.children"
        :key="child.id"
        :item="child"
      />
    </div>
  </div>
</template>

<script>
export default {
  name: 'TreeNode', // 重要：必须设置 name 属性
  props: {
    item: {
      type: Object,
      required: true
    }
  },
  data() {
    return {
      isOpen: false
    }
  },
  methods: {
    toggle() {
      this.isOpen = !this.isOpen
    }
  }
}
</script>
```

### 关键要点

1. **设置组件名称**
   - 递归组件必须设置 `name` 属性
   - 组件在模板中通过 `name` 引用自身

2. **传递递归数据**
   - 通过 props 传递子节点数据
   - 确保数据结构支持递归

3. **终止条件**
   - 通过 `v-if` 控制递归终止
   - 当没有子节点时停止递归

## 实际应用：多级菜单

### Menu 组件实现

让我们来看一个实际的多级菜单组件实现：

```vue
<template>
  <nav class="menu-container">
    <RouterLink
      :exact="item.exact"
      v-for="item in items"
      :key="item.link"

      <div class="icon">
```

### 将 Menu 改造为递归组件

如果需要支持多级菜单，可以这样改造：

```vue
<template>
  <nav class="menu-container">
    <div 
      v-for="item in items" 
      :key="item.name"
      class="menu-item"
    >
      <RouterLink
        :exact="item.exact"
        :to="{ name: item.name }"
        active-class="selected"
        exact-active-class=""
      >
        <div class="icon">
          <Icon :type="item.icon" />
        </div>
        <span>{{ item.title }}</span>
      </RouterLink>
      
      <!-- 递归渲染子菜单 -->
      <Menu 
        v-if="item.children && item.children.length"
        :items="item.children"
        class="submenu"
      />
    </div>
  </nav>
</template>

<script>
import Icon from "@/components/Icon";

export default {
  name: 'Menu',
  components: {
    Icon,
  },
  props: {
    items: {
      type: Array,
      default: () => []
    }
  }
}
</script>
```

## 递归组件的注意事项

### 1. 性能考虑

```javascript
// 问题：深层递归可能导致性能问题
// 解决方案：
// - 限制递归深度
// - 使用虚拟滚动
// - 懒加载子节点数据
```

### 2. 内存泄漏风险

```javascript
// 问题：递归组件可能产生内存泄漏
// 解决方案：
// - 确保在组件销毁时清理引用
// - 避免在递归组件中使用全局事件
// - 使用 v-if 而非 v-show 控制递归渲染
```

### 3. 调试困难

```javascript
// 问题：递归组件的调试比较困难
// 解决方案：
// - 添加递归深度显示
// - 使用开发者工具的组件面板
// - 在关键位置添加 console.log
```

## 常见应用场景

## 最佳实践

### 1. 命名规范

```javascript
// 使用描述性的组件名称
export default {
  name: 'TreeNode' // 明确表示这是一个树节点组件
}
```

### 2. Props 设计

```javascript
// Props 应该包含递归所需的所有数据
props: {
  data: Object,      // 当前节点数据
  level: Number,     // 当前层级
  maxLevel: Number   // 最大层级限制
}
```

### 3. 样式隔离

```vue
<style scoped lang="less">
// 使用 scoped 样式避免样式污染
// 递归组件的样式需要特别考虑继承问题
.org-node {
  margin-left: 20px; // 子节点缩进
}
</style>
```

### 4. 事件处理

```vue
<template>
  <TreeNode
    @node-click="handleNodeClick"
    @node-toggle="handleNodeToggle"
  />
</template>

<script>
export default {
  methods: {
    handleNodeClick(node) {
      // 处理节点点击
      this.$emit('node-click', node)
    },
    handleNodeToggle(node) {
      // 处理节点展开/收起
      this.$emit('node-toggle', node)
    }
  }
}
</script>
```
