## 章节概述

图片懒加载是一种性能优化技术，它可以显著提升页面加载速度和用户体验。本章将详细介绍如何实现一个功能完善的图片懒加载组件。

## 核心知识点

### 1. 图片懒加载原理

**什么是懒加载？**

懒加载（Lazy Loading）是一种延迟加载技术，只有当元素进入视口或即将进入视口时才进行加载。对于图片来说，这意味着：

- 初始加载时不加载所有图片
- 当图片滚动到可视区域时才加载
- 减少初始页面加载时间和带宽消耗

**应用场景：**

- 图片密集的页面（相册、电商产品列表）
- 长页面内容（博客文章、新闻列表）
- 移动端应用（节省流量）

### 2. Intersection Observer API

**基本用法：**

Intersection Observer API 提供了一种异步检测目标元素与祖先元素或视口相交情况的方法。

```javascript
// 创建观察者
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    // entry.isIntersecting 表示元素是否进入视口
    if (entry.isIntersecting) {
      // 加载图片
      loadImg(entry.target);
      // 停止观察已加载的元素
      observer.unobserve(entry.target);
    }
  });
}, {
  root: null, // null 表示使用视口作为根元素
  rootMargin: '0px', // 根元素的边距
  threshold: 0.1 // 10% 可见时触发
});

// 开始观察目标元素
observer.observe(imgElement);
```

**配置选项说明：**

- `root`: 用作视口的元素，默认为浏览器视口
- `rootMargin`: 根元素边距，可以提前触发加载
- `threshold`: 触发回调的可见比例阈值

### 3. ImageLoader 组件实现

**组件功能：**

- 占位符显示：加载前显示模糊的缩略图
- 平滑过渡：原图加载完成后淡入显示
- 状态管理：跟踪加载状态并提供事件通知
- 可配置性：支持自定义过渡时间

**组件代码：**

```vue
<template>
  <div class="image-loader-container">
    <!-- 占位图：原图未加载完成时显示 -->
    <img v-if="!everythingDone" class="placeholder" :src="placeholder" alt="" />
    
    <!-- 原图：通过透明度控制显示，加载完成后淡入 -->
    <img
      @load="handleLoad"
      :src="src"
      alt=""
      :style="{ opacity: originOpacity, transition: `${duration}ms` }"
    />
  </div>
</template>

<script>
export default {
  props: {
    // 原图地址（必需）
    src: {
      type: String,
      required: true,
    },
    // 占位图地址（必需）
    placeholder: {
      type: String,
      required: true,
    },
    // 过渡动画持续时间，默认500ms
    duration: {
      type: Number,
      default: 500,
    },
  },
  data() {
    return {
      originLoaded: false,     // 原图是否加载完成
      everythingDone: false,   // 是否一切都尘埃落定了（动画完成）
    };
  },
  computed: {
    // 根据原图加载状态计算透明度
    originOpacity() {
      return this.originLoaded ? 1 : 0;
    },
  },
  methods: {
    // 原图加载完成事件处理
    handleLoad() {
      this.originLoaded = true; // 显示原图（透明度变为1）
      
      // 等待过渡动画完成后移除占位图
      setTimeout(() => {
        this.everythingDone = true; // 移除占位图
        this.$emit("load");         // 触发加载完成事件
      }, this.duration);
    },
  },
};
</script>

<style scoped lang="less">
@import "~@/styles/mixin.less";

.image-loader-container {
  width: 100%;
  height: 100%;
  position: relative;
  overflow: hidden;
  
  img {
    .self-fill();           // 使用混合宏填充容器
    object-fit: cover;      // 保持图片比例填充
  }

  // 占位图样式：添加模糊效果
  .placeholder {
    filter: blur(2vw);
  }
}
</style>
```

**关键实现细节：**

1. **双图片叠加**：占位图和原图叠加在一起，通过透明度控制显示

2. **状态管理**：
   - `originLoaded`: 原图资源加载完成
   - `everythingDone`: 过渡动画完成，可以移除占位图

3. **平滑过渡**：使用 CSS transition 实现淡入效果

4. **模糊占位**：通过 `filter: blur(2vw)` 为占位图添加模糊效果

### 1. 占位图策略

**图片尺寸建议：**

- 占位图：1-5KB，建议使用极小的缩略图（如 100px 宽度）
- 原图：根据实际显示尺寸优化，避免过度加载

### 2. 性能优化

**预加载策略：**

```javascript
// 提前加载即将进入视口的图片
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      loadImg(entry.target);
    } else if (entry.intersectionRatio > 0) {
      // 预加载即将可见的图片
      preloadImg(entry.target);
    }
  });
}, {
  rootMargin: '200px' // 提前200px开始加载
});
```

## 实战练习

### 练习1：基础懒加载

创建一个包含多张图片的列表，实现基础的懒加载功能：

```vue
<template>
  <div class="image-list">
    <div v-for="item in images" :key="item.id" class="image-item">
      <img 
        :data-src="item.src" 
        class="lazy-image"
        alt=""
      />
    </div>
  </div>
</template>

<script>
export default {
  data() {
    return {
      images: [
        { id: 1, src: 'image1.jpg' },
        { id: 2, src: 'image2.jpg' },
        // ... 更多图片
      ]
    };
  },
  mounted() {
    this.initLazyLoad();
  },
  methods: {
    initLazyLoad() {
      const observer = new IntersectionObserver((entries) => {
        entries.forEach(entry => {
          if (entry.isIntersecting) {
            const img = entry.target;
            img.src = img.dataset.src;
            observer.unobserve(img);
          }
        });
      });
      
      document.querySelectorAll('.lazy-image').forEach(img => {
        observer.observe(img);
      });
    }
  }
};
</script>
```

## 常见问题

### Q1: 懒加载会影响 SEO 吗？

A: 现代搜索引擎可以执行 JavaScript，懒加载的图片通常能被正确抓取。但为了保险起见，重要图片建议使用 `<link rel="preload">` 预加载。

### Q2: 如何处理动态添加的图片？

A: 对于动态添加的图片，需要在添加后重新调用观察器的 observe 方法：

```javascript
addNewImage(imageSrc) {
  this.images.push(imageSrc);
  this.$nextTick(() => {
    const newImg = this.$refs.imageContainer.lastElementChild;
    this.observer.observe(newImg);
  });
}
```

### Q3: 占位图加载失败怎么办？

A: 可以使用 base64 编码的极小图片作为默认占位符：

```javascript
const defaultPlaceholder = 'data:image/svg+xml;base64,...';
```
