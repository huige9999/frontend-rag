## 组件化开发概述

### 什么是组件化？

组件化开发是将复杂的UI界面拆分成多个独立、可复用的组件，每个组件负责自己的功能和样式。这样可以提高代码的可维护性和复用性。

### 组件化的优势

1. **代码复用**：相同的组件可以在多个页面中使用
2. **便于维护**：每个组件独立管理，修改时不会影响其他部分
3. **团队协作**：不同开发者可以同时开发不同的组件
4. **提高效率**：使用现有组件可以快速构建新功能

## 组件目录结构

### 推荐的组件组织方式

```
src/
├── components/
│   ├── Avatar/
│   │   ├── index.vue       # 组件主文件
│   │   └── test.vue        # 组件测试文件
│   ├── Icon/
│   │   ├── index.vue
│   │   └── test.vue
│   ├── Pager/
│   │   ├── index.vue
│   │   └── test.vue
│   └── README.md           # 组件说明文档
```

### 组织结构说明

- 每个组件都有独立的文件夹
- `index.vue` 是组件的主文件
- `test.vue` 用于组件的单独测试
- `README.md` 提供组件的使用说明

## 常用组件示例

### 1. Avatar 组件（头像组件）

头像组件用于显示用户头像，支持自定义大小和图片地址。

```vue
<template>
  <img
    class="avatar-container"
    :src="url"
    :style="{
      width: size + 'px',
      height: size + 'px',
    }"
  />
</template>

<script>
export default {
  // 组件名称，便于调试和识别
  name: 'Avatar',
  
  props: {
    url: {
      type: String,      // url属性必须是一个字符串
      required: true,    // 属性必传
    },
    size: {
      type: Number,      // size属性必须是数字
      default: 150,     // 默认值为150px
    },
  },
};
</script>

<style scoped>
/* 使用scoped样式，避免污染全局样式 */
.avatar-container {
  border-radius: 50%;   /* 圆形头像 */
  object-fit: cover;    /* 保持图片比例，裁剪多余部分 */
  display: block;
}
</style>
```

**使用示例：**

```vue
<Avatar url="https://example.com/avatar.jpg" :size="200" />
```

**Props说明：**
- `url`（必传）：头像图片的URL地址
- `size`（可选）：头像尺寸，默认150px

### 2. Icon 组件（图标组件）

图标组件用于显示各种图标，通过映射表实现图标名称到CSS类名的转换。

```vue
<template>
  <i class="iconfont icon-container" :class="fontClass"></i>
</template>

<script>
// 图标类型到CSS类名的映射表
const classMap = {
  home: "iconzhuye",
  success: "iconzhengque",
  error: "iconcuowu",
  close: "iconguanbi",
  warn: "iconjinggao",
  info: "iconxinxi",
  blog: "iconblog",
  code: "iconcode",
  about: "iconset_about_hov",
  weixin: "iconweixin",
  mail: "iconemail",
  github: "icongithub",
  qq: "iconsign_qq",
  arrowUp: "iconiconfonticonfonti2copy",
  arrowDown: "iconiconfonticonfonti2",
  empty: "iconempty",
  chat: "iconliuyan",
};

// 导出所有可用的图标类型，便于其他模块使用
export const types = Object.keys(classMap);

export default {
  name: 'Icon',
  
  props: {
    type: {
      type: String,
      required: true,    // 必须指定图标类型
    },
  },
  
  computed: {
    // 根据type计算对应的CSS类名
    fontClass() {
      return classMap[this.type];
    },
  },
};
</script>

<style scoped>
/* 引入图标字体CSS */
@import "//at.alicdn.com/t/font_2164449_nalfgtq7il.css";

.iconfont {
  color: inherit;       /* 继承父元素颜色 */
  font-size: inherit;   /* 继承父元素字体大小 */
}
</style>
```

**使用示例：**

```vue
<Icon type="success" />
<Icon type="error" />
<Icon type="github" />
```

**Props说明：**
- `type`（必传）：图标类型，可选值：`home`, `success`, `error`, `close`, `warn`, `info`, `blog`, `code`, `about`, `weixin`, `mail`, `github`, `qq`, `arrowUp`, `arrowDown`, `empty`, `chat`

### 3. Pager 组件（分页组件）

分页组件用于处理大量数据的分页显示，支持页码跳转和边界控制。

```vue
<template>
  <!-- 只有总页数大于1时才显示分页器 -->
  <div class="pager-container" v-if="pageNumber > 1">
    <!-- 首页按钮 -->
    <a @click="handleClick(1)" :class="{ disabled: current === 1 }">
      |&lt;&lt;
    </a>
    
    <!-- 上一页按钮 -->
    <a @click="handleClick(current - 1)" :class="{ disabled: current === 1 }">
      &lt;&lt;
    </a>
    
    <!-- 页码列表 -->
    <a
      @click="handleClick(n)"
      v-for="(n, i) in numbers"
      :key="i"
      :class="{ active: n === current }"
    >
      {{ n }}
    </a>

    <!-- 下一页按钮 -->
    <a
      @click="handleClick(current + 1)"
      :class="{ disabled: current === pageNumber }"
    >
      &gt;&gt;
    </a>
    
    <!-- 末页按钮 -->
    <a
      @click="handleClick(pageNumber)"
      :class="{ disabled: current === pageNumber }"
    >
      &gt;&gt;|
    </a>
  </div>
</template>

<script>
export default {
  name: 'Pager',
  
  props: {
    current: {
      type: Number,
      default: 1,        // 当前页码，默认第1页
    },
    total: {
      type: Number,
      default: 0,        // 总数据条数
    },
    limit: {
      type: Number,
      default: 10,       // 每页显示条数
    },
    visibleNumber: {
      type: Number,
      default: 10,       // 可见的页码数量
    },
  },
  
  computed: {
    // 计算总页数
    pageNumber() {
      return Math.ceil(this.total / this.limit);
    },
    
    // 计算可见页码的最小值
    visibleMin() {
      let min = this.current - Math.floor(this.visibleNumber / 2);
      if (min < 1) {
        min = 1;
      }
      return min;
    },
    
    // 计算可见页码的最大值
    visibleMax() {
      let max = this.visibleMin + this.visibleNumber - 1;
      if (max > this.pageNumber) {
        max = this.pageNumber;
      }
      return max;
    },
    
    // 生成可见页码数组
    numbers() {
      let nums = [];
      for (let i = this.visibleMin; i <= this.visibleMax; i++) {
        nums.push(i);
      }
      return nums;
    },
  },
  
  methods: {
    // 处理页码点击事件
    handleClick(newPage) {
      // 边界检查：不能小于1
      if (newPage < 1) {
        newPage = 1;
      }
      // 边界检查：不能大于总页数
      if (newPage > this.pageNumber) {
        newPage = this.pageNumber;
      }
      // 如果点击的是当前页，不做任何操作
      if (newPage === this.current) {
        return;
      }
      // 向父组件抛出页码变化事件
      this.$emit("pageChange", newPage);
    },
  },
};
</script>

<style lang="less" scoped>
@import "~@/styles/var.less";

.pager-container {
  display: flex;
  justify-content: center;
  margin: 20px 0;
  
  a {
    color: @primary;
    margin: 0 6px;
    cursor: pointer;
    
    /* 禁用状态 */
    &.disabled {
      color: @lightWords;
      cursor: not-allowed;
    }
    
    /* 激活状态 */
    &.active {
      color: @words;
      font-weight: bold;
      cursor: text;
    }
  }
}
</style>
```

**使用示例：**

```vue
<Pager 
  :current="currentPage" 
  :total="100" 
  :limit="10"
  @pageChange="handlePageChange"
/>
```

**Props说明：**
- `current`：当前页码，默认1
- `total`：总数据条数，默认0
- `limit`：每页显示条数，默认10
- `visibleNumber`：可见页码数量，默认10

**事件：**
- `pageChange`：页码变化时触发，参数为新页码

## 编写组件说明文档

### README.md 的重要性

组件说明文档帮助团队成员了解组件的功能和使用方法，是组件化开发不可或缺的一部分。

### 文档应包含的内容

1. **组件简介**：组件的功能描述
2. **Props说明**：列出所有属性及其类型、默认值、是否必传
3. **Events说明**：列出所有事件及其参数
4. **使用示例**：提供完整的使用代码
5. **注意事项**：使用时需要注意的特殊情况

### 示例文档结构

```markdown
# Avatar 组件

## 功能说明

用于显示用户头像，支持自定义大小。

## Props

| 属性 | 类型 | 默认值 | 是否必传 | 说明 |
|------|------|--------|----------|------|
| url | String | - | 是 | 头像图片地址 |
| size | Number | 150 | 否 | 头像尺寸（px） |

## 使用示例

```vue
<Avatar url="https://example.com/avatar.jpg" :size="200" />
```

## 注意事项

- url属性必须传入有效的图片地址
- size属性值为数字，不需要单位
```

## 组件测试方法

### 使用 Vue CLI 进行原型测试

Vue CLI 提供了快速原型开发的功能，可以在不创建完整项目的情况下测试组件。

#### 安装工具

```bash
npm install -g @vue/cli-service-global
```

#### 测试单个组件

```bash
vue serve src/components/Avatar/test.vue
```

这会启动一个开发服务器，可以在浏览器中查看组件效果。

### test.vue 文件的作用

每个组件的 `test.vue` 文件用于：

1. 独立测试组件功能
2. 验证组件的Props和Events
3. 检查组件的样式效果
4. 作为组件使用示例

**test.vue 示例结构：**

```vue
<template>
  <div>
    <h1>Avatar 组件测试</h1>
    <Avatar url="https://example.com/avatar.jpg" :size="200" />
  </div>
</template>

<script>
import Avatar from "./index.vue";

export default {
  components: {
    Avatar,
  },
};
</script>
```

## 组件开发的最佳实践

### 1. 命名规范

- **组件文件夹**：使用大驼峰命名（如 `Avatar`、`Pager`）
- **组件名称**：与文件夹名称保持一致
- **Props名称**：使用小驼峰命名（如 `userName`、`pageSize`）

### 2. Props 验证

始终对Props进行类型验证和默认值设置：

```javascript
props: {
  // 基础类型检查
  propA: Number,
  
  // 多个可能的类型
  propB: [String, Number],
  
  // 必填字符串
  propC: {
    type: String,
    required: true,
  },
  
  // 带默认值的数字
  propD: {
    type: Number,
    default: 100,
  },
  
  // 带默认值的对象
  propE: {
    type: Object,
    default: function () {
      return { message: 'hello' };
    },
  },
}
```

### 3. 作用域样式

使用 `scoped` 属性避免样式污染：

```vue
<style scoped>
/* 这些样式只会应用于当前组件 */
.avatar-container {
  border-radius: 50%;
}
</style>
```

### 4. 事件命名

- 使用 `$emit` 抛出事件
- 事件名使用小驼峰或短横线命名
- 提供清晰的参数说明

```javascript
this.$emit("pageChange", newPage);
```

### 5. 组件文档

为每个组件编写完整的README文档，包括：
- 功能说明
- API文档（Props、Events、Slots）
- 使用示例
- 注意事项

## 组件通信

### 父子组件通信

#### 父组件向子组件传递数据（Props）

```vue
<!-- 父组件 -->
<template>
  <ChildComponent :message="parentMessage" />
</template>

<!-- 子组件 -->
<script>
export default {
  props: {
    message: String
  }
}
</script>
```

#### 子组件向父组件传递数据（Events）

```vue
<!-- 子组件 -->
<script>
export default {
  methods: {
    handleClick() {
      this.$emit('update', newValue);
    }
  }
}
</script>

<!-- 父组件 -->
<template>
  <ChildComponent @update="handleUpdate" />
</template>
```

### 兄弟组件通信

可以通过事件总线或状态管理（Vuex）实现兄弟组件之间的通信。
