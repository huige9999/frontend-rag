### 路由插件的基本使用

```js
import Vue from 'vue'
import VueRouter from 'vue-router'

// 1. 在Vue中安装路由插件
Vue.use(VueRouter); 

// 2. 创建路由实例，配置路由规则
const router = new VueRouter({
  routes: [
    // 路由配置
  ]
})

// 3. 在Vue实例中注入路由
new Vue({
  router,
  // ...其他配置
})
```

## 二、基本使用

### 2.1 路由配置

在 `router/routes.js` 中配置路由规则：

```js
// router/routes.js
import Home from "@/views/Home";
import About from "@/views/About";
import Blog from "@/views/Blog";
import Project from "@/views/Project";
import Message from "@/views/Message";

export default [
  { 
    name: "Home",    // 路由名称，用于命名路由
    path: "/",        // 路由路径
    component: Home   // 对应的组件
  },
  { name: "About", path: "/about", component: About },
  { name: "Blog", path: "/article", component: Blog },
  { name: "Project", path: "/project", component: Project },
  { name: "Message", path: "/message", component: Message },
];
```

在 `router/index.js` 中创建路由实例：

```js
// router/index.js
import Vue from "vue";
import VueRouter from "vue-router";
import routes from "./routes";

Vue.use(VueRouter); // 使用一个vue插件

const router = new VueRouter({
  routes, // 路由匹配规则
  mode: "history", // 路由模式
});

export default router;
```

### 2.2 路由视图

在应用的主组件中使用 `<RouterView />` 组件来显示匹配到的路由组件：

```vue
<!-- App.vue -->
<template>
  <div class="app-container">
    <Layout>
      <template #left>
        <div class="aside">
          <SiteAside />
        </div>
      </template>
      <template #default>
        <!-- 路由出口：匹配到的组件会渲染到这里 -->
        <RouterView />
      </template>
    </Layout>
  </div>
</template>

<script>
import Layout from "./components/Layout";
import SiteAside from "./components/SiteAside";

  components: {
    Layout,
    SiteAside,
  },
};
</script>
```

## 三、路由模式

路由模式决定了：
1. 路由从哪里获取访问路径
2. 路由如何改变访问路径

`vue-router` 提供了三种路由模式：

### 3.1 Hash 模式（默认值）

路由从浏览器地址栏中的 hash 部分获取路径，改变路径也是改变的 hash 部分。该模式兼容性最好。

```
http://localhost:8081/#/blog      → 路径：/blog
http://localhost:8081/about#/blog → 路径：/blog
```

**特点：**
- 兼容性最好，支持所有浏览器
- URL 中带有 `#` 符号
- 不需要服务器端配置

### 3.2 History 模式

路由从浏览器地址栏的 `location.pathname` 中获取路径，改变路径使用 H5 的 `history api`。该模式可以让地址栏最友好，但是需要浏览器支持 `history api`。

```
http://localhost:8081/#/blog   → 路径：/
http://localhost:8081/blog     → 路径：/blog
http://localhost:8081/about     → 路径：/about
```

**特点：**
- URL 最友好，没有 `#` 符号
- 需要浏览器支持 HTML5 History API
- 需要服务器端配置，否则刷新会 404

**服务器配置示例：**

```nginx
# nginx 配置
location / {
    try_files $uri $uri/ /index.html;
}
```

### 3.3 Abstract 模式

路由从内存中获取路径，改变路径也只是改动内存中的值。这种模式通常应用到非浏览器环境中。

```
内存：/        → /
内存：/about   → /about
内存：/blog    → /blog
```

**特点：**
- 用于非浏览器环境（如 Node.js、移动 App）
- 路径变化不会反映到地址栏

## 四、路由导航

### 4.1 RouterLink 组件

`vue-router` 提供了全局组件 `<RouterLink>`，它的渲染结果是一个 `<a>` 元素。

```vue
<!-- 基础用法 -->
<RouterLink to="/blog">文章</RouterLink>

<!-- hash 模式生成 -->
<a href="#/blog">文章</a>

<!-- history 模式生成 -->
<!-- 为避免刷新页面，vue-router 添加了点击事件并阻止默认行为，使用 history api 更改路径 -->
<a href="/blog">文章</a>
```

### 4.2 导航激活状态

默认情况下，`vue-router` 会用 **当前路径** 匹配 **导航路径**：

- **模糊匹配**：如果当前路径以导航路径开头，则为导航的 `a` 元素添加类名 `router-link-active`
- **精确匹配**：如果当前路径完全等于导航路径，则添加类名 `router-link-exact-active`

**示例：** 当前访问路径是 `/blog`

| 导航路径 | 添加的类名                                        |
| -------- | ------------------------------------------------- |
| /        | router-link-active                               |
| /blog    | router-link-active router-link-exact-active       |
| /about   | 无                                                |
| /message | 无                                                |

### 4.3 精确匹配配置

可以为 `<RouterLink>` 添加 `exact` 属性，要求必须精确匹配才能添加 `router-link-active` 类名。

**示例：** 当前访问路径是 `/blog`

| 导航路径 | exact | 类名                                        |
| -------- | ----- | ------------------------------------------- |
| /        | true  | 无                                          |
| /blog    | false | router-link-active router-link-exact-active |
| /about   | true  | 无                                          |
| /message | true  | 无                                          |

**示例：** 当前访问路径是 `/blog/detail/123`

| 导航路径 | exact | 类名               |
| -------- | ----- | ------------------ |
| /        | true  | 无                 |
| /blog    | false | router-link-active |
| /about   | true  | 无                 |
| /message | true  | 无                 |

### 4.4 自定义激活类名

可以通过属性更改默认的激活类名：

- `active-class`：更改模糊匹配的类名
- `exact-active-class`：更改精确匹配的类名

```vue
<RouterLink 
  to="/blog" 
  active-class="selected"
  exact-active-class="exact-selected"
>
  文章
</RouterLink>
```

### 4.5 实际应用示例

在导航菜单中使用 RouterLink：

```vue

  <nav class="menu-container">

      <div class="icon">
```

## 五、命名路由

使用命名路由可以解除系统与路径之间的耦合，便于维护和重构。

### 5.1 配置命名路由

```js
const router = new VueRouter({
  routes: [
    { 
      name: 'foo',     // 路由名称
      path: '/foo', 
      component: Foo 
    },
    { 
      name: 'bar', 
      path: '/bar', 
      component: Bar 
    },
  ],
});
```

### 5.2 使用命名路由

```vue
<!-- 使用命名路由，向 to 属性传递路由信息对象 -->
<!-- RouterLink 会根据传递的信息以及路由配置生成对应的路径 -->
<RouterLink :to="{ name: 'foo' }">go to foo</RouterLink>

<!-- 等价于 -->
<RouterLink to="/foo">go to foo</RouterLink>
```

### 5.3 命名路由的优势

1. **解耦**：不需要硬编码路径，修改路由配置时不需要修改导航代码
2. **可读性**：使用语义化的名称比路径更易理解
3. **维护性**：统一的命名规范便于项目维护

## 七、最佳实践

1. **路由模块化**：将路由配置分离到单独的文件中
2. **使用命名路由**：提高代码可读性和可维护性
3. **合理选择路由模式**：开发环境用 hash，生产环境用 history
4. **统一管理路由配置**：在 `routes.js` 中集中配置所有路由
5. **合理使用精确匹配**：根据页面层级关系设置 `exact` 属性

## 八、常见问题

### 8.1 History 模式刷新 404

**问题**：使用 history 模式时，刷新页面会出现 404 错误。

**原因**：服务器无法匹配到对应的路由路径。

**解决**：配置服务器将所有请求重定向到 `index.html`。

### 8.2 路由导航不激活

**问题**：导航链接没有显示激活状态。

**原因**：可能是 `exact` 属性设置不当。

**解决**：根据路由层级关系合理设置 `exact` 属性。

## 九、总结

Vue Router 是 Vue.js 官方的路由管理器，通过路由可以实现单页面应用的无刷新页面切换。核心概念包括：

- **路由配置**：定义路径和组件的映射关系
- **路由模式**：决定 URL 的表现形式和行为
- **路由导航**：实现页面之间的跳转
- **命名路由**：提高代码的可维护性
- **激活状态**：提供视觉反馈

掌握这些基础知识后，可以进一步学习嵌套路由、动态路由、路由守卫等高级特性。
