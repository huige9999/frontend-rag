# Vue从入门到实战 - 搭建工程

## 章节概述

本章将详细介绍如何使用Vue CLI搭建Vue工程，包括项目结构、单文件组件（SFC）、预编译等核心概念。

## 学习目标

- 理解Vue CLI的作用和重要性
- 掌握单文件组件（SFC）的结构和使用
- 了解Vue工程的预编译机制
- 学会创建和配置Vue项目

## 1. Vue CLI简介

### 1.1 什么是Vue CLI

`vue-cli`是Vue.js官方提供的脚手架工具，用于快速搭建Vue工程。它内部基于`webpack`构建，并预置了诸多插件和加载器，达到开箱即用的效果。

**官方文档**: https://cli.vuejs.org/zh/

### 1.2 Vue CLI预置的核心功能

除了基本的webpack插件和加载器外，`vue-cli`还预置了：

- **babel**: JavaScript编译器，支持ES6+语法
- **webpack-dev-server**: 开发服务器，支持热更新
- **eslint**: 代码质量检查工具
- **postcss**: CSS后处理器
- **less-loader**: Less预处理器支持

### 1.3 项目结构概览

```
my-site/
├── public/              # 静态资源目录
├── src/                 # 源代码目录
│   ├── components/      # 组件目录
│   ├── App.vue         # 根组件
│   └── main.js         # 入口文件
├── package.json        # 项目配置
├── babel.config.js     # Babel配置
└── vue.config.js       # Vue CLI配置
```

## 2. 单文件组件（SFC）

### 2.1 SFC概念

单文件组件（Single File Component）即一个文件就包含了一个组件所需的全部代码。这是Vue推荐的组件组织方式。

### 2.2 SFC结构

```html
<template>
  <!-- 组件模板代码 - HTML结构 -->
</template>

<script>
export default {
  // 组件配置 - JavaScript逻辑
}
</script>

<style>
/* 组件样式 - CSS样式 */
</style>
```

### 2.3 各部分说明

**template部分**: 
- 定义组件的HTML结构
- 使用Vue的模板语法
- 只能有一个根元素

**script部分**: 
- 导出组件配置对象
- 包含data、methods、computed等选项
- 可以导入其他依赖

**style部分**: 
- 定义组件的样式
- 可以使用scoped属性限定样式作用域
- 支持预处理器如less、scss

## 3. 项目配置文件详解

### 3.1 package.json

项目的核心配置文件，定义了项目依赖和脚本：

```json
{
  "name": "my-site",
  "version": "0.1.0",
  "private": true,
  "scripts": {
    "serve": "vue-cli-service serve",    // 启动开发服务器
    "build": "vue-cli-service build"     // 构建生产版本
  },
  "dependencies": {
    "core-js": "^3.6.5",     // JavaScript标准库
    "vue": "^2.6.11"         // Vue核心库
  },
  "devDependencies": {
    "@vue/cli-plugin-babel": "~4.5.0",    // Babel插件
    "@vue/cli-service": "~4.5.0",          // CLI服务
    "less": "^3.0.4",                       // Less预处理器
    "less-loader": "^5.0.0",               // Less加载器
    "vue-template-compiler": "^2.6.11"     // 模板编译器
  }
}
```

**配置说明**:
- `scripts`: 定义可执行的npm命令
- `dependencies`: 生产环境依赖
- `devDependencies`: 开发环境依赖

### 3.2 babel.config.js

Babel配置文件，用于JavaScript语法转换：

```javascript
module.exports = {
  presets: [
    '@vue/cli-plugin-babel/preset'  // 使用Vue CLI预设的Babel配置
  ]
}
```

## 4. 核心代码文件

### 4.1 入口文件 (main.js)

Vue应用的入口文件，负责创建Vue实例：

```javascript
// 入口文件
import Vue from "vue";           // 导入Vue核心库
import App from "./App.vue";     // 导入根组件

// 创建Vue实例并挂载到#app元素
new Vue({
  render: (h) => h(App),         // 渲染函数，将App组件渲染为虚拟DOM
}).$mount("#app");                // 手动挂载到页面中的#app元素
```

**代码说明**:
- `import Vue from "vue"`: 导入Vue构造函数
- `import App from "./App.vue"`: 导入根组件
- `render: (h) => h(App)`: 渲染函数，h是createElement函数
- `$mount("#app")`: 将Vue实例挂载到DOM元素

### 4.2 根组件 (App.vue)

应用的根组件，通常作为其他组件的容器：

```html
<template>
  <div>
    <h1>App组件</h1>
    <!-- 使用Avatar组件，传递url和size属性 -->
    <Avatar
      url="http://mdrs.yuanjin.tech/FgMwAPYq17So9nwVH44ltDHo7u3c"
      :size="150"
    />
    <Avatar
      url="https://qiheizhiya.oss-cn-shenzhen.aliyuncs.com/image/avatar8.jpg"
      :size="50"
    />
  </div>
</template>

<script>
// 导入Avatar组件
import Avatar from "./components/Avatar";

export default {
  name: "App",        // 组件名称，用于调试和递归调用
  components: {
    Avatar,           // 注册子组件
  },
};
</script>
```

**组件注册说明**:
- 局部注册：在components选项中注册
- 全局注册：使用Vue.component()方法
- 推荐使用局部注册，便于代码维护

### 4.3 子组件示例 (Avatar.vue)

一个典型的可复用组件示例：

```html
<template>
  <!-- 动态绑定src和样式 -->
  <img
    class="avatar-img"
    :src="url"                    <!-- 绑定图片源 -->
    :style="{                     <!-- 动态样式绑定 -->
      width: size + 'px',
      height: size + 'px',
    }"
  />
</template>

<script>
export default {
  props: {                        // 组件属性定义
    url: {
      type: String,              // url属性必须是一个字符串
      required: true,            // 属性必传
    },
    size: {
      type: Number,              // size属性必须是数字
      default: 100,             // 默认值（可选）
    },
  },
};
</script>

<style scoped>
/* scoped属性确保样式只作用于当前组件 */
.avatar-img {
  border-radius: 50%;            /* 圆形头像 */
  object-fit: cover;             /* 保持图片比例 */
  display: block;                /* 块级元素 */
}
</style>
```

**Props验证说明**:
- `type`: 属性类型（String, Number, Boolean, Array, Object等）
- `required`: 是否必传
- `default`: 默认值
- `validator`: 自定义验证函数

## 5. 预编译机制

### 5.1 什么是预编译

当`vue-cli`进行**打包**时，会直接把组件中的模板转换为`render`函数，这叫做模板预编译。

### 5.2 预编译的优势

1. **提高运行效率**: 运行时就不再需要编译模板了
2. **减少打包体积**: 打包结果中不再需要vue的编译代码

### 5.3 编译过程示意

```
.vue文件 (SFC)
    ↓
Vue Loader解析
    ↓
template → render函数  (预编译)
script → JavaScript代码
style → CSS代码
    ↓
webpack打包
    ↓
最终的JavaScript bundle
```

构建过程：

## 7. 组件开发最佳实践

### 7.1 组件命名规范

- 使用PascalCase（大驼峰）命名组件文件
- 组件name属性与文件名保持一致
- 避免使用HTML保留标签名

### 7.2 组件通信方式

1. **Props down**: 父组件向子组件传递数据
2. **Events up**: 子组件向父组件传递消息
3. **Vuex**: 跨组件状态管理
4. **provide/inject**: 跨层级组件通信

### 7.3 样式作用域

推荐使用`scoped`属性限定样式作用域：

```html
<style scoped>
/* 只作用于当前组件 */
.component-name {
  color: red;
}
</style>
```

## 8. 常见问题与解决方案

### 8.1 组件注册失败

**问题**: 组件未正确注册或导入

**解决方案**:
- 检查import路径是否正确
- 确认组件已在components中注册
- 查看控制台错误信息

### 8.2 样式不生效

**问题**: 样式被覆盖或不生效

**解决方案**:
- 使用scoped属性限定样式作用域
- 检查CSS选择器优先级
- 确认样式文件正确导入

### 8.3 热更新不工作

**问题**: 代码修改后页面不自动更新

**解决方案**:
- 重启开发服务器
- 检查webpack配置
- 清除浏览器缓存

## 9. 总结

本章重点介绍了Vue工程搭建的核心概念：

1. **Vue CLI**: 提供开箱即用的项目脚手架
2. **SFC**: 单文件组件，整合模板、逻辑和样式
3. **预编译**: 构建时将模板转换为渲染函数
4. **组件化**: 通过组件复用提高开发效率

通过这些概念和工具，我们可以快速搭建起结构清晰、易于维护的Vue应用。
