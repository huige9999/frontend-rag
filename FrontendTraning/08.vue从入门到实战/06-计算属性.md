# 06. 计算属性

## 什么是计算属性

计算属性是Vue中一种特殊的属性，它基于其他数据计算得出，并且会自动缓存计算结果。当依赖的数据发生变化时，计算属性会自动重新计算。

### 核心特点

1. **自动缓存**：计算属性会缓存结果，只有依赖变化时才重新计算
2. **响应式更新**：依赖的数据变化时自动更新
3. **简化模板逻辑**：将复杂的计算逻辑从模板中提取出来
4. **支持getter和setter**：可以自定义获取和设置行为

## 基本语法

### 完整写法（包含getter和setter）

```javascript
export default {
  data() {
    return {
      firstName: '张',
      lastName: '三'
    }
  },
  computed: {
    // 完整的计算属性写法
    fullName: {
      // getter：用于获取计算属性的值
      get() {
        console.log('计算 fullName')
        return this.firstName + this.lastName
      },
      // setter：用于设置计算属性的值
      set(newValue) {
        console.log('设置 fullName:', newValue)
        // 将新值分解并赋值给依赖的数据
        const names = newValue.split(' ')
        this.firstName = names[0]
        this.lastName = names[1] || ''
      }
    }
  }
}
```

### 简写（只有getter）

```javascript
export default {
  data() {
    return {
      firstName: '张',
      lastName: '三'
    }
  },
  computed: {
    // 简写形式，只有getter
    fullName() {
      console.log('计算 fullName')
      return this.firstName + this.lastName
    }
  }
}
```

## 计算属性 vs 方法

| 特性 | 计算属性 | 方法 |
|------|---------|------|
| 缓存机制 | 有缓存，依赖不变不重新计算 | 无缓存，每次调用都执行 |
| 使用方式 | 作为属性使用：`{{ fullName }}` | 作为函数调用：`{{ getFullName() }}` |
| 适用场景 | 根据数据得到其他数据 | 处理事件、执行操作 |
| 性能 | 更好（利用缓存） | 较差（每次都执行） |

### 对比示例

```vue
<template>
  <div>
    <!-- 使用计算属性 -->
    <h2>计算属性：{{ fullName }}</h2>
    <h2>计算属性：{{ fullName }}</h2>
    
    <!-- 使用方法 -->
    <h2>方法调用：{{ getFullName() }}</h2>
    <h2>方法调用：{{ getFullName() }}</h2>
  </div>
</template>

<script>
export default {
  data() {
    return {
      firstName: '张',
      lastName: '三'
    }
  },
  computed: {
    fullName() {
      console.log('计算属性被调用')
      return this.firstName + ' ' + this.lastName
    }
  },
  methods: {
    getFullName() {
      console.log('方法被调用')
      return this.firstName + ' ' + this.lastName
    }
  }
}
</script>
```

**输出结果：**
```
计算属性被调用
方法被调用
方法被调用
```

可以看到，计算属性只执行了一次，而方法执行了两次，这说明计算属性具有缓存特性。

## 计算属性的最佳实践

### ✅ 适合使用计算属性的场景

1. **复杂的数据处理**
```javascript
computed: {
  // 过滤和排序列表
  filteredList() {
    return this.items
      .filter(item => item.price > 100)
      .sort((a, b) => b.price - a.price)
  },
  
  // 格式化显示
  formattedPrice() {
    return '¥' + this.price.toFixed(2)
  }
}
```

2. **多数据组合**
```javascript
computed: {
  // 组合多个数据源
  userInfo() {
    return {
      name: this.user.name,
      age: this.user.age,
      department: this.department.name
    }
  }
}
```

### ❌ 不适合使用计算属性的场景

```javascript
// ❌ 错误：在计算属性中使用异步操作
computed: {
  async userData() {
    const response = await fetch('/api/user')
    return response.json()
  }
}

// ❌ 错误：在计算属性中使用随机数
computed: {
  randomValue() {
    return Math.random() // 每次依赖不变时不应该产生不同结果
  }
}

// ❌ 错误：在计算属性中使用当前时间
computed: {
  currentTime() {
    return new Date() // 时间应该实时变化，不适合缓存
  }
}
```

## 实战示例：购物车总价计算

```vue
<template>
  <div class="shopping-cart">
    <h2>购物车</h2>
    <div v-for="item in cartItems" :key="item.id" class="cart-item">
      <span>{{ item.name }}</span>
      <span>单价：¥{{ item.price }}</span>
      <span>数量：{{ item.quantity }}</span>
      <span>小计：¥{{ item.price * item.quantity }}</span>
    </div>
    
    <div class="cart-summary">
      <p>商品总数：{{ totalQuantity }}</p>
      <p>总价：¥{{ totalPrice }}</p>
      <p v-if="discount > 0">优惠：-¥{{ discountAmount }}</p>
      <p class="final-price">应付：¥{{ finalPrice }}</p>
    </div>
  </div>
</template>

<script>
export default {
  data() {
    return {
      cartItems: [
        { id: 1, name: '商品A', price: 199, quantity: 2 },
        { id: 2, name: '商品B', price: 299, quantity: 1 },
        { id: 3, name: '商品C', price: 99, quantity: 3 }
      ],
      discount: 0.1 // 10%折扣
    }
  },
  computed: {
    // 计算商品总数
    totalQuantity() {
      return this.cartItems.reduce((total, item) => {
        return total + item.quantity
      }, 0)
    },
    
    // 计算总价
    totalPrice() {
      return this.cartItems.reduce((total, item) => {
        return total + (item.price * item.quantity)
      }, 0)
    },
    
    // 计算折扣金额
    discountAmount() {
      return this.totalPrice * this.discount
    },
    
    // 计算最终价格
    finalPrice() {
      return (this.totalPrice - this.discountAmount).toFixed(2)
    }
  }
}
</script>

<style scoped>

}

}

.cart-summary {
  margin-top: 20px;

  background-color: #f5f5f5;
  border-radius: 4px;
}

.final-price {
  font-size: 18px;
  font-weight: bold;
  color: #f56c6c;
}
</style>
```

## 计算属性的进阶用法

### 计算属性的setter应用

```vue
<template>
  <div>
    <input v-model="firstName" placeholder="名">
    <input v-model="lastName" placeholder="姓">
    <input v-model="fullName" placeholder="全名">
    <p>全名：{{ fullName }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      firstName: '',
      lastName: ''
    }
  },
  computed: {
    fullName: {
      // getter：返回全名
      get() {
        return this.firstName + ' ' + this.lastName
      },
      // setter：设置全名时自动分解
      set(newValue) {
        const names = newValue.split(' ')
        this.firstName = names[0] || ''
        this.lastName = names[1] || ''
      }
    }
  }
}
</script>
```

### 计算属性的依赖追踪

```javascript
export default {
  data() {
    return {
      items: [1, 2, 3, 4, 5]
    }
  },
  computed: {
    // 只有 items 变化时才重新计算
    evenItems() {
      console.log('重新计算偶数项')
      return this.items.filter(item => item % 2 === 0)
    },
    
    // computed 也可以依赖其他 computed
    evenItemsCount() {
      return this.evenItems.length
    },
    
    // computed 也可以依赖 data 和 computed
    summary() {
      return `共 ${this.items.length} 项，其中偶数 ${this.evenItemsCount} 项`
    }
  }
}
```

## 面试题：计算属性和方法有什么区别？

**答案：**

1. **缓存机制**
   - 计算属性本质上是包含getter和setter的方法
   - 当获取计算属性时，实际上是在调用计算属性的getter方法
   - Vue会收集计算属性的依赖，并缓存计算属性的返回结果
   - 只有当依赖变化后才会重新进行计算
   - 方法没有缓存，每次调用方法都会导致重新执行

2. **参数限制**
   - 计算属性的getter和setter参数固定，getter没有参数，setter只有一个参数
   - 方法的参数不限

3. **使用场景**
   - 计算属性通常是根据已有数据得到其他数据
   - 在计算属性中不建议使用异步、当前时间、随机数等副作用操作
   - 方法用于处理事件、执行操作等

4. **语义区别**
   - **最重要的区别是含义上的区别**
   - 计算属性含义上也是一个数据，可以读取也可以赋值
   - 方法含义上是一个操作，用于处理一些事情

## 总结

### 计算属性的适用原则

1. **用于数据转换**：将原始数据转换为显示需要的格式
2. **用于数据组合**：将多个数据源组合成一个新的数据
3. **避免副作用**：不要在计算属性中进行异步操作或修改外部状态
4. **性能优先**：对于复杂的计算逻辑，优先使用计算属性而不是方法

### 记忆要点

- ✅ 计算属性有缓存，方法没有缓存
- ✅ 计算属性依赖不变不重新计算，方法每次都执行
- ✅ 计算属性是数据，方法是操作
- ✅ 计算属性可以包含getter和setter
- ❌ 计算属性中不能使用异步操作
- ❌ 计算属性中不能使用随机数或当前时间
