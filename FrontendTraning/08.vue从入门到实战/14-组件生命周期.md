# 14. 组件生命周期

## 学习目标

- 理解组件生命周期的概念和重要性
- 掌握Vue 3组件生命周期的各个阶段
- 学会在适当的时机执行相应的操作
- 了解生命周期在实际开发中的应用场景

## 什么是组件生命周期

组件生命周期是指一个组件从创建到销毁的整个过程。在这个过程中，Vue提供了多个生命周期钩子（Lifecycle Hooks），让开发者在特定时机执行代码。

## Vue 3 生命周期图解

```
创建阶段
   ↓
beforeCreate → created
   ↓
挂载阶段
   ↓
beforeMount → mounted
   ↓
更新阶段（多次）
   ↓
beforeUpdate → updated
   ↓
销毁阶段
   ↓
beforeUnmount → unmounted
```

## 生命周期钩子详解

### 1. 创建阶段

#### beforeCreate
- **时机**：实例刚在内存中被创建出来，组件属性计算之前
- **特点**：此时data、methods都不可用
- **应用场景**：一般不使用此钩子

#### created
- **时机**：实例已经创建完成，data、methods已可用
- **特点**：DOM还未生成，$ref不可用
- **应用场景**：
  - 发起Ajax请求获取初始数据
  - 初始化一些状态
  - 启动定时器

```javascript
export default {
  data() {
    return {
      news: []
    }
  },
  async created() {
    // 加载远程数据
    this.news = await getNews();
  }
}
```

### 2. 挂载阶段

#### beforeMount
- **时机**：模板编译完成，虚拟DOM已生成，还未挂载到页面
- **特点**：最后一次在渲染前修改数据的机会

#### mounted
- **时机**：组件DOM已经渲染完成并挂载到页面
- **特点**：可以访问DOM元素和$ref
- **应用场景**：
  - 操作DOM
  - 集成第三方库（如ECharts、Swiper等）
  - 获取元素尺寸

```javascript
export default {
  data() {
    return {
      containerWidth: 0,
      containerHeight: 0
    }
  },
  mounted() {
    // 直接操作DOM，获取容器尺寸
    this.containerWidth = this.$refs.container.clientWidth;
    this.containerHeight = this.$refs.container.clientHeight;
  }
}
```

### 3. 更新阶段

#### beforeUpdate
- **时机**：数据变化，DOM更新之前
- **特点**：可以在此进一步更改状态，不会触发重渲染

#### updated
- **时机**：DOM更新完成后
- **特点**：可以执行依赖DOM更新结果的操作
- **注意**：避免在此期间更改状态，可能导致无限循环

```javascript
export default {
  data() {
    return {
      count: 0
    }
  },
  updated() {
    console.log('组件更新完成');
  },
  methods: {
    increment() {
      this.count++;
    }
  }
}
```

### 4. 销毁阶段

#### beforeUnmount
- **时机**：组件即将被卸载
- **特点**：此时组件实例依然可用
- **应用场景**：
  - 清除定时器
  - 解绑事件监听
  - 销毁第三方实例

#### unmounted
- **时机**：组件已被卸载
- **特点**：DOM已被移除，所有指令和事件监听器都已解绑

```javascript
export default {
  data() {
    return {
      timer: null
    }
  },
  created() {
    // 创建定时器
    this.timer = setInterval(() => {
      console.log('定时器执行中...');
    }, 1000);
  },
  beforeUnmount() {
    // 清除定时器，防止内存泄漏
    clearInterval(this.timer);
  }
}
```

## 实际应用案例

### 案例1：时钟组件

```vue
<template>
  <h1>当前时间：{{ current }}</h1>
</template>

<script>
export default {
  data() {
    return {
      current: this.getCurrent(),
      timer: null,
    };
  },
  methods: {
    getCurrent() {
      return new Date().toLocaleTimeString();
    },
  },
  created() {
    console.log("created - 组件创建完成");
    // 在created中启动定时器
    this.timer = setInterval(() => {
      console.log("更新了时间");
      this.current = this.getCurrent();
    }, 1000);
  },
  beforeUnmount() {
    console.log("beforeUnmount - 组件即将销毁");
    // 在组件销毁前清除定时器
    clearInterval(this.timer);
  },
};
</script>
```

### 案例2：图片加载器组件

```vue
<template>
  <div class="image-loader-container">
    <!-- 显示占位图 -->
    <img v-if="!everythingDone" class="placeholder" :src="placeholder" alt="" />
    <!-- 原图 -->
    <img
      @load="handleLoad"
      :src="src"
      alt=""
      :style="{ opacity: originOpacity, transition: `${duration}ms` }"
    />
  </div>
</template>

<script>
export default {
  props: {
    src: {
      type: String,
      required: true,
    },
    placeholder: {
      type: String,
      required: true,
    },
    duration: {
      type: Number,
      default: 500,
    },
  },
  data() {
    return {
      originLoaded: false,    // 原图是否加载完成
      everythingDone: false,  // 是否一切都尘埃落定了
    };
  },
  computed: {
    originOpacity() {
      return this.originLoaded ? 1 : 0;
    },
  },
  methods: {
    handleLoad() {
      // 图片加载完成后，开始渐变动画
      this.originLoaded = true;
      setTimeout(() => {
        this.everythingDone = true;
        this.$emit("load");
      }, this.duration);
    },
  },
};
</script>

<style scoped lang="less">
.image-loader-container {
  width: 100%;
  height: 100%;
  position: relative;
  overflow: hidden;
  img {
    width: 100%;
    height: 100%;
    object-fit: cover;
  }
  .placeholder {
    filter: blur(2vw);  // 占位图模糊效果
  }
}
</style>
```

### 案例3：获取远程数据

<template>

      <li v-for="item in banners" :key="item.id">

## 生命周期最佳实践

### 1. 数据获取时机

```javascript
// ✅ 推荐：在created中获取数据
async created() {
  this.data = await fetchData();
}

// ❌ 不推荐：在mounted中获取数据（会导致页面先空白）
async mounted() {
  this.data = await fetchData();
}
```

### 2. DOM操作时机

```javascript
// ✅ 推荐：在mounted中操作DOM
mounted() {
  const width = this.$refs.container.clientWidth;
}

// ❌ 不推荐：在created中操作DOM（此时DOM未生成）
created() {
  const width = this.$refs.container.clientWidth; // undefined
}
```

### 3. 资源清理

```javascript
export default {
  data() {
    return {
      timer: null,
      eventHandler: null
    }
  },
  created() {
    // 创建资源
    this.timer = setInterval(() => {}, 1000);
    this.eventHandler = () => {};
    window.addEventListener('resize', this.eventHandler);
  },
  beforeUnmount() {
    // 清理资源，防止内存泄漏
    clearInterval(this.timer);
    window.removeEventListener('resize', this.eventHandler);
  }
}
```

## Vue 2 vs Vue 3 生命周期对比

| Vue 2          | Vue 3              | 说明           |
|----------------|-------------------|----------------|
| beforeCreate   | beforeCreate      | 实例创建前      |
| created        | created           | 实例创建后      |
| beforeMount    | beforeMount       | 挂载前         |
| mounted        | mounted           | 挂载后         |
| beforeUpdate   | beforeUpdate      | 更新前         |
| updated        | updated           | 更新后         |
| beforeDestroy  | beforeUnmount     | 销毁前（重命名） |
| destroyed      | unmounted         | 销毁后（重命名） |

## 组合式API中的生命周期

在Vue 3的组合式API中，生命周期钩子前面加上`on`：

```javascript
import { onMounted, onUnmounted } from 'vue';

export default {
  setup() {
    onMounted(() => {
      console.log('组件已挂载');
    });
    
    onUnmounted(() => {
      console.log('组件已卸载');
    });
  }
}
```

## 常见问题

### 1. 为什么异步请求要放在created而不是mounted？

- created时机更早，页面渲染更快
- mounted需要等待DOM渲染完成，会导致页面先空白
- 一般的异步请求不依赖DOM，放在created更合适

### 2. 如何避免内存泄漏？

- 在组件销毁时清除定时器
- 解绑事件监听器
- 销毁第三方库实例

### 3. 多次更新会不会导致性能问题？

- Vue有异步更新队列，会合并多次更新
- 避免在updated中修改数据导致死循环
- 复杂计算使用computed或watch

## 总结

组件生命周期是Vue开发中的重要概念：

1. **创建阶段**：初始化数据，发起请求
2. **挂载阶段**：操作DOM，集成第三方库
3. **更新阶段**：响应数据变化
4. **销毁阶段**：清理资源，防止内存泄漏

记住一个重要原则：**不要死记硬背，根据具体需求灵活使用**。
