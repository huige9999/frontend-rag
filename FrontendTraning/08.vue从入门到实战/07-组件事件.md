- 理解组件事件的概念和使用场景

## 组件事件概述

### 什么是组件事件

组件事件是Vue中实现子组件向父组件通信的重要机制。当子组件在某个时候发生了一件事，但自身无法处理时，可以通过事件的方式通知父组件进行处理。

### 事件相关概念

- **抛出事件**：子组件通过`this.$emit()`方法触发自定义事件，向父组件传递信息
- **事件参数**：子组件抛出事件时，传递给父组件的数据
- **注册事件**：父组件通过`@事件名`或`v-on:事件名`监听子组件的事件

## 自定义事件的使用

### 子组件抛出事件

```vue
<template>
  <div>
    <button @click="handleClick">点击我</button>
  </div>
</template>

<script>
export default {
  methods: {
    handleClick() {
      // 抛出自定义事件，可以传递参数
      this.$emit('my-event', '参数1', '参数2');
    }
  }
}
</script>
```

### 父组件监听事件

```vue
<template>
  <div>
    <ChildComponent @my-event="handleMyEvent" />
  </div>
</template>

<script>
export default {
  methods: {
    handleMyEvent(param1, param2) {
      console.log('收到事件参数：', param1, param2);
    }
  }
}
</script>
```

## 实战案例：分页组件

### 需求分析

开发一个通用的分页组件，具备以下功能：

- 显示当前页码和总页数
- 提供首页、上一页、下一页、末页导航
- 支持点击页码直接跳转
- 当前页码高亮显示
- 边界页码按钮禁用

### 组件设计

#### 属性（Props）

| 属性名        | 含义       | 类型   | 必填 | 默认值 |
| ------------- | ---------- | ------ | ---- | ------ |
| current       | 当前页码   | Number | 否   | 1      |
| total         | 总数据量   | Number | 否   | 0      |
| limit         | 页容量     | Number | 否   | 10     |
| visibleNumber | 可见页码数 | Number | 否   | 10     |

#### 事件（Events）

| 事件名     | 含义     | 事件参数 | 参数类型 |
| ---------- | -------- | -------- | -------- |
| pageChange | 页码变化 | 新的页码 | Number   |

### 组件实现

#### Pager.vue

```vue
<template>
  <!-- 只有总页数大于1时才显示分页器 -->
  <div class="pager-container" v-if="pageNumber > 1">
    <!-- 首页按钮 -->
    <a @click="handleClick(1)" :class="{ disabled: current === 1 }">
      |&lt;&lt;
    </a>
    <!-- 上一页按钮 -->
    <a @click="handleClick(current - 1)" :class="{ disabled: current === 1 }">
      &lt;&lt;
    </a>
    <!-- 页码按钮 -->
    <a
      @click="handleClick(n)"
      v-for="(n, i) in numbers"
      :key="i"
      :class="{ active: n === current }"
    >
      {{ n }}
    </a>
    <!-- 下一页按钮 -->
    <a
      @click="handleClick(current + 1)"
      :class="{ disabled: current === pageNumber }"
    >
      &gt;&gt;
    </a>
    <!-- 末页按钮 -->
    <a
      @click="handleClick(pageNumber)"
      :class="{ disabled: current === pageNumber }"
    >
      &gt;&gt;|
    </a>
  </div>
</template>

<script>
export default {
  // 组件属性定义
  props: {
    current: {
      type: Number,
      default: 1,
    },
    total: {
      type: Number,
      default: 0,
    },
    limit: {
      type: Number,
      default: 10,
    },
    visibleNumber: {
      type: Number,
      default: 10,
    },
  },
  computed: {
    // 计算总页数
    pageNumber() {
      return Math.ceil(this.total / this.limit);
    },
    // 计算可见页码的最小值
    visibleMin() {
      let min = this.current - Math.floor(this.visibleNumber / 2);
      if (min < 1) {
        min = 1;
      }
      return min;
    },
    // 计算可见页码的最大值
    visibleMax() {
      let max = this.visibleMin + this.visibleNumber - 1;
      if (max > this.pageNumber) {
        max = this.pageNumber;
      }
      return max;
    },
    // 生成可见页码数组
    numbers() {
      let nums = [];
      for (let i = this.visibleMin; i <= this.visibleMax; i++) {
        nums.push(i);
      }
      return nums;
    },
  },
  methods: {
    // 处理页码点击
    handleClick(newPage) {
      // 边界处理
      if (newPage < 1) {
        newPage = 1;
      }
      if (newPage > this.pageNumber) {
        newPage = this.pageNumber;
      }
      // 如果点击的是当前页，不做处理
      if (newPage === this.current) {
        return;
      }
      // 抛出页码变化事件，将新页码传递给父组件
      this.$emit("pageChange", newPage);
    },
  },
};
</script>

<style lang="less" scoped>
@import "~@/styles/var.less";
.pager-container {
  display: flex;
  justify-content: center;
  margin: 20px 0;
  a {
    color: @primary;
    margin: 0 6px;
    cursor: pointer;
    &.disabled {
      color: @lightWords;
      cursor: not-allowed;
    }
    &.active {
      color: @words;
      font-weight: bold;
      cursor: text;
    }
  }
}
</style>
```

### 父组件使用

#### App.vue

```vue
<template>
  <div>
    <h1>App组件</h1>
    <!-- 使用分页组件 -->
    <Pager
      :total="total"
      :current="current"
      @pageChange="handlePageChange"
    />
  </div>
</template>

<script>
import Pager from "./components/Pager";

export default {
  components: {
    Pager,
  },
  data() {
    return {
      current: 1,        // 当前页码
      total: 302,        // 总数据量
    };
  },
  methods: {
    // 处理页码变化事件
    handlePageChange(newPage) {
      this.current = newPage;
      console.log("当前页码：", newPage);
      // 在这里可以加载对应页码的数据
      this.loadData(newPage);
    },
    // 模拟加载数据
    loadData(page) {
      console.log(`正在加载第${page}页数据...`);
      // 实际开发中这里会调用API获取数据
    }
  },
};
</script>
```

## 事件修饰符

Vue提供了多种事件修饰符来处理事件：

### 常用修饰符

```vue
<!-- 阻止事件冒泡 -->
<ChildComponent @click.stop="handleClick" />

<!-- 阻止默认行为 -->
<form @submit.prevent="handleSubmit">

<!-- 只触发一次 -->
<ChildComponent @click.once="handleClick">

<!-- 添加事件监听器时使用事件捕获模式 -->
<ChildComponent @click.capture="handleClick">

<!-- 只当事件从侦听器绑定的元素本身触发时才触发回调 -->
<ChildComponent @click.self="handleClick">
```

## v-if 和 v-show 的区别

### 基本概念

**v-if**：条件性渲染，为true时渲染元素，为false时不渲染

```vue
<div v-if="isVisible">内容</div>
```

**v-show**：切换元素的display CSS属性

```vue
<div v-show="isVisible">内容</div>
```

### 区别对比

| 特性       | v-if                        | v-show                        |
| ---------- | --------------------------- | ----------------------------- |
| DOM生成    | 控制是否生成DOM元素         | 始终生成DOM元素               |
| 渲染原理   | 控制vnode生成              | 控制display属性               |
| 切换成本   | 高（需要销毁/重建DOM）     | 低（仅修改CSS属性）           |
| 初始渲染成本 | 低（条件为false时不渲染）  | 高（始终渲染所有内容）         |
| 树的稳定性 | 不稳定（DOM结构会变化）     | 稳定（DOM结构不变）           |

### 使用建议

**使用v-if的场景**：
- 显示状态变化较少
- 需要减少DOM节点数量
- 条件判断复杂且不会频繁变化

**使用v-show的场景**：
- 显示状态变化频繁
- 需要保持DOM树的稳定
- 简单的显示/隐藏切换

### 面试要点

```javascript
/*
v-if和v-show的本质区别：

v-if能够控制是否生成vnode，也就间接控制了是否生成对应的dom。
当v-if为true时，会生成对应的vnode，并生成对应的dom元素；
当其为false时，不会生成对应的vnode，自然不会生成任何的dom元素。

v-show始终会生成vnode，也就间接导致了始终生成dom。
它只是控制dom的display属性，当v-show为true时，不做任何处理；
当其为false时，生成的dom的display属性为none。

使用v-if可以有效的减少树的节点和渲染量，但也会导致树的不稳定；
而使用v-show可以保持树的稳定，但不能减少树的节点和渲染量。

因此，在实际开发中：
- 显示状态变化频繁的情况下应该使用v-show，以保持树的稳定
- 显示状态变化较少时应该使用v-if，以减少树的节点和渲染量
*/
```

## 最佳实践

### 1. 事件命名规范

```vue
<!-- 推荐：使用kebab-case命名 -->
<ChildComponent @page-change="handlePageChange" />
this.$emit('page-change', newPage);

<!-- 避免：使用camelCase命名 -->
<ChildComponent @pageChange="handlePageChange" />
```

### 2. 事件参数传递

```vue
<!-- 推荐：传递有意义的对象 -->
this.$emit('update', { id: 1, name: 'Item' });

<!-- 避免：传递多个散列参数 -->
this.$emit('update', 1, 'Item', 'other');
```

### 3. 事件处理

```vue
<!-- 父组件中 -->
<script>
export default {
  methods: {
    // 推荐：使用描述性的事件处理函数名
    handlePageChange(newPage) {
      // 处理逻辑
    },

    // 避免：使用过于简单的函数名
    change(newPage) {
      // 处理逻辑
    }
  }
}
</script>
```

### 4. 事件验证

```vue
<script>
export default {
  // 推荐：在组件中定义事件
  emits: {
    // 带验证的事件
    pageChange: (payload) => {
      if (typeof payload === 'number' && payload > 0) {
        return true;
      }
      console.warn('pageChange事件参数必须是正数');
      return false;
    }
  },
  methods: {
    handleClick(newPage) {
      this.$emit('pageChange', newPage);
    }
  }
}
</script>
```

## 总结

### 核心概念

1. **组件事件**：子组件向父组件通信的重要机制
2. **事件流程**：子组件抛出事件 → 父组件监听并处理
3. **事件参数**：可以在事件中传递数据

### 关键技术点

1. `this.$emit()`：抛出自定义事件
2. `@event-name`：监听自定义事件
3. `v-if` vs `v-show`：根据场景选择合适的条件渲染方式

### 实践建议

1. 合理使用组件事件实现子父组件通信
2. 事件命名要清晰明确
3. 事件参数要传递有意义的数据
4. 根据实际需求选择v-if或v-show
