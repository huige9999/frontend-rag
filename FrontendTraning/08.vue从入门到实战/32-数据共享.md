# 32. 数据共享

## 章节概述

在 Vue 应用开发中，当多个组件需要共享数据时，会带来一系列问题。本章节将介绍如何使用 Vuex 来解决这些数据共享问题。

## 学习目标

- 理解数据共享面临的挑战
- 掌握 Vuex 的基本概念和使用方法
- 学会如何使用 State、Mutation 和 Action 管理共享数据

## 核心问题

### 数据共享的挑战

在 Vue 中遇到**共享数据**时，会带来以下问题：

1. **如何保证数据的唯一性？**
   - 如果数据不唯一，会浪费大量内存资源，降低运行效率
   - 如果数据不唯一，可能出现数据不统一的情况，难以维护

2. **某个组件改动数据后，如何让其他用到该数据的组件知道数据变化了？**
   - 事件总线貌似可以解决，但需要手动维护监听，极其不方便
   - 事件总线的目的在于「通知」，而不是「共享数据」

### 传统方案的问题

一种容易想到的方案是将所有共享数据提升到根组件，通过 props 层层下发，通过事件层层上抛：

**缺点：**
- 需要编写大量代码层层下发数据，很多组件被迫拥有自己不需要的数据
- 需要编写大量代码层层上抛事件，很多组件被迫注册自己处理不了的事件

## 解决方案：独立数据仓库

设置一个**独立的数据仓库**是更好的解决方案：

```
┌─────────────┐
│   仓库      │
│ (数据存储)  │
└──────┬──────┘
       │
   ┌───┴───┐
   │       │
┌──▼──┐ ┌─▼────┐
│组件A│ │组件B │
└─────┘ └──────┘
```

**优势：**
- 组件需要什么共享数据，可以自由从仓库中获取，需要什么拿什么
- 组件可以自由改变仓库中的数据，仓库数据变化后，会自动通知用到对应数据的组件更新

## Vuex

Vuex 是 Vue 官方提供的状态管理库，用于解决 Vue 应用中的数据共享问题。

### 安装 Vuex

```bash
# 为了保证和课程一致，请安装3.6.2版本
npm i vuex@3.6.2
```

## 创建仓库

### 基本配置

```javascript
// store/index.js
import Vuex from "vuex";
import Vue from "vue";

// 应用 vuex 插件
Vue.use(Vuex);

const store = new Vuex.Store({
  // 仓库配置对象
  state: {
    // 仓库的初始状态（数据）
    count: 0, // 计数器，默认值为0
  },
});

export default store;
```

### 在 Vue 应用中注入仓库

```javascript
// main.js
import Vue from "vue";
import App from "./App.vue";
import store from "./store";

new Vue({
  store, // 向 vue 中注入仓库
  render: (h) => h(App),
}).$mount("#app");
```

### 在组件中使用仓库数据

```vue
<template>
  <div>
    <h1>{{ $store.state.count }}</h1>
  </div>
</template>

<script>
export default {
  // 可以通过 this.$store 访问仓库
};
</script>
```

Vuex 会自动将配置的状态数据设置为响应式数据，当数据变化时，依赖该数据的组件会自动渲染。

## 数据的变更（Mutation）

### 为什么需要 Mutation

虽然可以直接修改数据，但这样的做法在大型项目中会遇到问题：

> 如果有一天，你发现某个共享数据是错误的，而有一百多个组件都有可能变更过这块数据，你该如何知道是哪一步数据变更出现了问题？

为了更好地跟踪数据的变化，**Vuex 强烈建议使用 Mutation 来更改数据**。

### 定义 Mutation

```javascript
const store = new Vuex.Store({
  state: {
    count: 0,
  },
  mutations: {
    /**
     * 每个 mutation 是一个方法，描述了数据在某种场景下的变化
     * @param {Object} state - 当前的仓库状态数据
     */
    increase(state) {
      state.count++; // 增加计数
    },
    
    decrease(state) {
      state.count--; // 减少计数
    },
    
    /**
     * 计算n次幂
     * @param {Object} state - 当前的仓库状态数据
     * @param {*} payload - 负荷（负载），让数据产生变化时的附加信息
     * payload 可以是任何类型：数字、字符串、对象等
     * 在此 mutation 中，约定 payload 为数字，表示指数
     */
    power(state, payload) {
      state.count **= payload; // 计算幂次
    },
  },
});
```

### 提交 Mutation

在组件中通过 `store.commit` 方法提交 mutation：

```vue
<template>
  <div>
    <h1>{{ $store.state.count }}</h1>
    <button @click="handleIncrease">增加</button>
    <button @click="handlePower(3)">立方</button>
  </div>
</template>

<script>
export default {
  methods: {
    handleIncrease() {
      // 提交 mutation
      this.$store.commit("increase");
    },
    handlePower(n) {
      // 提交带 payload 的 mutation
      this.$store.commit("power", n);
    },
  },
};
</script>
```

### Mutation 的重要规则

1. **Mutation 中不得出现异步操作**
   - 在实际开发的规范中，甚至要求不得有副作用操作
   - 副作用操作包括：
     - 异步操作
     - 更改或读取外部环境的信息（如 localStorage、location、DOM 等）

2. **提交 Mutation 是数据改变的唯一原因**
   
   这样可以通过 Vue DevTools 清晰地追踪数据变化流程。

## 异步处理（Action）

如果在 Vuex 中要进行异步操作，需要使用 **Action**。

### 定义 Action

```javascript
// 辅助函数：延迟指定时间
function delay(duration) {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve();
    }, duration);
  });
}

const store = new Vuex.Store({
  state: {
    count: 0,
  },
  mutations: {
    increase(state) {
      state.count++;
    },
    decrease(state) {
      state.count--;
    },
    power(state, payload) {
      state.count **= payload;
    },
  },
  actions: {
    /**
     * Action 用于处理异步操作
     * @param {Object} ctx - 类似于 store 的对象，包含 commit、dispatch、state 等方法
     * @param {*} payload - 本次异步操作的额外信息
     */
    async asyncIncrease(ctx) {
      await delay(1000); // 延迟1秒
      ctx.commit("increase"); // 提交 mutation
    },
    
    async asyncDecrease(ctx) {
      await delay(1000);
      ctx.commit("decrease");
    },
    
    async asyncPower(ctx, payload) {
      await delay(1000);
      ctx.commit("power", payload); // 提交带参数的 mutation
    },
  },
});
```

### 分发 Action

在组件中通过 `store.dispatch` 方法分发 action：

```vue
<template>
  <div class="block">
    <h1>{{ $store.state.count }}</h1>
    <button @click="handleAsyncIncrease">异步增加</button>
    <button @click="handleAsyncDecrease">异步减少</button>
  </div>
</template>

<script>
export default {
  methods: {
    handleAsyncIncrease() {
      // 分发 action
      this.$store.dispatch("asyncIncrease");
    },
    handleAsyncDecrease() {
      this.$store.dispatch("asyncDecrease");
    },
  },
};
</script>
```

### 仓库配置

```javascript
// src/store/index.js
import Vuex from "vuex";
import Vue from "vue";

// 应用 vuex 插件
Vue.use(Vuex);

// 延迟辅助函数
function delay(duration) {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve();
    }, duration);
  });
}

// 创建并导出仓库
const store = new Vuex.Store({
  state: {
    count: 0, // 默认值为0
  },
  mutations: {
    increase(state) {
      state.count++;
    },
    decrease(state) {
      state.count--;
    },
    // payload：负荷/负载
    power(state, payload) {
      state.count **= payload;
    },
  },
  actions: {
    async asyncIncrease(ctx) {
      await delay(1000);
      ctx.commit("increase");
    },
    async asyncDecrease(ctx) {
      await delay(1000);
      ctx.commit("decrease");
    },
    async asyncPower(ctx, payload) {
      await delay(1000);
      ctx.commit("power", payload);
    },
  },
});

// 为了方便调试，将仓库挂载到 window 对象
window.store = store;

export default store;
```

```
┌─────────────────────────────────────────────────┐

│  ┌──────────┐      ┌──────────┐              │

│       │         │   (异步)    │              │
│       │         └──────┬──────┘              │
│       │                │                     │
│  ┌────▼────────────────▼──────────────────┐ │
│  │           组件自动更新 (响应式)          │ │
│  └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────────┘
```

## 核心概念总结

| 概念 | 说明 | 使用场景 |
|------|------|----------|
| **State** | 仓库中的状态数据，是响应式的 | 存储共享数据 |
| **Mutation** | 同步函数，用于修改 State | 所有数据修改操作 |
| **Action** | 异步函数，可以包含异步逻辑 | 需要异步操作后修改数据 |
| **Commit** | 提交 Mutation | `store.commit('mutation名', payload)` |
| **Dispatch** | 分发 Action | `store.dispatch('action名', payload)` |

## 最佳实践

1. **数据修改原则**
   - 所有数据修改必须通过 Mutation
   - Mutation 必须是同步的、无副作用的
   - 提交 Mutation 是数据改变的唯一原因

2. **异步操作原则**
   - 异步操作必须在 Action 中进行
   - Action 完成异步后，通过 Commit 提交 Mutation

3. **调试技巧**
   - 安装 Vue DevTools 可以查看 Vuex 状态
   - 可以追踪每次数据变化的历史记录

4. **项目组织**
   - 一个应用通常只需要创建一个 Store
   - 将 Store 配置单独放在 `store/index.js` 文件中
   - 在 `main.js` 中注入 Store

## 常见问题

### Q: 为什么不能直接修改 State？

A: 直接修改 State 会导致：
- 无法追踪数据变化来源
- Vue DevTools 无法记录变化历史
- 难以定位和调试问题

### Q: Mutation 和 Action 的区别？

A:
- **Mutation**: 必须同步执行，直接修改 State
- **Action**: 可以包含异步操作，通过 Commit 提交 Mutation 来间接修改 State

### Q: 什么时候使用 Action，什么时候使用 Mutation？

A:
- 需要异步操作时使用 Action
- 简单的同步数据修改直接使用 Mutation
