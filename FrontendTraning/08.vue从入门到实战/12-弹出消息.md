# 第12章 弹出消息

## 章节概述

本章学习如何在Vue应用中实现一个全局的弹出消息组件。通过掌握组件的动态渲染、CSS模块化、Vue实例扩展等核心概念，实现一个功能完善的消息提示系统。

- 理解并掌握CSS Module的使用方法

- 掌握Vue实例扩展的技巧

## 核心知识点

### 1. CSS Module

CSS Module是一种CSS模块化解决方案，能够避免样式冲突问题。

#### 使用方式

需要将样式文件命名为`xxx.module.ooo`格式：

- `xxx`为文件名
- `ooo`为样式文件后缀名，可以是`css`、`less`、`scss`等

#### 示例代码

```javascript
// 引入CSS Module
import styles from "./showMessage.module.less";

// 在JavaScript中使用
const typeClassName = styles[`message-${type}`]; // 得到类似 "message-success" 的类名
div.className = `${styles.message} ${typeClassName}`;
```

#### 优势

- **避免命名冲突**：CSS Module会自动生成唯一的类名
- **模块化**：每个组件的样式相互独立
- **更好的维护性**：样式与组件紧密关联

### 2. 获取组件渲染的DOM元素

在某些场景下，我们需要获取某个组件渲染后的DOM根元素，比如将组件渲染为HTML字符串插入到页面中。

#### 实现原理

```javascript
import Vue from "vue";

/**
 * 获取某个组件渲染的Dom根元素
 * @param {Component} comp - Vue组件
 * @param {Object} props - 传递给组件的props
 * @returns {HTMLElement} 组件渲染的DOM根元素
 */
export default function getComponentRootDom(comp, props) {
  // 创建一个临时的Vue实例
  const vm = new Vue({
    render: (h) => h(comp, { props }),
  });
  
  // 挂载但不插入DOM
  vm.$mount();
  
  // 返回渲染的DOM元素
  return vm.$el;
}
```

#### 使用场景

- 动态创建消息提示
- 将组件渲染到指定容器
- 生成静态HTML内容
- 第三方库集成

#### 实际应用

```javascript
import getComponentRootDom from "./getComponentRootDom";
import Icon from "@/components/Icon";

// 创建Icon组件的DOM
const iconDom = getComponentRootDom(Icon, {
  type: "success", // 传递props
});

// 将生成的DOM插入到页面
div.innerHTML = `<span class="${styles.icon}">${iconDom.outerHTML}</span>`;
```

### 3. Vue实例扩展

Vue实例扩展允许我们在Vue原型上添加全局方法或属性，从而在整个应用中访问它们。

![扩展vue实例](http://mdrs.yuanjin.tech/img/20201203172154.jpg)

#### 实现方式

```javascript
// main.js
import Vue from "vue";
import showMessage from "@/utils/showMessage";

// 将showMessage挂载到Vue原型上
Vue.prototype.$showMessage = showMessage;

// 在任何组件中都可以使用
this.$showMessage({
  content: "操作成功",
  type: "success",
});
```

#### 扩展的优势

- **全局访问**：所有组件实例都可以访问
- **简化调用**：不需要每次都import
- **统一管理**：全局工具方法集中管理

### 4. ref的使用

`ref`是Vue提供的用于访问DOM元素或组件实例的特殊属性。

#### 基本用法

```html
<template>
  <div>
    <p ref="para">some paragraph</p>
    <ChildComp ref="comp" />
    <button @click="handleClick">查看所有引用</button>
  </div>
</template>

<script>
import ChildComp from "./ChildComp";

export default {
  components: {
    ChildComp,
  },
  methods: {
    handleClick() {
      // 获取持有的所有引用
      console.log(this.$refs);
      /*
      输出结构：
      {
        para: p元素（原生DOM）,
        comp: ChildComp的组件实例
      }
      */
      
      // 可以直接操作子组件（不推荐）
      this.$refs.comp.a = 3;
      this.$refs.comp.m1();
    },
  },
};
</script>
```

#### 使用注意事项

> **重要提醒**：通过`ref`可以直接操作DOM元素，甚至直接改动子组件，这些都不符合Vue的设计理念。
>
> **除非迫不得已，否则不要使用ref**。应该优先使用props和events进行组件间通信。

#### 适用场景

- 管理焦点、文本选择或媒体播放
- 触发强制动画
- 集成第三方DOM库
- 访问子组件的公共方法

## 完整实现案例

### 消息提示组件完整实现

#### 1. showMessage.js - 核心逻辑

```javascript
import getComponentRootDom from "./getComponentRootDom";
import Icon from "@/components/Icon";
import styles from "./showMessage.module.less";

/**
 * 弹出消息的全局函数
 * @param {String} content 消息内容
 * @param {String} type 消息类型  info  error  success  warn
 * @param {Number} duration 多久后消失（毫秒）
 * @param {HTMLElement} container 容器，消息会显示到该容器的正中；如果不传，则显示到页面正中
 * @param {Function} callback 消息消失后的回调函数
 */
export default function(options = {}) {
  // 解构参数，设置默认值
  const content = options.content || "";
  const type = options.type || "info";
  const duration = options.duration || 2000;
  const container = options.container || document.body;
  
  // 创建消息容器元素
  const div = document.createElement("div");
  
  // 获取图标组件的DOM
  const iconDom = getComponentRootDom(Icon, {
    type, // info, success, warn, error
  });
  
  // 组装消息内容：图标 + 文字
  div.innerHTML = `<span class="${styles.icon}">${iconDom.outerHTML}</span><div>${content}</div>`;
  
  // 设置样式类名
  const typeClassName = styles[`message-${type}`]; // 根据类型获取对应样式
  div.className = `${styles.message} ${typeClassName}`;
  
  // 处理容器的定位
  // 如果容器是static定位，需要改为relative以便绝对定位子元素
  if (getComputedStyle(container).position === "static") {
    container.style.position = "relative";
  }
  
  // 将消息元素加入容器
  container.appendChild(div);
  
  // 强制浏览器重新计算（触发reflow），确保过渡动画能生效
  div.clientHeight;
  
  // 设置最终状态（淡入效果）
  div.style.opacity = 1;
  div.style.transform = `translate(-50%, -50%)`;
  
  // 定时移除消息
  setTimeout(() => {
    // 淡出效果
    div.style.opacity = 0;
    div.style.transform = `translate(-50%, -50%) translateY(-25px)`;
    
    // 监听过渡结束事件
    div.addEventListener(
      "transitionend",
      function() {
        div.remove(); // 移除DOM元素
        options.callback && options.callback(); // 执行回调函数
      },
      { once: true } // 事件只触发一次
    );
  }, duration);
}
```

## 常见问题

### Q1: 为什么需要强制触发reflow？

**A**: 浏览器需要从初始状态过渡到最终状态才会有动画效果。如果直接设置最终样式，浏览器会直接应用最终状态而跳过动画。通过访问`offsetHeight`、`clientHeight`等属性可以强制浏览器重新计算布局，确保过渡动画生效。

### Q2: CSS Module和scoped样式有什么区别？

**A**: 
- **CSS Module**: 在JavaScript层面处理类名，生成唯一类名，可以在JS中动态使用
- **scoped样式**: 通过添加data属性实现样式隔离，只能在模板中使用
- **选择建议**: 需要在JS中动态使用类名时选择CSS Module，否则使用scoped样式

### Q3: 何时应该使用ref？

**A**: ref应该作为最后手段使用。优先考虑：
- 使用props传递数据
- 使用events进行通信
- 使用provide/inject跨级通信
- 使用Vuex管理状态

只有在必须直接操作DOM或调用组件方法时才使用ref。

### Q4: 如何处理快速连续调用导致的消息堆积？

**A**: 可以实现消息队列或限制同时显示的消息数量：

```javascript
let messageCount = 0;
const MAX_MESSAGES = 3;

export default function(options = {}) {
  if (messageCount >= MAX_MESSAGES) {
    return; // 或加入队列等待
  }
  messageCount++;
  
  // ... 现有代码 ...
  
  div.addEventListener("transitionend", function() {
    messageCount--;
    div.remove();
  }, { once: true });
}
```
