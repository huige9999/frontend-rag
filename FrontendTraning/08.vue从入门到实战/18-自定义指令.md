# 第18章 自定义指令

## 本章导读

在前面的章节中，我们已经学习了Vue的内置指令，如`v-if`、`v-for`、`v-model`等。这些指令为我们提供了强大的功能。但在实际开发中，我们可能需要一些特定的功能，这时自定义指令就派上用场了。本章将详细介绍如何创建和使用自定义指令。

## 学习目标

- 理解自定义指令的概念和应用场景
- 掌握全局指令和局部指令的注册方式
- 熟悉指令的生命周期钩子函数
- 学会使用binding对象获取指令信息
- 能够创建实用的自定义指令

## 18.1 指令的定义方式

### 18.1.1 全局定义指令

全局指令在Vue实例化之前定义，定义后可在所有组件中使用。

```javascript
// 在main.js或单独的指令文件中
// 指令名称为：mydirec1
Vue.directive('mydirec1', {
  // 指令配置对象
  bind(el, binding) {
    // 绑定时调用
  },
  inserted(el, binding) {
    // 插入到DOM时调用
  }
})

// 指令名称为：mydirec2
Vue.directive('mydirec2', {
  // 指令配置对象
  update(el, binding) {
    // 更新时调用
  }
})
```

在组件中使用全局定义的指令：

```vue
<template>
  <div>
    <!-- 在任何组件中都可以使用全局指令 -->
    <MyComp v-mydirec1="js表达式" />
    <div v-mydirec2="js表达式">
      内容...
    </div>
    <img v-mydirec1="js表达式" />
  </div>
</template>
```

### 18.1.2 局部定义指令

局部指令在组件内部定义，仅在该组件中有效，类似于局部注册组件。

```vue
<template>
  <div>
    <!-- 仅在当前组件中可以使用 -->
    <MyComp v-mydirec1="js表达式" />
    <div v-mydirec2="js表达式">
      内容...
    </div>
    <img v-mydirec1="js表达式" />
  </div>
</template>

<script>
export default {
  // 在组件中定义局部指令
  directives: {
    // 指令名称：mydirec1
    mydirec1: {
      // 指令配置对象
      bind(el, binding) {
        console.log('mydirec1 bind')
      }
    },
    // 指令名称：mydirec2
    mydirec2: {
      // 指令配置对象
      inserted(el, binding) {
        console.log('mydirec2 inserted')
      }
    }
  }
}
</script>
```

### 18.1.3 提取指令配置

为了提高代码的可维护性和复用性，通常会将指令配置提取到单独的模块中。

```vue
<template>
  <div>
    <div v-loading="isLoading">内容区域</div>
    <button @click="isLoading = !isLoading">切换状态</button>
  </div>
</template>

<script>
// 导入指令配置对象
import loading from "@/directives/loading";

export default {
  data() {
    return {
      isLoading: true
    }
  },
  directives: {
    // 直接使用导入的配置对象
    loading
  }
}
</script>
```

## 18.2 指令配置对象

### 18.2.1 钩子函数

自定义指令的核心是配置对象中的钩子函数，Vue会在适当的时机调用这些钩子函数。

**常用钩子函数：**

```javascript
{
  bind(el, binding) {
    // 只调用一次，指令第一次绑定到元素时调用
    // 可以在这里进行一次性的初始化设置
  },
  inserted(el, binding) {
    // 被绑定元素插入父节点时调用
    // 此时元素已经在DOM中
  },
  update(el, binding) {
    // 所在组件的 VNode 更新时调用
    // 但是可能发生在其子 VNode 更新之前
  },
  componentUpdated(el, binding) {
    // 指令所在组件的 VNode 及其子 VNode 全部更新后调用
  },
  unbind(el, binding) {
    // 只调用一次，指令与元素解绑时调用
    // 可以在这里进行清理工作
  }
}
```

### 18.2.2 钩子函数参数

每个钩子函数都会被传递以下参数：

```javascript
{
  bind(el, binding, vnode, oldVnode) {
    // el: 指令所绑定的元素，可以用来直接操作DOM
    // binding: 一个对象，包含指令的详细信息
    // vnode: Vue编译生成的虚拟节点
    // oldVnode: 上一个虚拟节点（仅在update和componentUpdated钩子中可用）
  }
}
```

**el 参数：**
- `el` 是被绑定元素对应的真实DOM元素
- 可以直接使用`el`进行DOM操作

**binding 对象：**

binding对象包含以下属性：

```javascript
{
  name: '指令名称',           // 指令名称，不包括 v- 前缀
  value: '指令的绑定值',      // 指令的绑定值，例如：v-my-directive="1 + 1" 中，绑定值为 2
  oldValue: '上一个绑定值',    // 指令绑定的前一个值，仅在 update 和 componentUpdated 钩子中可用
  expression: "字符串形式的指令表达式", // 指令的表达式，例如：v-my-directive="1 + 1" 中，表达式为 "1 + 1"
  arg: '传给指令的参数',       // 传给指令的参数，例如：v-my-directive:foo 中，参数为 "foo"
  modifiers: '修饰符对象'      // 包含修饰符的对象，例如：v-my-directive.foo.bar 中，修饰符对象为 { foo: true, bar: true }
}
```

## 18.3 指令配置简化

在实际开发中，我们经常需要在`bind`和`update`时执行相同的操作。Vue允许我们简化这种情况：

```javascript
// 原始写法：两个钩子函数功能相同
{
  bind(el, binding) {
    // 绑定时的逻辑
    updateElement(el, binding.value)
  },
  update(el, binding) {
    // 更新时的逻辑（与bind相同）
    updateElement(el, binding.value)
  }
}

// 简化写法：直接提供一个函数
function(el, binding) {
  // 该函数会被同时设置到bind和update钩子中
  updateElement(el, binding.value)
}
```

## 18.4 实战案例：Loading指令

### 18.4.1 Loading组件

首先，我们创建一个Loading组件，用于显示加载状态。

```vue
<!-- Loading/index.vue -->
<template>
  <img class="loading-container" :src="loadingUrl" alt="加载中..." />
</template>

<script>
// 导入loading图标资源
import loadingUrl from "@/assets/loading.svg";

export default {
  name: "Loading",
  data() {
    return {
      loadingUrl,  // loading图片的URL
    };
  },
};
</script>

<style scoped lang="less">
@import "~@/styles/mixin.less";
.loading-container {
  .self-center();  // 使用混入实现居中定位
}
</style>
```

### 18.4.2 Loading指令实现

接下来，我们实现一个自定义指令，根据绑定的值来控制loading效果。

```javascript
// directives/loading.js
import loadingUrl from "@/assets/loading.svg";
import styles from "./loading.module.less";

// 辅助函数：从元素中获取loading图片元素
function getLoadingImage(el) {
  return el.querySelector("img[data-role=loading]");
}

// 辅助函数：创建loading图片元素
function createLoadingImg() {
  const img = document.createElement("img");
  img.dataset.role = "loading";  // 设置标识，便于后续查找
  img.src = loadingUrl;
  img.className = styles.loading;  // 设置样式类名
  return img;
}

// 导出指令配置函数（简化写法）
export default function(el, binding) {
  // 根据 binding.value 的值，决定创建或删除img元素
  
  // 获取当前元素中是否已存在loading图片
  const curImg = getLoadingImage(el);
  
  if (binding.value) {
    // 如果值为true，显示loading
    if (!curImg) {
      // 如果loading图片不存在，则创建并添加
      const img = createLoadingImg();
      el.appendChild(img);
    }
  } else {
    // 如果值为false，隐藏loading
    if (curImg) {
      // 如果loading图片存在，则移除
      curImg.remove();
    }
  }
}
```

```css
/* directives/loading.module.less */
@import "~@/styles/mixin.less";

.loading {
  .self-center();  // 使用混入实现绝对居中
}
```

### 18.4.3 使用Loading指令

在组件中注册并使用loading指令：

```vue
<template>
  <div style="position:relative">
    <!-- 使用v-loading指令，根据isLoading状态控制loading显示 -->
    <Loading v-if="isLoading" />
    <button @click="isLoading = !isLoading">切换显示/隐藏</button>
  </div>
</template>

<script>
import Loading from "./index.vue";

export default {
  components: {
    Loading,
  },
  data() {
    return {
      isLoading: true,  // 控制loading状态
    };
  },
};
</script>
```

### 18.4.4 指令的优势

使用自定义指令相比传统组件方式有以下优势：

1. **使用更简洁**：直接在元素上使用指令，无需额外的组件包裹
2. **性能更优**：直接操作DOM，减少组件层级
3. **代码复用**：可在多个元素上复用同一指令
4. **逻辑集中**：指令逻辑集中管理，便于维护

## 18.5 指令的高级应用

### 18.5.1 带参数的指令

```javascript
// 定义带参数的指令
Vue.directive('color', {
  bind(el, binding) {
    // binding.arg 获取指令参数
    const colorType = binding.arg || 'text';
    const colorValue = binding.value;
    
    switch(colorType) {
      case 'text':
        el.style.color = colorValue;
        break;
      case 'background':
        el.style.backgroundColor = colorValue;
        break;
      case 'border':
        el.style.borderColor = colorValue;
        break;
    }
  }
})
```

```vue
<template>
  <div>
    <!-- 不同参数控制不同样式属性 -->
    <p v-color:text="'red'">红色文字</p>
    <div v-color:background="'blue'">蓝色背景</div>
    <input v-color:border="'green'" />
  </div>
</template>
```

### 18.5.2 带修饰符的指令

```javascript
// 定义带修饰符的指令
Vue.directive('text', {
  bind(el, binding) {
    const text = binding.value || '';
    let result = text;
    
    // 处理修饰符
    if (binding.modifiers.uppercase) {
      result = text.toUpperCase();
    }
    if (binding.modifiers.lowercase) {
      result = text.toLowerCase();
    }
    if (binding.modifiers.reverse) {
      result = text.split('').reverse().join('');
    }
    
    el.textContent = result;
  }
})
```

```vue
<template>
  <div>
    <p v-text:uppercase="'hello world'">HELLO WORLD</p>
    <p v-text:lowercase="'HELLO WORLD'">hello world</p>
    <p v-text:reverse="'hello'">olleh</p>
    <p v-text:reverse.uppercase="'hello'">OLLEH</p>
  </div>
</template>
```

### 18.5.3 指令的值类型

指令的值可以是各种类型：

```vue
<template>
  <div>
    <!-- 字符串 -->
    <div v-my-directive="'string value'">
    
    <!-- 数字 -->
    <div v-my-directive="123">
    
    <!-- 对象 -->
    <div v-my-directive="{ color: 'red', size: 16 }">
    
    <!-- 数组 -->
    <div v-my-directive="[1, 2, 3]">
    
    <!-- 变量 -->
    <div v-my-directive="myVariable">
  </div>
</template>
```

## 18.6 指令的最佳实践

### 18.6.1 命名规范

- 指令名称使用小写字母和连字符：`v-my-directive`
- JavaScript中使用驼峰命名：`myDirective`

### 18.6.2 性能优化

- 避免在指令中进行复杂的计算
- 使用`binding.value`的变化来优化性能
- 在`unbind`钩子中进行清理工作

```javascript
export default {
  bind(el, binding) {
    // 初始化操作
    el.addEventListener('click', binding.value);
  },
  unbind(el, binding) {
    // 清理操作，避免内存泄漏
    el.removeEventListener('click', binding.value);
  }
}
```

### 18.6.3 错误处理

```javascript
export default function(el, binding) {
  try {
    // 主要逻辑
    if (typeof binding.value !== 'function') {
      throw new Error('v-log指令需要绑定一个函数');
    }
    // 正常处理...
  } catch (error) {
    console.error('v-log指令错误:', error.message);
  }
}
```

## 18.7 常用指令示例

### 18.7.1 权限控制指令

```javascript
// directives/permission.js
export default {
  inserted(el, binding) {
    const { value } = binding;
    const permissions = store.getters.permissions;
    
    if (value && !permissions.includes(value)) {
      // 如果没有权限，移除元素
      el.parentNode && el.parentNode.removeChild(el);
    }
  }
}
```

```vue
<template>
  <button v-permission="'admin'">管理员功能</button>
  <button v-permission="'user'">普通用户功能</button>
</template>
```

### 18.7.2 防抖指令

```javascript
// directives/debounce.js
export default {
  inserted(el, binding) {
    let timer;
    el.addEventListener('click', () => {
      if (timer) clearTimeout(timer);
      timer = setTimeout(() => {
        binding.value();
      }, binding.arg || 300);
    });
  }
}
```

```vue
<template>
  <button v-debounce:500="handleClick">点击我（防抖500ms）</button>
</template>
```

### 18.7.3 复制到剪贴板指令

```javascript
// directives/copy.js
export default {
  bind(el, binding) {
    el.copyData = binding.value;
    el.addEventListener('click', handleClick);
  },
  update(el, binding) {
    el.copyData = binding.value;
  },
  unbind(el) {
    el.removeEventListener('click', handleClick);
  }
}

function handleClick() {
  const input = document.createElement('input');
  input.value = this.copyData;
  document.body.appendChild(input);
  input.select();
  document.execCommand('Copy');
  document.body.removeChild(input);
  
  // 可以添加提示信息
  alert('复制成功！');
}
```
