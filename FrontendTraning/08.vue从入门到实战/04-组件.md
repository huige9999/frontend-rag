# 04. 组件

## 学习目标

- 理解组件的概念和作用
- 掌握组件的创建和注册方式
- 学会使用组件传递数据
- 理解组件树的概念
- 掌握单向数据流原则

## 组件的概念

组件的出现是为了实现以下两个目标：

1. **降低整体复杂度**，提升代码的可读性和可维护性
2. **提升局部代码的可复用性**

绝大部分情况下，一个组件就是页面中某个区域，组件包含该区域的：

- **功能**（JS代码）
- **内容**（模板代码）
- **样式**（CSS代码）

> 要在组件中包含样式，需要构建工具的支撑

## 组件开发

### 创建组件

组件是根据一个普通的配置对象创建的，所以要开发一个组件，只需要写一个配置对象即可。

该配置对象和Vue实例的配置是**几乎一样**的：

```js
// 组件配置对象
var myComp = {
  data() {
    return {
      // 组件数据
    }
  },
  template: `....` // 组件模板
}
```

值得注意的是，组件配置对象和Vue实例有以下几点差异：

- 无 `el` 配置
- `data` 必须是一个函数，该函数返回的对象作为数据
- 由于没有 `el` 配置，组件的虚拟DOM树必须定义在 `template` 或 `render` 中

#### 示例：创建一个按钮组件

```js
// MyButton.js
const template = `<button @click="count++">当前点击了{{count}}次</button>`;

export default {
  // data必须是函数，返回数据对象
  data() {
    return {
      count: 0, // 点击计数器
    };
  },
  template, // 组件模板
};
```

### 注册组件

注册组件分为两种方式：**全局注册**和**局部注册**

#### 全局注册

一旦全局注册了一个组件，整个应用中任何地方都可以使用该组件。

```js
// 参数1：组件名称，将来在模板中使用组件时，会使用该名称
// 参数2：组件配置对象
// 该代码运行后，即可在模板中使用组件
Vue.component('my-comp', myComp)
```

在模板中可以使用组件：

```html
<my-comp />
<!-- 或 -->
<my-comp></my-comp>
```

> 但在一些工程化的大型项目中，很多组件都不需要全局使用。
> 比如一个登录组件，只有在登录的相关页面中使用，如果全局注册，将导致构建工具无法优化打包
> **因此，除非组件特别通用，否则不建议使用全局注册**

#### 局部注册

局部注册就是哪里要用到组件，就在哪里注册。

```js
// 在要使用组件的组件或实例中加入配置
var otherComp = {
  components: {
    // 属性名为组件名称，模板中将使用该名称
    // 属性值为组件配置对象
    "my-comp": myComp
  },
  template: `
    <div>
      <my-comp></my-comp>
    </div>
  `
}
```

#### 工程化中的局部注册

在实际工程中，通常使用ES6模块的方式：

```js
// App.js - 根组件
import MyButton from "./components/MyButton.js";

export default {
  components: {
    MyButton, // ES6速写属性，等同于 MyButton: MyButton
  },
  template: `
    <div>
      <h1>App组件</h1>
      <MyButton />
    </div>
  `,
};
```

### 使用组件

在模板中使用组件特别简单，把组件名当作HTML元素名使用即可。

要注意以下几点：

#### 1. 组件必须有结束

组件可以自结束，也可以用结束标记结束，但必须要有结束。

```html
<!-- 正确：自结束 -->
<my-comp />

<!-- 正确：使用结束标记 -->
<my-comp></my-comp>

<!-- 错误：没有结束 -->
<my-comp>
```

#### 2. 组件的命名

无论使用哪种方式注册组件，组件的命名需要遵循规范。

组件可以使用：
- `kebab-case` 短横线命名法：`my-comp`
- `PascalCase` 大驼峰命名法：`MyComp`

```js
var otherComp = {
  components: {
    "my-comp": myComp,  // 方式1：短横线
    MyComp: myComp      // 方式2：大驼峰
  }
}
```

> 实际上，使用 `小驼峰命名法 camelCase` 也是可以识别的，只不过不符合官方要求的规范

使用 `PascalCase` 方式命名还有一个额外的好处，即可以在模板中使用两种组件名：

```js
var otherComp = {
  components: {
    MyComp: myComp
  }
}
```

模板中：
```html
<!-- 两种方式都可以 -->
<my-comp />
<MyComp />
```

因此，在使用组件时，为了方便，往往使用以下代码：

```js
var MyComp = {
  // 组件配置
}

var OtherComp = {
  components: {
    MyComp // ES6速写属性
  }
}
```

## 组件树

一个组件创建好后，往往会在各种地方使用它。它可能多次出现在Vue实例中，也可能出现在其他组件中。于是就形成了一个组件树。

```
App (根组件)
├── Header (头部组件)
│   ├── Logo (Logo组件)
│   └── Navigation (导航组件)
├── Content (内容组件)
│   ├── ArticleList (文章列表组件)
│   │   └── ArticleItem (文章项组件)
│   └── Sidebar (侧边栏组件)
└── Footer (页脚组件)
```

## 向组件传递数据

大部分组件要完成自身的功能，都需要一些额外的信息。

比如一个头像组件，需要告诉它头像的地址，这就需要在使用组件时向组件传递数据。

传递数据的方式有很多种，最常见的一种是使用**组件属性 component props**。

### 声明组件属性

首先在组件中声明可以接收哪些属性：

```js
var MyComp = {
  props: ["p1", "p2", "p3"], // 声明可接收的属性
  template: `
    <div>
      {{p1}}, {{p2}}, {{p3}}
    </div>
  `
}
```

### 传递属性值

在使用组件时，向其传递属性：

```js
var OtherComp = {
  components: {
    MyComp
  },
  data() {
    return {
      a: 1
    }
  },
  template: `
    <!-- 静态值直接赋值，动态值使用v-bind（简写:） -->
    <my-comp :p1="a" :p2="2" p3="3"/>
  `
}
```

### 单向数据流

**重要：在组件中，属性是只读的，绝不可以更改，这叫做单向数据流**

```js
var MyComp = {
  props: ["title"],
  template: `<div>{{title}}</div>`,
  created() {
    // 错误！不要修改props
    // this.title = "新标题"; 
  }
}
```

单向数据流的好处：

1. **防止子组件意外修改父组件状态**
2. **使数据流向更加清晰可预测**
3. **提升应用的可维护性**

如果需要修改数据，应该：
- 通过事件向父组件传递修改请求
- 在父组件中修改数据，通过props传递下来

## 工程结构示例

一个典型的Vue组件工程结构：

```
04. 组件/
├── 课件/
│   └── 笔记.md
└── 课堂代码/
    ├── index.html          # HTML入口
    ├── src/
    │   ├── main.js         # 应用入口
    │   ├── App.js          # 根组件
    │   └── components/
    │       └── MyButton.js # 子组件
    └── vue.browser.js      # Vue框架文件
```

### 入口文件 (index.html)

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Vue组件示例</title>
  </head>
  <body>
    <div id="app"></div>
    <script src="./src/main.js" type="module"></script>
  </body>
</html>
```

### 应用入口 (main.js)

```js
// 入口模块文件
import Vue from "./vue.browser.js";
import App from "./App.js";

// 创建Vue实例，渲染App组件
new Vue({
  render: (h) => h(App), // 渲染函数，将App组件转换为虚拟DOM
}).$mount("#app"); // 挂载到#app元素
```

### 根组件 (App.js)

```js
import MyButton from "./components/MyButton.js";

// Vue的根组件
const template = `
  <div>
    <h1>App组件</h1>
    <MyButton />
  </div>
`;

export default {
  components: {
    MyButton, // 局部注册子组件
  },
  template,
};
```

### 子组件 (components/MyButton.js)

```js
// 按钮组件模板
const template = `<button @click="count++">当前点击了{{count}}次</button>`;

export default {
  // 组件数据必须是函数
  data() {
    return {
      count: 0, // 点击次数
    };
  },
  template,
};
```

## 组件的最佳实践

### 1. 组件职责单一

每个组件应该只负责一个功能模块，保持组件的简洁和可复用性。

### 2. 合理划分组件

- 通用组件：如按钮、输入框等，可在项目中复用
- 业务组件：如用户信息卡片、商品列表等，与业务相关
- 页面组件：对应路由页面的顶层组件

### 3. 组件命名规范

- 组件文件名使用PascalCase：`MyButton.js`
- 组件注册时使用PascalCase：`components: { MyButton }`
- 模板中可使用kebab-case：`<my-button />`

### 4. Props验证

为了更好的代码提示和错误检查，可以使用对象形式的props定义：

```js
export default {
  props: {
    title: String,           // 类型限制
    likes: Number,           // 数字类型
    isPublished: Boolean,    // 布尔类型
    commentIds: Array,       // 数组类型
    author: Object,          // 对象类型
    callback: Function,      // 函数类型
    
    // 带默认值
    count: {
      type: Number,
      default: 0
    },
    
    // 必填项
    requiredProp: {
      type: String,
      required: true
    }
  }
}
```

## 总结

本章节我们学习了：

1. **组件的概念**：组件是可复用的Vue实例，包含功能、内容和样式
2. **组件的创建**：通过配置对象创建，data必须是函数
3. **组件的注册**：全局注册和局部注册，推荐使用局部注册
4. **组件的使用**：在模板中像HTML元素一样使用，必须有结束标签
5. **组件树**：组件的嵌套使用形成树形结构
6. **数据传递**：通过props传递数据，遵循单向数据流原则
7. **工程结构**：模块化的文件组织方式

组件化开发是现代前端开发的核心思想，掌握好组件的使用对于开发复杂应用至关重要。
