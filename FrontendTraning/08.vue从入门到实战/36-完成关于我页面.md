# 36. 完成「关于我」页面

## 章节概述

本章节将完成「关于我」页面的开发。这是一个相对简单的页面，主要功能是展示外部网页内容（如个人GitHub主页、简历页面等）。通过本章节的学习，你将掌握：

- 使用 iframe 嵌入外部网页
- Vuex 的状态管理
- 异步数据加载与加载状态控制
- 组件生命周期管理

## 核心知识点

### 1. 使用 iframe 嵌入外部内容

`<iframe>` 元素用于在当前HTML文档中嵌入另一个HTML文档。在「关于我」页面中，我们使用 iframe 来展示外部网页内容。

**iframe 常用属性：**
- `src`: 要嵌入的网页URL
- `frameborder`: 是否显示边框（0为无边框）
- `width`: 宽度
- `height`: 高度
- `@load`: 加载完成事件

### 2. Vuex 模块化管理

本章节使用 Vuex 来管理「关于我」页面的数据，包括：
- `data`: 存储iframe的src地址
- `loading`: 加载状态

### 3. 加载状态控制

通过双重加载状态控制，确保用户在数据加载和iframe渲染完成前看到加载动画：
- `loading`: API数据请求状态
- `srcLoaded`: iframe内容加载状态

## 代码实现

### 1. API 接口封装

```javascript
// src/api/about.js
import request from "./request";

/**
 * 获取关于我页面的iframe地址
 * @returns {Promise<string>} iframe的src地址
 */
export async function getAbout() {
  return await request.get("/api/about");
}
```

**说明：**
- 封装获取关于我页面数据的API接口
- 返回iframe需要加载的外部URL地址

### 2. Vuex Store 模块

```javascript
// src/store/about.js
import { getAbout } from "@/api/about";

export default {
  namespaced: true, // 启用命名空间
  state: {
    loading: false, // 加载状态
    data: "",       // iframe的src地址
  },
  mutations: {
    setLoading(state, payload) {
      state.loading = payload;
    },
    setData(state, payload) {
      state.data = payload;
    },
  },
  actions: {
    async fetchAbout(ctx) {
      // 如果已有数据，避免重复请求
      if (ctx.state.data) {
        return;
      }
      // 开始加载
      ctx.commit("setLoading", true);
      // 请求API获取数据
      const resp = await getAbout();
      // 保存数据
      ctx.commit("setData", resp);
      // 结束加载
      ctx.commit("setLoading", false);
    },
  },
};
```

**说明：**
- 使用 `namespaced: true` 启用命名空间，避免模块命名冲突
- `state` 存储加载状态和iframe地址数据
- `mutations` 用于同步修改状态
- `actions` 处理异步操作，包含数据缓存逻辑避免重复请求

### 3. About 页面组件

```vue
<!-- src/views/About/index.vue -->
<template>
  <div class="about-container" v-loading="loading || !srcLoaded">
    <!-- iframe嵌入外部网页 -->
    <iframe
      v-if="src"
      class="about-content"
      :src="src"
      frameborder="0"
      @load="srcLoaded = true"
    ></iframe>
  </div>
</template>

<script>
import { mapState } from "vuex";

export default {
  data() {
    return {
      // iframe内容是否加载完成
      srcLoaded: false,
    };
  },
  computed: mapState("about", {
    // 映射about模块的state到计算属性
    src: "data",     // iframe的src地址
    loading: "loading", // API加载状态
  }),
  created() {
    // 组件创建时获取数据
    this.$store.dispatch("about/fetchAbout");
  },
};
</script>

<style scoped lang="less">
.about-container {
  width: 100%;
  height: 100%;
  overflow: hidden;
  position: relative;
}

.about-content {
  width: 100%;
  height: 100%;
  display: block;
  box-sizing: border-box;
}
</style>
```

**关键点说明：**

1. **加载状态控制**
   - `v-loading="loading || !srcLoaded"`: 当API正在请求或iframe未加载完成时显示加载动画
   - 双重状态确保用户体验流畅

2. **iframe 嵌入**
   - `v-if="src"`: 只在有数据时渲染iframe
   - `@load="srcLoaded = true"`: iframe内容加载完成后标记为已完成
   - `frameborder="0"`: 去除边框

3. **Vuex 状态映射**
   - 使用 `mapState` 将 Vuex 的状态映射为组件的计算属性
   - 通过 `this.$store.dispatch` 触发 action 获取数据

4. **样式处理**
   - 设置容器和 iframe 的宽高为100%，使其充满整个页面
   - `overflow: hidden` 防止出现滚动条

## 实现流程

1. **组件创建** → About 组件实例化
2. **触发请求** → created 钩子中 dispatch 'about/fetchAbout' action
3. **API请求** → 调用 getAbout() 接口获取数据
4. **状态更新** → 将获取的URL保存到 Vuex store
5. **数据渲染** → 计算属性 src 更新，iframe 开始加载外部内容
6. **加载完成** → iframe 的 @load 事件触发，srcLoaded 设为 true
7. **显示内容** → 加载动画消失，iframe 内容展示

## 技术要点

### 1. 数据缓存策略

```javascript
if (ctx.state.data) {
  return; // 已有数据，避免重复请求
}
```

在 Vuex action 中检查是否已有数据，避免每次进入页面都重新请求，提升用户体验。

### 2. 双重加载状态

```javascript
v-loading="loading || !srcLoaded"
```

- `loading`: 控制API请求期间的加载状态
- `!srcLoaded`: 控制iframe内容加载期间的加载状态
- 两者任一为 true 都显示加载动画

### 3. iframe 加载事件

```javascript
@load="srcLoaded = true"
```

使用 iframe 的原生 load 事件来检测内容是否加载完成，确保用户看到完整内容。

### 4. Vuex 命名空间

```javascript
namespaced: true
```

启用命名空间后，在组件中访问模块时需要使用完整的路径：
- `this.$store.dispatch('about/fetchAbout')`
- `mapState('about', { ... })`

## 常见问题

### 1. iframe 显示空白？

可能原因：
- src 地址为空或无效
- 外部网站设置了 X-Frame-Options 禁止被嵌入

解决方案：
- 检查 API 返回的 URL 是否正确
- 确认目标网站允许被 iframe 嵌入

### 2. 加载动画一直显示？

可能原因：
- iframe 的 @load 事件未正确触发
- srcLoaded 状态未正确更新

解决方案：
- 确保 iframe 有 src 属性
- 检查 @load 事件是否正确绑定

### 3. 页面切换后数据丢失？

本实现中数据保存在 Vuex 中，切换页面不会丢失数据。如需每次进入页面都重新获取数据，可在 action 中移除数据缓存逻辑。

## 扩展思考

1. **如何处理 iframe 跨域问题？**
   - 某些网站设置了 X-Frame-Options 响应头禁止被嵌入
   - 可以考虑使用后端代理或提示用户在新窗口打开

2. **如何优化加载体验？**
   - 添加加载进度条
   - 提供加载超时提示
   - 添加骨架屏

3. **如何实现自适应高度？**
   - 由于跨域限制，iframe 自适应高度较难实现
   - 可考虑使用 postMessage 通信（需要目标网站配合）
