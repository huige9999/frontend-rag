# 王者荣耀官网英雄查询

## 章节概述

本章节通过实现王者荣耀官网英雄查询功能，训练程序思维和综合编程能力。该功能包括英雄列表展示、分类筛选和关键词搜索等功能。

- 掌握异步数据获取和处理

## 知识点详解

### 1. 异步数据获取

使用 `fetch` API 从服务器获取英雄数据：

```javascript
async function getHeroes() {
  return fetch('https://study.duyiedu.com/api/herolist')
    .then((resp) => resp.json())
    .then((resp) => resp.data.reverse());
}
```

**关键点：**
- `async/await` 处理异步操作
- `fetch` 发送网络请求
- `.reverse()` 反转数组，让新英雄排在前面

### 2. 数据驱动的DOM操作

根据数据动态生成HTML内容：

```javascript
function setHeroHTML(heroes) {
  doms.ul.innerHTML = heroes
    .map((h) => `
      <li>
        <a href="https://pvp.qq.com/web201605/herodetail/${h.ename}.shtml" target="_blank">
          <img src="https://game.gtimg.cn/images/yxzj/img201606/heroimg/${h.ename}/${h.ename}.jpg" alt="" />
          <span>${h.cname}</span>
        </a>
      </li>
    `)
    .join('');
}
```

**关键点：**
- 使用 `map` 方法将数据数组转换为HTML字符串
- 使用模板字符串生成HTML结构
- 一次性设置 `innerHTML` 提高性能

### 3. 数据筛选逻辑

根据不同条件筛选英雄数据：

```javascript
function searchHeros(radio) {
  let heroes;
  const type = radio.dataset.type;
  const value = radio.dataset.value;
  
  if (type === 'all') {
    heroes = allHeroes;
  } else if (type === 'pay_type') {
    heroes = allHeroes.filter((h) => h.pay_type === +value);
  } else {
    heroes = allHeroes.filter(
      (h) => h.hero_type === +value || h.hero_type2 === +value
    );
  }
  setHeroHTML(heroes);
}
```

**关键点：**
- 使用 `filter` 方法筛选数组
- `dataset` 访问自定义属性
- `+value` 字符串转数字
- 支持英雄类型双条件判断

## 英雄对象属性说明

### pay_type 属性

- 该值为 `10` 时，表示本周免费
- 该值为 `11` 时，表示新手推荐

### hero_type 和 hero_type2 属性

**说明：** 有些英雄同时具有两种英雄类型

**示例：** `{ hero_type: 6, hero_type2: 2}` 表示该英雄既是一个辅助，也是一个法师

**英雄类型对照表：**
- `1` - 战士
- `2` - 法师
- `3` - 坦克
- `4` - 刺客
- `5` - 射手
- `6` - 辅助

### JavaScript逻辑

```javascript
(async function () {
  /**
   * 从网络获取当前的英雄数据
   * @returns {Promise<Array>} 英雄数据数组
   */
  async function getHeroes() {
    return fetch('https://study.duyiedu.com/api/herolist')
      .then((resp) => resp.json())
      .then((resp) => resp.data.reverse());
  }

  // 缓存DOM元素引用
  const doms = {
    ul: document.querySelector('.list'),
    radios: document.querySelectorAll('.radio'),
    txtKeyword: document.querySelector('.input input'),
  };

  // 1. 初始化：加载所有的英雄数据，生成li，加入到ul中
  const allHeroes = await getHeroes();
  setHeroHTML(allHeroes);

  /**
   * 根据指定的英雄数组，生成对应的html，放入到ul中
   * @param {Array} heroes - 英雄数据数组
   */
  function setHeroHTML(heroes) {
    doms.ul.innerHTML = heroes
      .map((h) => `
        <li>
          <a href="https://pvp.qq.com/web201605/herodetail/${h.ename}.shtml" target="_blank">
            <img src="https://game.gtimg.cn/images/yxzj/img201606/heroimg/${h.ename}/${h.ename}.jpg" alt="" />
            <span>${h.cname}</span>
          </a>
        </li>
      `)
      .join('');
  }

  // 2. 交互事件绑定
  // 单选按钮点击事件
  for (const radio of doms.radios) {
    radio.addEventListener('click', function () {
      // 更新radio的选中样式
      setChoose(this);
      // 根据选中的条件筛选英雄
      searchHeros(this);
    });
  }

  // 关键词输入事件
  doms.txtKeyword.addEventListener('input', function () {
    // 根据关键词筛选英雄
    const heroes = allHeroes.filter((h) => h.cname.includes(this.value));
    setHeroHTML(heroes);
    // 重置选中状态为"全部"
    setChoose(document.querySelector(".radio[data-type='all']"));
  });

  /**
   * 根据radio中提供的自定义属性，查询英雄数据，然后设置html
   * @param {HTMLElement} radio - 被点击的单选按钮元素
   */
  function searchHeros(radio) {
    let heroes;
    // 获取自定义属性值
    const type = radio.dataset.type;
    const value = radio.dataset.value;
    
    if (type === 'all') {
      // 显示全部英雄
      heroes = allHeroes;
    } else if (type === 'pay_type') {
      // 按付费类型筛选
      heroes = allHeroes.filter((h) => h.pay_type === +value);
    } else {
      // 按英雄类型筛选（支持双类型英雄）
      heroes = allHeroes.filter(
        (h) => h.hero_type === +value || h.hero_type2 === +value
      );
    }
    setHeroHTML(heroes);
  }

  /**
   * 设置某个被选中的radio样式
   * @param {HTMLElement} radio - 要设置为选中状态的radio元素
   */
  function setChoose(radio) {
    // 移除之前的选中状态
    const checkedRadio = document.querySelector('.radio.checked');
    checkedRadio && checkedRadio.classList.remove('checked');
    // 添加当前选中状态
    radio.classList.add('checked');
  }
})();
```

## 编程思维要点

### 1. 数据驱动思维
- 将数据和视图分离
- 通过改变数据来驱动视图更新
- 避免直接操作DOM，优先操作数据

### 2. 模块化设计
- 将功能分解为独立的函数
- 每个函数只负责一个功能
- 提高代码可维护性和复用性

### 3. 事件驱动编程
- 理解事件冒泡和事件监听
- 合理使用不同事件类型
- 注意事件处理的性能优化

### 4. 缓存和优化
- 缓存DOM元素查询结果
- 缓存原始数据避免重复请求
- 使用批量更新提高性能

## 练习任务

1. **基础练习**：添加一个新的筛选条件，比如按首字母筛选英雄
2. **进阶练习**：实现分页功能，每页显示50个英雄
3. **高级练习**：添加英雄详情查看功能，点击英雄卡片显示详细信息
4. **挑战练习**：实现多条件组合筛选，支持同时筛选多个条件
