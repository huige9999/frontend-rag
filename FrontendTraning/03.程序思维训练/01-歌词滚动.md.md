## 知识点讲解

### 1. 歌词数据格式解析

歌词通常采用LRC格式，每行歌词包含时间标签和歌词内容：

```
[01:26.09]这沙滚滚水皱皱笑着浪荡
[01:28.09]贪痛快一个人漫游
[01:30.09]像俗尘一缕缈
```

需要将这种格式转换为便于程序处理的数据结构：

```javascript
// 转换后的数据结构
[
  { time: 86.09, words: '这沙滚滚水皱皱笑着浪荡' },
  { time: 88.09, words: '贪痛快一个人漫游' },
  { time: 90.09, words: '像俗尘一缕缈' }
]
```

### 2. 异步数据获取

使用fetch API从网络获取歌词数据：

```javascript
async function getLrc() {
  return await fetch('https://study.duyiedu.com/api/lyrics')
    .then(resp => resp.json())
    .then(resp => resp.data);
}
```

### 3. 数据解析逻辑

```javascript
// 1. 按行分割歌词字符串
// 2. 过滤空行
// 3. 提取时间和歌词内容
lrcData = lrc
  .split('\n')           // 按换行符分割
  .filter(s => s)        // 过滤空行
  .map(s => {
    const parts = s.split(']');                    // 分离时间和歌词
    const timeParts = parts[0].replace('[', '').split(':'); // 提取分秒
    return {
      time: +timeParts[0] * 60 + +timeParts[1],    // 转换为秒数
      words: parts[1]                              // 歌词内容
    };
  });
```

### 4. DOM结构生成

```javascript
// 生成歌词列表DOM
doms.ul.innerHTML = lrcData
  .map(lrc => `<li>${lrc.words}</li>`)
  .join('');
```

### 5. 事件监听与实时处理

监听audio元素的timeupdate事件，该事件在播放进度变化时触发：

```javascript
doms.audio.addEventListener('timeupdate', function() {
  setStatus(this.currentTime); // 传入当前播放时间
});
```

### 6. 歌词高亮与滚动逻辑

```javascript
function setStatus(time) {
  // 时间微调，让歌词显示更自然
  time += 0.5;

  // 1. 移除之前的高亮状态
  const activeLi = document.querySelector('.active');
  activeLi && activeLi.classList.remove('active');

  // 2. 根据当前时间找到对应的歌词行
  const index = lrcData.findIndex(lrc => lrc.time > time) - 1;
  if (index < 0) return;

  // 3. 高亮当前歌词
  doms.ul.children[index].classList.add('active');

  // 4. 计算并设置滚动位置
  // 让当前歌词行位于容器中心
  let top = size.liHeight * index + 
            size.liHeight / 2 - 
            size.containerHeight / 2;
  top = -top; // 向上滚动
  if (top > 0) top = 0; // 防止向下滚动

  doms.ul.style.transform = `translateY(${top}px)`;
}
```

### JavaScript逻辑

```javascript
(async function() {
  /**
   * 从网络获取歌词数据
   * @returns {Promise<string>} 返回LRC格式的歌词字符串
   */
  async function getLrc() {
    return await fetch('https://study.duyiedu.com/api/lyrics')
      .then(resp => resp.json())
      .then(resp => resp.data);
  }

  // DOM元素引用
  const doms = {
    ul: document.querySelector('.lrc'),
    audio: document.querySelector('audio'),
  };

  // 尺寸配置
  const size = {
    liHeight: 30,           // 单行歌词高度
    containerHeight: 420,   // 容器高度
  };

  let lrcData; // 存储解析后的歌词数据

  /**
   * 初始化函数
   * 获取并解析歌词，生成DOM结构
   */
  async function init() {
    // 1. 获取歌词数据
    const lrc = await getLrc();

    // 2. 解析LRC格式歌词
    // 转换为：[{ time: 86.09, words: '这沙滚滚水皱皱笑着浪荡' }, ...]
    lrcData = lrc
      .split('\n')              // 按行分割
      .filter(s => s)           // 过滤空行
      .map(s => {
        // 解析每行歌词：[01:26.09]这沙滚滚水皱皱笑着浪荡
        const parts = s.split(']');              // 分离时间和歌词
        const timeParts = parts[0]               // [01:26.09
          .replace('[', '')                       // 01:26.09
          .split(':');                            // ['01', '26.09']

        // 转换时间为秒数：1分26.09秒 = 86.09秒
        return {
          time: +timeParts[0] * 60 + +timeParts[1],
          words: parts[1],                        // 歌词内容
        };
      });

    // 3. 生成歌词列表DOM
    doms.ul.innerHTML = lrcData
      .map(lrc => `<li>${lrc.words}</li>`)
      .join('');
  }

  // 执行初始化
  await init();

  /**
   * 交互事件处理
   * 监听音频播放进度变化事件
   */
  doms.audio.addEventListener('timeupdate', function() {
    setStatus(this.currentTime); // 传入当前播放时间
  });

  /**
   * 根据播放时间设置歌词状态
   * @param {number} time - 当前播放时间（秒）
   */
  function setStatus(time) {
    // 时间微调：提前0.5秒显示歌词，体验更自然
    time += 0.5;

    // 步骤1：移除之前的高亮状态
    const activeLi = document.querySelector('.active');
    activeLi && activeLi.classList.remove('active');

    // 步骤2：根据时间找到对应的歌词行
    // 找到第一个时间大于当前时间的歌词，取其前一行
    const index = lrcData.findIndex(lrc => lrc.time > time) - 1;

    // 如果没有找到对应的歌词（比如刚开始播放），直接返回
    if (index < 0) return;

    // 步骤3：高亮当前歌词
    doms.ul.children[index].classList.add('active');

    // 步骤4：计算并设置滚动位置
    // 目标：让当前歌词行位于容器垂直中心位置
    let top = 
      size.liHeight * index +      // 当前行之前的所有行高度
      size.liHeight / 2 -           // 当前行半高
      size.containerHeight / 2;     // 容器半高

    top = -top; // 转换为向上滚动的负值

    // 防止向下滚动（top大于0时）
    if (top > 0) top = 0;

    // 应用滚动变换
    doms.ul.style.transform = `translateY(${top}px)`;
  }
})();
```

### 关键算法思想

#### 1. 查找算法

找到当前应该显示的歌词行：

```javascript
// 使用findIndex找到第一个时间大于当前时间的歌词
const index = lrcData.findIndex(lrc => lrc.time > time) - 1;
```

#### 2. 居中计算

让当前歌词位于容器中心：

```javascript
// 当前行顶部位置 = 当前行索引 * 行高
// 当前行中心位置 = 顶部位置 + 行高/2
// 目标滚动位置 = 当前行中心位置 - 容器中心位置
let top = size.liHeight * index + size.liHeight / 2 - size.containerHeight / 2;
```

## 常见问题

### Q1: 为什么需要时间微调（+0.5秒）？

**答:** 由于网络延迟、解析耗时等因素，歌词显示可能会滞后。通过提前0.5秒显示歌词，可以让用户体验更加流畅自然。

### Q2: 如何处理歌词格式不规范的情况？

**答:** 可以添加数据验证和容错处理：

```javascript
.map(s => {
  try {
    const parts = s.split(']');
    if (parts.length < 2) return null; // 跳过格式错误的行
    // ... 解析逻辑
  } catch (e) {
    return null;
  }
})
.filter(item => item !== null); // 过滤无效数据
```

### Q3: 如何优化大量歌词的性能？

**答:** 可以使用虚拟滚动技术，只渲染可见区域的歌词：

```javascript
// 只渲染可见区域的歌词（当前行前后各10行）
const visibleStart = Math.max(0, index - 10);
const visibleEnd = Math.min(lrcData.length, index + 10);
const visibleLyrics = lrcData.slice(visibleStart, visibleEnd);
```
