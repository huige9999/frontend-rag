## 核心内容

### 1. 提取公共组件

分析添加文章和编辑文章页面的代码，发现两者有很多相似之处。我们可以提取一个公共的 `EditArticle` 组件。

创建 `src/components/EditArticle.vue`：

```vue
<template>
  <div>
    <!-- 文章标题 -->
    <div class="block">文章标题</div>
    <div style="margin-bottom: 15px">
      <el-input v-model="form.title" placeholder="请输入文章标题"></el-input>
    </div>

    <!-- 文章编辑 -->
    <div class="block">文章编辑</div>
    <editor
      ref="toastuiEditor"
      :initialValue="form.editorText"
      height="600px"
      :options="editorOptions"
    />

    <!-- 文章描述 -->
    <div class="block">文章描述</div>
    <el-input type="textarea" v-model="form.description" :rows="6"></el-input>

    <!-- 文章头图 -->
    <Upload uploadTitle="文章头图" v-model="form.thumb" />

    <!-- 选择分类 -->
    <div class="block">选择分类</div>
    <el-select
      v-model="form.select"
      slot="prepend"
      placeholder="请选择文章分类"
    >
      <el-option
        v-for="item in blogType"
        :key="item.id"
        :label="item.name"
        :value="item.id"
      ></el-option>
    </el-select>

    <div>
      <!-- 发布按钮 -->
      <el-button
        type="primary"
        style="margin-top: 15px"
        @click="addArticleHandle"
      >{{btnContent}}</el-button>
    </div>
  </div>
</template>

<script>
import "@toast-ui/editor/dist/toastui-editor.css";
import { Editor } from "@toast-ui/vue-editor";
import Upload from "@/components/Upload.vue";
import { getBlogType } from "@/api/blogType.js";
import { addBlog, editBlog, findOneBlog } from "@/api/blog.js";
import "@toast-ui/editor/dist/i18n/zh-cn";

export default {
  props: ["mode"], // 接收模式参数：add 或 edit
  data() {
    return {
      form: {
        title: "", // 文章标题
        editorText: "", // 用户编辑的内容
        description: "", // 文章的描述
        thumb: "", // 文章的图片
        select: "", // 选择分类
      },
      blogType: [], // 存放博客分类
      imageUrl: "", // 图片在服务器上面的地址
      editorOptions: {
        language: "zh-CN", // 设置编辑器语言为中文
      },
      btnContent: '发布文章' // 按钮文本
    };
  },
  created() {
    // 1. 获取分类数据
    getBlogType().then(({ data }) => {
      this.blogType = data;
    });
    
    // 2. 根据模式执行不同逻辑
    if (this.mode === "edit") {
      // 编辑模式：获取文章详情并回填
      this.id = this.$route.params.id;
      findOneBlog(this.id).then(({ data }) => {
        this.form = data;
        this.form.select = data.category === null ? "" : data.category.id;
        this.$refs.toastuiEditor.invoke("setHTML", data.htmlContent);
      });
      this.btnContent = "确认修改";
    }
  },
  components: {
    editor: Editor,
    Upload,
  },
  methods: {
    addArticleHandle() {
      // 1. 获取富文本编辑器的 HTML 和 Markdown 内容
      let html = this.$refs.toastuiEditor.invoke("getHTML");
      let markdown = this.$refs.toastuiEditor.invoke("getMarkdown");

      // 2. 组装要传递给服务器的对象
      let obj = {
        title: this.form.title,
        description: this.form.description,
        createDate: new Date().getTime(),
        categoryId: this.form.select,
        toc: [],
        htmlContent: html,
        thumb: this.form.thumb,
        markdownContent: markdown,
      };

      // 3. 验证并提交数据
      if (obj.title && obj.description && obj.htmlContent && obj.categoryId) {
        if (this.mode === "add") {
          // 新增模式
          addBlog(obj).then(() => {
            this.$router.push("/blogList");
            this.$message.success("文章发布成功");
          });
        } else {
          // 编辑模式
          editBlog({ id: this.form.id, data: obj }).then(() => {
            this.$router.push('/blogList');
            this.$message.success('文章修改成功');
          })
        }
      } else {
        this.$message.error("请填写所有内容");
      }
    },
  },
};
</script>

<style lang="scss" scoped>
.block {
  margin: 15px 0;
  font-weight: 100;
}
</style>
```

## 技术要点解析

### 1. 组件复用的优势

**代码重复的问题**：

- 添加文章和编辑文章页面有 90% 的代码重复
- 维护成本高，修改一处需要同步修改
- 代码冗余，影响性能

**组件复用的优势**：

- 减少代码重复，提高可维护性
- 统一管理逻辑，降低维护成本
- 提高代码复用性和灵活性

### 2. Props 数据传递

**父组件传递数据**：

```vue
<!-- 父组件 -->
<template>
  <EditArticle mode="add"/>
</template>
```

**子组件接收数据**：

```javascript
// 子组件
export default {
  props: ["mode"], // 接收 mode 参数
  // 或者使用对象形式进行验证
  props: {
    mode: {
      type: String,     // 类型验证
      required: true,   // 必填验证
      validator: (value) => {
        return ['add', 'edit'].includes(value); // 自定义验证
      }
    }
  }
}
```

### 3. 条件渲染

**根据模式执行不同逻辑**：

```javascript
if (this.mode === "edit") {

}
```

### 4. 动态按钮文本

## 组件设计模式

### 1. 单一职责原则

每个组件只负责一个功能：

```vue
<!-- EditArticle.vue 只负责文章编辑表单 -->
<EditArticle mode="add"/>

<!-- Upload.vue 只负责图片上传 -->
<Upload v-model="form.thumb"/>
```

### 2. 配置化设计

通过 props 控制组件行为：

```vue
<EditArticle mode="add"/>    <!-- 添加模式 -->
<EditArticle mode="edit"/>   <!-- 编辑模式 -->
```

### 3. 数据流设计

```
父组件 ──props──> 子组件 ──$emit──> 父组件
    ↓                        ↑
   路由参数              表单数据
```
