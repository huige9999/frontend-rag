# 08. 鉴权功能制作

## 课程概述

本节课程实现博客管理系统的鉴权功能，包括路由守卫、权限验证、Token 验证、用户状态管理等，确保只有授权用户才能访问特定页面。

## 鉴权流程概述

### 1. vue-element-admin 原始鉴权流程

```
路由跳转 → 检查 Token → 有 Token？
                        ↓ 是          ↓ 否
                   检查用户信息      在白名单？
                        ↓             ↓ 是
                   有用户信息？      直接放行
                   ↓ 是      ↓ 否
                  放行    获取用户信息
                           ↓ 成功    ↓ 失败
                          放行     重定向登录
```

### 2. 改造后的鉴权流程

```
路由跳转 → 检查页面是否需要鉴权 → 不需要鉴权？
                                ↓ 是          ↓ 否
                            检查是否登录      有用户信息？
                            ↓ 是      ↓ 否     ↓ 是
                            重定向首页  放行   放行
                                        ↓ 否
                                    有 Token？
                                    ↓ 是      ↓ 否
                                验证 Token    重定向登录
                                ↓ 成功   ↓ 失败
                                放行    重定向登录
```

## 权限控制文件

### 1. permission.js 完整实现

`src/permission.js`：

```javascript
import router from './router'
import store from './store'
import { Message } from 'element-ui'
import NProgress from 'nprogress' // 进度条
import 'nprogress/nprogress.css' // 进度条样式
import getPageTitle from '@/utils/get-page-title'

NProgress.configure({ showSpinner: false }) // NProgress 配置

// 白名单：不需要鉴权的页面
const whiteList = ['/login']

// 路由前置守卫
router.beforeEach(async(to, from, next) => {
  // 开始进度条
  NProgress.start()

  // 设置页面标题
  document.title = getPageTitle(to.meta.title)

  // 获取本地用户信息
  const hasGetUserInfo = store.getters.user

  // 检查页面是否需要鉴权
  if (to.meta.auth) {
    // 页面需要鉴权
    if (hasGetUserInfo) {
      // 有用户信息，直接放行
      next()
    } else {
      // 没有用户信息，检查是否有 Token
      const hasToken = localStorage.getItem('adminToken')

      if (hasToken) {
        // 有 Token，验证 Token 有效性
        try {
          // 调用接口获取用户信息（相当于 whoami）
          await store.dispatch('user/getInfo')
          next()
        } catch (error) {
          // Token 无效
          await store.dispatch('user/resetToken')
          Message.error('登录过期，请重新登录')
          next(`/login?redirect=${to.path}`)
          NProgress.done()
        }
      } else {
        // 没有 Token，重定向到登录页
        next(`/login?redirect=${to.path}`)
        NProgress.done()
      }
    }
  } else {
    // 页面不需要鉴权
    if (to.path === '/login' && hasGetUserInfo) {
      // 已登录用户访问登录页，重定向到首页
      next({ path: '/' })
    } else {
      next()
    }
    NProgress.done()
  }
})

// 路由后置钩子
router.afterEach(() => {
  // 结束进度条
  NProgress.done()
})
```

## Vuex 状态管理

### 1. User Store 完善

`src/store/modules/user.js`：

```javascript
import { loginApi, logout, getInfo } from '@/api/user'
import { resetRouter } from '@/router'

const getDefaultState = () => {
  return {
    user: null  // 存储用户信息
  }
}

const state = getDefaultState()

const mutations = {
  SET_USER: (state, payload) => {
    state.user = payload
  }
}

const actions = {
  // 用户登录
  login({ commit }, userInfo) {
    return new Promise((resolve, reject) => {
      loginApi(userInfo).then(res => {
        const { data } = res
        if (data) {
          commit('SET_USER', data)
          resolve()
        } else {
          reject(res)
        }
      }).catch(error => {
        reject(error)
      })
    })
  },

  // 获取用户信息（验证 Token）
  getInfo({ commit, state }) {
    return new Promise((resolve, reject) => {
      getInfo(state.token).then(response => {
        const { data } = response
        if (!data) {
          return reject('验证失败，请重新登录')
        }
        commit('SET_USER', data)
        resolve(data)
      }).catch(error => {
        reject(error)
      })
    })
  },

  // 用户登出
  logout({ commit, state }) {
    return new Promise((resolve, reject) => {
      logout().then(() => {
        localStorage.removeItem('adminToken')
        commit('SET_USER', null)
        resetRouter()
        resolve()
      }).catch(error => {
        reject(error)
      })
    })
  },

  // 重置 Token
  resetToken({ commit }) {
    return new Promise(resolve => {
      localStorage.removeItem('adminToken')
      commit('SET_USER', null)
      resolve()
    })
  }
}

export default {
  namespaced: true,
  state,
  mutations,
  actions
}
```

### 2. 组件级权限

```vue
<template>
  <div>
    <!-- 只有管理员可见 -->
    <el-button v-if="isAdmin">删除</el-button>
    
    <!-- 有编辑权限可见 -->
    <el-button v-if="canEdit">编辑</el-button>
  </div>

<script>
export default {
  computed: {
    isAdmin() {
      return this.$store.getters.user?.role === 'admin'
    },
    canEdit() {
      return this.$store.getters.user?.permissions?.includes('edit')
    }
  }
}
</script>
```

## 常见问题

### 1. 无限循环

**问题**：路由守卫出现无限循环

**原因**：在守卫中没有正确调用 `next()`

**解决**：确保每种情况都调用了 `next()`

### 2. 进度条不消失

**问题**：页面加载完成后进度条仍然显示

**原因**：`NProgress.done()` 没有被调用

**解决**：在后置钩子中确保调用 `NProgress.done()`

### 3. Token 验证失败

**问题**：Token 存在但验证失败

**原因**：Token 过期或无效

**解决**：清除本地 Token 并重定向登录

### 4. 重定向丢失

**问题**：登录后没有跳转到原目标页面

**原因**：redirect 参数没有正确传递

**解决**：在登录页保存 redirect 参数并在登录成功后使用

## 安全建议

### 1. Token 管理

- Token 存储在 localStorage 中
- 设置合理的过期时间
- 实现 Token 自动刷新
- 退出登录时清除 Token

### 2. 权限验证

- 前端验证主要用于用户体验
- 后端必须进行权限验证
- 敏感操作需要二次验证
- 记录权限验证日志

### 3. 路由安全

- 不在 URL 中暴露敏感信息
- 404 页面放在路由配置最后
- 合理配置路由白名单
- 避免路由嵌套过深
