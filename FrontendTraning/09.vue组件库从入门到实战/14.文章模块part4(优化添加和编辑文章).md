# 第14章 文章模块 part4（优化添加和编辑文章）

## 章节概述

本章主要讲解如何通过组件复用来优化添加和编辑文章功能，提取公共组件以减少代码重复，提高代码可维护性。

## 核心内容

### 1. 组件复用的重要性

#### 1.1 代码重复的问题

在第13章中，我们发现添加文章（`addBlog`）和编辑文章（`editBlog`）两个页面的代码高度相似：
- 相同的表单结构
- 相同的输入组件
- 相同的验证逻辑
- 只有提交逻辑略有不同

这种代码重复会导致：
- 维护困难：修改一处需要同时修改多处
- 代码臃肿：大量重复代码占用空间
- 容易出错：可能漏修改某些重复的地方

#### 1.2 解决方案：组件提取

通过提取公共组件 `EditArticle.vue`，实现添加和编辑功能的复用。

### 2. 公共组件 EditArticle.vue

#### 2.1 组件设计思路

核心思想：通过 `mode` 属性区分当前是添加还是编辑模式。

- **mode="add"**：添加文章模式
- **mode="edit"**：编辑文章模式

#### 2.2 完整代码实现

```vue
<template>
  <div>
    <!-- 文章标题 -->
    <div class="block">文章标题</div>
    <div style="margin-bottom: 15px">
      <el-input v-model="form.title" placeholder="请输入文章标题"></el-input>
    </div>

    <!-- 文章编辑 -->
    <div class="block">文章编辑</div>
    <editor
      ref="toastuiEditor"
      :initialValue="form.editorText"
      height="600px"
      :options="editorOptions"
    />

    <!-- 文章描述 -->
    <div class="block">文章描述</div>
    <el-input type="textarea" v-model="form.description" :rows="6"></el-input>

    <!-- 文章头图 -->
    <Upload uploadTitle="文章头图" v-model="form.thumb" />

    <!-- 选择分类 -->
    <div class="block">选择分类</div>
    <el-select
      v-model="form.select"
      slot="prepend"
      placeholder="请选择文章分类"
    >
      <el-option
        v-for="item in blogType"
        :key="item.id"
        :label="item.name"
        :value="item.id"
      ></el-option>
    </el-select>

    <div>
      <!-- 动态按钮文本 -->
      <el-button
        type="primary"
        style="margin-top: 15px"
        @click="addArticleHandle"
        >{{btnContent}}</el-button
      >
    </div>
  </div>
</template>

<script>
import "@toast-ui/editor/dist/toastui-editor.css";
import { Editor } from "@toast-ui/vue-editor";
import Upload from "@/components/Upload.vue";
import { getBlogType } from "@/api/blogType.js";
import { addBlog, editBlog, findOneBlog } from "@/api/blog.js";
import "@toast-ui/editor/dist/i18n/zh-cn";

export default {
  props: ["mode"],  // 接收模式属性
  data() {
    return {
      form: {
        title: "",           // 文章标题
        editorText: "",      // 用户编辑的内容
        description: "",     // 文章的描述
        thumb: "",           // 文章的图片
        select: "",          // 选择分类
      },
      blogType: [],          // 存放博客分类
      imageUrl: "",          // 图片在服务器上面的地址
      editorOptions: {
        language: "zh-CN",   // 编辑器语言设置为中文
      },
      btnContent: '发布文章' // 按钮文本
    };
  },
  created() {
    // 获取分类数据
    getBlogType().then(({ data }) => {
      this.blogType = data;
    });

    // 根据模式判断是否需要加载文章内容
    if (this.mode === "edit") {
      // 获取文章内容并回填
      this.id = this.$route.params.id;
      findOneBlog(this.id).then(({ data }) => {
        this.form = data;
        this.form.select = data.category === null ? "" : data.category.id;
        this.$refs.toastuiEditor.invoke("setHTML", data.htmlContent);
      });
      this.btnContent = "确认修改";
    }
  },
  components: {
    editor: Editor,
    Upload,
  },
  methods: {
    addArticleHandle() {
      // 获取编辑器内容
      let html = this.$refs.toastuiEditor.invoke("getHTML");
      let markdown = this.$refs.toastuiEditor.invoke("getMarkdown");

      // 组装数据对象
      let obj = {
        title: this.form.title,
        description: this.form.description,
        createDate: new Date().getTime(),
        categoryId: this.form.select,
        toc: [],
        htmlContent: html,
        thumb: this.form.thumb,
        markdownContent: markdown,
      };

      // 表单验证和提交
      if (obj.title && obj.description && obj.htmlContent && obj.categoryId) {
        if (this.mode === "add") {
          // 添加文章
          addBlog(obj).then(() => {
            this.$router.push("/blogList");
            this.$message.success('文章发布成功');
          });
        } else {
          // 编辑文章
          editBlog({id: this.form.id, data: obj}).then(() => {
            this.$router.push('/blogList');
            this.$message.success('文章修改成功');
          });
        }
      } else {
        this.$message.error("请填写所有内容");
      }
    },
  },
};
</script>

<style lang="scss" scoped>
.block {
  margin: 15px 0;
  font-weight: 100;
}
</style>
```

### 5. 组件优化的关键点

#### 5.1 Props 传参

通过 props 接收模式参数：

```javascript
props: ["mode"]
```

#### 5.2 条件渲染

根据 mode 属性决定不同的行为：

```javascript
if (this.mode === "edit") {
  // 编辑模式的逻辑
} else {
  // 添加模式的逻辑
}
```

#### 5.3 动态按钮文本

根据模式显示不同的按钮文本：

```javascript
data() {
  return {
    btnContent: '发布文章'  // 默认文本
  }
}

created() {
  if (this.mode === "edit") {
    this.btnContent = "确认修改";
  }
}
```

#### 5.4 中文语言支持

引入中文语言包：

```javascript
import "@toast-ui/editor/dist/i18n/zh-cn";

// 在编辑器配置中设置
editorOptions: {
  language: "zh-CN",
}
```

## 重点知识点

### 1. 组件复用的优势

- **减少代码重复**：相同的逻辑只需要编写一次
- **提高可维护性**：修改时只需要修改一个组件
- **降低出错概率**：避免多处修改遗漏
- **代码更清晰**：逻辑更加集中和清晰

### 2. Props 的使用

Props 是父组件向子组件传递数据的方式：

```javascript
// 父组件
<EditArticle mode="add"/>

// 子组件
props: ["mode"]
```

### 3. 条件逻辑处理

通过条件判断实现不同模式的逻辑：

```javascript
if (this.mode === "add") {
  // 添加逻辑
} else {
  // 编辑逻辑
}
```

### 4. 编辑器语言配置

Toast UI Editor 支持多语言，通过配置实现中文界面：

```javascript
import "@toast-ui/editor/dist/i18n/zh-cn";

editorOptions: {
  language: "zh-CN",
}
```

## 优化对比

### 优化前

- addBlog 页面：约 140 行代码
- editBlog 页面：约 140 行代码
- 总计：约 280 行代码

### 优化后

- EditArticle 组件：约 150 行代码
- addBlog 页面：约 15 行代码
- editBlog 页面：约 15 行代码
- 总计：约 180 行代码

**代码减少约 35%，维护成本大幅降低**

## 技术要点

1. **组件提取**：识别重复代码并提取为公共组件
2. **Props 传参**：通过属性传递配置信息
3. **条件逻辑**：根据不同模式执行不同逻辑
4. **代码复用**：最大化代码复用减少重复
5. **国际化支持**：配置编辑器显示中文界面

## 注意事项

1. 组件提取时要考虑所有使用场景
2. Props 验证可以提高组件健壮性
3. 条件逻辑要考虑所有可能的情况
4. 保持组件的单一职责原则
5. 适当的组件拆分可以提高代码可读性

## 最佳实践

1. **识别重复代码**：当发现代码重复超过两次时，考虑提取组件
2. **Props 验证**：使用对象形式的 props 定义进行类型验证
3. **组件命名**：使用清晰、描述性的组件名称
4. **文档注释**：为组件添加使用说明和注释
5. **逐步重构**：先确保功能正常，再进行重构优化
