## MongoDB 数据库配置

### 1. MongoDB 下载

访问 MongoDB 官网：https://www.mongodb.com/

官网会自动根据你的操作系统推荐相应的版本，直接点击 Download 进行下载。

### 2. MongoDB 安装

#### Mac 安装

下载的是 .tgz 格式的压缩包，解压后将目录放入 `/usr/local`：

```bash
# 解压后移动到指定目录
sudo mv mongodb-macos-x86_64-<version> /usr/local/mongodb
```

#### 需要创建的文件和目录

- **data/** - 存储数据的地方，所有数据库的数据都在这个文件夹
- **mongod.log** - 日志文件
- **mongodb.conf** - 数据库配置文件

### 3. MongoDB 配置文件

创建 `/usr/local/mongodb/mongodb.conf`，内容如下：

```yaml
systemLog:
  destination: file  # 日志输出方式：file/syslog
  path: /usr/local/mongodb/mongod.log  # 日志路径
  logAppend: true  # 日志追加模式

net:
  port: 27017  # 监听端口
  bindIp: 127.0.0.1  # 绑定 IP，只监听本机
  maxIncomingConnections: 65536  # 最大连接数
  wireObjectCheck: true  # 校验客户端请求
 
processManagement:
  fork: true  # 后台运行

security:
  authorization: disabled  # 开启客户端认证

storage:
  dbPath: /usr/local/mongodb/data  # 数据存储路径
```

### 4. 启动 MongoDB

```bash
# 进入 MongoDB 的 bin 目录
cd /usr/local/mongodb/bin

# 使用配置文件启动
mongod -f /usr/local/mongodb/mongodb.conf
```

### 5. 连接 MongoDB

```bash
# 在 bin 目录下执行
mongo
```

## 数据恢复

### 服务器地址

https://gitee.com/dev-edu/mysite-server

### 数据恢复命令

```bash
mongorestore -h dbhost -d dbname --dir dbdirectory
```

**参数说明：**
- `-h` - MongoDB 服务器地址
- `-d` - 要恢复的数据库名称
- `--dir` - 备份数据所在位置

**示例：**

```bash
# 在 MongoDB bin 目录下执行
mongorestore -h 127.0.0.1:27017 -d mysite --dir ./dump/mysite
```

### 数据库说明

默认会生成一个名为 `mysite` 的数据库，包含以下集合：

- **users** - 用户表
- **blogs** - 文章表
- **projects** - 项目表

MongoDB 的特性：
- 连接不存在的数据库会自动创建
- 向不存在的集合添加数据会自动创建集合

## 关于数据库中的图片

### 图片存储机制

- 数据库中存储的是图片在服务器上的路径
- 真实的图片文件存储在服务器的 `upload` 目录

### 注意事项

提供的服务器代码中 `upload` 目录下的图片已被删除，因此：
- 数据库中有图片路径
- 但实际图片文件不存在
- 这是正常的，完成上传功能后上传自己的图片即可

## 接口文档

### Apifox 接口地址

https://app.apifox.com/project/2429912

### 主要接口

#### 1. 登录接口

- **URL**：`/api/admin/login`
- **方法**：POST
- **参数**：
  ```json
  {
    "loginId": "admin",
    "loginPwd": "123456",
    "captcha": "验证码"
  }
  ```

#### 2. 验证码接口

- **URL**：`/api/admin/captcha`
- **方法**：GET
- **返回**：SVG 格式的验证码图片

#### 3. 用户信息接口

- **URL**：`/api/admin/whoami`
- **方法**：GET
- **Headers**：
  ```
  authorization: Bearer <token>
  ```

## 前后端对接

### 1. 配置请求基础路径

修改 `src/utils/request.js`：

```javascript
const service = axios.create({
  baseURL: 'http://localhost:3000', // 后端服务器地址
  timeout: 5000
})
```

### 2. 创建接口文件

`src/api/user.js`：

```javascript
import request from '@/utils/request'

// 登录
export function loginApi(data) {
  return request({
    url: '/api/admin/login',
    method: 'post',
    data
  })
}

// 获取用户信息
export function getInfo(token) {
  return request({
    url: '/api/admin/whoami',
    method: 'get',
    headers: {
      authorization: `Bearer ${token}`
    }
  })
}
```

### 3. 处理跨域问题

如果遇到跨域问题，可以在后端添加 CORS 配置：

```javascript
// 后端代码
app.use(cors({
  origin: 'http://localhost:8080',
  credentials: true
}))
```

## 常见问题

### 1. MongoDB 连接失败

- 检查 MongoDB 是否启动
- 检查端口是否被占用
- 检查配置文件路径

### 2. 接口请求失败

- 检查服务器是否启动
- 检查端口是否正确
- 检查是否有跨域问题

### 3. 验证码显示异常

- 检查验证码接口是否返回正确
- 检查 SVG 渲染是否正确
