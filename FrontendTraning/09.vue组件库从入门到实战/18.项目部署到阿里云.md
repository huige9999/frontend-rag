# 第18章 项目部署到阿里云

## 章节概述

本章主要讲解如何将开发完成的前后端项目部署到阿里云服务器，包括服务器购买、环境配置、数据库安装、项目打包、Nginx 配置、域名绑定等完整流程。

## 核心内容

### 1. 项目部署概述

#### 1.1 部署的重要性

项目开发完成后，需要部署到服务器才能供用户访问。部署是将开发环境转移到生产环境的关键步骤。

#### 1.2 常见部署方式

**1. 后端部署（传统方式）**
- 前端将打包好的项目交给后端
- 后端将静态资源放到后端的静态资源目录
- 优点：简单直接
- 缺点：前后端耦合，扩展性差

**2. 前后端分离部署（现代方式）**
- 前端部署到前端服务器（如 Nginx）
- 后端部署到后端服务器
- 用户从前端服务器获取静态资源
- 从后端服务器获取数据
- 优点：解耦合，易于扩展

### 2. 部署流程概览

完整的部署流程包括：
1. 购买云服务器
2. 安装运行环境（Node.js）

### 6. 安装 MongoDB

#### 6.1 安装依赖库

```bash
yum -y install pcre*
yum -y install openssl*
```

#### 6.2 安装 MongoDB（传统方法）

**下载：**
```bash
curl -O https://fastdl.mongodb.org/linux/mongodb-linux-x86_64-4.2.1.tgz
```

**解压：**
```bash
tar zxvf mongodb-linux-x86_64-4.2.1.tgz
```

**移动：**
```bash
mv mongodb-linux-x86_64-4.2.1/ /usr/local/mongodb
```

**创建必要目录和文件：**
```bash
mkdir -p /usr/local/mongodb/data/db
touch /usr/local/mongodb/mongod.log
touch /usr/local/mongodb/mongodb.conf
```

#### 6.3 使用 yum 安装（推荐）

**添加 yum 源：**
```bash
vi /etc/yum.repos.d/mongodb-org-5.0.repo
```

**添加以下内容：**
```ini
[mongodb-org-5.0]
name=MongoDB Repository
baseurl=https://repo.mongodb.org/yum/redhat/7/mongodb-org/5.0/x86_64/
gpgcheck=1
enabled=1
gpgkey=https://www.mongodb.org/static/pgp/server-5.0.asc
```

**安装：**
```bash
yum install -y mongodb-org
```

#### 6.4 MongoDB 配置文件

```yaml
systemLog:
  destination: file
  path: /usr/local/mongodb/mongod.log
  logAppend: true

net:
  port: 27017
  bindIp: 0.0.0.0  # 允许远程连接

processManagement:
  fork: true  # 后台运行

security:
  authorization: enabled  # 开启认证

storage:
  dbPath: /usr/local/mongodb/data/db
  journal: 
    enabled: true
```

#### 6.5 启动 MongoDB

**传统方式：**
```bash
./mongod --config /usr/local/mongodb/mongodb.conf
```

**使用 systemctl：**
```bash
systemctl start mongod    # 启动
systemctl status mongod   # 查看状态
systemctl restart mongod  # 重启
```

### 7. 数据库用户管理

#### 7.1 创建管理员用户

```bash
use admin  # 切换到 admin 数据库
db.createUser({
  user:"root",
  pwd:"123456",
  roles:[{
    role:"userAdminAnyDatabase",
    db: "admin"
  }]
})
```

#### 7.2 创建数据库用户

```bash
# 先用管理员登录
use admin
db.auth("root","123456")

# 切换到目标数据库
use mysite

# 创建用户
db.createUser({
  user:"testadmin",
  pwd:"testadmin123",
  roles:[{
    role:"read",
    db: "mysite"
  },{
    role:"readWrite",
    db: "mysite"
  }]
})

# 验证登录
db.auth("testadmin","testadmin123")
```

#### 7.3 连接字符串格式

```
mongodb://username:password@ip:port/database?authSource=admin

示例：
mongodb://testadmin:testadmin123@88.888.88.888:27017/mysite?authSource=admin
```

### 11. Nginx 配置

#### 11.1 配置文件位置

```
/usr/local/nginx/conf/nginx.conf
```

#### 11.2 配置内容

```nginx
# 前台路由配置
location / {
    root   html;
    index  index.html index.htm;
    try_files $uri $uri/ /index.html;  # Vue Router history 模式
}

# API 代理
location /api {
  proxy_pass http://127.0.0.1:7001;
}

# 静态资源代理
location /res {
  proxy_pass http://127.0.0.1:7001;
}

location /static {
  proxy_pass http://127.0.0.1:7001;
}
```


## 重点知识点

### 1. 服务器选择

- **CentOS**：稳定但已停止维护
- **Alibaba Cloud Linux**：阿里云自研，兼容 CentOS，推荐使用

### 2. 部署架构

```
用户浏览器
    ↓
Nginx (静态资源)
    ↓
API 请求 → Node.js 后端服务
    ↓
MongoDB 数据库
```

### 3. 关键端口

- 80：HTTP
- 443：HTTPS
- 27017：MongoDB
- 7001：后端 API（示例）

### 4. 安全配置

1. **数据库加密**：设置用户名密码
2. **防火墙**：只开放必要端口
3. **安全组**：限制访问来源
4. **HTTPS**：使用 SSL 证书

### 5. 常见问题

**问题 1：Git 连接失败**
```
fatal: unable to access 'https://github.com/...': Encountered end of file
```
解决方案：开放 443 端口

**问题 2：Segmentation fault**
参考：https://blog.csdn.net/weixin_44320761/article/details/107671990

**问题 3：libssl.so.10 缺失**
参考：https://blog.csdn.net/xcyja/article/details/115357818

### 6. 性能优化

1. **静态资源压缩**：启用 gzip
2. **缓存配置**：设置合理的缓存策略
3. **CDN 加速**：使用 CDN 分发静态资源
4. **负载均衡**：使用多台服务器
