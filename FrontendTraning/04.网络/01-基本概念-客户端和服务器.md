> 对于前端而言，网络部分代码不多，以概念为主。
>

## 1. 客户端和服务器

在网络的世界里，两个应用程序之间会经常发生通信。

在大部分情况下，通信总是由一方发出一个消息开始，而另一方回复一个消息结束。

**发出消息的一方**称之为**客户端 Client**，发出消息的过程称之为**请求 Request**。

**回复消息的一方**称之为**服务器 Server**，回复消息的过程称之为**响应 Response**

![image-20220415151039230](http://mdrs.yuanjin.tech/img/20220415151044.png)

### 重要知识点

**1. 客户端和服务器都是应用程序**

不管是客户端，还是服务器，它们都是一个**应用程序**，而不是一台计算机。客户端和服务器可以分布在不同的计算机上，也可以在同一台计算机上，并不需要特殊看待。

**示例**：
- live server 插件就是一个服务器，它运行在本地的计算机上
- 大部分后端开发的就是服务器程序
- 前端的 Node 技术也能开发服务器程序

**2. C/S 结构和B/S结构**

客户端和服务器的这种交互模式称之为「经典 C/S 结构」（Client/Server）。
在这种结构中，如果客户端是浏览器，则我们称之为 B/S 结构（Browser/Server）。

**3. Web服务器**

服务器程序往往是为互联网产品提供服务，因此又称之为 web 服务器。

**4. 完整的交互流程**

一次完整的交互，总是从请求开始，响应结束。

## 2. URL地址

要完成一次请求和响应，首先需要让客户端找到服务器，不仅如此，还要找到服务器上我们想要的资源。

### 2.1 什么是URL

**URL**（Uniform Resource Locator，统一资源定位符）是一个字符串，用于表达互联网中某个资源的位置。

**URL地址示例**：
- 百度首页：`https://www.baidu.com/`
- 新闻页面：`https://mbd.baidu.com/newspage/data/landingsuper?context=%7B%22nid%22%3A%22news_9610068257663826418%22%7D&n_type=-1&p_from=-1`
- CSS文件：`https://meyerweb.com/eric/tools/css/reset/reset.css`
- JS文件：`https://cdn.bootcdn.net/ajax/libs/jquery/3.6.0/jquery.min.js`
- 图片文件：`https://gimg2.baidu.com/image_search/src=http%3A%2F%2Fup.enterdesk.com%2Fedpic_source%2F84%2F87%2F80%2F848780a296b66b382018fa7f675ecd06.jpg`

**URL地址可以找到的资源类型**：
- HTML页面
- 图片、视频、音频
- CSS、JavaScript代码
- 可供下载的文件
- 其他任何资源

### 2.2 URL的组成

一个完整的URL地址由多个部分组成：

```
协议 + 主机 + 端口 + 路径 + 参数 + hash
```

**示例说明**：

![image-20220415160543391](http://mdrs.yuanjin.tech/img/20220415160543.png)

### 2.3 各部分详解

#### 协议（Protocol/Schema）

**作用**：表示客户端希望用什么方式和服务器沟通

**常见值**：
- `http`：超文本传输协议
- `https`：安全的超文本传输协议

**小知识**：
1. 如果在浏览器的地址栏省略了协议，浏览器会自动为你补全
2. 可以在Chrome浏览器的地址栏点击右键，显示完整的地址
3. https协议比http协议更安全，但往往出现在线上，本地的服务器通常不会是https

#### 主机（Host）

**作用**：表示客户端希望在哪台计算机上寻找资源

**两种写法**：

**1. IP地址**
- IP地址是一个网络中计算机的唯一编号
- 通常，一个IP对应一台计算机
- 特殊IP地址：`127.0.0.1`，表示本机IP

**2. 域名**
- 域名类似IP地址的别名，把不容易记忆的数字变为容易记忆的单词
- 当使用域名访问时，会自动转换为IP地址
- 特殊域名：`localhost`，表示的IP地址是127.0.0.1

#### 端口（Port）

**作用**：表示客户端希望在哪个应用程序中寻找资源

**特点**：
- 每个服务器程序都会监听一个或多个端口
- 只有找到对应的端口，才能找到这个服务器程序
- **端口号是可选的**

**默认端口规则**：
- 如果使用http协议，默认端口号为80
- 如果使用https协议，默认端口号为443

#### 路径（Path）

**作用**：服务器上往往有许许多多的资源，每个资源都有自己的访问路径

**特点**：
- **路径是可选的**
- 若不填写，则路径为 `/`

#### 参数（Query/Param）

**作用**：某些资源可以根据需要呈现不同的内容，额外信息通过参数传递

**示例**：
```
http://duyiedu.com/news?page=1&limit=10
```
- `page=1&limit=10` 就是参数部分
- 可以包含多个参数，不同的参数之间使用 `&` 符号分割
- **参数是可选的**

#### Hash

**作用**：在网络通信中，hash没有什么用，它往往作为浏览器的锚链接出现

## 3. HTTP协议

通过url地址，能够在茫茫互联网中准确的找到自己想要的服务。但光找到服务还是不够，双方需要「用同一种语言」来对话。

这个「语言」就是协议，而互联网中最常见的协议就是**HTTP协议**。

> HTTPS是在HTTP协议基础上发展起来的，它增加了安全性，其他和HTTP协议完全一致

### 3.1 HTTP协议的基本特点

**1. 请求-响应模式**

HTTP是基于请求-响应的方式完成通信的，每一次通信都是由客户端向服务器发出请求，传递一些消息过去，然后经过服务器程序处理后，响应给客户端一些消息。

**2. 无状态协议**

每次请求-响应都是**独立**的，相互之间互不干扰。

HTTP的无状态会带来一些问题，这些问题我们会在后续的课程中讨论。

**3. 纯文本传输**

每次请求-响应传递的消息都是**纯文本（字符串）**，而且文本格式必须按照HTTP协议规定的格式书写。

![image-20220415172453837](http://mdrs.yuanjin.tech/img/20220415172453.png)

### 3.2 请求的消息格式

请求消息格式由三部分组成：

![image-20220415173353090](http://mdrs.yuanjin.tech/img/20220415173353.png)

- **请求行**：高度概括了客户端想要干什么
- **请求头**：描述了请求的一些额外信息
- **请求体**：包含了要给服务器传递的正文数据（**可以省略**）

#### 请求行

请求行是整个HTTP报文的第一行字符串，包含三个部分：

```
请求方法 路径+参数+hash 协议和版本
```

**重点关注：请求方法**

请求方法是一个单词，它表达了客户端的「动作」：

- **GET**：获取资源
- **POST**：提交消息
- **PUT**：修改数据
- **DELETE**：删除数据

**GET和POST的区别**：

1. GET和DELETE请求不能有请求体，而POST和PUT请求可以有请求体
2. 由于GET请求没有请求体，所以要传递数据只能把数据放到url的参数中

**浏览器中的GET请求场景**：
- 在地址栏输入地址并按下回车
- 点击了某个a元素
- 获取图片、音频、视频
- 获取CSS、JS、字体等文件

**浏览器自动发出的请求基本都是GET请求**，而POST请求需要开发者手动处理。

#### 请求头（Header）

请求头是一系列的键值对，里面包含了诸多和业务无关的信息。

**重要请求头**：

**1. Host**
- url地址中的主机

**2. User-Agent**
- 客户端的信息描述

**3. Content-Type**
- 请求体的消息是什么格式（如果没有请求体，这个字段无意义）

**Content-Type的常见取值**：

- `application/x-www-form-urlencoded`
  - 表示请求体的数据格式和url地址中参数的格式一样
  - 示例：`loginId=admin&loginPwd=123123`

- `application/json`
  - 表示请求体的数据是JSON格式
  - 示例：`{ "loginId": "admin", "loginPwd": "123123" }`

- `multipart/form-data`
  - 一种特殊的请求体格式，上传文件一般选择该格式

#### 请求体（Body）

包含业务数据的字符串。

**常见格式**：
- `application/x-www-form-urlencoded`：`属性名=属性值&属性名=属性值...`
- `application/json`：`{"属性名":"属性值", "属性名":"属性值"}`
- `multipart/form-data`：使用某个随机字符串作为属性之间的分隔符

### 3.3 响应的消息格式

服务器的响应格式包含三个部分：

![image-20220415182351508](http://mdrs.yuanjin.tech/img/20220415182351.png)

#### 响应行

响应行包含两个部分：

- **协议版本**：表示服务器打算和客户端用什么协议通信
- **状态码、状态消息**：表示服务器对当前请求的表态

**状态码分类**：

| 分类  | 分类描述                                       |
| :---- | :--------------------------------------------- |
| 1\*\* | 信息，服务器收到请求，需要请求者继续执行操作   |
| 2\*\* | 成功，操作被成功接收并处理                     |
| 3\*\* | 重定向，需要进一步的操作以完成请求             |
| 4\*\* | 客户端错误，请求包含语法错误或无法完成请求     |
| 5\*\* | 服务器错误，服务器在处理请求的过程中发生了错误 |

**常见状态码**：

1. **200 OK**
   - 一切正常

2. **301 Moved Permanently**
   - 资源已被永久重定向
   - "你的请求我收到了，但是呢，你要的东西不在这个地址了，我已经永远的把它移动到了一个新的地址"

3. **302 Found**
   - 资源已被临时重定向
   - "你的请求我收到了，但是呢，你要的东西不在这个地址了，我临时的把它移动到了一个新的地址"

4. **304 Not Modified**
   - 文档内容未被修改
   - "你的请求我收到了，你要的东西跟之前是一样的，没有任何的变化"

5. **400 Bad Request**
   - 语义有误，当前请求无法被服务器理解
   - "你给我发的是个啥啊，我听都听不懂"

6. **403 Forbidden**
   - 服务器拒绝执行
   - "你的请求我已收到，但是我就是不给你东西"

7. **404 Not Found**
   - 资源不存在
   - "你的请求我收到了，但我没有你要的东西"

8. **500 Internal Server Error**
   - 服务器内部错误
   - "你的请求我已收到，但这道题我不会，解不出来"

#### 响应头（Header）

响应头也是由很多个键值对组成的，具体有哪些键值对，完全取决于服务器程序。

**重要的响应头：Content-Type**

表示响应体的数据类型，在B/S模式中，浏览器会自动根据响应头中Content-Type的取值，决定如何处理响应体。

**常见Content-Type值**：

1. `text/plain`：普通的纯文本，浏览器通常会将响应体原封不动的显示到页面上
2. `text/html`：HTML文档，浏览器通常会将响应体作为页面进行渲染
3. `text/javascript`或`application/javascript`：JS代码，浏览器通常会使用JS执行引擎将它解析执行
4. `text/css`：CSS代码，浏览器会将它视为样式
5. `image/jpeg`：浏览器会将它视为jpg图片
6. `attachment`：附件，浏览器看到这个类型，通常会触发下载功能

#### 响应体（Body）

响应的主体内容。

## 4. 浏览器页面处理流程

当在浏览器地址栏中输入一个url地址，并按下回车后，会发生什么？

**示例地址**：`oss.duyiedu.com/test/index.html`

![image-20220428165629557](http://mdrs.yuanjin.tech/img/20220428165634.png)

**基本流程**：
1. 浏览器解析URL地址
2. 根据URL地址向服务器发送HTTP请求
3. 服务器处理请求并返回HTTP响应
4. 浏览器解析响应内容
5. 如果是HTML文档，继续请求其中的资源（CSS、JS、图片等）
6. 最终渲染出完整的页面

## 5. 常见问题

### 5.1 跨域错误

![image-20220510154337224](http://mdrs.yuanjin.tech/img/20220510154337.png)

**什么是跨域？**

浏览器为了安全，制定了一个规则：**页面的源和请求目标的源应该保持一致**，如果不一致，就产生了跨源或者叫跨域。

**源 = 协议 + 主机 + 端口**

**跨域判断示例**：

| 页面源                          | 目标源                      | 是否跨域 |
| ------------------------------- | --------------------------- | -------- |
| https://baidu.com/news.html     | http://103.231.13.42/1.jpg  | 是       |
| https://www.baidu.com/news.html | http://baidu.com:8080/1.jpg | 是       |
| https://baidu.com/news.html     | https://baidu.com/1.jpg     | 否       |

**同源策略**：
- 浏览器对img、link、script的限制比较宽松，一般允许跨域
- 浏览器对AJAX比较严格，一般不允许跨域

**解决跨域**：
- 如果服务器明确告知浏览器允许跨域，则浏览器会允许AJAX跨域请求

### 5.2 404错误

![image-20220510153910252](http://mdrs.yuanjin.tech/img/20220510153910.png)

**错误含义**：
- 服务器响应了一个404状态码
- 表示服务器告诉客户端：你要的资源并不存在

**解决方法**：
1. 首先检查请求的url地址是否正确
2. 如果url地址正确，则可能是服务器的问题，需要联系后端开发人员

### 5.3 Favicon错误

![image-20220510152948450](http://mdrs.yuanjin.tech/img/20220510152948.png)

**错误信息**：
- 加载资源失败：服务器响应了404状态码
- 请求地址：`:5500/favicon.ico`

**原因**：

很多浏览器在解析页面后，如果发现页面中并没有使用`link`元素加载站点图标，会尝试请求以下地址来获取图标：

```
站点协议://站点主机:站点端口/favicon.ico
```

如果这个地址无法获得图标，就会报出相应错误。该错误会在下一次刷新后消失。

### 5.4 网络连接问题

**网络断开**：
![image-20220510155614692](http://mdrs.yuanjin.tech/img/20220510155614.png)
- 检查网络连接
- 检查是否在调试工具中进行了网络断开调试

**域名不存在**：
![image-20220510155742272](http://mdrs.yuanjin.tech/img/20220510155742.png)
- 无法连接到服务器
- 检查域名是否正确
