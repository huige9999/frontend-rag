## 2. 章节逻辑说明

这门课是一条完整的**「后端全栈开发」主线**，从 Node 运行时原理一路讲到能独立交付一个带数据库、登录认证、各类常见场景的真实 Web 服务。

### 先讲什么：Node 运行时与底层能力（第1章）

先打地基：全局对象、模块化机制、内置模块（文件 IO / 流 / net / http / https）、事件循环与生命周期、EventEmitter。这一章建立「Node 能干什么、底层怎么跑」的认知。

**为什么先讲**：后面无论是连数据库还是写 Web 服务，本质都是在用 Node 的 IO 与网络能力，必须先掌握。

### 再讲什么：数据从哪来（第2、3章）

- **第2章**先用纯 SQL 打底，搞懂表记录的增删改、查询、联表、函数分组——这是关系型数据库的语言基础。
- **第3章**用 Node 接上数据库：从原生 mysql 驱动 → Sequelize ORM → 模型增删改查 → 配合模拟数据、数据抓取、MD5、moment、数据验证、虚拟字段、日志，逐步搭出一个完整的**数据访问层**。

**为什么这么安排**：先会写 SQL，再用 ORM 理解它背后做了什么；数据库是 Web 服务的「数据源」，必须先有，第4章才能做真实业务。

### 然后讲什么：Web 服务与真实场景（第4章）

这是课程篇幅最大、最贴近实战的一章：
1. **框架与工程**：express 基本使用 → nodemon → 中间件机制 → 常用中间件 → 路由，搭出 Web 服务骨架。
2. **认证体系**：cookie → 登录认证 → 跨域（JSONP / CORS / CORS 中间件）→ session → jwt → 服务器端与客户端登录实现，串起一条完整的认证链。
3. **高频业务场景**：日志、文件上传/下载、图片水印、防盗链、代理、模板引擎、二维码、验证码、客户端缓存、富文本框——每个都是后端常考常考常考的实战点。

**为什么放最后讲框架**：有了底层（第1章）和数据（第2、3章），框架只是把它们粘合起来的胶水，所以 Express 这套机制会讲得很顺。

### 最后讲什么：进阶与原理（第5章）

websocket / socket.io / 在线聊天室（实时通信），CSRF / XSS（Web 安全），NodeJS 组成原理、进程和线程（运行时深入）。

**为什么放最后**：这些是「会用之后回头看原理」的内容，实时通信和安全属于高阶主题，进程线程模型是对第1章事件循环的延伸闭环。

### 主线一句话总结

**运行时能力（怎么跑）→ 数据能力（数据从哪来）→ Web 服务（怎么对外提供服务）→ 进阶原理（实时/安全/底层）**，最终目标是能用 Node 独立做一个完整的后端服务。

## 3. 使用场景等元信息沉淀

- **学完能用在哪**：独立写 Node 后端服务（RESTful API + 数据库 + 登录认证 + 文件/图片处理）；为前端项目提供 BFF 层、Mock 服务器、代理服务器；理解 SSR、构建工具（webpack/vite 底层都是 Node）背后的运行环境。
- **和实际项目的关系**：第3章的「三层架构（routes → services → models）」是真实后端项目的标准分层，可直接复用到业务项目；第4章的认证与各场景是面试与工作中最高频的后端需求。
- **和其他课程的关系**：与《工程化》共享 Node 运行时知识；认证/跨域/缓存等概念在前端面试中是跨课程通用考点。
- **面试高频场景**：事件循环与 Node 生命周期（第1章）、跨域方案对比（第4章）、cookie/session/jwt 区别（第4章）、进程与线程（第5章）、XSS/CSRF（第5章）。
