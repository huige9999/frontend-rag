# 页面访问痕迹采集

页面访问痕迹采集是用户行为监控中最核心的部分之一，主要关注用户在页面之间的流转轨迹，包括页面浏览量（PV）、页面停留时长、页面跳转路径等信息。这些数据能够帮助我们理解用户的访问模式、优化页面内容和布局、提升用户体验。

## 前端埋点

所谓**埋点**是数据采集领域（尤其是用户行为数据采集领域）的术语，其实严格来说，我们之前对错误数据的采集，对性能数据的采集，都算是一种埋点。

**埋点方案：**

- 代码埋点：用户触发某个动作后手动上报数据，优点是准确性高，能满足自定义的场景，缺点有侵入性，和目标系统耦合大，不利于维护与复用。
- ~~可视化埋点：由可视化工具进行配置采集指定元素——查找 dom 并绑定事件，优点是简单，缺点是准确性较低，针对性和自定义埋点能力较弱。~~
- 全埋点（无埋点）：由前端自动采集全部事件并上报，前端也就没有埋点成本，由数据分析平台或后端过滤有用数据，优点是数据全面，缺点是数据量大，噪声数据多。

**用户关键行为相关的埋点类型**：

- 页面埋点：统计用户进入或离开页面的各种维度信息，如页面浏览次数（PV）、页面停留时间、路由切换等。
- 点击埋点：统计用户在应用内的每一次点击事件，如新闻的浏览次数、文件下载的次数、推荐商品的命中次数等
- 曝光埋点：统计具体区域是否被用户浏览到，如活动的引流入口的显示、投放广告的显示等。

## PV、UV

PV(page view) 是页面浏览量，UV(Unique visitor)用户访问量。PV 只要访问一次页面就算一次，UV 同一天内多次访问只算一次。

对于前端来说，只要每次进入页面上报一次 PV 就行，UV 的统计可以放在服务端来做。

### PV 采集实现

PV 采集的核心逻辑非常简单：在 SDK 初始化时调用 `pv()` 函数，上报当前页面信息即可。

```js
import { lazyReportCache } from "../report";

const connection = navigator.connection;

export function pv() {
  lazyReportCache('behavior', {
    subType: 'pv',
    referrer: document.referrer,
    effectiveType: connection ? connection.effectiveType : '', // 网络环境
    rtt: connection ? connection.rtt : '', // 往返时间
  })
}
```

PV 上报的数据包含以下字段：

| 字段 | 说明 |
|------|------|
| `subType` | 值为 `pv`，标识这是一条页面浏览记录 |
| `referrer` | 来源页面地址，即用户从哪个页面跳转过来的 |
| `effectiveType` | 当前网络连接类型（如 `4g`、`3g`），通过 `navigator.connection` 获取 |
| `rtt` | 往返时延（Round-Trip Time），单位为毫秒，反映当前网络延迟情况 |

> 注意：`navigator.connection` 是 Network Information API，不是所有浏览器都支持，使用前需要做兼容性判断。

## 页面停留时长

用户进入页面记录一个初始时间，用户离开页面时用当前时间减去初始时间，就是用户停留时长。

### beforeunload 方式

计算逻辑放在 `beforeunload` 事件里：

```js
export function pageStayTime() {
  const startTime = Date.now();

  window.addEventListener('beforeunload', () => {
    const stayTime = Date.now() - startTime;

    report('behavior', [{
      subType: 'pageStayTime',
      effectiveType: connection ? connection.effectiveType : '',
      stayTime
    }], true)
  }, true);
}
```

这里需要注意两个要点：

1. **使用 `report` 而非 `lazyReportCache`**：因为 `beforeunload` 事件中页面即将卸载，如果使用延迟上报，数据可能来不及发送就丢失了。所以第三个参数传 `true`，表示立即上报。
2. **使用 `sendBeacon` 进行上报**：在页面卸载场景下，`sendBeacon` 能保证数据在页面关闭后依然可以发送成功，而普通的 `fetch` 或 `XMLHttpRequest` 请求可能会被浏览器取消。

## 页面跳转

利用 `addEventListener()` 监听 `popstate`、`hashchange` 页面跳转事件。需要注意的是调用 `history.pushState()` 或 `history.replaceState()` 不会触发 `popstate` 事件。只有在做出浏览器动作时，才会触发该事件，如用户点击浏览器的回退按钮（或者在 Javascript 代码中调用 `history.back()` 或者 `history.forward()` 方法）。同理，`hashchange` 也一样。

### 页面跳转采集实现

```js
export function pageChange() {
  let from = document.referrer;

  // 监听 popstate 事件
  window.addEventListener('popstate', () => {
    const to = window.location.href;

    lazyReportCache('behavior', {
      subType: 'pageChange',
      from,
      to
    });

    from = to;
  }, true)

  // 监听 hashchange 事件
  let oldURL = document.referrer;
  window.addEventListener('hashchange', event => {
    const newURL = event.newURL;

    lazyReportCache('behavior', {
      subType: 'pageChange',
      from: oldURL,
      to: newURL
    });

    oldURL = newURL;
  }, true)
}
```

页面跳转上报的数据包含 `from`（来源页面）和 `to`（目标页面），可以清晰地追踪用户的页面跳转路径。需要维护一个 `from` 变量，在每次跳转后更新为当前页面地址，以便下次跳转时能正确记录来源。

## Vue 路由变更

对于使用 Vue 框架的单页面应用（SPA），页面跳转是通过前端路由实现的，不会触发 `popstate` 和 `hashchange` 事件。因此需要利用 Vue Router 提供的 `beforeEach` 钩子来监听路由变更。

### Vue 路由变更采集实现

```js
// 首先记录一开始的时间
window.pageViewStartTime = Date.now();

export function onVueRouter() {
  if (!config.vue || !config.vue.router) return;

  config.vue.router.beforeEach((to, from, next) => {
    // 如果是首次加载页面不需要统计
    if (!from.name) return next();

    const stayTime = Date.now() - window.pageViewStartTime;

    window.pageViewStartTime = Date.now();

    lazyReportCache('behavior', {
      subType: 'vueRouterChange',
      params: to.params,
      query: to.query,
      name: to.name || to.path,
      from: from.fullPath,
      to: to.fullPath,
      stayTime
    })

    next();
  });
}
```

Vue 路由变更上报的数据更加丰富：

| 字段 | 说明 |
|------|------|
| `subType` | 值为 `vueRouterChange`，标识 Vue 路由变更事件 |
| `params` | 目标路由的动态参数 |
| `query` | 目标路由的查询参数 |
| `name` | 目标路由名称（优先取 `name`，没有则取 `path`） |
| `from` | 来源路由完整路径 |
| `to` | 目标路由完整路径 |
| `stayTime` | 在上一个页面的停留时长（毫秒） |

注意几个要点：

1. **首次加载跳过**：通过 `if (!from.name) return next()` 判断，首次加载页面时 `from` 为空，此时不需要统计路由变更。
2. **全局时间变量**：使用 `window.pageViewStartTime` 记录每次进入页面的时间，路由变更时计算停留时长并重新记录时间。

## SDK 配置更新

在 `config/index.js` 中，新增了与 Vue 相关的配置项：

```js
const config = {
  // 项目名称
  appId: 'monitor-sdk-demo',
  userId: 'ys',
  // 上报地址
  reportUrl: 'http://localhost:3001/report/actions',
  // 是否全埋点
  trackerAll: false,
  // Vue 相关配置
  vue: {
    Vue: null,
    router: null
  },
  ua: navigator.userAgent,
};

export default config;

export function setConfig(options) {
  for (const key in config) {
    if (options[key]) {
      config[key] = options[key];
    }
  }
}
```

`vue` 配置对象包含两个字段：

- `Vue`：Vue 实例，用于 Vue 相关的扩展功能
- `router`：Vue Router 实例，用于 `onVueRouter()` 中注册路由钩子

使用时需要在初始化 SDK 时传入：

```js
import Vue from 'vue';
import router from './router';
import monitor from 'monitor-sdk';

monitor.init({
  appId: 'my-app',
  userId: 'user-001',
  reportUrl: 'http://localhost:3001/report/actions',
  vue: {
    Vue,
    router
  }
});
```

## SDK 入口更新

在 `index.js` 中，将行为采集相关的功能全部暴露出去：

```js
import { setConfig } from "./config";
import performance from "./performance";
import error from "./error";
import { tracker, autoTracker } from "./action";
import { pv, pageStayTime, pageChange, onVueRouter } from "./behavior";

const monitor = {
  init(options = {}) {
    setConfig(options);      // 配置全局参数
    error();                 // 错误监听处理
    performance();           // 性能监听处理
    autoTracker();           // 自动埋点
    pv();                    // 页面浏览量采集
  },
  tracker,                   // 手动埋点
  pageStayTime,              // 页面停留时长
  pageChange,                // 页面跳转
  onVueRouter,               // Vue 路由变更
};

export default monitor;
```

SDK 入口的设计体现了**自动采集与手动采集相结合**的思路：

- `pv()` 和 `autoTracker()` 在 `init` 时自动执行，无需手动调用
- `pageStayTime()`、`pageChange()`、`onVueRouter()` 则由开发者根据需要手动调用，给予灵活的控制权

## 数据上报统一格式

所有页面访问痕迹相关的数据，其上报类型（type）统一为 `behavior`，通过 `subType` 字段区分具体的行为类型：

| subType | 说明 | 上报方式 |
|---------|------|----------|
| `pv` | 页面浏览量 | 延迟上报（lazyReportCache） |
| `pageStayTime` | 页面停留时长 | 立即上报（report + isImmediate） |
| `pageChange` | 页面跳转 | 延迟上报（lazyReportCache） |
| `vueRouterChange` | Vue 路由变更 | 延迟上报（lazyReportCache） |

最终上报的数据结构如下：

```json

  "currentPage": "http://localhost:8080/about",
  "ua": "Mozilla/5.0 ..."
}
```

## 总结

页面访问痕迹采集主要围绕四个维度展开：

1. **PV 采集**：页面加载时自动上报，记录来源页面和网络环境信息
2. **页面停留时长**：通过记录进入页面和离开页面的时间差来计算，离开时使用 `sendBeacon` 立即上报
3. **页面跳转**：监听 `popstate` 和 `hashchange` 事件，记录跳转的来源和目标页面
4. **Vue 路由变更**：利用 Vue Router 的 `beforeEach` 钩子，不仅能记录路由变更，还能同时获取页面停留时长

这些数据组合在一起，就能完整地还原用户在应用中的访问轨迹，为后续的用户行为分析、产品优化提供数据支撑。
