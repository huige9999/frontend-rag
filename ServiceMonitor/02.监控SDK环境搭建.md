# 02. 监控SDK环境搭建

## 一、课程目标

- 搭建监控SDK的开发环境

## 三、SDK项目配置

### 3.1 package.json

SDK项目使用ES模块规范（`"type": "module"`），并依赖Rollup及相关的Babel插件进行打包。

```json
{
  "name": "monitor-sdk",
  "version": "1.0.0",
  "main": "src/index.js",
  "scripts": {
    "build": "rollup -c",
    "dev": "rollup -c -w"
  },
  "devDependencies": {
    "@babel/preset-env": "^7.22.9",
    "@rollup/plugin-babel": "^6.0.3",
    "@rollup/plugin-commonjs": "^25.0.2",
    "@rollup/plugin-node-resolve": "^15.1.0",
    "rollup": "^3.26.2"
  },
  "type": "module"
}
```

**关键点说明：**

| 字段 | 说明 |
|------|------|
| `"main": "src/index.js"` | 指定SDK的入口文件 |
| `"type": "module"` | 项目使用ES Module模块规范 |
| `"build"` | 执行一次Rollup打包 |
| `"dev"` | 启动watch模式，文件变更时自动重新打包 |

**核心依赖说明：**

| 依赖包 | 作用 |
|--------|------|
| `rollup` | JavaScript模块打包器，适合类库打包 |
| `@rollup/plugin-babel` | Rollup的Babel插件，用于ES6+语法转换 |
| `@babel/preset-env` | Babel预设，自动根据目标环境转换语法 |
| `@rollup/plugin-commonjs` | 将CommonJS模块转换为ES Module |
| `@rollup/plugin-node-resolve` | 帮助Rollup识别 `node_modules` 中的第三方模块 |

### 3.2 .babelrc

Babel配置文件，指定使用 `@babel/preset-env` 预设进行语法转换：

```json
{
  "presets": [
    "@babel/preset-env"
  ]
}
```

## 四、Rollup打包配置解析

### 4.1 完整配置文件

```js
import commonjs from "@rollup/plugin-commonjs"
import resolve from "@rollup/plugin-node-resolve"
import babel, { getBabelOutputPlugin } from "@rollup/plugin-babel"

export default {
  input: 'src/index.js',
  output: [
    {
      file: './lib/index.js',
      format: 'iife',
      name: 'monitor',
    },
    {
      file: './lib/index.esm.js',
      format: 'esm',
    },
    {
      file: './lib/index.umd.js',
      format: 'umd',
      name: 'monitorSDK',
    },
  ],
  watch: {
    exclude: 'node_modules/**',
  },
  plugins: [
    getBabelOutputPlugin({
      presets: ['@babel/preset-env'],
      allowAllFormats: true,
    }),
    commonjs(),
    resolve(),
    babel({
      babelHelpers: 'bundled',
      exclude: 'node_modules/**',
      extensions: ['.js'],
    }),
  ]
}
```

### 4.2 配置详解

#### 4.2.1 input — 入口文件

```js
input: 'src/index.js',
```

指定SDK的打包入口文件为 `src/index.js`。

#### 4.2.2 output — 多格式输出

Rollup支持同时输出多种模块格式，这里是核心配置：

```js
output: [
  {
    file: './lib/index.js',
    format: 'iife',        // 立即执行函数格式
    name: 'monitor',       // 全局变量名
  },
  {
    file: './lib/index.esm.js',
    format: 'esm',         // ES Module格式
  },
  {
    file: './lib/index.umd.js',
    format: 'umd',         // 通用模块定义格式
    name: 'monitorSDK',    // UMD全局变量名
  },
]
```

**三种输出格式对比：**

| 格式 | 适用场景 | 加载方式 |
|------|----------|----------|
| **IIFE** | 通过 `<script>` 标签直接引入 | 暴露为全局变量 `window.monitor` |
| **ESM** | 现代打包工具（Webpack/Vite）或原生ES Module | `import monitor from 'monitor-sdk'` |
| **UMD** | 兼容所有环境（AMD/CMD/CommonJS/全局变量） | 自动选择合适的模块系统 |

**IIFE输出示例（打包后）：**

```js
(function () {
  'use strict';

  var fn = function fn() {
    console.log('fn');
    return [1, 2, 3, 4].map(function (item) {
      return item + 1;
    });
  };
  fn();
})();
```

**ESM输出示例（打包后）：**

```js
var fn = function fn() {
  console.log('fn');
  return [1, 2, 3, 4].map(function (item) {
    return item + 1;
  });
};
fn();
```

**UMD输出示例（打包后）：**

```js
(function (factory) {
  typeof define === 'function' && define.amd ? define(factory) : factory();
})(function () {
  'use strict';

  var fn = function fn() {
    console.log('fn');
    return [1, 2, 3, 4].map(function (item) {
      return item + 1;
    });
  };
  fn();
});
```

> 可以看到，通过Babel的转换，箭头函数 `=>` 和 `const` 都被转译成了ES5语法的 `function` 和 `var`，确保了兼容性。

#### 4.2.3 watch — 文件监听

```js
watch: {
  exclude: 'node_modules/**',
}
```

开发模式下排除 `node_modules` 目录的监听，避免不必要的重复打包。

#### 4.2.4 plugins — 插件配置

```js
plugins: [
  getBabelOutputPlugin({
    presets: ['@babel/preset-env'],
    allowAllFormats: true,    // 允许对所有输出格式做Babel转换
  }),
  commonjs(),                 // CommonJS -> ES Module 转换
  resolve(),                  // 解析 node_modules 中的模块
  babel({                     // 对源码进行Babel转换
    babelHelpers: 'bundled',
    exclude: 'node_modules/**',
    extensions: ['.js'],
  }),
]
```

**插件执行顺序与作用：**

1. **`getBabelOutputPlugin`** — 在输出阶段对生成的代码进行Babel转换（而非在编译阶段），`allowAllFormats: true` 使其支持所有输出格式（IIFE/ESM/UMD）
2. **`commonjs()`** — 将CommonJS模块转换为ES Module，使Rollup能正确处理CJS依赖
3. **`resolve()`** — 让Rollup能找到并打包 `node_modules` 中的第三方模块
4. **`babel()`** — 在编译阶段对源码进行Babel转换，`babelHelpers: 'bundled` 将helper函数内联打包

## 五、SDK入口文件设计

### 5.1 入口文件 src/index.js

```js
import { fn } from "./module";

fn();
```

入口文件导入模块并执行。后续课程中，这里会扩展为SDK的初始化逻辑。

### 5.2 模块文件 src/module.js

```js
export const fn = () => {
  console.log('fn');
  return [1, 2, 3, 4].map(item => item + 1);
}
```

模块文件导出一个箭头函数，其中使用了ES6语法（`const`、箭头函数、`map`）。这个文件用于验证Rollup打包和Babel语法转换是否正常工作。

## 七、使用说明

### 7.1 SDK开发流程

```bash
# 1. 进入SDK项目目录
cd monitor-demo/monitor-sdk

# 2. 安装依赖
npm install

# 3. 启动开发模式（watch监听）
npm run dev

# 4. 或执行一次性打包
npm run build
```

### 7.2 Vue2测试项目启动

```bash
# 1. 进入Vue2测试项目目录
cd monitor-demo/vue2-demo

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run serve
```

### 7.3 SDK引用方式

SDK打包后生成在 `lib/` 目录下，根据不同使用场景选择对应的格式：

**方式一：script标签引入（IIFE格式）**

```html
<script src="./lib/index.js"></script>
<script>
  // 通过全局变量 monitor 访问SDK
  monitor.init({ ... });
</script>
```

**方式二：ES Module引入**

```js
import monitor from './lib/index.esm.js';
monitor.init({ ... });
```

**方式三：UMD格式引入（兼容各种模块系统）**

```js
// CommonJS
const monitorSDK = require('./lib/index.umd.js');

// AMD
define(['./lib/index.umd.js'], function(monitorSDK) { ... });
```

## 八、为什么选择Rollup

在本课程中选择Rollup而非Webpack作为SDK的打包工具，原因如下：

| 对比项 | Rollup | Webpack |
|--------|--------|---------|
| 定位 | 类库/工具库打包 | 应用程序打包 |
| 输出代码 | 简洁、无额外运行时代码 | 包含模块运行时（webpack runtime） |
| Tree-shaking | 原生支持，效果更好 | 需要配置，效果一般 |
| 输出格式 | 原生支持IIFE/ESM/UMD/AMD/CJS | 需要额外配置 |
| 打包体积 | 更小 | 相对较大 |

> **总结：** 对于SDK这种类库项目，Rollup是更优的选择。它打包出的代码更加干净简洁，没有额外的运行时代码注入，适合作为被其他项目引用的监控SDK。
