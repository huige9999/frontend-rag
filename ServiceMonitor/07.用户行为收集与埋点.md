# 用户行为监控

## 前端埋点

所谓**埋点**是数据采集领域（尤其是用户行为数据采集领域）的术语，其实严格来说，我们之前对错误数据的采集，对性能数据的采集，都算是一种埋点。

**埋点方案：**

- 代码埋点：用户触发某个动作后手动上报数据，优点时准确性高，能满足自定义的场景，缺点有侵入性，和目标系统耦合大，不利于维护与复用。
- ~~可视化埋点：由可视化工具进行配置采集指定元素——查找 dom 并绑定事件，优点是简单，缺点是准确性较低，针对性和自定义埋点能力较弱。~~
- 全埋点（无埋点）：由前端自动采集全部事件并上报，前端也就没有埋点成本，由数据分析平台或后端过滤有用数据，优点是数据全面，缺点是数据量大，噪声数据多。

**用户关键行为相关的埋点类型**：

- 页面埋点：统计用户进入或离开页面的各种维度信息，如页面浏览次数（PV）、页面停留时间、路由切换等。
- 点击埋点：统计用户在应用内的每一次点击事件，如新闻的浏览次数、文件下载的次数、推荐商品的命中次数等
- 曝光埋点：统计具体区域是否被用户浏览到，如活动的引流入口的显示、投放广告的显示等。

---

## 埋点方案实现

本节我们实现两种埋点方式：**手动埋点（代码埋点）** 和 **自动埋点（全埋点）**，并提供灵活的配置切换能力。

### 配置项更新

在配置模块中，新增了与行为监控相关的两个关键字段：`trackerAll` 和 `reportUrl`。

```js
// src/config/index.js

const config = {
  // 项目名称
  appId: 'monitor-sdk-demo',
  userId: 'ys',
  // 上报地址
  reportUrl: 'http://localhost:3001/report/actions',
  // 是否全埋点
  trackerAll: false,
  vue: {
    Vue: null,
    router: null
  },
  ua: navigator.userAgent,
};

export default config;

export function setConfig(options) {
  for (const key in config) {
    if (options[key]) {
      config[key] = options[key];
    }
  }
}
```

**关键配置说明：**

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `trackerAll` | Boolean | `false` | 是否开启全埋点模式。`true` 时自动上报所有用户交互事件，`false` 时仅上报带有 `data-tracker` 属性的元素 |
| `reportUrl` | String | `'http://localhost:3001/report/actions'` | 数据上报的服务端地址 |

### 用户行为采集模块

行为采集模块（`src/action/index.js`）是本次实现的核心，提供两个导出函数：

- `tracker`：手动埋点函数，由开发者主动调用
- `autoTracker`：自动埋点函数，SDK 初始化时自动注册事件监听

```js
// src/action/index.js

import { lazyReportCache } from '../report'
import { getPaths } from '../util/paths'
import config from "../config"

export const tracker = (event) => {
  // 如果是全自动上报，则不需要手动上报
  if (config.trackerAll) return;

  const target = event.target;

  const data = {
    eventType: event.type,
    tagName: target.tagName,
    x: event.x,
    y: event.y,
    paths: getPaths(event),
    value: target.value || target.innerText,
  }

  lazyReportCache('action', data);
}

export const autoTracker = () => {
  ['click', 'keydown', 'blur', 'focus', 'touchstart', 'touchend'].forEach(eventType => {
    let timer = null;
    document.addEventListener(eventType, event => {
      clearTimeout(timer);
      // 防抖
      timer = setTimeout(() => {
        const target = event.target;
        const dataTracker = target.getAttribute('data-tracker');

        // 如果配置项中config.trackerAll为true，则所有元素都需要上报
        // 如果元素上有data-tracker属性，则上报
        if (config.trackerAll || dataTracker) {
          const data = {
            eventType: event.type,
            tagName: target.tagName || 'window',
            x: event.x,
            y: event.y,
            paths: getPaths(event),
            value: target.value || target.innerText || '',
          }
          lazyReportCache('action', data);
        }
      }, 500)
    }, false)
  })
}
```

**代码解析：**

#### 1. 手动埋点 `tracker`

手动埋点函数接收一个原生 DOM 事件对象 `event`，从中提取以下关键信息：

- `eventType`：事件类型（如 `click`、`input` 等）
- `tagName`：触发事件的元素标签名（如 `BUTTON`、`INPUT`）
- `x / y`：事件触发时的鼠标/触点坐标
- `paths`：DOM 事件冒泡路径的 CSS 选择器字符串（用于精确定位元素在 DOM 树中的位置）
- `value`：元素的值（如 input 的 value）或文本内容（如按钮的 innerText）

> 注意：当 `config.trackerAll` 为 `true` 时，手动埋点会直接 return，避免与自动埋点产生重复数据。

#### 2. 自动埋点 `autoTracker`

自动埋点通过 `document.addEventListener` 对以下六种事件类型进行全局监听：

| 事件类型 | 触发场景 |
|----------|----------|
| `click` | 鼠标点击 |
| `keydown` | 键盘按下 |
| `blur` | 元素失去焦点 |
| `focus` | 元素获得焦点 |
| `touchstart` | 触摸开始（移动端） |
| `touchend` | 触摸结束（移动端） |

**上报策略：** 采用 **防抖（debounce）** 机制，设置 500ms 的延迟。在连续快速操作时，只有最后一次操作会被上报，有效减少冗余数据。

**过滤机制：** 自动埋点并非无条件上报所有事件，而是通过以下两种条件之一决定是否上报：

1. **全埋点模式**（`config.trackerAll = true`）：所有元素的所有监听事件都会上报
2. **标记模式**（`config.trackerAll = false`）：仅当触发元素具有 `data-tracker` 自定义属性时才上报

标记模式的 HTML 使用示例：

```html
<!-- 只有带 data-tracker 属性的按钮点击会被自动上报 -->
<button data-tracker="submit-btn">提交</button>
<button>取消</button>  <!-- 点击不会被上报 -->
```

#### 3. DOM 路径解析工具

行为采集模块依赖 `getPaths` 工具函数来获取事件冒泡路径的 CSS 选择器字符串，这在后续分析用户行为时能精确定位用户操作的 DOM 元素。

```js
// src/util/paths.js

export const getComposedPathEle = (e) => {
  if (!e) return [];
  // 如果支持path属性，直接返回path属性
  // 如果不支持，就通过composedPath方法获取
  let pathArr = e.path || (e.composedPath && e.composedPath());
  if ((pathArr || []).length) {
    return pathArr;
  }

  // composedPath方法不兼容，手动获取
  let target = e.target;
  const composedPath = [];

  while (target && target.parentNode) {
    composedPath.push(target);
    target = target.parentNode;
  }

  composedPath.push(document, window);

  return composedPath;
}

export const getComposedPath = (e) => {
  if (!e) return [];
  const composedPathEle = getComposedPathEle(e);

  const composedPath = composedPathEle.reverse().slice(2).map(ele => {
    let selector = ele.tagName.toLowerCase();
    if (ele.id) {
      selector += `#${ele.id}`;
    }
    if (ele.className) {
      selector += `.${ele.className}`;
    }
    return selector;
  });

  return composedPath;
}

export const getPaths = (e) => {
  if (!e) return '';
  const composedPath = getComposedPath(e);
  const selectors = composedPath.join(' > ');
  return selectors;
}
```

该工具的解析流程：

1. `getComposedPathEle`：获取事件冒泡路径上的所有 DOM 元素，兼容处理了 `event.path`、`event.composedPath()` 和手动遍历三种方式
2. `getComposedPath`：将 DOM 元素数组转换为 CSS 选择器数组（包含标签名、id、className）
3. `getPaths`：将选择器数组用 ` > ` 连接，生成如 `div#app > form.login > button#submit.btn` 的完整路径字符串

### SDK 入口集成

在 SDK 入口文件中，将行为监控模块集成进来：

```js
// src/index.js

import { setConfig } from "./config";
import performance from "./performance";
import error from "./error";
import { tracker, autoTracker } from "./action";

const monitor = {
  init(options = {}) {
    console.log('init');
    setConfig(options);     // 配置全局参数
    error();                // 错误监听处理
    performance();          // 性能监听处理
    autoTracker();          // 自动埋点
  },
  tracker
};

export default monitor;
```

**集成要点：**

- `autoTracker()` 在 `init` 时自动调用，注册全局事件监听，开始采集用户行为数据
- `tracker` 挂载到 monitor 对象上导出，供开发者在需要的地方手动调用
- 调用顺序：先配置 `setConfig`，再启动错误监控和性能监控，最后启动行为监控

### 使用方式

#### 全埋点模式

```js
import monitor from 'monitor-sdk';

monitor.init({
  appId: 'my-app',
  userId: 'user-001',
  reportUrl: 'https://your-server.com/report',
  trackerAll: true,  // 开启全埋点，所有交互事件自动上报
});
```

#### 标记埋点模式

```js
import monitor from 'monitor-sdk';

monitor.init({
  appId: 'my-app',
  userId: 'user-001',
  reportUrl: 'https://your-server.com/report',
  trackerAll: false,  // 仅上报带 data-tracker 属性的元素（默认值）
});
```

```html
<!-- 在需要埋点的元素上添加 data-tracker 属性 -->
<button data-tracker="add-to-cart">加入购物车</button>
<a data-tracker="download-link" href="/file.pdf">下载文件</a>
```

#### 手动埋点

```html
<button id="myBtn">特殊操作</button>
```

```js
import monitor from 'monitor-sdk';

monitor.init({
  appId: 'my-app',
  userId: 'user-001',
  reportUrl: 'https://your-server.com/report',
});

// 手动绑定埋点
document.getElementById('myBtn').addEventListener('click', (event) => {
  monitor.tracker(event);  // 手动上报此次点击事件
});
```

### 上报数据结构

每条行为数据通过 `lazyReportCache('action', data)` 延迟上报，最终的服务端接收到的数据格式如下：

```json
{
  "id": "unique-session-id",
  "appId": "monitor-sdk-demo",
  "userId": "ys",
  "currentTime": 1717700000000,
  "type": "action",
  "data": [
    {
      "eventType": "click",
      "tagName": "BUTTON",
      "x": 320,
      "y": 480,
      "paths": "div#app > form.login > button#submit",
      "value": "提交"
    }
  ],
  "currentPage": "https://example.com/login",
  "ua": "Mozilla/5.0 ..."
}
```

### 数据流程总结

```
用户交互事件（click/keydown/touchstart/...）
        |
        v
  autoTracker / tracker
        |
        v
  提取事件信息（eventType, tagName, paths, x, y, value）
        |
        v
  lazyReportCache('action', data)  —— 延迟3秒批量上报
        |
        v
  report()  —— 使用 navigator.sendBeacon 发送数据
        |
        v
  服务端接收并存储
```
