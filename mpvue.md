## 2. 章节逻辑说明

这门课的主线是**「学 mpvue 框架机制（01~13）→ 用 50 个案例疯狂练手（14~63）→ 接入小程序云开发收尾（64~68）」**。一条「先打地基，再用海量项目把语法锤熟，最后补云端能力」的强实战主线。mpvue 本质是「用 Vue2 语法写微信小程序」，所以基础段大量复用 Vue2 语法，案例段则是把这套语法在各种真实业务里反复用。

### 第一段：mpvue 基础与 Vue 语法迁移（01~13）

分三层：

- **01~03 认识 / 入门 / 生命周期**：先搞清 mpvue 是什么、怎么起步（脚手架 `vue init mpvue/mpvue-quickstart`）、和 Vue2 生命周期的关系。
- **04~09 Vue2 模板语法迁移**：模板语法、计算属性、Class/Style 绑定、条件/列表渲染、事件处理、表单绑定——这套几乎就是 Vue2 的核心语法，mpvue 原样支持。
- **10~13 mpvue / 小程序特性**：自定义组件、**原生小程序组件使用**（这是 mpvue 区别于纯 Vue 的关键）、**flyio 网络请求**、全局存储与页面跳转传参（小程序路由机制）。

**为什么这么安排**：mpvue = Vue2 语法 + 小程序运行时。所以先迁移你已有的 Vue 语法（04~09），再补 mpvue 特有的「小程序组件、网络、跳转」能力（10~13），就能开始写真实小程序了。`flyio`（12）和「全局存储/跳转传参」（13）是后续所有案例的基础设施。

### 第二段：综合案例实战（14~63）

这是全课最重的部分——**50 个案例**，覆盖资讯、媒体、工具、表单、电商、社交、布局等几乎所有常见小程序业务类型。每个案例都是一次完整的「需求 → 拆页面 → 调接口 → 绑数据 → 写交互」流程，本质是用项目把第一段的语法和能力反复练熟。

- **资讯/内容展示**（今日新闻、水浒传、云图库）：列表渲染 + 接口请求 + 详情页。
- **媒体/娱乐**（音乐风云榜、抽奖小游戏）：音频播放、动画/随机逻辑。
- **交互/通讯**（机器人聊天、侧滑菜单+旅游风景）：聊天 UI、手势/侧滑交互。
- **实用工具**（汇率转换、IOS 计算器、微天气、微方向）：计算逻辑、API 对接（天气）。
- **表单/数据交互**（表单提交与照片存储、城市列表、用户注册表单和拨打电话）：表单绑定（09 的进阶）+ 小程序原生能力（拍照、拨打电话）。
- **电商/平台**（大好商城、我的电视云操作、ITBook、医疗机构团购站）：商品/列表/详情/下单的完整电商链路。
- **个人中心/社交**（我的个人中心列表、妈妈课堂、送祝福）：个人页 + 内容型应用。
- **布局专项**（weui 抽奖布局 1~5）：用 5 章专攻一套 weui 风格的复杂布局，是 CSS/布局能力的集中训练。

**为什么塞这么多案例**：这是本课最鲜明的特点——不追求每个知识点深挖，而是用「量大管饱的真实项目」让语法变成本能。复习时不需要全做，按业务类型各挑一个代表案例回看即可。

### 第三段：小程序云开发进阶（64~68）

- **64~65 云数据库与 mpvue 操作**：小程序云开发的数据库 CRUD。
- **66~67 云函数在 mpvue 中的使用**：云函数的编写与调用。
- **68 云存储在 mpvue 中的运用**：文件/图片上传到云存储。

**为什么放最后**：云开发是小程序「免服务器」的高级能力，前置依赖是前面所有基础和接口请求能力都熟练，所以放在收尾段作为进阶提升。

### 主线一句话总结

**mpvue 基础 + Vue2 语法迁移（01~13）→ 50 个综合案例练手（14~63）→ 小程序云开发收尾（64~68）**，目标是「用 Vue2 语法熟练开发各类微信小程序，并掌握免服务器的云开发能力」。

## 3. 使用场景等元信息沉淀

- **学完能用在哪**：用 Vue2 语法独立开发各类微信小程序（资讯、电商、工具、社交等），并能用云开发免去自建后端。
- **和实际项目的关系**：每个案例都是可直接运行的小程序工程，挑业务类型相近的案例作为项目脚手/参考非常高效。
- **和其他课程的关系**：前置是《vue 从入门到实战》（mpvue 的语法层就是 Vue2，必须先熟）；本课与《微信小程序》（原生语法）是「同一目标的两种写法」——mpvue 是 Vue 语法派，原生小程序是 wxml/wxss 派，二选一或互补均可。云开发段（64~68）与小程序云开发生态相关。
- **面试/教学高频场景**：mpvue 与原生小程序的区别与取舍、生命周期、flyio 网络请求封装、页面跳转传参、小程序云开发（云函数/云数据库/云存储）。

## 4. 临时补充

- 本课最鲜明的特点：**案例量极大（50 个）**，是典型的「以练代学」。复习策略建议：基础段（01~13）全看，案例段（14~63）按业务类型各挑 1~2 个代表回看即可，不必逐个重做。
- mpvue 的核心价值是「复用 Vue2 生态与语法写小程序」，但需注意 **mpvue 已停止维护**（社区活跃度低），新项目更多用原生小程序或 uni-app。但课程里的小程序运行时机制（组件、生命周期、跳转、云开发）是通用的，仍值得学。
- 案例章的 `README.md` 全是脚手架默认模板（`> A Mpvue project`），不含实际知识点——**真正的教学内容在 `src/` 源码里**，复习时直接读源码。
- 原仓库目录名里的数字是课程顺序，但 `ls` 出来是按字符串排序（`1.` `10.` `2.` 错乱），本 md 已按真实课程编号顺序重排。
- 部分案例存在缺号（如 IOS 计算器只有 `-1` 和 `-3`，无 `-2`），属原仓库历史遗留，整理时保留原编号。
