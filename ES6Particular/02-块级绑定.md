# 块级绑定

## 2-1. 声明变量的问题

使用var声明变量存在以下问题：

### 1. 允许重复的变量声明：导致数据被覆盖

```javascript
var a = 1;

function print(){
    console.log(a)
}

//假设这里有一千行代码

var a = 2;

print(); // 输出2，a被重新声明覆盖
```

### 2. 变量提升：怪异的数据访问

```javascript
if (Math.random() < 0.5) {
    var a = "abc";
    console.log(a);
}
else {
    console.log(a); // 可以访问a，但值为undefined
}

console.log(a); // 可以访问a
```

### 3. 变量提升：闭包问题

```javascript
var div = document.getElementById("divButtons")

for (var i = 1; i <= 10; i++) {
    var btn = document.createElement("button");
    btn.innerHTML = "按钮" + i;
    div.appendChild(btn);
    btn.onclick = function () {
        console.log(i); // 输出11，因为循环结束后i变为11
    }
}

// 循环结束后，i：11
```

### 4. 全局变量挂载到全局对象：全局对象成员污染问题

```javascript
var abc = "123";
console.log(window.abc); // 输出"123"，全局变量挂载到window对象

var console = "abc";
console.log(console) // 报错，console被覆盖

var name = "abc";
// name是window对象的特殊属性，会被特殊处理
```

## 2-2. 使用let声明变量

ES6不仅引入let关键字用于解决变量声明的问题，同时引入了块级作用域的概念。

**块级作用域**：代码执行时遇到花括号，会创建一个块级作用域，花括号结束，销毁块级作用域。

### 1. 解决全局变量挂载到全局对象的问题

let声明的变量不会挂载到全局对象：

```javascript
let a = 123;
console.log(window.a); // 输出undefined，let声明的变量不会挂载到window
```

### 2. 解决重复声明的问题

let声明的变量，不允许当前作用域范围内重复声明：

```javascript
let a = 123;
let a = 456; // 报错：检查到当前作用域（全局作用域）已声明了变量a
```

在不同作用域中可以声明同名变量：

```javascript
let a = 123;

function test(){
    let a = 456; // 函数作用域，可以声明
    console.log(a) // 输出456
}

test();

{
    let a = 456; // 块级作用域，可以声明
    console.log(a); // 输出456，使用的是块级作用域中的a
}
console.log(a) // 输出123
```

### 3. 解决变量提升的问题

使用let不会有变量提升，因此，不能在定义let变量之前使用它：

```javascript
console.log(a) // 报错：Cannot access 'a' before initialization
let a = 123;
```

**底层实现**：let声明的变量实际上也会有提升，但是，提升后会将其放入到"暂时性死区"（Temporal Dead Zone）。如果访问的变量位于暂时性死区，则会报错："Cannot access 'a' before initialization"。当代码运行到该变量的声明语句时，会将其从暂时性死区中移除。

### 4. 块级作用域的限制

在块级作用域中用let定义的变量，在作用域外不能访问：

```javascript
if (Math.random() < 0.5) {
    let a = 123; // 定义在当前块级作用域
    console.log(a) // 当前块级作用域中的a
}
else {
    // 这是另外一个块级作用域，该作用域中找不到a
    console.log(a) // 报错
}

console.log(a); // 报错
```

### 5. 解决循环中的闭包问题

在循环中，用let声明的循环变量，会特殊处理：每次进入循环体，都会开启一个新的作用域，并且将循环变量绑定到该作用域（每次循环，使用的是一个全新的循环变量）：

```javascript
let div = document.getElementById("divButtons");

for (let i = 1; i <= 10; i++) {
    let button = document.createElement("button")
    button.innerHTML = "按钮" + i;
    button.onclick = function () {
        console.log(i) // 使用的是当前作用域中的i，输出正确的值
    }
    div.appendChild(button)
}

// 在循环中使用let声明的循环变量，在循环结束后会销毁
```

## 2-3. 使用const声明常量

const和let完全相同，仅在于用const声明的变量，必须在声明时赋值，而且不可以重新赋值。

### 基本用法

```javascript
const a = 1;
a = 2; // 报错：不能重新赋值
console.log(a)
```

### 为什么应该尽量使用const

在开发中，应该尽量使用const来声明变量，以保证变量的值不会随意篡改：

1. 根据经验，开发中的很多变量，都是不会更改，也不应该更改的。
2. 后续的很多框架或者是第三方JS库，都要求数据不可变，使用const可以一定程度上保证这一点。

```javascript
const div = document.getElementById("game");
// 一般来说，div变量是不会更改的
```

### 注意的细节

#### 1. 常量不可变的含义

常量不可变，是指声明的常量的内存空间不可变，并不保证内存空间中的地址指向的其他空间不可变：

```javascript
const a = {
    name: "kevin",
    age: 123
};
a.name = "abc"; // 可以修改对象属性
console.log(a) // 输出 {name: "abc", age: 123}
```

#### 2. 常量的命名

**特殊的常量**：该常量从字面意义上，一定是不可变的，比如圆周率、月地距离或其他一些绝不可能变化的配置。通常，**该常量的名称全部使用大写，多个单词之间用下划线分割**：

```javascript
const PI = 3.14;
const MOON_EARTH_DISTANCE = 3245563424; // 月地距离
```

**普通的常量**：使用和之前一样的命名即可（小驼峰命名）。

#### 3. 在for循环中使用const

在for循环中，循环变量不可以使用常量：

```javascript
for (const i = 0; i < 10; i++) { // 报错：不能给常量赋值
    // ...
}
```

但在for-in或for-of循环中可以使用const：

```javascript
var obj = {
    name: "kevin",
    age: 1
}

for (const prop in obj) {
    console.log(prop) // 输出属性名
}
```
