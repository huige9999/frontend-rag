# 第5章 对象

## 5-1. 新增的对象字面量语法

### 1. 成员速写

如果对象字面量初始化时，成员的名称来自于一个变量，并且和变量的名称相同，则可以进行简写。

**示例代码：**

```javascript
function createUser(loginId, loginPwd, nickName) {
    const sayHello = function () {
        console.log("loginId", this.loginId, "nickname", this.nickName)
    }
    return {
        loginId,
        loginPwd,
        nickName,
        sayHello,
        id: Math.random()
    }
}
const u = createUser("abc", "123", "aaa");
u.sayHello();
```

**说明：** 当属性名与变量名相同时，可以直接写变量名，无需写成 `loginId: loginId` 的形式。

### 2. 方法速写

对象字面初始化时，方法可以省略冒号和function关键字。

**示例代码：**

```javascript
const prop1 = "name2";
const prop2 = "age2";
const prop3 = "sayHello2";

const user = {
    [prop1]: "姬成",
    [prop2]: 100,
    [prop3](){
        console.log(this[prop1], this[prop2])
    }
}

user[prop3]();
```

**说明：** 方法速写允许直接写 `methodName() {}` 而不是 `methodName: function() {}`。

### 3. 计算属性名

有的时候，初始化对象时，某些属性名可能来自于某个表达式的值，在ES6，可以使用中括号来表示该属性名是通过计算得到的。

**示例代码：**

```javascript
const prop1 = "name2";
const prop2 = "age2";

const user = {
    [prop1]: "姬成",
    [prop2]: 100
}
```

**说明：** 使用 `[表达式]` 的方式可以动态计算属性名，这在某些场景下非常有用。

## 5-2. Object的新增API

### 1. Object.is

用于判断两个数据是否相等，基本上跟严格相等（===）是一致的，除了以下两点：

1. NaN和NaN相等
2. +0和-0不相等

**示例代码：**

```javascript
console.log(NaN === NaN); // false
console.log(+0 === -0);  // true

console.log(Object.is(NaN, NaN)) // true
console.log(Object.is(+0, -0))   // false
```

**说明：** `Object.is()` 提供了一种更精确的相等性判断方式，特别处理了NaN和±0的特殊情况。

### 2. Object.assign

用于混合对象。

**示例代码：**

```javascript
const obj1 = {
    a: 123,
    b: 456,
    c: "abc"
}

const obj2 = {
    a: 789,
    d: "kkk"
}

// 将obj2的数据，覆盖到obj1，并且会对obj1产生改动，然后返回obj1
// const obj = Object.assign(obj1, obj2);

// 创建新对象，避免修改原对象
const obj = Object.assign({}, obj1, obj2);

console.log(obj)        // { a: 789, b: 456, c: "abc", d: "kkk" }
console.log(obj===obj1) // false
console.log(obj1)       // { a: 123, b: 456, c: "abc" }
console.log(obj2)       // { a: 789, d: "kkk" }
```

**说明：** `Object.assign()` 从右向左复制属性，后面的对象会覆盖前面的对象同名属性。常用于对象合并和创建对象副本。

### 3. Object.getOwnPropertyNames 的枚举顺序

Object.getOwnPropertyNames方法之前就存在，只不过，官方没有明确要求，对属性的顺序如何排序，如何排序，完全由浏览器厂商决定。

ES6规定了该方法返回的数组的排序方式如下：

- 先排数字，并按照升序排序
- 再排其他，按照书写顺序排序

**示例代码：**

```javascript
const obj = {
    d: 1,
    b: 2,
    a: 3,
    0: 6,
    5: 2,
    4: 1
}
const props = Object.getOwnPropertyNames(obj)
console.log(props) // ["0", "4", "5", "d", "b", "a"]
```

**说明：** ES6规范了属性枚举顺序，数字属性按升序排列，其他属性按书写顺序排列。

### 4. Object.setPrototypeOf

该函数用于设置某个对象的隐式原型。

**示例代码：**

```javascript
const obj1 = {
    a: 1
}

const obj2 = {
    b: 2
}

// 相当于：obj1.__proto__ = obj2
Object.setPrototypeOf(obj1, obj2)

console.log(obj1) // obj1现在可以访问obj2的属性b
```

**说明：** `Object.setPrototypeOf()` 可以动态设置对象的原型链，相当于修改对象的 `__proto__` 属性。

## 5-3. 面向对象简介

### 面向对象编程思想

面向对象：一种编程思想，跟具体的语言无关。

**对比面向过程：**

- **面向过程：** 思考的切入点是功能的步骤
- **面向对象：** 思考的切入点是对象的划分

### 经典案例：大象装冰箱

**面向过程的思路：**
1. 冰箱门打开
2. 大象装进去
3. 冰箱门关上

**面向对象的思路：**
划分出两个对象：大象、冰箱，冰箱有打开门、关闭门、装东西的功能。

**示例代码：**

```javascript
/**
 * 大象
 */
function Elephant() {

}

/**
 * 冰箱
 */
function Frige() {

}

Frige.prototype.openDoor = function () {
    console.log("冰箱门打开");
}

Frige.prototype.closeDoor = function () {
    console.log("冰箱门关上");
}

Frige.prototype.join = function(something){
    this.openDoor();
    //装东西
    console.log("装进去了：" + something.constructor.name);
    this.closeDoor();
}

// 面向过程的方式
// var frig = new Frige();
// frig.openDoor();
// var ele = new Elephant();
// frig.join(ele);
// frig.closeDoor();

// 面向对象的方式
var frig = new Frige();
frig.join(new Elephant());
```

**说明：** 面向对象的核心是将问题分解为各个对象，每个对象有自己的属性和行为，对象之间通过交互完成任务。这种方式更符合人类思维习惯，代码更易维护和扩展。
