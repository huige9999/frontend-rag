# 代理与反射

## 12-1. 属性描述符

Property Descriptor（属性描述符）是一个普通对象，用于描述一个属性的相关信息。

### 获取属性描述符

通过 `Object.getOwnPropertyDescriptor(对象, 属性名)` 可以得到一个对象的某个属性的属性描述符。

### 属性描述符的属性

- **value**：属性值
- **configurable**：该属性的描述符是否可以修改
- **enumerable**：该属性是否可以被枚举
- **writable**：该属性是否可以被重新赋值

`Object.getOwnPropertyDescriptors(对象)` 可以得到某个对象的所有属性描述符。

### 设置属性描述符

如果需要为某个对象添加属性时或修改属性时，配置其属性描述符，可以使用下面的代码：

```javascript
Object.defineProperty(对象, 属性名, 描述符);
Object.defineProperties(对象, 多个属性的描述符)
```

**示例：**

```javascript
const obj = {
    a: 1,
    b: 2
}

Object.defineProperties(obj, {
    a: {
        value: 3,
        configurable: false,
        enumerable: false,
        writable: false
    }
})

obj.a = 10;  // 由于 writable 为 false，赋值不会生效
console.log(obj);  // { a: 3, b: 2 }
```

### 存取器属性

属性描述符中，如果配置了 `get` 和 `set` 中的任何一个，则该属性不再是一个普通属性，而变成了存取器属性。

`get` 和 `set` 配置均为函数，如果一个属性是存取器属性：
- 读取该属性时，会运行 `get` 方法，将 `get` 方法得到的返回值作为属性值
- 给该属性赋值时，则会运行 `set` 方法

存取器属性最大的意义，在于可以控制属性的读取和赋值。

**示例：基本用法**

```javascript
const obj = {
    b: 2
}
Object.defineProperty(obj, "a", {
    get() {
        console.log("运行了属性a的get函数")
        return obj._a;
    },
    set(val){
        console.log("运行了属性a的set函数", val)
        obj._a = val;
    }
})

obj.a = 10;        // 输出: 运行了属性a的set函数 10
console.log(obj.a); // 输出: 运行了属性a的get函数, 然后输出 10
```

**示例：数据验证**

```javascript
obj = {
    name: "adsf"
}

Object.defineProperty(obj, "age", {
    get() {
        return obj._age;
    },
    set(val) {
        if (typeof val !== "number") {
            throw new TypeError("年龄必须是一个数字")
        }
        if (val < 0) {
            val = 0;
        } else if (val > 200) {
            val = 200;
        }
        obj._age = val;
    }
})

obj.age = 25;      // 正常设置
obj.age = "abc";   // 抛出 TypeError: 年龄必须是一个数字
```

**示例：双向绑定**

```javascript
const spanName = document.getElementById("name")
const spanAge = document.getElementById("age")

const user = {}

Object.defineProperties(user, {
    name: {
        get() {
            return spanName.innerText;
        },
        set(val) {
            spanName.innerText = val;
        }
    },
    age: {
        get() {
            return +spanAge.innerText;
        },
        set(val) {
            if (typeof val !== "number") {
                throw new TypeError("年龄必须是一个数字")
            }
            if (val < 0) {
                val = 0;
            } else if (val > 200) {
                val = 200;
            }
            spanAge.innerText = val;
        }
    }
})

// 现在 user 对象与页面元素实现了双向绑定
```

## 12-2. Reflect

### Reflect 是什么？

Reflect 是一个内置的 JS 对象，它提供了一系列方法，可以让开发者通过调用这些方法，访问一些 JS 底层功能。

由于它类似于其他语言的**反射**，因此取名为 Reflect。

### 它可以做什么？

使用 Reflect 可以实现诸如：
- 属性的赋值与取值
- 调用普通函数
- 调用构造函数
- 判断属性是否存在与对象中

等功能。

### 为什么需要 Reflect？

有一个重要的理念，在 ES5 就被提出：**减少魔法、让代码更加纯粹**。

这种理念很大程度上是受到函数式编程的影响。

ES6 进一步贯彻了这种理念，它认为：
- 对属性内存的控制
- 原型链的修改
- 函数的调用

等等这些都属于底层实现，属于一种魔法，因此需要将它们提取出来，形成一个正常的 API，并高度聚合到某个对象中，于是就造就了 Reflect 对象。

### Reflect 提供的 API

#### Reflect.set(target, propertyKey, value)

设置对象 target 的属性 propertyKey 的值为 value，等同于给对象的属性赋值。

```javascript
const obj = {
    a: 1,
    b: 2
}

// obj.a = 10;  // 传统写法
Reflect.set(obj, "a", 10);  // Reflect 写法
```

#### Reflect.get(target, propertyKey)

读取对象 target 的属性 propertyKey，等同于读取对象的属性值。

```javascript
console.log(Reflect.get(obj, "a"))  // 输出: 10
```

#### Reflect.apply(target, thisArgument, argumentsList)

调用一个指定的函数，并绑定 this 和参数列表。等同于函数调用。

```javascript
function method(a, b) {
    console.log("method", a, b);
}

// method(3, 4);  // 传统写法
Reflect.apply(method, null, [3, 4])  // Reflect 写法
```

#### Reflect.deleteProperty(target, propertyKey)

删除一个对象的属性。

```javascript
const obj = {
    a: 1,
    b: 2
}

// delete obj.a;  // 传统写法
Reflect.deleteProperty(obj, "a");  // Reflect 写法
console.log(obj);  // { b: 2 }
```

#### Reflect.defineProperty(target, propertyKey, attributes)

类似于 Object.defineProperty，不同的是如果配置出现问题，返回 false 而不是报错。

#### Reflect.construct(target, argumentsList)

用构造函数的方式创建一个对象。

```javascript
function Test(a, b) {
    this.a = a;
    this.b = b;
}

// const t = new Test(1, 3);  // 传统写法
const t = Reflect.construct(Test, [1, 3]);  // Reflect 写法
console.log(t)  // Test { a: 1, b: 3 }
```

#### Reflect.has(target, propertyKey)

判断一个对象是否拥有一个属性。

```javascript
const obj = {
    a: 1,
    b: 2
}

// console.log("a" in obj);  // 传统写法
console.log(Reflect.has(obj, "a"));  // Reflect 写法, 输出: true
```

## 12-3. Proxy 代理

代理：提供了修改底层实现的方式。

### 基本语法

```javascript
// 代理一个目标对象
// target：目标对象
// handler：是一个普通对象，其中可以重写底层实现
// 返回一个代理对象
new Proxy(target, handler)
```

### Proxy 的拦截器

handler 对象中可以定义多种拦截器方法：

#### set(target, propertyKey, value)

拦截属性赋值操作。

#### get(target, propertyKey)

拦截属性读取操作。

#### has(target, propertyKey)

拦截 `in` 操作符。

**示例：完整的 Proxy 代理**

```javascript
const obj = {
    a: 1,
    b: 2
}

const proxy = new Proxy(obj, {
    set(target, propertyKey, value) {
        console.log(`设置属性 ${propertyKey} = ${value}`);
        Reflect.set(target, propertyKey, value);
    },
    get(target, propertyKey) {
        if (Reflect.has(target, propertyKey)) {
            return Reflect.get(target, propertyKey);
        } else {
            return -1;  // 属性不存在时返回 -1
        }
    },
    has(target, propertyKey) {
        return false;  // 拦截 in 操作符，始终返回 false
    }
});

proxy.a = 10;          // 输出: 设置属性 a = 10
console.log(proxy.a);  // 输出: 10
console.log(proxy.d);  // 输出: -1 (属性不存在)
console.log("a" in proxy);  // 输出: false (被 has 拦截器拦截)
```

## 12-4. 应用 - 观察者模式

### 什么是观察者模式？

有一个对象，是观察者，它用于观察另外一个对象的属性值变化，当属性值变化后会收到一个通知，可能会做一些事。

### 实现方式 1：使用 Object.defineProperty

```javascript
// 创建一个观察者
function observer(target) {
    const div = document.getElementById("container");
    const ob = {};
    const props = Object.keys(target);
    
    for (const prop of props) {
        Object.defineProperty(ob, prop, {
            get() {
                return target[prop];
            },
            set(val) {
                target[prop] = val;
                render();  // 属性变化时重新渲染
            },
            enumerable: true
        })
    }
    
    render();

    function render() {
        let html = "";
        for (const prop of Object.keys(ob)) {
            html += `
                <p><span>${prop}：</span><span>${ob[prop]}</span></p>
            `;
        }
        div.innerHTML = html;
    }

    return ob;
}

const target = {
    a: 1,
    b: 2
}
const obj = observer(target)

// 修改 obj 的属性会自动更新页面显示
```

### 实现方式 2：使用 Proxy（推荐）

```javascript
// 创建一个观察者
function observer(target) {
    const div = document.getElementById("container");
    const proxy = new Proxy(target, {
        set(target, prop, value) {
            Reflect.set(target, prop, value);
            render();  // 属性变化时重新渲染
        },
        get(target, prop) {
            return Reflect.get(target, prop);
        }
    })
    
    render();

    function render() {
        let html = "";
        for (const prop of Object.keys(target)) {
            html += `
                <p><span>${prop}：</span><span>${target[prop]}</span></p>
            `;
        }
        div.innerHTML = html;
    }

    return proxy;
}

const target = {
    a: 1,
    b: 2
}
const obj = observer(target)

// 修改 obj 的属性会自动更新页面显示
```

使用 Proxy 的优势：
- 更加简洁
- 不需要遍历所有属性
- 动态添加的属性也能被监听

## 12-5. 应用 - 偷懒的构造函数

使用 Proxy 可以自动将构造函数的参数赋值给实例对象的属性。

```javascript
class User {
    // 不需要手动写 this.firstName = firstName 等
}

function ConstructorProxy(Class, ...propNames) {
    return new Proxy(Class, {
        construct(target, argumentsList) {
            const obj = Reflect.construct(target, argumentsList)
            propNames.forEach((name, i) => {
                obj[name] = argumentsList[i];
            })
            return obj;
        }
    })
}

const UserProxy = ConstructorProxy(User, "firstName", "lastName", "age")

const obj = new UserProxy("袁", "进", 18);
console.log(obj)  // User { firstName: '袁', lastName: '进', age: 18 }

// 可以复用于其他类
class Monster {

}

const MonsterProxy = ConstructorProxy(Monster, "attack", "defence", "hp", "rate", "name")

const m = new MonsterProxy(10, 20, 100, 30, "怪物")
console.log(m);  // Monster { attack: 10, defence: 20, hp: 100, rate: 30, name: '怪物' }
```

## 12-6. 应用 - 可验证的函数参数

使用 Proxy 可以在函数调用时自动验证参数类型。

### 实现方式 1：使用 Proxy

```javascript
function sum(a, b) {
    return a + b;
}

function validatorFunction(func, ...types) {
    const proxy = new Proxy(func, {
        apply(target, thisArgument, argumentsList) {
            types.forEach((t, i) => {
                const arg = argumentsList[i]
                if (typeof arg !== t) {
                    throw new TypeError(`第${i+1}个参数${argumentsList[i]}不满足类型${t}`);
                }
            })
            return Reflect.apply(target, thisArgument, argumentsList);
        }
    })
    return proxy;
}

const sumProxy = validatorFunction(sum, "number", "number")
console.log(sumProxy(1, 2))  // 输出: 3
console.log(sumProxy("1", 2))  // 抛出 TypeError: 第1个参数1不满足类型number
```

### 工作原理

1. `validatorFunction` 接收一个函数和期望的参数类型列表
2. 返回一个 Proxy 对象，拦截函数的调用
3. 当函数被调用时，先验证每个参数的类型
4. 如果验证通过，使用 `Reflect.apply` 调用原函数
5. 如果验证失败，抛出 TypeError

这种模式可以用于：
- 参数类型验证
- 参数范围验证
- 参数格式验证
- 函数调用的日志记录
- 函数性能监控
