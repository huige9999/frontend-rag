# 第13章 增强的数组功能

本章介绍ES6中新增的数组API以及扩展的类型化数组功能，帮助开发者更高效地处理数组数据和内存操作。

## 13.1 新增的数组API

ES6为Array构造函数和数组原型添加了许多实用的新方法，让数组操作更加便捷。

### 静态方法

#### Array.of(...args)

使用指定的数组项创建一个新数组，解决了`new Array()`在不同参数数量下的行为不一致问题。

```javascript
// 对比 new Array() 和 Array.of()
const arr1 = new Array(5);        // 创建长度为5的空数组 [empty × 5]
const arr2 = Array.of(5);         // 创建包含元素5的数组 [5]
const arr3 = Array.of(1, 2, 3);   // 创建数组 [1, 2, 3]

console.log(arr1); // [empty × 5]
console.log(arr2); // [5]
console.log(arr3); // [1, 2, 3]
```

#### Array.from(arg)

通过给定的类数组或可迭代对象创建一个新的数组。这是将类数组对象（如DOM集合）转换为真正数组的推荐方式。

```javascript
// 将DOM集合转换为数组
const divs = document.querySelectorAll("div");
const result = Array.from(divs);
console.log(result); // 真正的数组，可以使用所有数组方法

// 其他类数组对象的转换
const arrayLike = { 0: 'a', 1: 'b', 2: 'c', length: 3 };
const arr = Array.from(arrayLike);
console.log(arr); // ['a', 'b', 'c']

// 使用映射函数
const nums = Array.from([1, 2, 3], x => x * 2);
console.log(nums); // [2, 4, 6]
```

### 实例方法

#### find(callback)

用于查找满足条件的第一个元素，返回元素的值。

```javascript
const arr = [
    { name: "a", id: 1 },
    { name: "b", id: 2 },
    { name: "c", id: 3 },
    { name: "d", id: 4 },
    { name: "e", id: 5 }
];

// 找到id为5的对象
const result = arr.find(item => item.id === 5);
console.log(result); // { name: "e", id: 5 }

// 找到第一个年龄大于18的用户
const users = [
    { name: "张三", age: 17 },
    { name: "李四", age: 19 },
    { name: "王五", age: 20 }
];
const adult = users.find(user => user.age > 18);
console.log(adult); // { name: "李四", age: 19 }
```

#### findIndex(callback)

用于查找满足条件的第一个元素的下标，返回索引值。

```javascript
const arr = [
    { name: "a", id: 1 },
    { name: "b", id: 2 },
    { name: "c", id: 3 },
    { name: "d", id: 4 },
    { name: "e", id: 5 }
];

// 找到id为5的对象索引
const resultIndex = arr.findIndex(item => item.id === 5);
console.log(resultIndex); // 4

// 找不到时返回-1
const notFoundIndex = arr.findIndex(item => item.id === 100);
console.log(notFoundIndex); // -1
```

#### fill(data)

用指定的数据填充满数组所有的内容，可以指定起始和结束位置。

```javascript
// 创建长度为100的数组，每项填充"abc"
const arr = new Array(100);
arr.fill("abc");
console.log(arr); // ["abc", "abc", ..., "abc"] (100个)

// 指定填充范围
const arr2 = [1, 2, 3, 4, 5];
arr2.fill(0, 2, 4); // 从索引2开始，到索引4（不包含）结束
console.log(arr2); // [1, 2, 0, 0, 5]

// 常用于数组初始化
const matrix = new Array(5).fill(0);
console.log(matrix); // [0, 0, 0, 0, 0]
```

#### copyWithin(target, start?, end?)

在数组内部完成复制，将数组中某一段数据复制到另一位置。

```javascript
const arr = [1, 2, 3, 4, 5, 6];

// 从下标2开始，改变数组的数据，数据来自于下标0位置开始
arr.copyWithin(2);        // [1, 2, 1, 2, 3, 4]
// 解释：将索引0及之后的数据复制到索引2位置

// 从下标2开始，数据来自于下标1位置开始
arr.copyWithin(2, 1);     // [1, 2, 2, 3, 4, 5]
// 解释：将索引1及之后的数据复制到索引2位置

// 从下标2开始，数据来自于下标1到3（不包含）
arr.copyWithin(2, 1, 3);  // [1, 2, 2, 3, 5, 6]
// 解释：将索引1到3的数据复制到索引2位置
```

**参数说明：**
- `target`: 复制到的目标位置
- `start`: 可选，复制源起始位置，默认为0
- `end`: 可选，复制源结束位置（不包含），默认为数组长度

#### includes(data)

判断数组中是否包含某个值，使用`Object.is`进行匹配，可以正确检测`NaN`。

```javascript
const arr = [45, 21, 356, 66, 6, NaN, 723, 54];

// 使用indexOf的局限性
console.log(arr.indexOf(66) >= 0);      // true
console.log(arr.indexOf(NaN) >= 0);     // false - indexOf无法检测NaN

// includes可以正确检测NaN
console.log(arr.includes(66));          // true
console.log(arr.includes(NaN));         // true - includes可以检测NaN

// 检测字符串
const fruits = ['apple', 'banana', 'orange'];
console.log(fruits.includes('banana'));  // true
console.log(fruits.includes('grape'));   // false
```

**includes vs indexOf：**
- `includes`返回布尔值，`indexOf`返回索引
- `includes`可以正确检测`NaN`，`indexOf`不能
- `includes`更语义化，代码更易读

## 13.2 [扩展]类型化数组

类型化数组（Typed Arrays）是ES6引入的重要特性，用于优化多个数字的存储，特别适合处理大量数值数据。

### 数字存储的前置知识

在深入理解类型化数组之前，需要了解计算机如何存储数字：

**1. 固定位数存储**
计算机必须使用固定的位数来存储数字，无论数字大小，占用的内存空间是固定的。

**2. 无符号整数范围**
n位的无符号整数能表示的数字是2^n个，取值范围是：0 ~ 2^n - 1

**3. 有符号整数范围**
n位的有符号整数能表示的数字是2^n个，取值范围是：-2^(n-1) ~ 2^(n-1) - 1

**4. 浮点数表示法**
用于表示整数和小数，目前分为两种标准：
- **32位浮点数（单精度）**：1位符号 + 8位阶码 + 23位尾数
- **64位浮点数（双精度）**：1位符号 + 11位阶码 + 52位尾数

**5. JavaScript的数字存储**
JS中的所有数字，均使用双精度浮点数保存。

### 类型化数组类型

类型化数组提供多种类型，可以根据数据特点选择最合适的类型：

| 类型 | 描述 | 字节数 | 取值范围 |
|------|------|--------|----------|
| Int8Array | 8位有符号整数 | 1 | -128 ~ 127 |
| Uint8Array | 8位无符号整数 | 1 | 0 ~ 255 |
| Int16Array | 16位有符号整数 | 2 | -32768 ~ 32767 |
| Uint16Array | 16位无符号整数 | 2 | 0 ~ 65535 |
| Int32Array | 32位有符号整数 | 4 | -2^31 ~ 2^31-1 |
| Uint32Array | 32位无符号整数 | 4 | 0 ~ 2^32-1 |
| Float32Array | 32位浮点数 | 4 | IEEE 754标准 |
| Float64Array | 64位浮点数 | 8 | IEEE 754标准 |

### 创建类型化数组

```javascript
// 方式1：指定长度创建
const arr1 = new Int32Array(10);  // 创建长度为10的32位有符号整数数组
console.log(arr1.length);         // 10
console.log(arr1.byteLength);     // 40 (10个元素，每个4字节)

// 方式2：使用of方法创建
const arr2 = Uint8Array.of(12, 5, 6, 7);
console.log(arr2);                // Uint8Array [12, 5, 6, 7]
console.log(arr2.byteLength);     // 4 (每个元素1字节)

// 方式3：from方法从可迭代对象创建
const arr3 = Int16Array.from([100, 200, 300]);
console.log(arr3);                // Int16Array [100, 200, 300]

// 方式4：从其他类型化数组创建
const arr4 = Int32Array.of(35111, 7, 3, 11);
const arr5 = new Int8Array(arr4);  // 数据会被转换和截断
console.log(arr4 === arr5);        // false - 不是同一个数组
console.log(arr5);                 // Int8Array [-57, 7, 3, 11]
// 注意：35111超出Int8范围，被截断为-57
```

### 类型化数组的属性和方法

```javascript
const arr = Int8Array.of(125, 7, 3, 11);

// 基本属性
console.log(arr.length);      // 4 - 元素数量
console.log(arr.byteLength);  // 4 - 占用字节数

// 数组访问
console.log(arr[0]);          // 125
arr[1] = 100;                 // 修改元素
console.log(arr);             // Int8Array [125, 100, 3, 11]

// 遍历
for (const item of arr) {
    console.log(item);        // 依次输出：125, 100, 3, 11
}

// 使用数组方法（返回同类型的新数组）
const arr2 = arr.map(item => item * 2);
console.log(arr2);            // Int8Array [-6, -56, 6, 22]
// 注意：125*2=250超出Int8范围，变为-6（溢出）
```

### 类型化数组的特点和限制

```javascript
const arr = Int8Array.of(125, 7, 3, 11);

// 1. 不能增加数据 - 长度固定
arr[4] = 1000;  // 无效 - 不报错但也不生效
console.log(arr.length);  // 仍然是4

// 2. 不能删除数据
delete arr[0];  // 无效
console.log(arr);  // Int8Array [125, 7, 3, 11]

// 3. 方法返回同类型数组
const mapped = arr.map(x => x * 2);
console.log(mapped instanceof Int8Array);  // true
console.log(mapped);  // Int8Array [-6, 14, 6, 22]

// 4. 数值溢出会自动处理
const bigNum = Int8Array.of(127);
bigNum[0] = 128;  // 128超出范围
console.log(bigNum[0]);  // -128（溢出循环）
```

**类型化数组的优势：**
- 内存占用更小，存储效率更高
- 适合处理大量数值数据（如图像处理、科学计算）
- 支持与WebGL、二进制文件等底层API交互

**使用场景：**
- Canvas图像处理
- WebGL图形渲染
- 二进制数据解析
- 音视频数据处理
- 大量数值计算

## 13.3 [扩展]ArrayBuffer

ArrayBuffer是一个用于存储固定大小内存数据的对象，它是类型化数组和DataView的基础。

### ArrayBuffer基础

```javascript
// 创建指定字节数的内存空间
const bf = new ArrayBuffer(10);  // 创建10个字节的内存空间

// 获取字节数
console.log(bf.byteLength);  // 10

// 切片创建新的ArrayBuffer
const bf2 = bf.slice(3, 5);  // 从第3字节开始，到第5字节（不包含）
console.log(bf2.byteLength);  // 2
```

### 读写ArrayBuffer的两种方式

#### 方式1：使用DataView

DataView提供了一个与平台无关的数据视图，适合需要混用多种存储格式的场景。

```javascript
const bf = new ArrayBuffer(10);  // 10个字节的内存空间

// 创建DataView，可以指定起始位置和长度
const view = new DataView(bf, 3, 4);  // 从第3字节开始，长度4字节

// 写入数据
view.setInt16(1, 3);  // 在偏移量1的位置写入16位整数3

// 读取数据
const value = view.getInt16(1);
console.log(value);  // 3

// DataView的优势：可以控制字节序
const view2 = new DataView(bf);
view2.setInt16(0, 0x1234, true);   // 小端序
view2.setInt16(2, 0x1234, false);  // 大端序
```

**DataView的特点：**
- 可以精确控制读写位置和数据类型
- 支持设置字节序（大端序/小端序）
- 适合处理复杂数据结构
- API相对繁琐，性能略低于类型化数组

#### 方式2：使用类型化数组

类型化数组是对ArrayBuffer的视图，提供了更简洁的API，是更常用的方式。

```javascript
const bf = new ArrayBuffer(10);  // 10个字节的内存空间

// 创建不同的类型化数组视图
const arr1 = new Int8Array(bf);   // 每个元素1字节，长度10
const arr2 = new Int16Array(bf);  // 每个元素2字节，长度5

// 它们共享同一块内存
console.log(arr1 === arr2);           // false - 不是同一个数组对象
console.log(arr1.buffer === arr2.buffer);  // true - 共享同一个ArrayBuffer

// 通过一个数组修改会影响另一个
arr1[0] = 10;  // 修改第一个字节
console.log(arr1);  // Int8Array [10, 0, 0, 0, 0, 0, 0, 0, 0, 0]
console.log(arr2);  // Int16Array [10, 0, 0, 0, 0]
// arr2[0] = 10，因为前两个字节组成一个16位整数
```

### 类型化数组与ArrayBuffer的关系

```javascript
// 如果没有手动指定ArrayBuffer，类型化数组会自动创建
const arr = new Int16Array(5);  // 创建长度为5的16位整数数组
console.log(arr.buffer.byteLength);  // 10 (5个元素 × 2字节)

// 手动指定ArrayBuffer
const bf = new ArrayBuffer(8);
const arr = new Int16Array(bf);
arr[0] = 2344;  // 操作了前两个字节
console.log(arr);  // Int16Array [2344, 0, 0, 0]
```

### 内存共享示例

```javascript
const bf = new ArrayBuffer(6);  // 6字节内存

// 创建不同的视图
const int8View = new Int8Array(bf);
const int16View = new Int16Array(bf);

// 通过Int8Array写入
int8View[0] = 1;    // 第1字节
int8View[1] = 2;    // 第2字节
int8View[2] = 3;    // 第3字节
int8View[3] = 4;    // 第4字节

console.log(int8View);   // Int8Array [1, 2, 3, 4, 0, 0]
console.log(int16View);  // Int16Array [513, 1024, 0]
// 解释：513 = 2*256 + 1（小端序）
//       1024 = 4*256 + 3
```

**小端序（Little-Endian）：**
- 低位字节存储在低地址
- 在上面的例子中，字节1和2组成16位整数：2（高字节）和1（低字节）
- 计算方式：高字节 × 256 + 低字节 = 2 × 256 + 1 = 513

### 实际应用场景

类型化数组和ArrayBuffer主要用于：

1. **二进制数据处理**
   - 文件解析（如图片、音频、视频文件）
   - 网络协议数据包处理
   - 二进制文件格式读写

2. **高性能计算**
   - 大量数值计算（科学计算、图形处理）
   - 游戏开发中的物理引擎
   - WebGL图形渲染

3. **与其他系统交互**
   - WebGL着色器数据传输
   - WebAssembly内存交互
   - 硬件设备通信

这些特性使JavaScript能够处理更复杂的底层操作，突破了传统JavaScript在数据处理上的限制。
