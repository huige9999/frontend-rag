# 03-15 获取实时信息part3

## 课程概述

本课程介绍了如何让大模型获取实时信息的解决方案，主要通过工具调用(Function Calling)的方式实现。课程内容涵盖了大模型工具调用的基本原理、提示词工程以及具体的代码实现。

## 核心概念

### 工具调用原理
大模型不会直接使用工具，而是返回【我要使用 XXX 工具】的信息，然后由系统解析并执行相应的工具调用，最后将结果返回给大模型生成最终答案。

### 提示词方案
早期采用提示词的方式来实现工具调用，通过精心设计的提示词让大模型判断是否需要调用工具以及调用哪个工具。

// 对接百度翻译API
async function translate({ input }) {
  // 学习阶段，仅支持中文翻译成英文
  const text = input.replace(/.*翻译.*?[：:]?\s*/, "").trim();
  const salt = Date.now();
  const sign = crypto
    .createHash("md5")
    .update(ID + text + salt + KEY)
    .digest("hex");

  const url = `https://fanyi-api.baidu.com/api/trans/vip/translate?q=${encodeURIComponent(
    text
  )}&from=zh&to=en&appid=${ID}&salt=${salt}&sign=${sign}`;

  const res = await fetch(url);
  const data = await res.json();
  
  if (data.trans_result?.length > 0) {
    return `翻译结果：${data.trans_result[0].dst}`;
  } else {
    return `翻译失败：${JSON.stringify(data)}`;
  }
}
```

// 天气查询核心函数
async function getWeather({ city, date }) {
  const formattedDate = formatDate(date);
  if (!formattedDate) {
    return `无法识别日期格式："${date}"，请使用"今天"、"明天"或"后天"`;
  }

  const locationId = await getCityLocation(city);
  if (!locationId) {
    return `无法识别城市："${city}"`;
  }

  const url = `https://devapi.qweather.com/v7/weather/7d?location=${locationId}&key=${HEFENG_API_KEY}`;
  const res = await fetch(url);
  const data = await res.json();

  const match = data.daily.find((d) => d.fxDate === formattedDate);
  if (!match) {
    return `暂无 ${formattedDate} 的天气数据`;
  }

  return `📍 ${city}（${formattedDate}）天气：${match.textDay}，气温 ${match.tempMin}°C ~ ${match.tempMax}°C`;
}
```

### 3. 提示词模板 (promptTemplates.js)

#### 工具调用判断提示词
```javascript
function buildFunctionCallPrompt(userInput) {
  return `
    你是一个中文智能助手，具有工具调用能力。请严格按照以下规则回复：
    
    1. 如果不需要调用函数工具，请直接返回字符串："无函数调用"
    2. 如果需要调用函数，请返回标准JSON数组格式
    
    目前你支持的函数列表如下：
    - getWeather：获取城市的天气信息
      * city: 城市名称（必须是中文，如"北京"、"上海"、"成都"）
      * date: 日期（必须是中文，只能是："今天"、"明天"、"后天"）
    - translate：用于中英文互译
      * input: 需要翻译的文本
      
    用户输入：【${userInput}】
  `.trim();
}
```

#### 工具结果回答提示词
```javascript
function buildAnswerPrompt(userInput, results) {
  const toolsText = results
    .map((item) => {
      const { function: fn, args, result } = item;
      return `函数：${fn}\n参数：${JSON.stringify(args)}\n返回结果：${result}`;
    })
    .join("\n\n");

  return `
    你是一个友好的中文智能助手，已经执行了相关函数调用，以下是函数返回结果：
    
    ${toolsText}
    
    请基于上述工具结果，严格使用中文自然语言回答用户的问题。
    
    用户原始输入是：【${userInput}】
  `.trim();
}
```

### 4. 路由处理逻辑 (routes/index.js)

```javascript
router.post("/ask", async (req, res) => {
  const question = req.body.question || "";
  
  // 1. 判断是否需要调用工具
  const functionCallPrompt = buildFunctionCallPrompt(question);
  const functionCallResult = await callLLM(functionCallPrompt);
  
  let finalResponse = "";
  
  if (functionCallResult.trim() === "无函数调用") {
    // 常规对话
    const prompt = [
      "你是一个中文智能助手，请严格使用中文来回答用户的问题",
      ...conversations.map(
        (item) => `${item.role === "user" ? "用户" : "助手"}：${item.content}`
      ),
      `用户的问题：${question}`,
    ].join("\n");
    
    finalResponse = await callLLMStream(prompt, (chunk) => {
      res.write(`${JSON.stringify({ response: chunk })}\n`);
    });
  } else {
    // 2. 解析并执行工具调用
    const toolCalls = JSON.parse(functionCallResult);
    const toolsResult = [];
    
    for (const tool of toolCalls) {
      const { function: functionName, args } = tool;
      if (toolsMap[functionName]) {
        const result = await toolsMap[functionName](args);
        toolsResult.push({
          function: functionName,
          args,
          result,
        });
      }
    }
    
    // 3. 基于工具结果生成最终答案
    const answerPrompt = buildAnswerPrompt(question, toolsResult);
    finalResponse = await callLLMStream(answerPrompt, (chunk) => {
      res.write(`${JSON.stringify({ response: chunk })}\n`);
    });
  }
  
  // 4. 保存会话历史
  conversations.push(
    { role: "user", content: question },
    { role: "assistant", content: finalResponse }
  );
  
  res.end();
});
```

## 技术要点

### 1. 工具调用的完整流程
1. 用户提问 → 构建工具判断提示词
2. LLM判断是否需要调用工具 → 返回JSON格式的工具调用信息
3. 系统解析JSON → 执行对应的工具函数
4. 将工具结果构建成回答提示词 → LLM生成最终答案
5. 返回答案给用户并保存会话历史

### 2. 提示词工程要点
- 明确指定返回格式（JSON数组）
- 详细说明每个工具的参数要求
- 强调中文使用和日期格式限制
- 提供正确和错误的格式示例

### 3. 错误处理
- 工具不存在时的处理
- 工具调用失败的处理
- JSON解析错误的处理
- API调用失败的友好提示

### 4. 会话管理
- 使用数组存储会话历史
- 限制历史记录数量（最多20条）
- 支持清空会话记录
- 支持查看会话历史
