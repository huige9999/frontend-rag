# 嵌入

## 课程概述

本课程介绍嵌入（Embedding）技术，这是一种寻找词和词之间相似性的NLP技术，通过将词汇表示为数值向量来捕捉语义信息。

## 核心概念

### 什么是嵌入？

嵌入（Embedding）是一种将词汇表示为数值向量的NLP技术：
- **文本嵌入**：将文本转换为向量表示
- **文本向量化**：与嵌入是同一概念
- **核心价值**：通过向量在各个维度上的特征相似度来判断语义接近程度

### 嵌入的基本原理

#### 从词元到向量

1. **分词处理**：文本经过Tokenizer变成词元ID列表

示例："我喜欢小猫和小狗"

| 词语 | 词元ID |
| ---- | ------ |
| 我   | 1001   |
| 喜欢 | 1002   |
| 小猫 | 2001   |
| 和   | 1003   |
| 小狗 | 2002   |

2. **嵌入矩阵**：为每个词元嵌入高维向量信息
   - 向量维度通常为成百上千维
   - 每个维度代表一个语义方向

#### 简化的4维嵌入示例

| 向量维度 | 语义方向（隐含） | 小猫的值 | 粗略含义   |
| -------- | ---------------- | -------- | ---------- |
| 第1维    | 是否是动物？     | 0.72     | 是动物     |
| 第2维    | 亲和力/可爱度    | 0.35     | 稍微可爱   |
| 第3维    | 体型大小（抽象） | 0.11     | 比较小     |
| 第4维    | 家养 vs 野外     | 0.80     | 更偏向家养 |

#### 嵌入向量的表示

| Token ID | 词语 | 嵌入向量（4维）          |
| -------- | ---- | ------------------------ |
| 2001     | 小猫 | [0.72, 0.35, 0.11, 0.80] |
| 2002     | 小狗 | [0.70, 0.38, 0.14, 0.78] |
| 1002     | 喜欢 | [0.10, 0.93, 0.21, 0.11] |

### 嵌入的优势

| 对比项   | 小猫 🐱                   | 小狗 🐶                   |
| -------- | ------------------------ | ------------------------ |
| 词元ID   | 2001                     | 2002                     |
| 嵌入向量 | [0.72, 0.35, 0.11, 0.80] | [0.70, 0.38, 0.14, 0.78] |
| 结论     | 语义相似的动物           | 语义相似的动物           |

**嵌入带来的4大好处**：

1. **传递语义相似性**：相似词向量靠近
2. **支持上下文学习**：嵌入可更新
3. **可参与数学运算**：词向量支持加减
4. **输入给神经网络**：向量可计算梯度

### 向量空间可视化

在二维空间投影后，相似词会靠得更近：
- "小猫"和"小狗"等语义相近词靠得更近
- "喜欢"等语义不同的词离得稍远
- "讨厌""桌子"等不相关词离得比较远

## Word2Vec算法

### 基本介绍

- **开发时间**：2013年
- **开发者**：托马斯·米克洛夫和Google团队
- **核心特点**：将词汇表中的每个词表示成固定长度的向量
- **优势**：使在大规模数据集上训练变得可行

### 两种训练方式

#### 1. CBOW模型（Continuous Bag-of-Words）

**核心思想**：给定上下文（前后词），预测中心词（目标词）

**示例**：
```
我 喜欢 吃 苹果
```

输入上下文（"我"，"吃"）→ 预测中心词："喜欢"

#### 2. Skip-Gram模型

**核心思想**：给定中心词，预测上下文词

**示例**：
```
我 喜欢 吃 苹果
```

输入中心词："喜欢" → 预测上下文词："我"、"吃"

### 两种模型对比

| 比较项       | CBOW                         | Skip-Gram                  |
| ------------ | ---------------------------- | -------------------------- |
| 训练目标     | 预测中心词                   | 预测上下文                 |
| 输入         | 上下文词（多个）             | 中心词（一个）             |
| 输出         | 中心词（一个）               | 上下文词（多个）           |
| 训练速度     | 快                           | 慢                         |
| 对稀有词表现 | 一般                         | 更好                       |
| 语义表达能力 | 差一些                       | 更强                       |
| 适合场景     | 大词频、高效率、短上下文窗口 | 稀有词、多语义任务、大语料 |

### 代码实现

#### 训练Word2Vec模型

```python
from gensim.models import Word2Vec

# 示例语料
sentences = [
    ["i", "love", "deep", "learning"],
    ["i", "love", "nlp"],
    ["deep", "learning", "is", "fun"],
    ["nlp", "is", "a", "part", "of", "ai"],
    ["word2vec", "is", "a", "powerful", "embedding"]
]

# 训练模型
# sg=1 表示使用 Skip-Gram 模型
# sg=0 表示使用 CBOW 模型
model = Word2Vec(sentences, vector_size=100, window=2, min_count=1, sg=1)

# 保存模型
model.save("word2vec.model")
print("模型训练完成并已保存。")
```

#### 测试Word2Vec模型

```python
from gensim.models import Word2Vec

# 加载模型
model = Word2Vec.load("word2vec.model")

# 获取某个词的词向量
print("词向量（nlp）：")
print(model.wv["nlp"])

# 找出最相似的词
print("\n与 'nlp' 最相似的词：")
print(model.wv.most_similar("nlp", topn=3))

# 计算两个词的相似度
print("\n'deep' 和 'learning' 的相似度：")
print(model.wv.similarity("deep", "learning"))
```

### 关键参数说明

- `sg`：训练算法选择（0=CBOW，1=Skip-Gram）
- `vector_size`：向量维度大小
- `window`：上下文窗口大小
- `min_count`：最小词频阈值

## 应用场景

1. **语义搜索**：基于语义相似度的文档检索
2. **文本分类**：将文本转换为向量进行分类
3. **推荐系统**：基于内容的推荐
4. **问答系统**：理解问题与答案的语义关系
5. **机器翻译**：捕捉不同语言间的语义对应

## 小结

嵌入技术通过将词汇转换为数值向量，使计算机能够理解和处理文本的语义信息。Word2Vec作为经典的嵌入算法，提供了CBOW和Skip-Gram两种训练方式，各有优势。嵌入技术是现代NLP和LLM的基础，为后续的深度学习模型提供了重要的语义表示能力。
