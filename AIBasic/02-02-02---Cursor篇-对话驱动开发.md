# [Cursor篇] 对话驱动开发

## 课程概述

传统的软件开发模式包括：
- 测试驱动开发（TDD）
- 行为驱动开发（BDD）
- 领域驱动开发（DDD）
- 事件驱动开发（EDD）

AI 的出现使其出现了新的开发模式：**对话驱动开发**

## 两种对话驱动开发模式

### PDD（Prompt-Driven Development）：提示词驱动开发
- 开发者编写提示词，AI 根据提示生成代码
- 开发者保持强控制权，AI 作为工具使用
- 核心能力是编写清晰、高质量的提示词
- 适用于局部代码生成（函数、接口、配置）

**示例：**
> 帮我写一个注册的接口，带邮箱和密码的验证。

### AI-DD（AI-Driven Development）：AI 驱动开发
- AI 根据上下文、环境、目标主动参与编程
- 控制权部分转移给 AI，AI 成为合作者
- 重点是构建 AI 协作环境（持续对话、自动分析）
- 全局参与（项目结构建议、模块拆分、重构建议）

**示例：**
> 该项目是否有可以优化的地方？

### 两种模式对比

| 维度             | PDD（Prompt-Driven）                       | AI-DD（AI-Driven）                                           |
| ---------------- | ------------------------------------------ | ------------------------------------------------------------ |
| **驱动方式**     | 开发者写 prompt，AI 根据提示生成代码       | AI 根据上下文、环境、目标主动参与编程                        |
| **控制权**       | **开发者强控制**：AI 是工具                | **控制权部分转移给 AI：AI 是合作者**                         |
| **主要手段**     | 编写清晰、高质量 prompt 是核心能力         | 构建 AI 协作环境（如持续对话、自动分析）是重点               |
| **输出范围**     | 局部代码生成（函数、接口、配置）           | 全局参与（项目结构建议、模块拆分、重构建议）                 |
| **依赖模型能力** | 通常是调用 GPT、Claude 等接口              | 可能集成多个模型 + 插件 + 工具链                             |
| **典型工具**     | Copilot Chat、Cursor Prompt Block、CodeGPT | Cursor Max Mode、Code Interpreter、AutoDev 工具链、GPT Agents |

## Cursor 中的对话模式

Cursor 提供了两种主要的对话模式：

- **Ask 模式**：PDD 驱动
- **Agent 模式**：AI 驱动

## ASK 模式

Ask 模式允许你通过 AI 搜索与提问的方式来探索和了解代码库，但**不会对代码做出任何修改**。

### 特点
- **只读模式（read-only）**
- 拥有完整的读取权限，可以查看项目文件和上下文
- 不会自动执行 AI 提出的任何更改建议
- 默认启用搜索工具，可以更方便地定位和读取项目中的文件内容

### 适用场景
- 提问代码相关的问题
- 探索项目结构和逻辑
- 学习代码库的实现方式
- 阅读和理解陌生代码库
- 和 AI 一起制定解决方案，但暂时不动代码

### 配置选项

| 配置项                            | 说明                                                         |
| --------------------------------- | ------------------------------------------------------------ |
| **Model（模型）**                 | 可以预设一个默认使用的 AI 模型（如 GPT-4 或 Claude）         |
| **Keybinding（快捷键）**          | 设置一个快捷键，便于随时切换到 Ask 模式                      |
| **Search Codebase（代码库搜索）** | 启用后，Cursor 可以自动搜索**项目中的文件作为上下文**，无需手动 `@ 文件`添加上下文 |

## Agent 模式

Agent 是一个具备高度**自主性**的 AI 编码助手，能够独立探索、规划并执行复杂的代码库变更任务，并拥有全套开发工具的支持。

### 核心特点
- **高度自主性**：能够独立完成任务，减少用户引导
- **全套工具支持**：拥有读取、编辑、执行命令等完整开发权限
- **自动修改**：会自动更改文件，但需要用户确认是否接受更改

### 核心能力

在 Agent 模式下，AI 可以自主完成以下操作：
- 浏览和理解整个代码库结构
- 读取项目文档和说明
- 搜索网页（例如查找第三方库用法）
- 编辑文件内容
- 执行终端命令（如安装依赖、运行脚本）

### 工作流程

1. **理解需求**：Agent 分析任务描述和项目上下文，理解目标
2. **探索代码库**：搜索相关文件、文档，必要时访问网页
3. **制定修改方案**：将任务拆解为若干步骤，制定可执行的修改计划
4. **执行修改**：根据计划修改文件，可能建议引入新依赖或执行命令
5. **验证结果**：检查改动是否符合预期，尝试自动修复错误
6. **任务完成**：总结修改内容，帮助了解变更点

### 配置选项

- **选择模型**：为 Agent 模式选择默认的大语言模型
- **编辑快捷键**：设置或修改快捷键，方便快速切换功能
- **启用自动运行与自动修复**：自动执行修改计划，并在检测到错误时自动尝试修复

## Chat 窗口操作

### 回滚和撤销

在 Chat 窗口中，右下角有一个 **Restore 按钮**，可以回滚到之前的版本。

在回滚之后，还可以使用撤销按钮，撤销回滚操作。

## 实战示例

### 示例代码：哈夫曼树与冒泡排序

```javascript
// 哈夫曼树节点定义
class HuffmanNode {
  constructor(char, freq, left = null, right = null) {
    this.char = char; // 字符
    this.freq = freq; // 频率
    this.left = left; // 左子节点
    this.right = right; // 右子节点
  }
}

// 构建哈夫曼树
function buildHuffmanTree(charFreqs) {
  // charFreqs: [{char: 'a', freq: 5}, ...]
  const nodes = charFreqs.map(({ char, freq }) => new HuffmanNode(char, freq));

  while (nodes.length > 1) {
    // 按频率升序排序
    nodes.sort((a, b) => a.freq - b.freq);
    // 取出最小的两个节点
    const left = nodes.shift();
    const right = nodes.shift();
    // 合并成新节点
    const newNode = new HuffmanNode(null, left.freq + right.freq, left, right);
    nodes.push(newNode);
  }
  return nodes[0]; // 返回根节点
}

// 生成哈夫曼编码
function generateHuffmanCodes(root) {
  const codes = {};
  function traverse(node, code) {
    if (!node) return;
    if (node.char !== null) {
      codes[node.char] = code;
      return;
    }
    traverse(node.left, code + '0');
    traverse(node.right, code + '1');
  }
  traverse(root, '');
  return codes;
}

// 示例用法
const charFreqs = [
  { char: 'a', freq: 5 },
  { char: 'b', freq: 9 },
  { char: 'c', freq: 12 },
  { char: 'd', freq: 13 },
  { char: 'e', freq: 16 },
  { char: 'f', freq: 45 },
];

const huffmanTree = buildHuffmanTree(charFreqs);
const codes = generateHuffmanCodes(huffmanTree);
console.log('哈夫曼编码:', codes);

// 冒泡排序实现
function bubbleSort(arr) {
  const n = arr.length;
  for (let i = 0; i < n - 1; i++) {
    for (let j = 0; j < n - 1 - i; j++) {
      if (arr[j] > arr[j + 1]) {
        // 交换
        [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
      }
    }
  }
  return arr;
}

// 冒泡排序示例
const arr = [5, 2, 9, 1, 5, 6];
console.log('冒泡排序前:', arr);
console.log('冒泡排序后:', bubbleSort([...arr]));
```

## 学习要点

1. **理解对话驱动开发的概念**：PDD 和 AI-DD 的区别与应用场景
2. **掌握 Cursor 的两种模式**：Ask 模式用于理解代码，Agent 模式用于自动修改
3. **学会有效使用 Agent**：让 AI 帮助完成复杂的编码任务
4. **熟悉 Chat 窗口操作**：掌握回滚和撤销功能，保护代码安全

## 适用场景

- **Ask 模式**：学习新代码库、理解项目结构、制定解决方案
- **Agent 模式**：重构代码、添加新功能、修复 Bug、优化性能
