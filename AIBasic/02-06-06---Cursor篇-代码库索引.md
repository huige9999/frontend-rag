# 02-06-06 Cursor篇-代码库索引

## 概述

代码库索引是 Cursor 的核心功能，通过为项目创建嵌入向量（Embeddings），帮助 AI 更准确地理解项目结构和代码。

## 核心特性

代码库索引的工作原理：

- **向量化处理**：为代码文件创建嵌入向量
- **云端存储**：索引数据存储在云端
- **本地保护**：代码本身只在本地存储
- **安全保障**：确保项目安全性

## 主要优势

### 传统开发 vs 代码库索引

**传统开发**：
- 开发者需要记住整个项目结构
- 手动搜索文件和代码
- 难以理解跨文件关系

**使用代码库索引后**：
- **AI 回答更准确**：基于当前项目的索引库回答问题
- **文件搜索更快**：快速定位相关代码
- **跨文件理解**：AI 能更好地理解文件间的关系
- **重构建议更精准**：提供更准确的代码重构建议

## 最佳实践

### 1. 定期检查索引状态
确保索引保持最新状态，及时发现索引问题。

### 2. 合理设置忽略规则
通过 `.cursorignore` 文件配置，避免索引无关文件。

#### 示例项目结构
```
/project
  /frontend
  /backend
  /docs
  /scripts
```

#### 配置示例
```
/backend
/docs
/scripts
*.md
node_modules
```

### 3. 大型项目模块化索引
对于大型项目，考虑按模块进行索引管理：
- 前端项目单独索引
- 后端项目单独索引
- 文档和脚本可选择性索引

### 4. 团队协作规范
- 团队统一 `.cursorignore` 配置
- 建立索引管理规范
- 定期同步索引策略

### 5. 快速理解项目
利用代码库索引功能快速：
- 了解新项目结构
- 定位关键代码
- 理解模块关系

### 典型项目应用

**个人博客前台管理系统**（Vue 项目）：
- 技术栈：Vue.js
- 开发环境：完整的前端开发配置
- 索引策略：排除构建产物和配置文件，专注于源代码

## 工作流程

1. **项目初始化**：首次打开项目时，Cursor 会自动创建索引
2. **持续更新**：代码修改后，索引自动更新
3. **智能查询**：AI 基于索引提供准确的代码建议
4. **优化调整**：根据项目需要调整索引策略

## 注意事项

- 确保网络连接稳定，索引需要云端处理
- 大型项目首次索引可能需要较长时间
- 定期清理 `.cursorignore` 中不需要的规则
- 敏感信息会通过忽略规则自动排除

## 总结

代码库索引是 Cursor 提升开发效率的核心功能，通过向量化技术和云端存储，为 AI 提供了深入理解项目的能力，使代码开发更加智能和高效。
