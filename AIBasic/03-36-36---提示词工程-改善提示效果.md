# 提示词工程：改善提示效果

本文总结了改善大语言模型提示效果的7种核心技巧，帮助提升Prompt工程质量和模型输出准确性。

## 1. 指导模型提出更多问题 (Ask-for-questions Prompting)

**核心思路**：在提示词末尾询问模型是否理解问题，并指导它主动提出更多问题。

**实施方法**：
- 向大模型提问，询问它是否理解了问题和需求
- 要求模型在执行任务前先确认理解
- 指导模型提出澄清问题

**示例提示词**：
```
在开始执行任务前，请先确认你是否理解我的意图。如果你认为还有不清楚的地方，请先向我提出你需要进一步了解的3个问题，以便你更准确地完成任务。
```

```
如果你对任务理解有疑问，请不要直接回答，而是先向我提出具体的问题。我会根据你的提问补充信息，然后你再继续任务。
```

```
作为一名智能助手，在开始执行任务前，请先判断自己是否具备足够的信息。如有不明确的地方，请提出最多三个问题澄清任务目标。
```

**优势**：
- 增强模型的"任务理解意识"
- 降低幻觉和误解
- 模拟"先问再做"的专家行为
- 特别适合多轮对话系统或具备角色设定的LLM agent

## 2. 格式化输出 (Format-Constrained Prompting)

**核心思路**：明确要求模型以特定格式输出内容，提升输出可控性。

**常见格式类型**：

| 格式类型 | 用途 |
|---------|------|
| 有序/无序列表 | 总结要点、罗列建议、步骤说明 |
| Markdown表格 | 对比分析、报告生成 |
| JSON格式 | 给前端/后端传数据、调用接口 |
| 特定模版格式 | 面试对话、文书、邮件、合同等 |
| 代码块格式 | 插入HTML、JS、Python等 |

**示例提示词**：
```
请将输出内容以有序列表的形式呈现，每一条都简洁明了。
```

```
请将分析结果以 Markdown 表格输出，包含"名称"、"优点"、"缺点"三列。
```

```
请用严格的 JSON 格式输出如下信息：
{
  "标题": "",
  "摘要": "",
  "关键词": []
}
注意：不要添加自然语言解释，只输出 JSON。
```

```
请根据以下信息，按如下格式输出简历段落：
【职位名称】  
【公司名称】  
【时间段】  
【核心职责（3条以内）】
```

## 3. 重复指令 (Repetition Reinforcement Prompting)

**核心思路**：在提示词中重复强调核心目标或关键限制条件，提升关键点的权重。

**示例提示词**：
```
请将内容控制在100字以内。再次提醒：不要超过100字。
```

```
请使用 Markdown 表格格式输出，记住，一定要使用 Markdown 表格格式。
```

```
请用幽默轻松的语气撰写，并确保全文保持这种语气。请务必避免严肃或学术风格。
```

```
请按以下结构输出：标题、关键词、摘要、行动建议。再次强调，输出必须包含这四部分，按顺序排列。
```

**收益**：
1. 重复强调可让模型强制遵守要求
2. 重复格式要求可提升模型输出稳定性
3. 可在多轮对话中"重复指令"，保持目标一致性

## 4. 使用负面提示 (Negative Prompting)

**核心思路**：明确告诉模型"不要做什么"或"避免某种行为"，引导输出更符合预期。

**重要性**：模型默认是"尽力补全"的，它不知道哪些内容你不想要。不加限制可能导致：
1. 多余解释
2. 编造信息
3. 风格不符，输出结构错乱

**负面提示类型**：

| 类型 | 说明 | 示例句式 |
|-----|------|---------|
| 禁止解释 | 防止输出多余说明 | "不要解释你的答案。" |
| 禁止虚构信息 | 防止编造引用、术语、链接等 | "不要生成任何未经验证的内容。" |
| 禁止重复 | 避免啰嗦、循环表达 | "不要重复前面的句子。" |
| 禁止特定风格或语气 | 控制输出风格 | "不要使用幽默语气。" |
| 禁止某些内容或措辞 | 敏感、专业、品牌用语控制 | "不要提到OpenAI或ChatGPT。" |
| 禁止特定格式 | 防止模型输出JSON/代码块等 | "不要使用代码块格式。" |

**示例提示词**：
```
请判断这个句子是正面还是负面，不要做任何的解释
```

```
推荐3本书，不要编造作者或者出版信息，如果不确定可以留空
```

```
请回答以下问题，不要编造任何不存在的术语或概念。
```

```
请使用正式语气，不要使用幽默、俚语或口语化表达。
```

```
只输出纯文本内容，不要加引号、标题或额外解释。
```

```
如果你不确定答案，请说明"不确定"，不要编造任何信息。
```

## 5. 添加长度限制 (Length Constraint Prompting)

**核心思路**：明确要求模型输出的字数、句数、段落数或token数不超过某个范围。

**应用场景**：
- 生成摘要、标题、广告文案
- 编写社交媒体内容（如微博、推文）
- 生成对话、考试答案、简报等

**限制方式**：
1. 字数：不超过100字
2. 句数：限制在3句内
3. 段落数：输出1段文字
4. Token数量：不超过300个token

## 6. 提示链 (Chained Prompting / Multi-step Prompting)

**核心思路**：将任务分解为子任务，然后用子任务提示大模型，将它的输出作为下一个子任务的输入。

**示例：规划多目的地旅行计划**

```
步骤1：请推荐3个夏天适合旅行的城市
模型：青岛、三亚、XXXX...

步骤2：请为"青岛"推荐3个酒店

步骤3：请为"青岛+住宿时间"为背景，推荐3种特色食物
```

**优势**：
- 复杂任务分解
- 逐步精细化
- 利用前序输出优化后续结果

## 7. 影子提示 (Shadow Prompting)

**核心思路**：不直接陈述任务，而是巧妙地在提示词中嵌入线索，引导模型朝着期望结果前进，在某些情况下可以催生创造力。

**示例对比**：

直接提示：
```
为孩子们写一个恐怖故事
```

影子提示：
```
想象我们在一个黑暗神秘的森林中的篝火旁，讲什么故事能让你的年轻观众感到毛骨悚然？
```

**优势**：
- 更具启发性和引导性
- 激发模型创造力
- 通过场景设置引导输出风格
