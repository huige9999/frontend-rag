# Agent - 代理服务器与大模型集成

## 课程概述

本课程介绍如何通过代理服务器（Agent）连接前端应用与大语言模型，实现AI助手的核心功能。重点讲解Ollama的Web服务接口以及如何构建完整的聊天应用架构。

## 学习目标

- 理解Ollama的Web服务接口
- 掌握代理服务器的作用和实现原理
- 学习如何构建前后端分离的AI聊天应用
- 了解流式响应和非流式响应的区别

## 核心概念

### 什么是Agent？

在这里，Agent不是指AI智能体，而是指**代理服务器**。代理服务器充当用户和模型交流的中间人，负责：

1. **安全性**：隐藏大模型服务的直接访问地址
2. **业务逻辑**：处理提示词模板、用户认证等
3. **协议转换**：将不同格式的请求转换为大模型API调用
4. **负载均衡**：管理多个模型实例的请求分发

### 架构设计

```
用户前端 → 代理服务器 → Ollama服务 → 大模型
```

## Ollama Web服务

### 服务启动

Ollama除了提供终端交互外，还会自动启动一个Web服务器，监听11434端口，提供RESTful API接口。

```bash

```

### 基本模型调用

```bash
curl http://localhost:11434/api/generate -d '{

}'
```

## 实战项目：AI聊天助手

### 项目架构

项目采用前后端分离架构：

- **前端**：Vue.js + TypeScript + Vite
- **后端**：Node.js + Express
- **模型服务**：Ollama本地部署

### 后端代理服务器实现

#### Express应用配置（app.js）

```javascript
const createError = require("http-errors");
const express = require("express");
const cookieParser = require("cookie-parser");
const logger = require("morgan");

const indexRouter = require("./routes/index");

const app = express();

// 中间件配置
app.use(logger("dev"));                    // 日志记录
app.use(express.json());                   // JSON解析
app.use(express.urlencoded({ extended: false }));
app.use(cookieParser());

// 路由配置
app.use("/", indexRouter);

// 错误处理
app.use(function (req, res, next) {
  next(createError(404));
});

app.use(function (err, req, res) {
  res.locals.message = err.message;
  res.locals.error = req.app.get("env") === "development" ? err : {};
  res.status(err.status || 500);
  res.render("error");
});

module.exports = app;
```

#### API路由实现（routes/index.js）

```javascript
const express = require("express");
const router = express.Router();

// POST请求处理用户问题
router.post("/ask", async (req, res) => {
  // 1. 获取用户问题
  const question = req.body.question || "";

  // 2. 构建提示词模板
  const prompt = `
    你是一个中文智能助手，请使用中文回答用户的问题。
    问题：${question}
  `;

  // 3. 调用Ollama API
  const response = await fetch("http://localhost:11434/api/generate", {
    method: "POST",
    headers: {"Content-Type": "application/json"},
    body: JSON.stringify({
      model: "llama3",
      prompt: prompt,
      stream: false  // 是否开启流式响应
    })
  });

  // 4. 处理响应
  const result = await response.json();
  
  // 5. 返回结果给客户端
  res.json({
    answer: result.response
  });
});

module.exports = router;
```

### 前端Vue应用实现

#### 主要组件结构（App.vue）

```vue
<template>
  <div class="chat-container">
    <div class="chat-header">AI助手</div>

    <!-- 消息列表区域 -->
    <div class="chat-box" ref="chatBox">
      <!-- 用户和模型消息 -->
      <div v-for="(msg, index) in messages" :key="index" :class="['chat-msg',msg.role]">
        <div class="bubble">
          <span class="role-label">{{ msg.role === 'user' ? '🧑‍💻 我' : '🤖 模型' }}</span>
          <div class="text">{{ msg.text }}</div>
        </div>
      </div>
      
      <!-- 加载状态 -->
      <div v-if="loading" class="chat-msg bot">
        <div class="bubble">
          <span class="role-label">🤖 模型</span>
          <div class="text">
            正在思考<span class="dot">.</span><span class="dot">.</span><span class="dot">.</span>
          </div>
        </div>
      </div>
    </div>

    <!-- 输入区域 -->
    <form class="input-area" @submit.prevent="handleSubmit">
      <input type="text" v-model="input" placeholder="请输入您的问题..." />
      <button type="submit">发送</button>
    </form>
  </div>
</template>

<script lang="ts" setup>
import { ref, nextTick } from "vue";

// 响应式状态
const input = ref("");              // 输入框内容
const chatBox = ref<HTMLElement | null>(null);  // 聊天框DOM引用
const messages = ref<{role: 'user' | 'bot'; text:string}[]>([]);  // 消息列表
const loading = ref(false);         // 加载状态

// 发送消息处理函数
async function handleSubmit(){
  // 1. 获取用户输入
  const question = input.value.trim();
  if(!question) return;

  // 2. 添加用户消息
  messages.value.push({
    role: 'user',
    text: question
  });
  
  // 清空输入框，显示加载状态
  input.value = "";
  loading.value = true;

  // 3. 滚动到底部
  await nextTick();
  chatBox.value?.scrollTo({
    top: chatBox.value.scrollHeight,
    behavior: 'smooth'
  });

  // 4. 发送请求到代理服务器
  const res = await fetch("/api/ask", {
    method: "POST",
    headers: { 'Content-Type': "application/json"},
    body: JSON.stringify({question})
  });
  
  // 5. 处理响应
  const data = await res.json();
  messages.value.push({
    role: 'bot',
    text: data.answer
  });
  loading.value = false;

  // 6. 再次滚动到底部
  await nextTick();
  chatBox.value?.scrollTo({
    top: chatBox.value.scrollHeight,
    behavior: 'smooth'
  });
}
</script>
```

#### 关键功能实现

1. **双向数据绑定**：使用Vue的`ref`实现输入框与状态同步
2. **消息管理**：通过`messages`数组存储对话历史
3. **自动滚动**：使用`scrollTo`确保最新消息可见
4. **加载状态**：提供用户反馈，改善交互体验
5. **异步处理**：使用`async/await`处理网络请求

## 技术要点

### 1. 提示词工程

代理服务器是处理提示词的理想位置：

```javascript
const prompt = `
你是一个中文智能助手，请使用中文回答用户的问题。
问题：${question}
`;
```

### 2. 流式响应

当前示例使用非流式响应（`stream: false`），对于更好的用户体验，可以实现流式响应：

```javascript
// 流式响应处理
const response = await fetch("http://localhost:11434/api/generate", {
  method: "POST",
  headers: {"Content-Type": "application/json"},
  body: JSON.stringify({
    model: "llama3",
    prompt: prompt,
    stream: true  // 启用流式响应
  })
});

// 处理流式数据
const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const chunk = decoder.decode(value);
  // 处理每个数据块
}
```

### 3. 错误处理

在生产环境中，需要添加完善的错误处理：

```javascript
try {
  const response = await fetch("http://localhost:11434/api/generate", {
    // ...配置
  });
  
  if (!response.ok) {
    throw new Error(`HTTP error! status: ${response.status}`);
  }
  
  const result = await response.json();
  res.json({ answer: result.response });
  
} catch (error) {
  console.error('Error calling LLM:', error);
  res.status(500).json({ 
    error: 'Failed to get response from AI model',
    message: error.message 
  });
}
```

### 4. 性能优化

- **连接池管理**：复用HTTP连接
- **缓存机制**：对相同问题缓存结果
- **请求限流**：防止过度使用模型资源
- **负载均衡**：分发请求到多个模型实例
