# RAG实践 Part2 - 智能客服系统实现

## 课程概述

本课程基于RAG（检索增强生成）技术，实现一个能够根据外挂知识库回答用户问题的智能客服系统。系统集成了向量嵌入、相似度计算和LLM调用等核心技术。

- **Node.js + Express**: 服务器框架
- **pdf-parse**: PDF文档解析

- **流式响应**: SSE (Server-Sent Events)

### 前端技术
- **Vue 3**: 前端框架
- **TypeScript**: 类型支持
- **Vite**: 构建工具

## 核心架构

### 1. 数据索引阶段

数据索引是将外挂知识库转换为向量形式的过程：

```
PDF文档 → 文本分割 → 向量嵌入 → 存储
```

**关键实现**：

```javascript
// PDF解析和向量生成
async function generateEmbeddings() {
  // 1. 加载外挂知识库
  const buffer = fs.readFileSync("./香蕉手机参数配置.pdf");
  const data = await pdfParse(buffer);

  // 2. 文档分割处理
  const paragraphs = data.text
    .split(/\n\s*\n/)
    .map((text, idx) => ({
      id: `chunk-${idx}`,
      content: text.trim(),
    }))
    .filter((p) => p.content.length > 20);

  // 3. 向量转换
  const withEmbedding = [];
  for (const p of paragraphs) {
    const embedding = await getEmbedding(p.content);
    withEmbedding.push({
      ...p,
      embedding,
    });
  }

  // 4. 存储到文件（企业中应使用向量数据库）
  fs.writeFileSync(
    EMBEDDING_PATH,
    JSON.stringify(withEmbedding, null, 2),
    "utf8"
  );

  return withEmbedding;
}
```

**向量嵌入函数**：

```javascript
// 使用本地nomic-embed-text模型生成向量
async function getEmbedding(text) {
  const res = await fetch("http://localhost:11434/api/embeddings", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      model: "nomic-embed-text",
      prompt: text,
    }),
  });
  const result = await res.json();
  return result.embedding;
}
```

### 2. 数据查询阶段

查询阶段包括向量检索和相似度计算：

```
用户问题 → 向量化 → 相似度计算 → 阈值判断 → 内容生成
```

**余弦相似度计算**：

```javascript
/**
 * 余弦相似度计算
 * @param {*} vecA 向量A
 * @param {*} vecB 向量B
 * @returns 相似度分数 (0-1之间)
 */
function cosineSimilarity(vecA, vecB) {
  const dot = vecA.reduce((sum, val, i) => sum + val * vecB[i], 0);
  const normA = Math.sqrt(vecA.reduce((sum, val) => sum + val * val, 0));
  const normB = Math.sqrt(vecB.reduce((sum, val) => sum + val * val, 0));
  return dot / (normA * normB);
}
```

**向量检索函数**：

```javascript
/**
 * 基于嵌入向量的搜索
 * @param {*} query 用户问题
 * @param {*} embeddedDocs 外挂知识库向量
 * @param {*} topK 返回前K个结果
 */
async function searchByEmbedding(query, embeddedDocs, topK = 3) {
  // 1. 将用户问题转为向量
  const queryEmbedding = await getEmbedding(query);

  // 2. 计算与知识库的相似度
  const scored = embeddedDocs.map((chunk) => {
    const score = cosineSimilarity(queryEmbedding, chunk.embedding);
    return {
      ...chunk,
      score,
    };
  });

  // 3. 返回相似度最高的topK条结果
  return scored.sort((a, b) => b.score - a.score).slice(0, topK);
}
```

## 核心功能实现

### 1. LLM调用模块

**超时机制的fetch封装**：

```javascript
async function fetchWithTimeout(url, options = {}, timeout = LLM_TIMEOUT_MS) {
  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), timeout);
  
  try {
    const response = await fetch(url, {
      ...options,
      signal: controller.signal,
    });
    clearTimeout(timeoutId);
    return response;
  } catch (err) {
    clearTimeout(timeoutId);
    if (err.name === "AbortError") 
      throw new Error("请求超时，请稍候重试");
    throw err;
  }
}
```

**流式响应处理**：

```javascript
async function callLLM(messages, tools = null, callback) {
  const requestBody = {
    model: LLM_MODEL,
    messages,
    stream: true,
  };

  if (tools) requestBody.tools = tools;

  const response = await fetchWithTimeout(LLM_ENDPOINT, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: `Bearer ${process.env.DEEPSEEK_API_KEY}`,
    },
    body: JSON.stringify(requestBody),
  });

  const reader = response.body.getReader();
  const decoder = new TextDecoder("utf-8");

  let fullResponse = "";
  let toolCalls = [];

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value, { stream: true });
    const lines = chunk.split("\n").filter((line) => line.trim());

    for (const line of lines) {
      if (!line.startsWith("data: ")) continue;
      
      const jsonStr = line.slice(6);
      if (jsonStr === "[DONE]") continue;

      try {
        const data = JSON.parse(jsonStr);
        const delta = data.choices?.[0]?.delta;

        if (delta) {
          if (delta.content) {
            fullResponse += delta.content;
            callback?.(delta.content);
          }

          if (delta.tool_calls) {
            // 处理工具调用
            for (const toolCall of delta.tool_calls) {
              const existingCall = toolCalls.find(
                (call) => call.index === toolCall.index
              );

              if (existingCall) {
                // 合并工具调用参数
                if (toolCall.function?.name) {
                  existingCall.function.name = toolCall.function.name;
                }
                if (toolCall.function?.arguments) {
                  existingCall.function.arguments += toolCall.function.arguments;
                }
              } else {
                // 新的工具调用
                toolCalls.push({
                  index: toolCall.index,
                  id: toolCall.id,
                  type: "function",
                  function: {
                    name: toolCall.function?.name,
                    arguments: toolCall.function?.arguments,
                  },
                });
              }
            }
          }
        }
      } catch (e) {
        console.error("JSON解析失败", e.message);
      }
    }
  }

  if (toolCalls.length > 0)
    return { content: fullResponse, tool_calls: toolCalls };

  return fullResponse;
}
```

### 2. 主要路由处理

**问答接口**：

```javascript
router.post("/ask", async (req, res) => {
  // 设置SSE流响应
  res.setHeader("Content-Type", "text/event-stream");
  res.setHeader("Cache-Control", "no-cache");

  // 1. 加载知识库向量
  const embeddedDocs = await loadCachedEmbeddings();
  
  // 2. 获取用户问题
  const question = req.body.question || "";
  
  // 3. 向量检索
  const relevantDocs = await searchByEmbedding(question, embeddedDocs);
  
  // 4. 阈值判断和消息构建
  let userMessage = question;
  const allDocsRelevant =
    relevantDocs.length > 0 &&
    relevantDocs.every((doc) => doc.score > RELEVANCE_THRESHOLD);

  if (allDocsRelevant) {
    const relevantContent = relevantDocs
      .filter((doc) => doc.score > RELEVANCE_THRESHOLD)
      .map((doc) => doc.content)
      .join("\n\n");

    userMessage = `参考以下资料回答问题：

${relevantContent}

问题：${question}`;
  }

  // 5. 构建会话消息
  const messages = [...conversations, { role: "user", content: userMessage }];

  // 6. 调用LLM
  const response = await callLLM(messages, toolList, (chunk) => {
    res.write(`${JSON.stringify({ response: chunk })}\n`);
  });

  // 7. 处理工具调用（如果有）
  if (response.tool_calls) {
    // 工具调用逻辑
    const toolResults = [];
    for (const toolCall of response.tool_calls) {
      const functionName = toolCall.function.name;
      const args = JSON.parse(toolCall.function.arguments);

      if (toolsMap[functionName]) {
        const result = await toolsMap[functionName](args);
        toolResults.push({
          tool_call_id: toolCall.id,
          role: "tool",
          content: result,
        });
      }
    }

    // 二次调用LLM获取最终回答
    messages.push(
      { role: "assistant", content: response.content, tool_calls: response.tool_calls },
      ...toolResults
    );
    
    const finalResponse = await callLLM(messages, toolList, (chunk) => {
      res.write(`${JSON.stringify({ response: chunk })}\n`);
    });

    // 保存会话历史
    conversations.push(
      { role: "user", content: question },
      { role: "assistant", content: response.content, tool_calls: response.tool_calls },
      ...toolResults,
      { role: "assistant", content: finalResponse }
    );
  } else {
    // 无工具调用，直接保存会话
    conversations.push(
      { role: "user", content: question },
      { role: "assistant", content: response }
    );
  }

  // 限制会话历史长度
  if (conversations.length > 20)
    conversations.splice(0, conversations.length - 20);

  res.end();
});
```

## 常见问题

### Q: 如何调整相似度阈值？
A: 修改`routes/index.js`中的`RELEVANCE_THRESHOLD`常量，数值越高判断越严格。

### Q: 支持哪些文档格式？
A: 当前主要支持PDF格式，可通过修改`generateEmbeddings`函数扩展其他格式。

### Q: 如何处理超大文档？
A: 建议对文档进行预处理，分章节存储，或使用专业的向量数据库。

### Q: 系统响应慢怎么办？
A: 检查本地模型是否正常运行，考虑使用GPU加速，或调整向量缓存策略。
