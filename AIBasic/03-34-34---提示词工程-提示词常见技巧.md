# 提示词常见技巧

## 学习目标

- 掌握少样本学习（Few-shot Learning）的应用
- 理解逐步思考（Chain of Thought）的原理
- 学会运用N-shot Learning技巧提升提示效果

## 少样本学习

### 概念定义

少样本学习（Few-shot Learning），也称之为少样本提示，这是一种范式，在这种范式中，模型通过提示词获得你希望模型执行的任务示例。

### 基本原理

给大模型提供2～5条任务-结果对（上下文），接着给大模型一个新的任务，让大模型模仿前面的例子给出答案。

### 核心优势

1. **快速适应新任务** - 无需重新训练模型
2. **灵活应对多种格式** - 适应不同的输出要求
3. **提高准确性/稳定性** - 减少模型输出波动
4. **更可控的输出结构** - 确保输出格式符合预期

### 应用示例

#### 1. 情感分析

```
例子：
评论：这个产品真不错，用得很顺手！ → 情感：正面
评论：完全失望，质量太差了。 → 情感：负面
评论：快递到了。 → 情感：中性

评论：外观还行，性能一般般。 → 情感：
```

#### 2. 命名实体识别

```
例子：
句子：马云是阿里巴巴的创始人。  
实体：[马云: 人名], [阿里巴巴: 组织]

句子：乔布斯出生在旧金山。  
实体：[乔布斯: 人名], [旧金山: 地点]

句子：李雷明天去北京出差。  
实体：
```

#### 3. 结构化数据转换

```
例子：
姓名：张三，年龄：28，职业：工程师  
→ {"name": "张三", "age": 28, "job": "工程师"}

姓名：李四，年龄：35，职业：设计师  
→ {"name": "李四", "age": 35, "job": "设计师"}

姓名：王五，年龄：40，职业：产品经理  
→
```

## 逐步思考

### 问题背景

> 369 * 1235 是多少？

早期的大模型是无法回答这个问题的，给出的答案千奇百怪。

### 为什么大模型不能直接解决这个问题？

因为大模型是从左到右，一个词元一个词元的去预测，每一个数字都是预测出来的，而非计算出来的。

### 解决方案

**逐步思考**：不是让大模型直接给出答案，而是通过提示词让大模型进行**中间推理**。

#### 实施方法

在提示词之后添加"Let's think step by step（让我们一步一步思考）"

#### 理论基础

这种策略被称之为 **零样本思维链策略（Zero-Shot Chain of Thought Strategy）**，最早由 Kojima 等人于 2022 年发表在一篇名为"Large Language Models Are Zero-Shot Reasoners"的科学论文中。

- **零样本**：不提供任何的样本
- **思维链**：让大模型一步一步进行推理，然后得到答案

### 应用示例

#### 1. 数学计算

```
Q: 小明有 3 个苹果，小红给了他 2 个苹果，然后他吃掉了 1 个。问他现在有几个苹果？
请一步步思考。
```

#### 2. 逻辑推理

```
Q: 如果所有猫都会爬树，汤姆是猫，那么汤姆会爬树吗？
请一步步思考。
```

#### 3. 时间推理

```
Q: 今天是星期三，那么 10 天后是星期几？
请一步步思考。
```

#### 4. 复杂计算

```
Q: 小王有 4 支红笔和 5 支蓝笔，他给了朋友 2 支蓝笔和 1 支红笔。现在他还有几支笔？
请一步步思考。
```

#### 5. 条件推理

```
Q: 如果下雨了，地会湿；如果地湿了，孩子们就不能出去玩。现在下雨了。那么孩子们能出去玩吗？
请一步步思考。
```

### 常用提示词模板

1. 请一步一步思考
2. 请解释你的推理过程
3. 请详细说明你是如何得到这个答案的

## N-shot Learning 体系

### 学习方式对比

#### 少样本学习（Few-shot Learning）
给一些少量的样本，大模型会基于你所给的样本，给出问题的答案

#### 零样本学习（Zero-shot Learning）
完全不给样本，让大模型自己一步一步思考，得出答案

#### 单样本学习（One-shot Learning）
给一个样本，然后询问大模型问题

### One-shot Learning 示例

```
Prompt:
例子：
Input: "Good morning" → Output: "早上好"

请翻译：
Input: "Thank you" → Output:
```

## 总结

### 核心要点

1. **少样本学习**：通过提供示例来引导模型输出，提高准确性和可控性
2. **逐步思考**：通过中间推理步骤，提升模型在复杂任务上的表现
3. **N-shot Learning**：根据任务需求选择合适的样本数量策略

### 实践建议

- 对于需要特定输出格式的任务，使用少样本学习
- 对于复杂推理任务，使用逐步思考策略
- 灵活结合不同技巧，发挥最佳效果
