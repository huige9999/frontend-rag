# 项目收官

### 首页

**1. 最近浏览的文档**

这一块儿需要更新数据表，例如在用户表中新增一个 recentDocs 字段

```js
{
  _id: "userId123",
  username: "xiejie",
  recentDocs: [
    { docId: "abc123", visitedAt: ISODate("2025-05-04T05:00:00Z") },
    { docId: "xyz456", visitedAt: ISODate("2025-05-03T22:00:00Z") },
    ...
  ]
}
```

之后每次用户打开文档页面时（例如 `/doc/[id]`），触发如下逻辑：

```js
// 后端伪代码
async function updateRecentDocs(userId, docId) {
  const user = await User.findById(userId);
  const updated = user.recentDocs.filter(d => d.docId !== docId); // 去重
  updated.unshift({ docId, visitedAt: new Date() }); // 插入最前
  user.recentDocs = updated.slice(0, 20); // 最多保留20条
  await user.save();
}

```

之后添加相应的接口

```js
// GET /api/user/recent-docs
const user = await User.findById(userId);
const docIds = user.recentDocs.map(d => d.docId);
const docs = await DocModel.find({ _id: { $in: docIds } });

// 按访问顺序返回（可加排序）
return docs.sort(...); // 可根据 visitedAt 排序
```



**2. 我收藏的文档**

实现思路基本和上面类似，在用户表中新增 `starredDocs` 字段

```js
{
  _id: "userId123",
  username: "xiejie",
  recentDocs: [
    { docId: "abc123", visitedAt: ISODate("2025-05-04T05:00:00Z") },
    { docId: "xyz456", visitedAt: ISODate("2025-05-03T22:00:00Z") },
    ...
  ]
  starredDocs: [
    { docId: "abc123", addedAt: ISODate("2025-05-01T05:00:00Z") },
    { docId: "xyz456", addedAt: ISODate("2025-04-28T12:00:00Z") },
    ...
  ]
}
```

当用户点击"收藏"按钮时，后端需要处理文档的添加逻辑：

```js
// 后端伪代码
async function addToStarredDocs(userId, docId) {
  const user = await User.findById(userId);
  
  // 防止重复收藏，先去重
  const updated = user.starredDocs.filter(d => d.docId !== docId);
  updated.unshift({ docId, addedAt: new Date() }); // 插入最前
  user.starredDocs = updated.slice(0, 20); // 限制最多保留20条
  await user.save();
}
```

当用户点击"移除收藏"时，后端需要处理删除的逻辑：

```js
// 后端伪代码
async function removeFromStarredDocs(userId, docId) {
  const user = await User.findById(userId);
  
  // 删除文档
  user.starredDocs = user.starredDocs.filter(d => d.docId !== docId);
  await user.save();
}
```

为了展示用户收藏的文档，可以增加以下接口：

```js
// GET /api/user/starred-docs
const user = await User.findById(userId);
const docIds = user.starredDocs.map(d => d.docId);
const docs = await DocModel.find({ _id: { $in: docIds } });

// 按添加时间排序（或其他排序方式）
return docs.sort((a, b) => b.addedAt - a.addedAt); // 按添加时间降序
```



**3. 我创建的文档**

```js
const mongoose = require("mongoose");

// 定义对应的 Schema
const documentSchema = new mongoose.Schema(
  {
    id: String, // mongodb 自动生成的 id
    title: String, // 文档标题
    desc: String, // 文档概述
    content: String, // 文档内容
    createTime: String, // 创建时间
    updateTime: String, // 更新时间
    collaborators: [
      {
        type: mongoose.Schema.Types.ObjectId,
        ref: "userModel",
      },
    ],

    // 可选：文档创建者（单个用户）
    owner: {
      type: mongoose.Schema.Types.ObjectId,
      ref: "userModel",
    },
  },
 	// ...
);
// ...
```

也就是说，文档表中每一条文档记录都有对应的创建者 id，我们只需要搜索文档创建者 id 和当前用户 id 对应的文档记录即可。



**4. 我协作的文档**

我协作的文档，指的是：

>用户出现在 `document.collaborators` 中，但不是 `document.owner`

因此应该执行如下的查询：

```js
DocumentModel.find({
  collaborators: userId, // 匹配当前用户是协作者
  owner: { $ne: userId }, // 排除用户是文档创建者的情况（避免与"我创建的文档"重复）
});

```

示例代码：

```js
// routes/doc.js 或 controllers/document.js
const DocumentModel = require("../model/documentModel");

/**
 * 获取当前用户协作的文档列表
 */
router.get("/api/docs/collaborated", async (req, res) => {
  const userId = req.user._id; // 从 JWT 中解析出的用户 ID

  try {
    const docs = await DocumentModel.find({
      collaborators: userId,
      owner: { $ne: userId },
    })
      .populate("owner", "username avatar") // 也可展示作者信息
      .sort({ updateTime: -1 });

    res.json({
      code: 0,
      message: "",
      data: docs,
    });
  } catch (err) {
    res.status(500).json({ code: 1, message: "获取协作文档失败", error: err.message });
  }
});
```



### 团队

团队这一趴分为：

1. 团队成员
2. 团队文档
3. 邀请成员

"团队成员"功能模块的核心目标是：**展示某个团队下的所有成员信息，并支持邀请、移除、查看成员状态等操作**。目前在我们的后端服务器中，其实没有针对团队的数据模型定义，这里需要新增加一个：

```js
// models/teamModel.js
const mongoose = require("mongoose");

const teamSchema = new mongoose.Schema(
  {
    name: String, // 团队名称
    description: String, // 描述
    owner: { type: mongoose.Schema.Types.ObjectId, ref: "userModel" }, // 团队创建者
    members: [{ type: mongoose.Schema.Types.ObjectId, ref: "userModel" }], // 所有成员
    createTime: String,
  },
  {
    versionKey: false,
  }
);

mongoose.model("teamModel", teamSchema, "teams");
module.exports = mongoose.model("teamModel");
```

有了上面的数据模型设计后，团队模块的核心功能都变得很简单了。



**展示团队成员列表**

接口：`GET /api/team/:teamId`

```js
TeamModel.findById(teamId)
  .populate("members", "username avatar") // 获取成员头像/昵称
  .then(team => {
    res.json({ code: 0, data: team.members });
  });

```



**邀请成员加入团队**

接口：`POST /api/team/:teamId`

```js
const { teamId } = req.params;
const { inviteeUserId } = req.body;

const team = await TeamModel.findById(teamId);

// 防止重复邀请
if (!team.members.includes(inviteeUserId)) {
  team.members.push(inviteeUserId);
  await team.save();
}
```



**移除成员**

接口：`DELETE /api/team/:teamId`

```js
team.members = team.members.filter(id => id.toString() !== targetUserId);
await team.save();
```



**获取某个用户所在的所有团队**

```js
TeamModel.find({ members: userId })
```



当然，有了团队模块之后，增加文档协作者那一块儿的逻辑也需要有变化，比如【必须是一个团队才能成为文档协作者】等，不过这还是看你怎么设计。



### 回收站

同样需要调整数据模型，添加一个 **逻辑** 删除字段 `isDeleted` 和可选的 `deletedAt`

```js
const documentSchema = new mongoose.Schema(
  {
    ...
    isDeleted: {
      type: Boolean,
      default: false,
    },
    deletedAt: {
      type: String, // ISO 时间字符串
      default: null,
    },
  },
  { versionKey: false }
);
```

这里使用"逻辑删除"，而不是直接 `remove()` 真正删除文档，这样用户才能查看回收站、恢复文档等。



有了这个字段后，实现我的回收站功能就很简单了，直接获取当前用户**逻辑删除**的所有文档

接口：`GET /api/docs/recycle`

```js
const docs = await DocumentModel.find({
  owner: userId,       // 当前用户是创建者
  isDeleted: true,     // 已删除
});
```



而清空回收站则是 **彻底物理删除** 用户删除的文档。这一次就会真真正正的从数据库移除记录。

接口：`DELETE /api/docs/recycle/clear`

```js
await DocumentModel.deleteMany({
  owner: userId,
  isDeleted: true,
});
```

注意：清空操作应有确认弹窗或二次确认提示。
