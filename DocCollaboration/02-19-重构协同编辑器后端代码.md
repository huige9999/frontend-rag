# 重构协同编辑器后端代码

> 本节课主要通过代码实现来学习如何重构协同编辑器的后端代码，使用 Node.js + Socket.io 搭建 WebSocket 服务器，配合 Yjs 实现多房间文档同步和 Awareness 状态广播。后端负责管理每个房间的 Y.Doc 副本，转发增量更新和协作者状态信息。

## 关键代码

### server.js - WebSocket 协同服务器

后端服务器的核心逻辑，基于 Socket.io 构建。主要功能包括：
- 使用 `docs` Map 管理多个房间的文档和 Awareness 状态
- 新客户端加入房间时，推送完整的文档状态（sync-doc）
- 接收客户端的增量更新，应用到服务器端 Y.Doc 副本，并广播给房间内其他客户端
- 转发 Awareness 光标状态信息
- 处理客户端断开连接

```js
// 服务器文件
const http = require("http"); // 引入 node.js 原生的 http 模块
const { Server } = require("socket.io");
const Y = require("yjs");
const { Awareness } = require("y-protocols/awareness");

// 创建一个 http 服务器
const server = http.createServer();
// 基于 http 服务器创建 websocket 服务器
const io = new Server(server, {
  cors: "*",
});

// 这个 map 用于存储每个房间对应的文档和光标状态信息
// 类似于 { doc, awareness }
// 每个方法会独立的管理自己的文档和光标状态
const docs = new Map();

// 当有客户端连接到当前的 websocket 服务器的时候，会触发该事件
io.on("connection", (socket) => {
  let currentRoom = null; // 记录当前 socket 加入的时候的房间名

  socket.on("join-room", (roomName) => {
    currentRoom = roomName;

    // 将 socket 加入到指定的房间，方便后面进行群发
    socket.join(roomName);

    if (!docs.has(roomName)) {
      // 进入此分支，说明当前的这个房间是第一次被访问
      // 第一次访问，我们需要创建全新的 yjs 副本
      const doc = new Y.Doc();
      const awareness = new Awareness(doc);
      // 接下来保存到当前的房间里面
      docs.set(roomName, { doc, awareness });
    }

    // 根据 roomName 获取对应的 doc 和 awareness
    const { doc } = docs.get(roomName);

    // 1. 文档同步 - 服务器需要将文档的状态发送给新加入的客户端
    const state = Y.encodeStateAsUpdate(doc);
    socket.emit("sync-doc", state.buffer);

    // 2. 监听客户端发送过来的增量更新
    socket.on("update", (update) => {
      const binary = new Uint8Array(update);
      // 应用到服务器的 doc 上面
      Y.applyUpdate(doc, binary);
      // 接下来还需要将这个更新广播给当前房间的其他人
      socket.to(roomName).emit("update", binary.buffer);
    });

    // 3. 监听客户端发来的 awareness 光标状态
    socket.on("awareness-update", (update) => {
      // 这边不需要解析，直接广播给其他客户端即可
      socket.to(roomName).emit("awareness-update", update);
    });

    // 4. 断开连接
    socket.on("disconnect", () => {
      if (currentRoom) socket.leave(currentRoom);
    });
  });
});

server.listen("1234", () => {
  console.log(`Websocket 服务器已启动，监听1234端口`);
});
```

### CustomSocketProvider.ts - 前端自定义通信提供者（配套前端代码）

与后端服务器配套的前端通信类，通过 Socket.io 与服务器通信，处理文档同步和 Awareness 状态同步。

```ts
import { io, Socket } from "socket.io-client";
import * as Y from "yjs";
import {
  Awareness,
  encodeAwarenessUpdate,
  applyAwarenessUpdate,
} from "y-protocols/awareness";

export class CustomSocketProvider {
  private socket: Socket;
  public awareness: Awareness;

  constructor(serverUrl: string, roomName: string, ydoc: Y.Doc) {
    this.socket = io(serverUrl, {
      transports: ["websocket"],
    });

    this.awareness = new Awareness(ydoc);

    // 连接成功后加入房间
    this.socket.on("connect", () => {
      this.socket.emit("join-room", roomName);
    });

    // 接收完整文档同步
    this.socket.on("sync-doc", (update: ArrayBuffer) => {
      Y.applyUpdate(ydoc, new Uint8Array(update));
    });

    // 接收增量更新
    this.socket.on("update", (update: ArrayBuffer) => {
      Y.applyUpdate(ydoc, new Uint8Array(update));
    });

    // 本地文档变更发送给服务器
    ydoc.on("update", (update: Uint8Array) => {
      this.socket.emit("update", update.buffer);
    });

    // 本地 Awareness 状态变更发送给服务器
    this.awareness.on(
      "update",
      ({ added, updated, removed }: { added: number[]; updated: number[]; removed: number[] }) => {
        const update = encodeAwarenessUpdate(this.awareness, [
          ...added,
          ...updated,
          ...removed,
        ]);
        this.socket.emit("awareness-update", update);
      }
    );

    // 接收其他用户的 Awareness 状态
    this.socket.on("awareness-update", (update: ArrayBuffer) => {
      applyAwarenessUpdate(
        this.awareness,
        new Uint8Array(update),
        this.socket.id
      );
    });
  }

  destroy() {
    this.socket.disconnect();
  }
}
```
