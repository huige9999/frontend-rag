## 2. 章节逻辑说明

这是一门**短小精悍的 git 入门课**，只两章，主线极简：**「先建立 git 的整体认知与基本工作流（01）→ 再讲分支与协作（02）」**，刚好够「一个人能用、一群人能协作」。

- **01 git基本概念**：什么是版本控制、git 的工作区/暂存区/仓库三层结构、add/commit 等基本工作流。解决「我自己怎么用 git 管理代码」。
- **02 git分支**：分支是什么、创建/切换/合并分支、多人协作时分支怎么用。解决「多人协作时怎么用 git」。

**为什么这么安排**：git 的核心是「工作区↔暂存区↔仓库」的三层流转，必须先讲清（01），才谈得上分支这种「仓库之上的并发协作机制」（02）。概念 → 分支是 git 学习的标准递进。

### 主线一句话总结

**基本概念与工作流（01）→ 分支与协作（02）**，目标是用最短路径让前端能独立管理代码版本并参与团队协作。

---

## 3. 使用场景等元信息沉淀

- **学完能用在哪**：日常用 git 管理本地代码、提交到远程仓库、在团队里用分支协作开发。
- **和实际项目的关系**：git 是所有真实项目开发的必备工具，本课覆盖的是「够用的基础」，进阶（rebase/cherry-pick/子模块等）需另学。
- **和其他课程的关系**：属于「效率和工具」类基础课，是后续《工程化》《vue 项目实战》里 clone/提交/多人协作的前提。
- **面试/教学高频场景**：工作区/暂存区/仓库三层模型（01）、分支创建与合并（02）是 git 高频考点；更深的内容见独立的《git 详细版》课程。
