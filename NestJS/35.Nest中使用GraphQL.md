# 35. Nest中使用GraphQL

## 课件内容

### 项目初始化

新建个项目

```typescript
nest new nest-graphql -g -p pnpm
```

安装 `graphql` 和 `apollo` 相关的包

```typescript
pnpm add @nestjs/graphql@12 @nestjs/apollo@12 @apollo/server graphql
```

`@nestjs/graphql`和`@nestjs/apollo`现在使用12的版本，因为和nest cli直接生成的`@nestjs/core`和`@nestjs/common`版本还不兼容

### 配置 GraphQLModule

首先，在 `app.module.ts`中引入 `GraphQLModule`：

```typescript
import { Module } from '@nestjs/common';
import { AppController } from './app.controller';
import { AppService } from './app.service';
import { ApolloDriver } from '@nestjs/apollo';
import { GraphQLModule } from '@nestjs/graphql';
import { ApolloServerPluginLandingPageLocalDefault } from '@apollo/server/plugin/landingPage/default';
@Module({
  imports: [
    GraphQLModule.forRoot({
      driver: ApolloDriver,
      typePaths: ['./**/*.graphql'],
      playground: false,
      plugins: [ApolloServerPluginLandingPageLocalDefault()],
    }),
  ],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}
```

其中：

- **driver**：指定驱动
- **typePaths**：指定schema位置，也就是后缀为`.graphql`的文件
- **playground: false, plugins: [ApolloServerPluginLandingPageLocalDefault()]**：这两个完全是为了能够有上节课一样的playground模版，用起来简单方便

### 定义 Schema

在 VSCode 中可以下载 GraphQL 插件实现语法高亮。在 src 下创建 **schema.graphql** 文件：

```graphql
type Employee {
  id: String
  name: String
  sex: String
  age: Int
}

type Department {
  id: String
  name: String
  employees: [Employee]
}

type Query {
  employees: [Employee]
  departments: [Department]
  employeesByDepartmentName(name: String!): [Employee]
}

type Res {
  success: Boolean
  id: String
}

type Mutation {
  addEmployee(name: String!, age: Int!, sex: String!): Res
  updateEmployee(id: String!, name: String!, age: Int!, sex: String!): Res
  deleteEmployee(id: String!): Res
}
```

### 创建 Resolver

通过命令行直接创建 resolver：

```bash
nest g resolver employee --no-spec
```

创建后自动注册到 `app.module.ts` 的 providers 中。

### 实现 Resolver 逻辑

使用 `@Query`、`@Mutation`、`@Args` 装饰器实现对应的查询和变更逻辑。

启动程序 `pnpm run start:dev`，即可通过 `/graphql` 路由访问 GraphQL Playground。

### 跨域与前端对接

在后端开启跨域支持：

```typescript
async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.enableCors();
  await app.listen(process.env.PORT ?? 3000);
}
```

前端项目修改 ApolloClient 的 uri 地址即可对接：

```typescript
const client = new ApolloClient({
  uri: 'http://localhost:3000/graphql',
  cache: new InMemoryCache(),
});
```

## 关键代码

### 1. GraphQL Module 配置（app.module.ts）

核心配置项：`driver` 指定 ApolloDriver，`typePaths` 指定 `.graphql` schema 文件路径。

```typescript
GraphQLModule.forRoot({
  driver: ApolloDriver,
  typePaths: ['./**/*.graphql'],
  playground: false,
  plugins: [ApolloServerPluginLandingPageLocalDefault()],
}),
```

### 2. Resolver 基础用法（nest-graphql 内存版本）

使用 `@Resolver()` 装饰器标记类，`@Query` / `@Mutation` 装饰器映射到 schema 中定义的操作，`@Args` 获取参数：

```typescript
@Resolver()
export class EmployeeResolver {
  @Query('employees')
  async getEmployees() {
    return employees;
  }

  @Mutation('addEmployee')
  async addEmployee(
    @Args('name') name: string,
    @Args('age') age: number,
    @Args('sex') sex: string,
  ) {
    employees.push({ id: Math.ceil(Math.random() * 100) + '', name, age, sex });
    return { success: true, id: Math.ceil(Math.random() * 100) + '' };
  }
}
```

### 3. Schema 定义（schema.graphql）

schema 中定义了类型（type）、查询（Query）、变更（Mutation）和输入类型（input）：

```graphql
type Employee {
  id: Int
  name: String
  sex: String
  age: Int
  department: Department        # 多对一：员工关联部门
}

type Department {
  id: Int
  name: String
  employees: [Employee]          # 一对多：部门包含多个员工
}

input CreateEmployeeInput {      # 新增员工时的输入类型
  name: String
  age: Int
  sex: String
  department: DepartmentInput
}
input DepartmentInput {
  id: Int
}

type Query {
  employees: [Employee]
  departments: [Department]
  employeesByDepartmentName(name: String!): [Employee]
}

type Mutation {
  addEmployee(input: CreateEmployeeInput): Res
  updateEmployee(id: Int!, name: String!, age: Int!, sex: String!): Res
  deleteEmployee(id: Int!): Res
}
```

### 4. Resolver 结合 TypeORM + MySQL（nest-graphql-mysql 版本）

Resolver 中注入 `EntityManager` 操作数据库，通过 `relations` 加载关联关系：

```typescript
@Resolver()
export class EmployeeResolver {
  @InjectEntityManager()
  private employeeEntityManager: EntityManager;

  @Query('employees')
  async getEmployees() {
    return await this.employeeEntityManager.find(Employee, {
      relations: { department: true },   // 加载关联的部门信息
    });
  }

  @Query('employeesByDepartmentName')
  async getEmployeesByDepartmentName(@Args('name') name: string) {
    return await this.employeeEntityManager.find(Employee, {
      where: { department: { name } },   // 按部门名称筛选
    });
  }

  @Mutation('addEmployee')
  async addEmployee(@Args('input') input: CreateEmployeeInput) {
    const entity = await this.employeeEntityManager.save(Employee, {
      ...input,
      department: { id: input.department.id },  // 通过部门ID建立关联
    });
    return { success: true, id: entity.id };
  }
}
```

### 5. InputType 定义

使用 `@InputType()` 和 `@Field()` 装饰器定义 GraphQL 的输入类型，对应 schema 中的 input：

```typescript
@InputType()
export class CreateEmployeeInput {
  @Field() name: string;
  @Field(() => Int) age: number;
  @Field() sex: string;
  @Field(() => DepartmentInput) department: DepartmentInput;
}

@InputType()
export class DepartmentInput {
  @Field(() => Int) id: number;
}
```

### 6. Entity 关联关系（TypeORM）

Employee 多对一关联 Department：

```typescript
@Entity()
export class Employee {
  @PrimaryGeneratedColumn() id: number;
  @Column() name: string;
  @Column() age: number;
  @Column() sex: string;

  @ManyToOne(() => Department, (department) => department.employees)
  @JoinColumn({ name: 'departmentId' })
  department: Department;
}
```

Department 一对多关联 Employee（级联保存）：

```typescript
@Entity()
export class Department {
  @PrimaryGeneratedColumn() id: number;
  @Column() name: string;

  @OneToMany(() => Employee, (employee) => employee.department, { cascade: true })
  employees: Employee[];
}
```
