# 16.对于ORM的理解

## 一、对于ORM的理解

面向对象编程和关系型数据库，都是目前最流行的技术，但是它们的模型是不一样的。

面向对象编程把所有实体看成对象（object），关系型数据库则是采用实体之间的关系（relation）连接数据。很早就有人提出，关系也可以用对象表达，这样的话，就能使用面向对象编程，来操作关系型数据库。

**简单说，ORM 就是通过实例对象的语法，完成关系型数据库的操作的技术，是"对象-关系映射"（Object/Relational Mapping） 的缩写。**

ORM 把数据库映射成对象

- 数据库的表（table） --> 类（class）
- 记录（record，行数据）--> 对象（object）
- 字段（field）--> 对象的属性（attribute）

举例来说，下面是一行 SQL 语句。

```sql
SELECT id, first_name, last_name, phone, birth_date, sex
FROM persons 
WHERE id = 10
```

如果程序直接操作SQL，大致语法如下：

```typescript
res = db.execSql(sql);
name = res[0]["FIRST_NAME"];
```

如果改成 ORM 的写法大致如下：

```typescript
p = Person.get(10);
name = p.first_name;
```

一比较就可以发现，ORM 使用对象，封装了数据库操作，因此可以不碰 SQL 语言。开发者只使用面向对象编程，与数据对象直接交互，不用关心底层数据库。

总结起来，ORM 有下面这些优点。

> - 数据模型都在一个地方定义，更容易更新和维护，也利于重用代码。
> - ORM 有现成的工具，很多功能都可以自动完成，比如数据消毒、预处理、事务等等。
> - 它迫使你使用 MVC 架构，ORM 就是天然的 Model，最终使代码更清晰。
> - 基于 ORM 的业务代码比较简单，代码量少，语义性好，容易理解。
> - 你不必编写性能不佳的 SQL。

但是，ORM 也有很突出的缺点。

> - ORM 库不是轻量级工具，需要花精力学习和设置。
> - 对于复杂的查询，ORM 要么是无法表达，要么是性能不如原生的 SQL。
> - ORM 抽象掉了数据库层，开发者无法了解底层的数据库操作，也无法定制一些特殊的 SQL。

## 二、TypeORM

**TypeORM** 是一个 ORM，可以在 NodeJS、浏览器、Cordova、PhoneGap、Ionic、React Native、NativeScript、Expo 和 Electron 平台上运行，并且可以与 TypeScript 和 JavaScript（ES5、ES6、ES7、ES8）一起使用。

TypeORM 受到其他 ORM 的很大影响，例如 Hibernate、Doctrine 和 Entity Framework。

### 直接开始

直接运行命令创建TypeORM项目

```typescript
npx typeorm init --name typeorm-test --database mysql2
```

没有安装TypeORM包的话，使用npx命令时，系统会自动检测是否已经安装这个包。选择"y"后，脚本将自动安装相关的依赖包。不过项目并不会自动帮你安装mysql的依赖包，所以项目创建好之后，还需在项目中安装mysql2的相关依赖：

```typescript
npm i mysql2 --save
```

### 创建模型及实体

要处理数据库，首先当然要创建表，在TypeORM中，我们通过定义实体来告诉他如何创建一个数据表。比如，我们要创建一个用户表，就需要定义用户的实体模型：

```typescript
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ length: 100 })
  name: string;

  @Column({ length: 100 })
  nickname: string;

  @Column()
  age: number;

  @Column({ length: 11 })
  phone: string;

  @Column("text")
  desc: string;

  @Column("double", {
    default: 0,
  })
  other: number;
}
```

我们仅仅定义一个User模型并不够，还需要使用装饰器`@Entity`将其定义为一个实体。

用于装饰列的装饰器包括 `@Column`、`@PrimaryGeneratedColumn`和`@PrimaryColumn`。`@PrimaryColumn`用于定义普通主键列，而`@PrimaryGeneratedColumn`用于定义自增主键列。

string类型默认映射为数据库中的`varchar(255)`，`number`类型默认映射为`int`整数类型。如果我们希望某个字段是文本(Text)类型或者(Double)类型，可以通过`@Column`装饰器指定。

### 连接数据库

为了和数据库进行交互，我们需要设置一个数据源(DataSource)。数据源中包含了与数据库连接相关的配置。

```typescript
export const AppDataSource = new DataSource({
  type: "mysql",
  host: "localhost",
  port: 3306,
  username: "root",
  password: "123456",
  database: "typeorm_test",
  synchronize: true,
  logging: true,
  entities: [User],
  migrations: [],
  subscribers: [],
});
```

关键配置说明：

- **type**：数据库类型，支持 postgres、oracle、sqlite 等
- **database**：连接的数据库名（数据库需要自己先创建）
- **host**、**port**：数据库服务器的主机和端口号
- **username**、**password**：登录数据库的用户名和密码
- **synchronize**：同步建表，当 database 里没有和 Entity 对应的表时，自动生成建表SQL并执行
- **logging**：启动日志记录，会打印生成的SQL语句
- **entities**：指定与数据库表对应的 Entity，支持路径通配符如 `./**/entity/*.ts`

其他配置：**migrations**（修改表结构的SQL）、**subscribers**（Entity生命周期订阅者）、**poolSize**（连接池最大数量）、**connectorPackage**（驱动包）

创建DataSource实例之后，需要调用该实例的**initialize**方法来创建数据库连接。

### 使用实体管理器操作数据库

一旦数据库连接成功，我们就可以使用实体管理器（manager）来执行数据库操作：

```typescript
AppDataSource.initialize()
  .then(async () => {
    // 插入数据
    const user1 = new User();
    user1.name = "jack";
    user1.nickname = "张三";
    user1.phone = "13311111111";
    user1.age = 25;
    user1.desc = "This is a desc1";
    await AppDataSource.manager.save(user1);

    // 查询所有数据
    const users = await AppDataSource.manager.find(User);

    // 更新数据
    user1.name = "tom";
    await AppDataSource.manager.save(user1);

    // 条件查询
    const user = await AppDataSource.manager.findOneBy(User, { id: 2 });
  })
  .catch((error) => console.log(error));
```

### 使用Repository操作CRUD

除了使用EntityManager，我们还可以使用Repository来管理每个实体：

```typescript
async function main() { 
  await AppDataSource.initialize();
  const userRepository = AppDataSource.getRepository(User);

  // 新增
  const user1 = new User();
  user1.name = "jack";
  user1.nickname = "张三";
  await userRepository.save(user1);

  // 查询所有
  const users = await userRepository.find();

  // 条件查询
  const user = await userRepository.findOneBy({ id: 2 });

  // 更新
  user.name = "tom";
  await userRepository.save(user);

  // 删除
  await userRepository.delete(user);
}
```

### 使用QueryBuilder操作CRUD

QueryBuilder是TypeORM中最强大的功能之一，提供了更加底层的数据库查询操作，灵活度高，支持组合更复杂的SQL查询语句。

创建QueryBuilder有三种方式：

```typescript
// 方式一：DataSource创建
const user = await AppDataSource.createQueryBuilder()
  .select("user")
  .from(User, "user")
  .where("user.id = :id", { id: 1 })
  .getOne();

// 方式二：Repository创建
const user = await AppDataSource.getRepository(User)
  .createQueryBuilder("user")
  .where("user.id = :id", { id: 1 })
  .getOne();

// 方式三：EntityManager创建
const user = await AppDataSource.manager
  .createQueryBuilder(User, "user")
  .where("user.id = :id", { id: 1 })
  .getOne();
```

**新增数据：**

```typescript
const user = new User();
user.name = "lily";
user.nickname = "丽丽";

const result = await AppDataSource.createQueryBuilder()
  .insert()
  .into(User)
  .values(user)
  .execute();
```

**查询数据：**

```typescript
// 查询所有（映射到实体对象）
const saveUsers = await queryBuilder.select("u").from(User, "u").getMany();

// 查询原始数据库字段（当实体字段名与数据库字段名不一致时使用）
const saveUsers = await queryBuilder.select().from(User, "u").getRawMany();
```

**更新数据：**

```typescript
const resultUpdate = await queryBuilder
  .update(User)
  .set(userUpdate)
  .where("id = :id", { id: 1 })
  .execute();
```

**删除数据：**

```typescript
const resultDelete = await queryBuilder
  .delete()
  .from(User)
  .where("id = :id", { id: 1 })
  .execute();
```

## 三、TypeORM处理多表关系

### 一对一关系

以用户表和身份证表为例。使用 `@OneToOne` 装饰器建立关联，`@JoinColumn()` 定义外键列：

```typescript
@Entity()
export class IdCard { 
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ length: 18 })
  cardNo: string;

  @Column()
  name: string;

  @OneToOne(() => User)
  @JoinColumn()
  user: User;
} 
```

插入数据时，关键是确保关联：

```typescript
const user = new User();
user.name = "hmm";

const idCard = new IdCard();
idCard.cardNo = "123456789012345678";
idCard.user = user;  // 关联两个实体

await userRepository.save(user);
await idCardRepository.save(idCard);
```

查询时需要设置 relations：

```typescript
const idCards = await idCardRepository.find({
  relations: { user: true },
});
```

#### 双向关联

默认关系是单向的（只有IdCard能访问User）。要实现双向关联：

```typescript
// IdCard 实体
@OneToOne(() => User, (user) => user.card)
@JoinColumn()
user: User;

// User 实体
@OneToOne(() => IdCard, (idCard) => idCard.user)
card: IdCard;
```

#### 级联处理

设置 cascade 后，可以通过一个 Repository 同时保存关联实体：

```typescript
@OneToOne(() => IdCard, (idCard) => idCard.user, {
  cascade: true,
})
card: IdCard;
```

```typescript
user.card = idCard;
await userRepository.save(user);  // 自动级联保存 idCard
```

### 一对多/多对一关系

以部门和员工为例，`@OneToMany` 必须与 `@ManyToOne` 配对使用，外键设置在"多"的一方：

```typescript
// 部门实体（一方）
@Entity()
export class Department { 
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ length: 100 })
  name: string;

  @OneToMany(() => Employee, employee => employee.department, {
    cascade: true
  })
  employees: Employee[];
}

// 员工实体（多方）
@Entity()
export class Employee {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ length: 100 })
  name: string;

  @ManyToOne(() => Department, department => department.employees)
  @JoinColumn() 
  department: Department;
}
```

插入数据：

```typescript
depart.employees = [emp1, emp2];
await departmentRepository.save(depart);
```

### 多对多关系

以订单和商品为例，使用 `@JoinTable()` 生成中间表：

```typescript
// 订单实体
@Entity()
export class Order {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ length: 100 })
  name: string;

  @ManyToMany(() => Product, product => product.orders, {
    cascade: true
  })
  products: Product[];
}

// 商品实体
@Entity()
export class Product {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ length: 100 })
  name: string;

  @ManyToMany(() => Order, order => order.products)
  @JoinTable()
  orders: Order[];
}
```

也可以自定义中间表的名称和列名：

```typescript
@JoinTable({
  name: 'order_products',
  joinColumn: {
    name: 'order_id',
    referencedColumnName: 'id',
  },
  inverseJoinColumn: {
    name: 'product_id',
    referencedColumnName: 'id',
  },
})
orders: Order[];
```

## 四、在Nest中使用TypeORM

### 项目初始化

```shell
nest n nest-typeorm-mysql -p -pnpm
nest g res user
pnpm add typeorm mysql2 @nestjs/typeorm -s
```

### 在AppModule中注册TypeORM

```typescript
@Module({
  imports: [
    TypeOrmModule.forRoot({
      type: 'mysql',
      host: 'localhost',
      username: 'root',
      port: 3306,
      password: '123456',
      database: 'nest_typeorm',
      entities: [__dirname + '/**/*.entity{.ts,.js}'],
      synchronize: true,
      timezone: 'Z',
    }),
    UserModule,
  ],
})
export class AppModule {}
```

> 注意：`synchronize: true` 便于开发，但在生产环境中必须禁用，避免随意改变数据库结构。

### EntityManager操作实体

在Service层注入EntityManager：

```typescript
@Injectable()
export class UserService {
  @InjectEntityManager()
  private entityManager: EntityManager;

  async create(createUserDto: CreateUserDto) {
    return await this.entityManager.save(User, createUserDto);
  }

  async findAll() {
    return await this.entityManager.find(User);
  }

  async findOne(id: number) {
    return await this.entityManager.findOneBy(User, { id });
  }

  async update(id: number, updateUserDto: UpdateUserDto) {
    return await this.entityManager.save(User, { id, ...updateUserDto });
  }

  async remove(id: number) {
    return await this.entityManager.delete(User, { id });
  }
}
```

### Repository操作实体

使用Repository不需要每次都传入实体对象。注入时使用 `@InjectRepository`，同时需要在模块中注册：

**user.module.ts：**

```typescript
@Module({
  imports: [TypeOrmModule.forFeature([User])],
  controllers: [UserController],
  providers: [UserService],
})
export class UserModule {}
```

**user.service.ts：**

```typescript
@Injectable()
export class UserService {
  @InjectRepository(User)
  private userRepository: Repository<User>;

  async create(createUserDto: CreateUserDto) {
    return await this.userRepository.save(createUserDto);
  }

  async findAll() {
    return await this.userRepository.find();
  }

  async findOne(id: number) {
    return await this.userRepository.findOneBy({ id });
  }

  async update(id: number, updateUserDto: UpdateUserDto) {
    return await this.userRepository.save({ id, ...updateUserDto });
  }

  async remove(id: number) {
    return await this.userRepository.delete({ id });
  }
}
```

### QueryBuilder操作实体

当需要更复杂的SQL时（如使用 OR 条件），可以使用QueryBuilder：

```typescript
async findAll(name: string, sex: string) {
  return this.userRepository
    .createQueryBuilder('u')
    .where('u.name = :name or u.sex = :sex', { name, sex })
    .getMany();
}
```

## 关键代码

### TypeORM 实体定义核心装饰器

```typescript
@Entity()
export class User {
  @PrimaryGeneratedColumn()        // 自增主键
  id: number;

  @Column({ length: 100 })         // 普通列，可指定类型/长度/默认值
  name: string;

  @Column("text")                  // 指定为 text 类型
  desc: string;
}
```

### 数据源配置

```typescript
export const AppDataSource = new DataSource({
  type: "mysql",
  host: "localhost",
  port: 3306,
  username: "root",
  password: "123456",
  database: "typeorm_test",
  synchronize: true,               // 自动同步建表
  logging: true,                   // 打印SQL日志
  entities: [User],                // 或使用通配符路径
});
```

### NestJS 中 TypeORM 模块注册

```typescript
// 根模块：forRoot 注册全局数据源
TypeOrmModule.forRoot({
  type: 'mysql',
  host: 'localhost',
  database: 'nest_typeorm',
  entities: [__dirname + '/**/*.entity{.ts,.js}'],
  synchronize: true,
})

// 子模块：forFeature 注册实体对应的 Repository
TypeOrmModule.forFeature([User])
```

### 三种CRUD操作方式对比

```typescript
// EntityManager —— 每次需传入实体类
await this.entityManager.find(User);
await this.entityManager.save(User, dto);

// Repository —— 无需每次传入实体，需在模块注册 TypeOrmModule.forFeature
await this.userRepository.find();
await this.userRepository.save(dto);

// QueryBuilder —— 适合复杂查询
this.userRepository.createQueryBuilder('u')
  .where('u.name = :name or u.sex = :sex', { name, sex })
  .getMany();
```

### 多表关系装饰器速查

```typescript
// 一对一
@OneToOne(() => User, (user) => user.card)
@JoinColumn()                     // 外键所在方
user: User;

// 一对多（多方持有外键）
@OneToMany(() => Employee, emp => emp.department, { cascade: true })
employees: Employee[];

@ManyToOne(() => Department, dept => dept.employees)
@JoinColumn()
department: Department;

// 多对多（@JoinTable 自动生成中间表）
@ManyToMany(() => Product, product => product.orders, { cascade: true })
products: Product[];

@ManyToMany(() => Order, order => order.products)
@JoinTable({
  name: 'order_products',         // 自定义中间表名
  joinColumn: { name: 'order_id', referencedColumnName: 'id' },
  inverseJoinColumn: { name: 'product_id', referencedColumnName: 'id' },
})
orders: Order[];
```
