# 25. Nest中实现Session与JWT

## 一、Session 实现

### 1. 安装依赖

```shell
pnpm add express-session @types/express-session
```

### 2. 在 main.ts 中启用 Session

在 `main.ts` 中通过 `app.use()` 注册 `express-session` 中间件：

```typescript
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';
import session from 'express-session';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.use(
    session({
      secret: 'my-secret',
      resave: false,
      saveUninitialized: false,
    }),
  );
  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();
```

**配置参数说明：**

- **secret**：用于签名 session ID 的密钥
- **resave**：是否强制保存每个请求的 session，即使 session 没有修改过。设置为 `false` 可以减少不必要的 session 保存操作，提高性能。
- **saveUninitialized**：是否保存未初始化的 session（即新建的 session 对象，但是没有设置任何数据）。通常设为 `false`，以防止存储不必要的 session。

### 3. 在 Controller 中使用 Session

NestJS 提供了两种方式获取 Session：

**方式一：使用 `@Session()` 装饰器**

```typescript
@Get('session1')
getSession(@Session() session: Record<string, any>) {
  console.log(session);
  session.views = session.views ? session.views + 1 : 1;
  return session.views;
}
```

**方式二：通过 `@Req()` 装饰器从 Request 中获取**

```typescript
@Get('session2')
getSession2(@Req() req: Request) {
  const session = req.session;
  console.log(session);
  session.views = session.views ? session.views + 1 : 1;
  return session.views;
}
```

两种方式效果相同。方式二通过 `req.session` 获取，需要更细致的 TypeScript 类型处理。

### 4. 扩展 Session 类型声明

当使用 `@Req()` 方式访问 session 时，为了获得更好的类型提示，需要在 `src/types` 目录下创建 `express-session.d.ts` 文件来扩展 `SessionData` 接口：

```typescript
import { Session } from 'express-session';

declare module 'express-session' {
  interface SessionData {
    user?: {
      username: string;
      role: string;
    };
    views?: number;
  }
}
```

> 引入 `Session` 类是因为 `SessionData` 本身其实是 `Session` 类中的一个接口，所以如果要扩展 `SessionData`，需要引用它所依赖的 `Session` 类，才能确保 TypeScript 正确推断类型。

## 二、JWT 实现

### 1. 安装依赖

```shell
pnpm add @nestjs/jwt
```

### 2. 在 AppModule 中注册 JwtModule

```typescript
import { Module } from '@nestjs/common';
import { AppController } from './app.controller';
import { AppService } from './app.service';
import { JwtModule } from '@nestjs/jwt';

@Module({
  imports: [
    JwtModule.register({
      secret: 'secretKey',
      signOptions: { expiresIn: '7d' },
    }),
  ],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}
```

通过 `JwtModule.register()` 静态配置，指定密钥 `secret` 和过期时间 `expiresIn`。

### 3. 在 Controller 中使用 JwtService

注入 `JwtService` 后，可以使用 `sign()` 签发 token，`verify()` 验证 token：

**签发 Token（jwt1 接口）：**

```typescript
@Get('jwt1')
getJwt1(@Res({ passthrough: true }) res: Response) {
  const token = this.jwtService.sign({ count: 1 });
  res.setHeader('Authorization', `Bearer ${token}`);
  return 'hello jwt1';
}
```

**验证并刷新 Token（jwt2 接口）：**

```typescript
@Get('jwt2')
getJwt2(
  @Headers('authorization') authorization: string,
  @Res({ passthrough: true }) res: Response,
) {
  if (authorization) {
    try {
      const token = authorization.split(' ')[1];
      const payload = this.jwtService.verify(token);
      const newToken = this.jwtService.sign({ count: payload.count + 1 });
      res.setHeader('Authorization', `Bearer ${newToken}`);
    } catch (e) {
      throw new UnauthorizedException();
    }
  } else {
    throw new UnauthorizedException();
  }
  return 'hello jwt2';
}
```

> 在 Authorization 头信息前面加上 `Bearer`，主要是 **OAuth 2.0** 规范（[RFC 6750](https://datatracker.ietf.org/doc/html/rfc6750)）明确规定了使用 `Bearer` 作为令牌类型。

## 关键代码

### 1. main.ts - 注册 Session 中间件

```typescript
import * as session from 'express-session';

// 在 bootstrap 中注册
app.use(
  session({
    secret: 'my-secret',
    resave: false,
    saveUninitialized: false,
  }),
);
```

核心逻辑：通过 `app.use()` 将 `express-session` 中间件挂载到 NestJS 应用，配置 secret、resave、saveUninitialized 三个关键参数。

### 2. app.module.ts - 注册 JwtModule

```typescript
JwtModule.register({
  secret: 'secretKey',
  signOptions: { expiresIn: '7d' },
}),
```

核心逻辑：使用 `JwtModule.register()` 进行静态配置，设置密钥和 token 有效期（7天）。

### 3. app.controller.ts - Session 与 JWT 的完整使用

```typescript
@Inject()
private jwtService: JwtService;

// Session 方式一：@Session() 装饰器
@Get('session1')
getSession1(@Session() session: Record<string, any>): string {
  session.views = session.views ? session.views + 1 : 1;
  return session.views;
}

// Session 方式二：@Req() 装饰器
@Get('session2')
getSession2(@Req() req: Request) {
  const session = req.session;
  session.views = session.views ? session.views + 1 : 1;
}

// JWT 签发
@Get('jwt1')
getJwt1(@Res({ passthrough: true }) res: Response) {
  const token = this.jwtService.sign({ count: 1 });
  res.setHeader('Authorization', `Bearer ${token}`);
  return 'hello jwt1';
}

// JWT 验证 + 续签
@Get('jwt2')
getJwt2(
  @Headers('authorization') authorization: string,
  @Res({ passthrough: true }) res: Response,
) {
  if (authorization) {
    try {
      const token = authorization.split(' ')[1];          // 提取 token
      const payload = this.jwtService.verify(token);      // 验证 token
      const newToken = this.jwtService.sign({ count: payload.count + 1 }); // 签发新 token
      res.setHeader('Authorization', `Bearer ${newToken}`);
    } catch (e) {
      throw new UnauthorizedException();
    }
  } else {
    throw new UnauthorizedException();
  }
  return 'hello jwt2';
}
```

核心逻辑：
- Session 通过 `@Session()` 或 `@Req()` 两种方式获取，直接读写属性即可
- JWT 通过 `JwtService.sign()` 签发、`JwtService.verify()` 验证
- Token 放在 `Authorization: Bearer <token>` 头中传递
- 验证失败抛出 `UnauthorizedException`

### 4. express-session.d.ts - Session 类型扩展

```typescript
import { Session } from 'express-session';

declare module 'express-session' {
  interface SessionData {
    user?: {
      username: string;
      role: string;
    };
    views?: number;
  }
}
```

核心逻辑：通过 TypeScript 模块扩展（declaration merging），为 `express-session` 的 `SessionData` 接口添加自定义字段，获得类型安全。
