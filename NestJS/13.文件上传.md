# 13. 文件上传

## 单文件上传

在Nest中，上传单个文件的功能由FileInterceptor()拦截器和`@UploadFile()`装饰器共同实现。当FileInterceptor绑定到控制器方法时，它负责拦截请求中包含的文件，并将这些文件保存到指定的位置。而`@UploadFile()`装饰器用于从请求中提取已上传的文件。

```typescript
@Post('upload')
@UseInterceptors(FileInterceptor('file', { dest: './uploads' }))
uploadFile(@UploadedFile() file: Express.Multer.File, @Body() body) {
  console.log(file);
  return {
    message: '上传成功',
    file: file.filename,
  };
}
```

> **注意：**
>
> `FileInterceptor`在包`@nestjs/platform-express`中，
>
> `Express.Multer.File`是上传文件的类型，不过会报类型错误，需要引入相关的类型文件
>
> `pnpm add @types/multer -D`

除了在拦截器上直接指定文件上传后的保存位置之外，`Multer`中间件提供了文件存储引擎，我们可以在`app.module.ts`中通过动态模块进行配置：

```typescript
MulterModule.register({
  storage: diskStorage({
    destination: './uploads',
    filename: (req, file, cb) => {
      const randomName = Array(32)
        .fill(null)
        .map(() => Math.round(Math.random() * 16).toString(16))
        .join('');
      const filename = `${randomName}${extname(file.originalname)}`;
      cb(null, filename);
    },
  }),
})
```

其中：

```typescript
Array(32)
.fill(null)
.map(() => Math.round(Math.random() * 16).toString(16))
.join('')
```

生成了一个由 32 个十六进制字符组成的随机字符串。`extname`函数需要从`path`中引入。

## 多文件上传

把 `UploadedFile` 换成 `UploadedFiles`，把`FileInterceptor`换成`FilesInterceptor`，都是多加一个 s就可以了：

```typescript
@Post('uploads1')
@UseInterceptors(
  FilesInterceptor('files', 3, {
    dest: 'uploads',
  }),
)
uploadFiles1(
  @UploadedFiles() files: Array<Express.Multer.File>,
  @Body() body,
) {
  console.log('files', files);
  return {
    message: '上传成功',
    files,
  };
}
```

不过稍微注意的是，上传多文件并不会自动帮你创建文件夹。

除了上面的方式之外，还能以下面的方式传递多文件——通过`FileFieldsInterceptor`分别处理多个文件的字段：

```typescript
@Post('uploads2')
@UseInterceptors(
  FileFieldsInterceptor([
    { name: 'files1', maxCount: 3 },
    { name: 'files2', maxCount: 3 },
  ]),
)
uploadFiles2(
  @UploadedFiles()
  files: { file1?: Express.Multer.File[]; file2?: Express.Multer.File[] },
  @Body() body,
) {
  console.log(files);
  return {
    message: '上传成功',
    files,
  };
}
```

## 任意字段文件上传

如果并不知道有哪些字段是 file，可以用 `AnyFilesInterceptor`：

```typescript
@Post('anyUploads')
@UseInterceptors(
  AnyFilesInterceptor({
    dest: 'uploads',
  }),
)
uploadAnyFiles(
  @UploadedFiles() files: Array<Express.Multer.File>,
  @Body() body,
) {
  console.log('files', files);
  return {
    message: '上传成功',
    files,
  };
}
```

`AnyFilesInterceptor`还有storage字段，我们可以直接单独的处理文件存储，将存储逻辑抽离到独立文件中，然后在拦截器中引入使用。

## 上传限制

要对上传的文件做一些限制，比如文件大小、类型等，这部分就可以利用 `pipe` 处理。

### 自定义 Pipe 校验

可以自己创建一个 Pipe：

```
nest g pipe fileValidation --no-spec --flat
```

在里面做一些简单处理：

```typescript
@Injectable()
export class FileValidationPipe implements PipeTransform {
  transform(value: Express.Multer.File, metadata: ArgumentMetadata) {
    if (value.size > 10 * 1024) {
      throw new HttpException(
        '文件上传大小不能超过10k',
        HttpStatus.BAD_REQUEST,
      );
    }
    return value;
  }
}
```

在 controller 中应用：将 Pipe 作为参数传给 `@UploadedFile()` 装饰器即可。

### 使用内置的 ParseFilePipe

像文件大小、类型的校验这种逻辑太过常见，我们完全可以用Nest已经封装好的：

```typescript
@UploadedFile(
  new ParseFilePipe({
    validators: [
      new MaxFileSizeValidator({ maxSize: 1000 }),
      new FileTypeValidator({ fileType: 'image/jpeg' }),
    ],
    exceptionFactory: (err) => {
      throw new HttpException(
        '文件大小或类型不正确，上传失败---' + err,
        400,
      );
    },
  })
)
```

其中，`MaxFileSizeValidator` 是校验文件大小、`FileTypeValidator` 是校验文件类型。

### 自定义 FileValidator

我们也可以自己实现这样的 validator，只要继承 `FileValidator` 就可以：

```typescript
export class MyFileValidator extends FileValidator {
  isValid(file: Express.Multer.File): boolean | Promise<boolean> {
    if (file.size > 10 * 1024) return false;
    if (file.mimetype !== 'image/jpeg') return false;
    return true;
  }

  buildErrorMessage(file: Express.Multer.File): string {
    if (file.size > 10 * 1024) {
      return `文件 ---${file.originalname}--- 大小超出 10k`;
    } else if (file.mimetype !== 'image/jpeg') {
      return `文件 ---${file.originalname}--- 类型不是 image/jpeg`;
    }
  }
}
```

## 关键代码

### 1. MulterModule 注册（app.module.ts）

在模块中全局配置文件存储引擎，使用 `diskStorage` 自定义保存路径和随机文件名：

```typescript
MulterModule.register({
  storage: diskStorage({
    destination: './uploads',
    filename: (req, file, cb) => {
      const randomName = Array(32)
        .fill(null)
        .map(() => Math.round(Math.random() * 16).toString(16))
        .join('');
      const filename = `${randomName}${extname(file.originalname)}`;
      cb(null, filename);
    },
  }),
})
```

### 2. 完整 Controller 示例（app.controller.ts）

涵盖四种上传方式的控制器：

```typescript
// 单文件上传 + 自定义校验
@Post('upload1')
@UseInterceptors(FileInterceptor('file'))
uploadFile1(
  @UploadedFile(
    new ParseFilePipe({
      validators: [new MyFileValidator({})],
    }),
  )
  file: Express.Multer.File,
) { ... }

// 多文件上传（同名字段，最多3个）
@Post('upload2')
@UseInterceptors(FilesInterceptor('files', 3))
uploadFile2(@UploadedFiles() files: Array<Express.Multer.File>) { ... }

// 多文件上传（不同字段名）
@Post('upload3')
@UseInterceptors(
  FileFieldsInterceptor([
    { name: 'files1', maxCount: 3 },
    { name: 'files2', maxCount: 3 },
  ]),
)
uploadFile3(
  @UploadedFiles() files: { file1?: Express.Multer.File[]; file2?: Express.Multer.File[] },
) { ... }

// 任意字段文件上传 + 自定义存储
@Post('upload4')
@UseInterceptors(AnyFilesInterceptor({ storage: storage }))
uploadFile4(@UploadedFiles() files: Array<Express.Multer.File>) { ... }
```

### 3. 独立存储配置（storage.ts）

将 `multer.diskStorage` 配置抽离为独立模块，供 `AnyFilesInterceptor` 等场景直接引用：

```typescript
const storage = multer.diskStorage({
  destination: './uploads',
  filename: (req, file, cb) => {
    const randomName = Array(32)
      .fill(null)
      .map(() => Math.round(Math.random() * 16).toString(16))
      .join('');
    cb(null, `${randomName}${extname(file.originalname)}`);
  },
});
export { storage };
```

### 4. 自定义文件校验器（MyFileValidator.ts）

继承 `FileValidator`，实现 `isValid` 和 `buildErrorMessage` 方法，可灵活定义文件大小和类型的校验逻辑：

```typescript
export class MyFileValidator extends FileValidator {
  isValid(file: Express.Multer.File): boolean | Promise<boolean> {
    if (file.size > 10 * 1024) return false;
    if (file.mimetype !== 'image/jpeg') return false;
    return true;
  }
  buildErrorMessage(file: Express.Multer.File): string {
    if (file.size > 10 * 1024) {
      return `文件 --- ${file.originalname} --- 太大,超出10K`;
    } else if (file.mimetype !== 'image/jpeg') {
      return `文件 --- ${file.originalname} --- 类型不正确,只能上传jpeg图片`;
    }
  }
}
```

### 5. 自定义 Pipe 校验（file-validation-pipe.pipe.ts）

通过实现 `PipeTransform` 接口，在 `transform` 方法中对文件进行校验，不合法时抛出 `HttpException`：

```typescript
export class FileValidationPipePipe implements PipeTransform {
  transform(value: Express.Multer.File, metadata: ArgumentMetadata) {
    if (value.size > 10 * 1024) {
      throw new HttpException('文件上传大小不能超过10K', HttpStatus.BAD_REQUEST);
    }
    return value;
  }
}
```
