# 23-平行坐标系

## 简介
本节课学习 ECharts 平行坐标系（parallel）的使用方法，通过北京、上海、广州三地的空气质量数据，掌握 parallel 坐标系和 parallelAxis（平行坐标轴）的配置技巧。

## 关键代码

### parallel 与 parallelAxis 完整配置

```javascript
option = {
  parallel: {
    layout: 'horizontal',  // 布局方向：水平排列
    parallelAxisDefault: {  // 所有坐标轴的默认配置
      nameTextStyle: {
        color: 'gold',
        fontSize: 20,
      }
    }
  },
  parallelAxis: [
    {
      dim: 0,           // 维度索引，对应数据的第0列
      name: '日期',
      realtime: false,   // 拖拽时是否实时更新视图
      areaSelectStyle: { // 框选样式配置
        width: 40,
        borderWidth: 2,
        borderColor: 'red',
        color: 'rgba(0,100,200,0.5)',
        opacity: 1,
      },
      nameLocation: 'start',  // 坐标轴名称位置
      nameTextStyle: {
        color: 'red',
        fontSize: 20,
      },
      nameGap: 25,       // 名称与轴线距离
      nameRotate: 45,    // 名称旋转角度
    },
    { dim: 1, name: 'AQI指数' },
    { dim: 2, name: 'PM2.5' },
    { dim: 3, name: 'PM10' },
    { dim: 4, name: '一氧化碳' },
    { dim: 5, name: '二氧化氮' },
    { dim: 6, name: '二氧化硫' },
    {
      dim: 7,
      name: '等级',
      type: 'category',  // 类目型坐标轴
      data: ['优', '良', '轻度污染', '中度污染', '重度污染', '严重污染']
    }
  ],
  series: [
    {
      name: '北京',
      type: 'parallel',
      data: dataBJ,
    },
    {
      name: '上海',
      type: 'parallel',
      data: dataSH,
    },
    {
      name: '广州',
      type: 'parallel',
      data: dataGZ,
    }
  ],
};
```

### 代码说明
- `parallel` - 平行坐标系的全局配置，`layout` 控制排列方向（`'horizontal'` / `'vertical'`）
- `parallel.parallelAxisDefault` - 为所有坐标轴设置统一的默认样式，避免重复配置
- `parallelAxis` - 定义每条平行坐标轴，数组索引对应数据维度
- `parallelAxis.dim` - 指定该轴对应的维度（数据列索引）
- `parallelAxis.realtime` - 框选时是否实时更新视图，数据量大时建议设为 `false`
- `parallelAxis.areaSelectStyle` - 框选交互的视觉样式（宽度、边框、颜色等）
- `parallelAxis.nameLocation` - 轴名称显示位置（`'start'` / `'middle'` / `'end'`）
- `parallelAxis.type: 'category'` - 将维度设为类目轴，适用于"等级"等离散值
- `series.type: 'parallel'` - 系列类型为平行坐标系，每条数据表现为穿过各轴的折线
- 数据格式为二维数组，每个子数组长度需与 `parallelAxis` 的维度数量一致
