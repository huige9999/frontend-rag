# 29-数据集1-存储

## 简介
本节课学习 ECharts 数据集（dataset）的数据存储方式，掌握三种不同的数据源格式（二维数组、对象数组、按列组织的对象）以及 `dimensions` 维度定义的用法。

## 关键代码

### 数据集的三种 source 格式

```javascript
option = {
  legend: {},
  tooltip: {},
  yAxis: {},
  xAxis: {
    type: 'category',
  },
  series: [
    { type: 'bar' },
    { type: 'bar' },
    { type: 'bar' },
  ],
  dataset: {
    // 格式一：二维数组（第一行为维度名）
    source: [
      ['product', '2014', '2015', '2016'],
      ['iphone4', 43.3, 85.8, 93.7],
      ['iphone5', 83.1, 73.4, 55.1],
      ['iphone6', 86.4, 65.2, 82.5],
      ['iphone7', 72.4, 53.9, 39.1]
    ],

    // 格式二：对象数组
    source: [
      { product: 'iphone4', '2014': 43.3, '2015': 85.8, '2016': 93.7 },
      { product: 'iphone5', '2014': 83.1, '2015': 73.4, '2016': 55.1 },
      { product: 'iphone6', '2014': 86.4, '2015': 65.2, '2016': 82.5 },
      { product: 'iphone7', '2014': 72.4, '2015': 53.9, '2016': 39.1 },
    ],

    // 格式三：按列组织的对象（键值对）
    source: {
      product: ['iphone4', 'iphone5', 'iphone6', 'iphone7'],
      '2014': [43.3, 83.1, 86.4, 72.4],
      '2015': [85.8, 73.4, 65.2, 53.9],
      '2016': [93.7, 55.1, 82.5, 39.1],
    },

    sourceHeader: true,  // 第一行是否作为维度名（仅二维数组有效）

    // 维度定义
    dimensions: [
      'product', '2014', '2015',
      { name: '2016', type: 'number' }  // 可单独指定维度类型
    ],
  }
};
```

### 代码说明
- `dataset` - 数据集组件，用于统一管理图表数据，使数据和样式配置分离
- `dataset.source` - 数据源，支持三种格式：
  - **二维数组**：第一行为维度名，后续行为数据，最常用
  - **对象数组**：每条数据是一个对象，键为维度名
  - **按列对象**：每个键对应一列数据，适合从后端接口直接获取的数据
- `dataset.sourceHeader` - 是否将第一行/第一条数据作为维度名，默认 `true`
- `dataset.dimensions` - 显式定义维度，可以是字符串（维度名）或对象（含 `name` 和 `type`）
- 维度类型 `type` 支持 `'number'`、`'float'`、`'int'`、`'ordinal'`（类目）、`'time'`（时间）
- 使用 `dataset` 后，`series` 中无需再单独指定 `data`，ECharts 会自动从 `dataset` 中读取
- `series.type` 指定图表类型，多个系列共享同一数据集
