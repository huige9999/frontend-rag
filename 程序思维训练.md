# 程序思维训练

## 1. 目录结构整理

```
程序思维训练
├── 01. 歌词滚动                  （音乐歌词滚动效果）
├── 02. 王者荣耀官网英雄查询       （英雄列表查询）
└── 03. 美团省市级联               （省市区级联选择）
```

> 注：每章统一为「素材 + 课堂代码」结构，本课以项目驱动，无理论章节。

---

## 2. 章节逻辑说明

这门课没有概念章节，**完全是项目驱动**：用 3 个经典小案例，训练「把一个需求拆成 DOM 操作 + 数据处理 + 交互逻辑」的编程思维。三个案例难度递增、覆盖的交互类型互补。

- **01 歌词滚动**：音乐播放器常见的「歌词随播放进度高亮并滚动」效果。训练点：定时器、滚动定位、状态同步。
- **02 王者荣耀官网英雄查询**：仿官网的英雄列表/查询。训练点：数据驱动渲染 DOM、过滤查询、列表渲染。
- **03 美团省市级联**：省→市→区的级联下拉选择。训练点：多层数据结构、联动逻辑、事件联动。

**为什么这么安排**：从「单一时间轴驱动」（歌词滚动）到「数据列表渲染与查询」（英雄查询）再到「多级联动」（省市级联），三个案例分别代表三种最常见的前端交互范式，逐个把「需求 → 代码」的思维练熟。

### 主线一句话总结

**歌词滚动（定时与滚动）→ 英雄查询（数据渲染与过滤）→ 省市级联（多级联动）**，目标不是学新 API，而是训练「拿到需求如何拆解、如何用原生 JS 把交互做出来」的编程思维。

---

## 3. 使用场景等元信息沉淀

- **学完能用在哪**：独立用原生 JS 实现常见的页面交互效果；强化「需求拆解 → 数据结构 → DOM 操作」的思维链条，为后续学框架（框架本质也是这套逻辑的封装）打底。
- **和实际项目的关系**：歌词滚动、级联选择、列表查询都是真实业务里高频出现的交互；课程训练的是「不依赖框架也能写出」的底层能力。
- **和其他课程的关系**：上承《JavaScript 语言提升》（用的是 ES6+ 语法），是学 Vue/React 前很好的「原生 JS 实战过渡」——理解了原生怎么写，再学框架「数据驱动视图」会觉得顺理成章。
- **面试/教学高频场景**：省市级联是经典手写题；歌词滚动/列表查询常见于作业与小型作品。

---

## 4. 临时补充

- 这门课代码全在各章「课堂代码」目录，复习时直接看代码实现即可，重点是「思路」而非 API 记忆。
- 三个案例虽然小，但分别代表了「时间驱动 / 数据驱动 / 事件联动」三类前端交互原型，建议作为以后写任何新交互时「找参照」的样板。
