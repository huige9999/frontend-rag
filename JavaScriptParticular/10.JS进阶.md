## 一、原型和原型链

### 笔记

- 所有对象都是通过 `new 函数` 创建
- 所有的函数也是对象
  - 函数中可以有属性
- 所有对象都是引用类型

### 原型 prototype

所有函数都有一个属性：`prototype`，称之为函数原型。

默认情况下，`prototype` 是一个普通的 Object 对象。

默认情况下，`prototype` 中有一个属性 `constructor`，它也是一个对象，它指向构造函数本身。

### 隐式原型 `__proto__`

所有的对象都有一个属性：`__proto__`，称之为隐式原型。

默认情况下，隐式原型指向创建该对象的函数的原型。

当访问一个对象的成员时：

1. 看该对象自身是否拥有该成员，如果有直接使用
2. 在原型链中依次查找是否拥有该成员，如果有直接使用

> 猴子补丁：在函数原型中加入成员，以增强起对象的功能，猴子补丁会导致原型污染，使用需谨慎。

### 原型链

特殊点：

1. `Function.__proto__` 指向自身的 `prototype`
2. `Object.prototype.__proto__` 指向 `null`

### 代码示例

**创建对象与原型关系**

```js
function test() {

}

var obj = new test();
// obj.__proto__ === test.prototype
```

**通过隐式原型获取构造函数名称**

```js
function A() {}

function B() {}

function create() {
    if (Math.random() < 0.5) {
        return new A();
    } else {
        return new B();
    }
}

var obj = create();
// 如何得到创建obj的构造函数名称
console.log(obj.__proto__.constructor.name);
```

**原型属性共享**

```js
function A() {}

var obj1 = new A();
var obj2 = new A();
obj1.abc = 123;
obj2.__proto__.bcd = 456;

console.log(obj1.abc, obj2.abc);    // 123, undefined
console.log(obj1.__proto__.bcd, obj2.__proto__.bcd); // 456, 456
```

> `obj1.abc` 是自身属性，`obj2` 没有自身属性 `abc` 所以是 `undefined`；而通过 `obj2.__proto__.bcd` 设置的是原型属性，所有实例共享。

**猴子补丁 - 重写原型方法**

```js
function A() {}
A.prototype.toString = function() {
    return "123";
}
var obj = new A();
```

### 面试题

**面试题1：原型链查找**

```js
var F = function () {}
Object.prototype.a = function () {}
Function.prototype.b = function () {}

var f = new F();

console.log(f.a, f.b, F.a, F.b);
// fn    undefined    fn    fn
```

> `f` 能访问 `Object.prototype.a`（通过原型链），但 `f` 的原型链上没有 `Function.prototype`，所以 `f.b` 为 `undefined`。`F` 是函数，其原型链同时经过 `Function.prototype` 和 `Object.prototype`。

**面试题2：实例属性 vs 原型属性**

```js
function A() {}
function B(a) {
    this.a = a;
}
function C(a) {
    if (a) {
        this.a = a;
    }
}
A.prototype.a = 1;
B.prototype.a = 1;
C.prototype.a = 1;

console.log(new A().a);   // 1  (自身没有a，查找原型得到1)
console.log(new B().a);   // undefined (构造函数B中 this.a = a，a为undefined赋值给自身)
console.log(new C(2).a);  // 2  (传入了a，自身属性为2)
```

**面试题3：原型链判断**

```js
function User() {}
User.prototype.sayHello = function() {}

var u1 = new User();
var u2 = new User();

console.log(u1.sayHello === u2.sayHello);                              // true
console.log(User.prototype.constructor);                                // User Function
console.log(User.prototype === Function.prototype);                     // false
console.log(User.__proto__ === Function.prototype);                     // true
console.log(User.__proto__ === Function.__proto__);                     // true
console.log(u1.__proto__ === u2.__proto__);                             // true
console.log(u1.__proto__ === User.__proto__);                           // false
console.log(Function.__proto__ === Object.__proto__);                   // true
console.log(Function.prototype.__proto__ === Object.prototype.__proto__); // false
console.log(Function.prototype.__proto__ === Object.prototype);          // true
```

## 二、原型链的应用

### 笔记

#### 基础方法

W3C不推荐直接使用系统成员 `__proto__`。

**Object.getPrototypeOf(对象)**

获取对象的隐式原型。

**Object.prototype.isPrototypeOf(对象)**

判断当前对象(this)是否在指定对象的原型链上。

**对象 instanceof 函数**

判断函数的原型是否在对象的原型链上。

**Object.create(对象)**

创建一个新对象，其隐式原型指向指定的对象。

**Object.prototype.hasOwnProperty(属性名)**

判断一个对象 **自身** 是否拥有某个属性。

#### 应用

**类数组转换为真数组**

```js
Array.prototype.slice.call(类数组);
```

**实现继承**

默认情况下，所有构造函数的父类都是 `Object`。

圣杯模式。

### 代码示例

**圣杯模式实现继承**

```js
function User(firstName, lastName, age) {
    this.firstName = firstName;
    this.lastName = lastName;
    this.age = age;
    this.fullName = this.firstName + " " + this.lastName;
}

User.prototype.sayHello = function() {
    console.log(`大家好，我叫${this.fullName}，今年${this.age}岁了`);
}

function VIPUser(firstName, lastName, age, money) {
    User.call(this, firstName, lastName, age);
    this.money = money;
}
myPlugin.inherit(VIPUser, User);

VIPUser.prototype.upgrade = function() {
    console.log(`使用了${100}元软妹币，升级了！`);
    this.money -= 100;
}

var vUser = new VIPUser("姬", "成", 1, 100);
```

**循环对象属性（包含原型链属性）**

```js
var p = {
    x: 123,
    y: 456
};

var obj = Object.create(p);
obj.abc = "656432434af";
for (var prop in obj) {
    // if (obj.hasOwnProperty(prop)) {
    //     console.log(prop);  // 只输出自身属性
    // }
    console.log(prop);  // 输出自身属性 + 原型链上的属性
}
```

> `for...in` 会遍历对象自身及原型链上所有可枚举属性。使用 `hasOwnProperty` 可过滤只输出自身属性。

## 三、[扩展] 属性描述符

### 笔记

属性描述符：它表达了一个属性的相关信息（元数据），它本质上是一个对象。

属性描述符参考：[MDN - Object.defineProperty](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Object/defineProperty)

1. 数据属性
2. 存取器属性
   1. 当给它赋值，会自动运行一个函数
   2. 当获取它的值时，会自动运行一个函数

**Object.getOwnPropertyDescriptor**

获取某个对象的某个属性的属性描述符对象（该属性必须直接属于该对象）。

### 代码示例

**数据属性**

```js
var obj = {
    x: 1  // 数据属性
};

obj.y = 2; // 数据属性
```

**使用 defineProperty 定义不可写属性**

```js
var obj = {
    x: 1,
    y: 2
};

Object.defineProperty(obj, "name", {
    value: "abc",
    writable: false,    // 不可重写
    enumerable: true    // 可迭代/遍历
})

for (var prop in obj) {
    console.log(prop);
}
```

**存取器属性基本用法**

```js
var obj = {};

Object.defineProperty(obj, "x", {
    get: function() {
        // 当读取属性x时，运行的函数
        console.log("读取属性x");
        // 该函数的返回值，将作为属性的值
        return 2;
    },
    set: function(val) {
        // 当给该属性赋值时，运行的函数
        // val：表示要赋的值
        console.log("给属性赋值为" + val);
    }
});

obj.x = 3;          // 相当于运行了 set(3)
console.log(obj.x); // 先运行 get()，返回 2
```

**存取器属性 - 年龄范围限制**

```js
function User(name, age) {
    this.name = name;
    // 年龄的取值范围是 0 - 100
    var _age;
    Object.defineProperty(this, "age", {
        get: function() {
            return _age;
        },
        set: function(val) {
            if (val < 0) {
                val = 0;
            } else if (val > 100) {
                val = 100;
            }
            _age = val;
        }
    })
    this.age = age;
}

var u = new User("abc", -1);
u.age = u.age + 10000;
console.log(u.age); // 100（被限制在 0-100 范围内）
```

**存取器属性实战 - 弹跳小球边界检测**

```js
var div = document.querySelector(".move");
var config = {
    _x: 0,
    _y: 0,
    xDis: 2,
    yDis: 2,
    duration: 16,
    width: 100,
    height: 100
}

Object.defineProperty(config, "x", {
    get: function() {
        return this._x;
    },
    set: function(val) {
        if (val < 0) {
            val = 0;
        } else if (val > document.documentElement.clientWidth - this.width) {
            val = document.documentElement.clientWidth - this.width;
        }
        this._x = val;
        div.style.left = val + "px";
    }
})

Object.defineProperty(config, "y", {
    get: function() {
        return this._y;
    },
    set: function(val) {
        if (val < 0) {
            val = 0;
        } else if (val > document.documentElement.clientHeight - this.height) {
            val = document.documentElement.clientHeight - this.height;
        }
        this._y = val;
        div.style.top = val + "px";
    }
})

// 通过 setInterval 配合边界判定实现弹跳动画
```

## 四、执行上下文

### 笔记

**函数执行上下文**：一个函数运行之前，创建的一块内存空间，空间中包含有该函数执行所需要的数据，为该函数执行提供支持。

**执行上下文栈**：call stack，所有执行上下文组成的内存空间。

**栈**：一种数据结构，先进后出，后进先出。

**全局执行上下文**：所有JS代码执行之前，都必须有该环境。

JS引擎始终执行的是栈顶的上下文。

### 执行上下文中的内容

#### 1. this指向

1. 直接调用函数，`this` 指向全局对象
2. 在函数外，`this` 指向全局对象
3. 通过对象调用或 `new` 一个函数，`this` 指向调用的对象或新对象

#### 2. VO 变量对象

- **Variable Object (VO)**：VO 中记录了该环境中所有声明的参数、变量和函数
- **Global Object (GO)**：全局执行上下文中的 VO
- **Active Object (AO)**：当前正在执行的上下文中的 VO

VO 确定过程：

1. 确定所有形参值以及特殊变量 `arguments`
2. 确定函数中通过 `var` 声明的变量，将它们的值设置为 `undefined`，如果 VO 中已有该名称，则直接忽略
3. 确定函数中通过字面量声明的函数，将它们的值设置为指向函数对象，如果 VO 中已存在该名称，则覆盖

当一个上下文中的代码执行的时候，如果上下文中不存在某个属性，则会从之前的上下文寻找。

### 代码示例

**执行上下文栈的运行顺序**

```js
console.log("global 1");

function A() {
    console.log("A1");

    function B() {
        console.log("B");
    }

    B();
    console.log("A2");
}
A();
console.log("global 2");

// 输出顺序: global 1 -> A1 -> B -> A2 -> global 2
```

**this指向**

```js
console.log(this.name);
var a = 2;
function A() {
    this.abc = 123;

    function B() {
        console.log(this); // window（直接调用，this指向全局对象）
    }
    B();
}
var a = new A();

console.log(a.abc); // 123
```

**VO变量对象 - 变量提升与函数提升**

```js
function A(a, b) {
    console.log(a, b); // 1, function b(){}
    var b = 123;
    console.log(a, b); // 1, 123

    function b() {
        var d = 123;
    }
}

A(1, 2);
```

> 第一个 `console.log` 时，函数 `b` 的声明提升覆盖了形参 `b=2` 和 `var b`，所以输出函数本身。第二个 `console.log` 时，`b` 已被赋值为 `123`。

**完整的VO确定过程**

```js
var g1 = 123;
var a = 2;
function A(a, b) {
    console.log(a, b, g1); // function a(){}, function b(){}, 123
    var b = 123;

    function b() {}

    var a = function() {}

    console.log(a, b); // function(){}, 123
}
var g2 = 456;
var g3 = function() {}

A(1, 2);
```

> 第一步：形参 `a=1, b=2`；第二步：`var a` 和 `var b` 已存在则忽略；第三步：函数字面量 `function b(){}` 和 `function a(){}`（var a 是函数表达式，不提升）覆盖已有的 `a` 和 `b`。所以第一次输出是两个函数和全局变量 `g1=123`。

**栈溢出（递归无终止条件）**

```js
function A() {
    var a = 1;
    A(); // 无限递归，最终栈溢出（Stack Overflow）
}
A();
```

## 五、作用域链

### 笔记

1. VO 中包含一个额外的属性，该属性指向创建该 VO 的函数本身
2. 每个函数在创建时，会有一个隐藏属性 `[[scope]]`，它指向创建该函数时的 AO
3. 当访问一个变量时，会先查找自身 VO 中是否存在，如果不存在，则依次查找 `[[scope]]` 属性

> 某些浏览器会优化作用域链，函数的 `[[scope]]` 中仅保留需要用到的数据。

### 代码示例

**作用域链查找**

```js
var g = 0;

function A() {
    var a = 1;

    function B() {
        var b = 2;
        var C = function() {
            var c = 3;
            console.log(c, b, a, g); // 3, 2, 1, 0 （沿作用域链逐层查找）
        }
        C();
    }

    B();
}

A();
```

**闭包 - 作用域链的经典应用**

```js
var count = 100;
function A() {
    var count = 0;
    return function() {
        count++;
        console.log(count);
    }
}

var test = A();

test(); // 1
test(); // 2
test(); // 3

console.log(count); // 100（全局变量不受影响）
```

> 内部函数通过作用域链引用了外部函数 `A` 的 `AO` 中的 `count`，形成了闭包。

**作用域在函数创建时确定（静态作用域）**

```js
var a = 1;

function A() {
    console.log(a); // 1（函数创建时作用域已确定，不会因调用位置改变）
}

function Special() {
    var a = 5;
    var B = A;
    B();
}

Special(); // 输出 1，而不是 5
```

> JavaScript 使用词法作用域（静态作用域），函数的作用域在定义时确定，而非调用时。

**试题4：IIFE 解决循环中的闭包问题**

```js
// 问题代码：var i 只有全局一份，setTimeout 回调都引用同一个 i
// function A() {
```

## 六、事件循环

### 笔记

**异步**：某些函数不会立即执行，需要等到某个时机成熟后才会执行，该函数叫做异步函数。

浏览器的线程：

1. JS执行引擎：负责执行JS代码
2. 渲染线程：负责渲染页面
3. 计时器线程：负责计时
4. 事件监听线程：负责监听事件
5. http网络线程：负责网络通信

**事件队列**：一块内存空间，用于存放执行时机到达的异步函数。当JS引擎空闲（执行栈没有可执行的上下文），它会从事件队列中拿出第一个函数执行。

**事件循环**：event loop，是指函数在执行栈、宿主线程、事件队列中的循环移动。

### 代码示例

**异步函数与同步代码的执行顺序**

```js
setTimeout(function A() {
    console.log("异步代码")
}, 0)  // 即使延迟为0，也要等同步代码执行完

for (var i = 0; i < 10000; i++) {
    console.log(i);
}
// 所有同步代码执行完后，才执行 setTimeout 中的异步函数
```

**事件循环在事件处理中的体现**

```js
document.querySelector("button").onclick = function A() {
    setTimeout(function B() {
        console.log("异步代码")
    }, 0);
    loop();
}

function loop() {
    for (var i = 0; i < 10000; i++) {
        console.log(i);
    }
}

loop(); // 先执行初始的同步 loop()
// 点击按钮后：先执行 A -> 执行 loop() -> 执行完同步代码后 -> 从事件队列取出 B 执行
```

## 七、骚操作 - 对象混合和克隆

### 代码示例

**对象混合（mixin）**

```js
// 使用 Object.assign 混合对象
var obj1 = {
    x: 1,
    y: 3,
    z: 5
};

var obj2 = {
    x: "abc",
    z: "bcd",
    h: "aaa"
};

var obj = Object.assign({}, obj1, obj2);
console.log(obj); // { x: "abc", y: 3, z: "bcd", h: "aaa" }
```

**对象混合应用于默认配置**

```js
// option参数是一个对象，表示配置对象
function complicate(option) {
    // 如果没有传递，则使用默认值
    var defaultOption = {
        a: "default-a",
        b: "default-b",
        c: "default-c",
        d: "default-d",
        e: "default-e"
    };

    var opt = myPlugin.mixin(defaultOption, option);
    console.log(opt);
}

complicate({
    a: 123,
    b: 34
});
// 结果: { a: 123, b: 34, c: "default-c", d: "default-d", e: "default-e" }
```

> 对象混合常用于合并默认配置和用户传入的配置，用户的配置会覆盖默认值。

## 八、骚操作 - 函数防抖和函数节流

### 函数节流（throttle）

保证一个时间段内只执行一次。

```js
// 保证一个时间段内执行一次
window.onresize = function() {
    handle();
}

function test() {
    console.log("a");
}

var handle = myPlugin.throttle(test, 1000, false);
handle();  // 只有第一次会立即执行
handle();  // 在 1000ms 内不会再次执行
handle();
```

> 函数节流：在指定的时间间隔内，无论触发多少次，只执行一次。适用于 `resize`、`scroll` 等高频事件。

### 函数防抖（debounce）

在连续触发结束后只执行一次。

```js
// 高阶函数：在函数内部返回一个新的函数

// 窗口大小变化防抖
var handle = myPlugin.debounce(function(width) {
    console.log(width);
}, 1000);
window.onresize = function() {
    handle(document.documentElement.clientWidth);
}

// 输入框搜索防抖
var inp = document.querySelector("input");
var handle = myPlugin.debounce(function(val, inp) {
    console.log("搜索" + val);
    console.log(inp);
}, 1000);
inp.oninput = function() {
    handle(this.value, this);
}
```

> 函数防抖：在连续触发事件后，等待一定时间没有新事件才执行。适用于搜索输入框等场景，避免频繁发送请求。

## 九、[扩展] 骚操作 - 科里化

### 概念

科里化（Currying，得名于数学家 Haskell Curry）：固定某个函数的一些参数，得到该函数剩余参数的一个新函数，如果没有剩余参数，则调用。

### 代码示例

**基本科里化**

```js
function f(x, y, z) {
    return (x + y) * z;
}

// 求：(2+3)*5  (2+5)*6  (2+4)*7  (2+4)*16
var g = myPlugin.curry(f, 2)         // 固定第一个参数 x=2
console.log(g(3, 5));                 // 25 = (2+3)*5
console.log(g(5, 6));                 // 42 = (2+5)*6

var y = g(4);                         // 固定 x=2, y=4
console.log(y(7));                    // 42 = (2+4)*7
console.log(y(16));                   // 96 = (2+4)*16
```

**科里化实战 - 创建DOM元素**

```js
function createElement(container, name, props, styles, content) {
    var ele = document.createElement(name);
    container.appendChild(ele);

    for (var prop in props) {
        ele[prop] = props[prop];
    }

    for (var prop in styles) {
        ele.style[prop] = styles[prop];
    }
    if (content) {
        ele.innerHTML = content;
    }
}

var div = document.querySelector(".container");
var create = myPlugin.curry(createElement, div, "div", {}, {
    height: "100px",
    background: "#008c8c",
    margin: "30px",
    color: "#fff"
});

// 之后只需要传入内容即可创建元素
create("asdfasdf");
create("2344");
create("啊打发法是发啊手动阀啊");
```

> 科里化的优势：可以将多参数函数转化为一系列单参数函数，方便复用和组合，减少重复传参。

## 十、[扩展] 骚操作 - 函数管道

### 概念

函数管道（pipe）：将多个单参函数组合起来，形成一个新的函数，这些函数中，前一个函数的输出，是后一个函数的输入。

### 代码示例

**实现小驼峰命名法转换**

```js
// 将字符串中每一个单词首字母大写
function everyFirstLetterUp(str) {
    return str.replace(/\b(\w)(\w*)\b/g, function($, $1, $2) {
        return $1.toUpperCase() + $2;
    });
}

// 将字符串中除首字母外的其他字母小写
function otherLetterLower(str) {
    return str.replace(/\b(\w)(\w*)\b/g, function($, $1, $2) {
        return $1 + $2.toLowerCase();
    });
}

// 去掉字符串中所有的空白字符
function removeEmpty(str) {
    return str.replace(/\s+/g, "");
}

// 将字符串中第一个单词字符小写
function firstLetterLower(str) {
    return str.replace(/\w/, function($) {
        return $.toLowerCase();
    });
}

// 截取字符串
function curString(number, str) {
    return str.substr(0, number);
}

// 通过函数管道组合：小驼峰命名法
var smallCamel = myPlugin.pipe(
    everyFirstLetterUp,
    firstLetterLower,
    otherLetterLower,
    removeEmpty,
    myPlugin.curry(curString, 10)
);

console.log(smallCamel("    MY     fIrST    naME"));       // myFirstNam
console.log(smallCamel("  asdfasf  sdfasf   ggggggg  hGDGF")); // asdfasfSd
```

> 函数管道将多个处理步骤串联起来，数据像水流过管道一样经过每个函数的处理。结合科里化，可以将多参数函数转化为单参数函数加入管道。
