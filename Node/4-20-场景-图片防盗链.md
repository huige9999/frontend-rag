# 4-20 场景 - 图片防盗链

## 知识要点

- 通过检查 HTTP 请求头中的 `referer` 和 `host` 实现图片防盗链
- 只处理图片类型请求（.jpg, .jpeg, .png, .gif）
- 当检测到盗链时，通过 URL 重写返回默认图片
- 防盗链中间件应在静态资源中间件之前注册

## 防盗链中间件

```js
// routes/imgProtectMid.js
const url = require("url");
const path = require("path");

module.exports = (req, res, next) => {
  const host = req.headers.host; // 获取本网站的主机名（包括端口号）
  let referer = req.headers.referer;

  // 只处理图片请求
  const extname = path.extname(req.path);
  if (![".jpg", ".jpeg", ".png", ".gif"].includes(extname)) {
    next();
    return;
  }
  if (referer) {
    referer = url.parse(referer).host;
  }
  if (referer && host !== referer) {
    req.url = "/img/logo.jpg"; // URL 重写，返回默认图片
  }
  next();
};
```

## 注册顺序

防盗链中间件必须在静态资源中间件之前：

```js
app.use(require("./imgProtectMid")); // 先注册防盗链
app.use(express.static(staticRoot));  // 再注册静态资源
```

## 防盗链原理

1. 获取请求的 `host`（当前服务器主机名）
2. 获取请求的 `referer`（来源页面地址）
3. 如果是图片请求且 `referer` 的 host 与当前 host 不同，说明是盗链
4. 将请求 URL 重写为默认图片路径
