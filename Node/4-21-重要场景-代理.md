# 4-21 重要场景 - 代理

## 知识要点

- 使用 `http-proxy-middleware` 实现 API 代理
- 代理可以将请求转发到其他服务器
- 通过 `pathRewrite` 重写请求路径
- 代理常用于解决跨域问题或将请求分发到不同的后端服务

## 代理中间件

```js
// routes/proxyMid.js
const { createProxyMiddleware } = require("http-proxy-middleware");
const context = "/data";

module.exports = createProxyMiddleware(context, {
  target: "http://yuanjin.tech:5100",
  pathRewrite: function (path, req) {
    console.log(path.substr(context.length));
    return path.substr(context.length);
  },
});
```

## 注册代理中间件

```js
// routes/init.js
app.use(require("./proxyMid"));
```

## 代理工作流程

1. 客户端请求 `/data/xxx`
2. 代理匹配 `/data` 前缀
3. 通过 `pathRewrite` 去掉 `/data` 前缀，得到 `/xxx`
4. 将请求转发到 `http://yuanjin.tech:5100/xxx`
5. 将响应返回给客户端

## 配置选项

| 选项 | 说明 |
|------|------|
| `target` | 目标服务器地址 |
| `pathRewrite` | 路径重写函数 |
| `changeOrigin` | 是否修改请求头中的 Origin |
| `onProxyReq` | 代理请求回调 |
| `onProxyRes` | 代理响应回调 |
