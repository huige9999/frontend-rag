# 4-13 JWT (Json Web Token)

## 知识要点

- JWT（Json Web Token）是一种统一的、安全的令牌格式
- JWT 由三部分组成：header、payload、signature
- JWT 可存储在 cookie、localStorage 或通过 header 传输
- 使用 `jsonwebtoken` 库签发和验证 JWT
- JWT 适用于多种终端设备（浏览器、移动端、桌面应用等）

## JWT 令牌结构

`header.payload.signature`

- **header**：令牌头部，记录类型和签名算法（如 `HS256`）
- **payload**：令牌负荷，记录主体信息（用户信息等）
- **signature**：令牌签名，使用密钥对前两部分签名，防止篡改

## 使用 jsonwebtoken 库

### 签发 JWT

```js
const jwt = require("jsonwebtoken");
const secrect = "yuanjin";

const token = jwt.sign({ id: userId }, secrect, {
  expiresIn: 3600 * 24, // 过期时间（秒）
});
```

### 验证 JWT

```js
try {
  const result = jwt.verify(token, secrect);
  // result 即为 payload 中的数据
  return result;
} catch (err) {
  // 验证失败
  return null;
}
```

### JWT 的工作流程

1. 客户端登录成功
2. 服务器签发 JWT 令牌
3. 客户端存储令牌（cookie / localStorage）
4. 后续请求附带令牌（cookie / authorization header）
5. 服务器验证令牌有效性
