# 5-6 NodeJS组成原理

## 知识要点

### Node.js 组成结构（从上到下）

#### 1. 用户代码
- JS 代码，由开发者编写
- 运行在最上层

#### 2. 第三方库
- 大部分仍然是 JS 代码，由其他开发者编写
- 通过 npm 安装使用

#### 3. 本地模块代码（Node.js 内置模块）
- JS 代码
- 如 `fs`、`http`、`path`、`crypto` 等

#### 4. V8 引擎
- C/C++ 代码，核心作用：**把 JS 代码解释成为机器码**
- 可以通过 V8 引擎的某种机制扩展其功能
- V8 引擎的扩展和对扩展的编译，是通过 **gyp 工具** 完成的

### node-gyp 工具

某些第三方库（如 `node-sass`、`bcrypt` 等）包含 C/C++ 原生代码，需要使用 `node-gyp` 工具进行构建：

```bash
npm install -g node-gyp
```

在 Windows 环境下，`node-gyp` 还需要：
- Python 2.7 或 3.x
- Visual Studio Build Tools

## 关键代码

### 验证 Node.js 模块包装函数（index.js）

在 Node.js 中，每个模块都被包装在一个函数中，可以通过 `arguments` 来验证：

```js
console.log(arguments.length);
// 输出：5
// 说明模块代码被包装在一个函数中，该函数有5个参数：
// exports, require, module, __filename, __dirname
```

### Node.js 模块包装函数

Node.js 内部将每个模块的代码包装在如下函数中：

```js
(function(exports, require, module, __filename, __dirname) {
  // 你的模块代码实际上在这里面运行
  // 这就是为什么你可以直接使用 require、module 等变量的原因
});
```
