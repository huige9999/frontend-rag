# 4-25 场景 - 客户端缓存

## 知识要点

- HTTP 缓存协议：`Cache-Control`、`ETag`、`Last-Modified`、`Date`
- 强缓存（max-age）vs 协商缓存（304 Not Modified）
- 为静态资源配置长期缓存，HTML 文件不缓存
- 通过 `express.static` 的 `setHeaders` 自定义缓存策略

## HTTP 缓存流程

1. 服务器在响应头中附带缓存指令（`Cache-Control: max-age=3600`、`ETag`、`Last-Modified`）
2. 浏览器将资源缓存到本地
3. 再次请求时先检查缓存是否过期
4. 缓存有效：直接使用缓存（不请求服务器）
5. 缓存过期：发送带 `If-Modified-Since` 和 `If-None-Match` 的请求
6. 服务器判断资源是否变化：没变返回 304，变了返回新资源

## Express 中配置缓存

为非 HTML 文件设置长期缓存：

```js
app.use(
  express.static(staticRoot, {
    setHeaders(res, path) {
      if (!path.endsWith(".html")) {
        // 非 HTML 文件缓存 100 年
        res.header("Cache-Control", `max-age=${3600 * 24 * 365 * 100}`);
      }
    },
  })
);
```

## 缓存策略说明

- HTML 文件不缓存（每次都请求最新版本）
- 其他静态资源（JS/CSS/图片）缓存 100 年
- 配合文件名 hash（如 `app.68297cd8.css`）实现更新
- 当 HTML 引用新的 hash 文件名时，浏览器会请求新资源

## Cache-Control 常用值

| 值 | 说明 |
|----|------|
| `max-age=秒` | 缓存有效期 |
| `no-cache` | 可以缓存但每次需验证 |
| `no-store` | 不缓存 |
| `public` | 公开资源 |
| `private` | 私有资源 |
