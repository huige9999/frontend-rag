# 5-7 进程和线程

## 知识要点

### 1. 进程

一个应用程序通过操作系统启动时，操作系统会为其分配一个进程。

- 进程拥有**独立的、可伸缩的**内存空间，原则上不受其他进程干扰
- 进程之间可以通信（如 IPC 协议）
- **CPU 在不同的进程之间切换执行**
- 主进程：由操作系统直接开启
- 子进程：由另一个进程开启，继承父进程部分信息，但保持相对独立

### 2. 线程

操作系统启动一个进程时，会自动为其分配一个线程（主线程）。**程序一定在线程上运行！**

- 主线程可以创建多个子线程
- CPU 在进程的多个线程中切换执行
- 线程与进程最大的区别：**线程的内存空间没有隔离**，共享进程的内存空间

### 3. 什么时候使用线程？

使用线程的主要目的是**充分利用多核 CPU**：

**最理想的线程效果：**
1. 线程数等于 CPU 核数
2. 线程永不阻塞（没有 IO，只有大量运算）
3. 线程相对独立，几乎不使用共享数据

> 线程一般处理 **CPU 密集型操作**（运算操作），IO 密集型操作适合使用异步而非线程。

## 关键代码

### 子进程示例（index copy.js）

使用 `child_process` 模块开启子进程：

```js
const childProcess = require("child_process");

childProcess.exec(`ls`, (err, out, stdErr) => {
  // err：开启进程过程中的错误
  // out：子进程的正常输出
  // stdErr：子进程的错误输出
  // 子进程的错误绝不会影响父进程，内存空间完全隔离
  console.log(out);
});
```

### 创建子线程（index copy 2.js）

使用 `worker_threads` 模块创建子线程：

```js
const { Worker } = require("worker_threads");
const path = require("path");

// 创建子线程，指定入口文件和传递数据
const worker = new Worker(path.resolve(__dirname, "worker.js"), {
  workerData: 123,
});

worker.on("exit", () => {
  console.log("子线程退出了");
});
```

### 单线程素数计算（index copy 3.js）

不使用多线程，直接在主线程中计算素数：

```js
const isPrime = require("./isPrime");
const arr = require("./numbers.json");

console.time();
const newArr = [];
for (const n of arr) {
  if (isPrime(n)) {
    newArr.push(n);
  }
}
console.timeEnd();
console.log(newArr);
```

### 多线程素数计算（index.js）

将数组按 CPU 核数分割，分配给多个子线程并行计算素数：

```js
const { Worker } = require("worker_threads");
const os = require("os");
const arr = require("./numbers.json");

const cpuNumber = os.cpus().length; // 获取 CPU 核数
const len = Math.ceil(arr.length / cpuNumber); // 每个线程处理的数据量

console.time();
let numbers = cpuNumber; // 剩余未完成的线程数
const newArr = []; // 最终结果

for (let i = 0; i < cpuNumber; i++) {
  const data = arr.slice(i * len, (i + 1) * len);
  const worker = new Worker("./worker.js", {
    workerData: data,
  });

  worker.on("message", (result) => {
    newArr.push(...result);
    numbers--;
    if (numbers === 0) {
      // 所有线程处理完毕
      console.timeEnd();
      console.log(newArr);
    }
    worker.terminate(); // 终止子线程
  });
}
```

### 子线程逻辑（worker.js）

子线程接收数据，计算素数后返回结果：

```js
const isPrime = require("./isPrime");
const {
  parentPort,
  workerData,
  threadId,
} = require("worker_threads");

const name = `线程${threadId}`;
const newArr = [];

for (const n of workerData) {
  if (isPrime(n)) {
    newArr.push(n);
  }
}

console.log(`${name}处理完成，并把结果给予了主线程`);
parentPort.postMessage(newArr); // 向主线程发送结果
```

### 素数判断函数（isPrime.js）

判断一个数是否为素数：

```js
module.exports = function (n) {
  // 素数：只能被1和自身整除的数
  if (n < 2) {
    return false;
  }
  for (let i = 2; i < n; i++) {
    if (n % i === 0) {
      return false;
    }
  }
  return true;
};
```

### 线程 API 速查

**父线程 API：**

```js
const { Worker } = require("worker_threads");
const worker = new Worker(入口文件, { workerData: 传递的数据 });

worker.on("exit", () => {});       // 子线程退出
worker.on("message", (msg) => {}); // 收到子线程消息
worker.postMessage(任意消息);       // 向子线程发送消息
worker.terminate();               // 终止子线程
```

**子线程 API：**

```js
const {
  isMainThread,   // 是否是主线程
  parentPort,      // 与父线程通信的端口
  workerData,      // 启动时传递的数据
  threadId,        // 线程唯一编号
} = require("worker_threads");

parentPort.on("message", (msg) => {}); // 收到父线程消息
parentPort.postMessage(data);          // 向父线程发送消息
```
