# 4-8 断点调试

## 知识要点

- Node.js 支持通过 Chrome DevTools 进行断点调试
- 启动调试模式：`node --inspect index.js` 或 `node --inspect-brk index.js`
- 使用 VS Code 内置调试器配置 launch.json
- 本节代码与 4-7 完全一致，重点是调试工具的使用方法

## 调试方法

### 方法一：Chrome DevTools

1. 使用 `--inspect` 标志启动应用：

```shell
node --inspect index.js
```

2. 打开 Chrome 浏览器，访问 `chrome://inspect`
3. 点击 "Open dedicated DevTools for Node"
4. 在 Sources 面板中找到代码文件，设置断点进行调试

### 方法二：VS Code 调试

在 `.vscode/launch.json` 中配置：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Launch Program",
      "program": "${workspaceFolder}/index.js"
    }
  ]
}
```

然后按 F5 启动调试。

### --inspect 与 --inspect-brk 的区别

| 标志 | 说明 |
|------|------|
| `--inspect` | 启动调试器，代码正常运行 |
| `--inspect-brk` | 启动调试器，在第一行代码处断住 |
