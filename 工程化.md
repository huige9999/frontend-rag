## 2. 章节逻辑说明

这门课是前端从「写脚本」走向「工程化开发」的入门，主线是**「模块化标准（01~02）→ 包管理（03）→ 工具链版本（04-05 Volta）→ 预编译器 Less（04）→ 构建工具 webpack（05）→ 综合案例（06）」**，把「现代前端工程」的几大支柱逐个建立起来。

### 先讲什么：模块化标准（01~02）

- **01 CommonJS**：node 天生支持的社区模块化标准（`require`/`module.exports`），解决全局污染、依赖混乱、代码难以细分。
- **02 ES Module**：ES6 官方模块化标准（`import`/`export`），浏览器和新版 node 都支持，配综合练习。

**为什么先讲模块化**：模块化是「工程化」的起点——没有模块化，代码没法拆分、依赖没法管理，后面包管理、构建工具都无从谈起。

### 再讲什么：包管理 + 工具链版本（03、04-05）

- **03 包管理器**：npm 怎么装包、管依赖。
- **04-05 使用Volta 管理工具链版本**：用 Volta 统一团队 node/npm 等工具链版本，避免「我本地能跑」的环境问题。

**为什么这么安排**：有了模块化，下一步就是「怎么用别人写好的模块」——包管理装依赖，Volta 管版本，解决「依赖从哪来、怎么保证团队一致」。

### 然后讲什么：预编译器与构建工具（04 Less、05 构建工具）

- **04 Less**：CSS 预编译器，让 CSS 支持变量、嵌套、混入。
- **05 构建工具的使用**：webpack 全流程（打包、开发服务器、缓存等），配「工程化全图 / 打包 / 开发服务器 / 缓存」drawio 图。

**为什么这么安排**：模块化和包管理解决「JS 怎么组织」，Less 解决「CSS 怎么工程化」，webpack 则是把所有这些「源码 + 预编译 + 模块」打包成浏览器能跑的产物——构建工具是工程化的「集大成者」，放最后顺理成章。

### 最后讲什么：综合案例（06）

- **06 分页电影列表案例**：把模块化 + 构建 + Less 串成一个真实小项目（带分步 drawio 思路图 + 课堂代码），作为整门课的落地收口。

### 主线一句话总结

**模块化标准（01~02）→ 包管理与工具链（03、04-05 Volta）→ CSS 预编译 Less（04）→ 构建工具 webpack（05）→ 分页电影列表综合案例（06）**，目标是从「裸写 HTML/JS」进阶到「能用 npm + Less + webpack 搭一个现代前端工程」。

## 3. 使用场景等元信息沉淀

- **学完能用在哪**：用 CommonJS/ESM 组织代码、用 npm 管依赖、用 Volta 锁版本、用 Less 写样式、用 webpack 搭建和打包工程。
- **和实际项目的关系**：模块化/包管理/构建是所有现代前端项目的底座；webpack 全流程图是排查「打包慢、产物大」问题的理论依据。
- **和其他课程的关系**：模块化有独立的《模块化详细版》、包管理有《包管理器详细版》、webpack 有《Webpack 详细版》、CSS 有《现代CSS》——本课是「工程化入门串联」，深度交给各详细版课程。
- **面试/教学高频场景**：CommonJS 与 ESM 区别（01~02）、npm 依赖管理（03）、webpack 打包/开发服务器/缓存原理（05）。
